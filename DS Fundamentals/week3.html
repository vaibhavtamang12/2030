P1 — Python Fundamentals You Must Know

📦

Variables & Types

A variable is a named box that holds a value. Types: int (whole numbers), float (decimals), str (text), bool (True/False).

📋

Lists

An ordered collection: [1, 2, 3]. You can put anything inside. Key: they're slow for math — that's why we use NumPy arrays instead.

🗂

Dictionaries

Key-value pairs: {'price': 100.5, 'ticker': 'AAPL'}. Think of it as a lookup table. Basis for pandas DataFrames conceptually.

🔄

Loops

for item in list: — repeats code. The enemy of performance in quant work. You'll learn to replace most loops with vectorized operations.

⚙️

Functions

def my_func(x): return x*2. Reusable blocks of code. Think of them as mini-machines: input goes in, output comes out.

📥

Imports

import numpy as np — loads an external library. The as np part gives it a short alias so you type np.array() instead of numpy.array().

# Python basics in 30 lines — run this first
import math

# Variables
price = 150.25    # float

ticker = 'AAPL'   # string
value = price * shares  # 15025.0

# List (ordered, slow for math)
prices = [100, 102, 98, 105, 103]

# Loop — calculates returns (we'll vectorize this later!)
returns = []
for i in range(1, len(prices)):

# Function

    return (p_new - p_old) / p_old

print(calc_return(100, 105))  # 0.05 = 5%

P3 — Statistics Essentials

📊

Mean (Average)

Sum of all values ÷ count. The "center" of your data. In finance: average daily return.

📏

Standard Deviation (Std)

How spread out values are from the mean. In finance: volatility. High std = risky asset.

🔔

Bell curve. Many financial models assume returns are normally distributed (a simplification, but useful starting point).

🔗

How two assets move together. Range: -1 to +1. Diversification = combining low-correlation assets.

NumPy — The Engine TOPIC 1
NumPy is the foundation of numerical computing in Python. Everything else in this guide is built on it.
🧠 The Analogy

Python lists are like carrying groceries one bag at a time. NumPy arrays are like a conveyor belt — they move all items at once. This is why NumPy is 100–500× faster than pure Python for mathematical operations.

1.1 — Why NumPy Exists

❌ Python Lists (Slow)
  • Dynamic typing — Python checks types at runtime
  • Stored as pointers (memory jumps)
  • No parallel CPU operations (SIMD)
  • Loops in Python = slow
✅ NumPy Arrays (Fast)
  • Fixed dtype — C-level operations
  • Contiguous memory (cache-friendly)
  • BLAS/LAPACK optimized
  • Vectorized: operates on whole arrays

import numpy as np
prices = np.array([100, 102, 98, 105, 103, 107])

prices[0]    # 100  — first element
prices[-1]   # 107  — last element

idx = np.array([0, 2, 4])  # pick specific days

1.4 — Broadcasting — The Magic Trick

🧠 Analogy

Broadcasting is like multiplying every price in a column by a scalar, but NumPy figures out how to stretch smaller arrays to match larger ones — without copying data.

import numpy as np

prices = np.array([[100, 102, 98],
                   [200, 205, 195]])  # shape (2,3)

# Scalar broadcast — applies to every element
prices * 1.1  # shape (2,3) × scalar = (2,3)

# Row-wise normalisation — subtract each asset's mean
means = prices.mean(axis=1, keepdims=True)  # shape (2,1)
norm = prices - means  # (2,3) - (2,1) → broadcasts to (2,3)

# Column broadcast — different weight per day
weights = np.array([1.0, 0.9, 1.1])  # shape (3,)
adj = prices * weights  # (2,3) × (3,) → (2,3) ✓

1.5 — Essential NumPy Operations for Quant Work

import numpy as np

rng = np.random.default_rng(42)
returns = rng.normal(0.001, 0.02, size=252)  # 1yr daily returns

# ── Statistics ──
print(returns.mean())           # average daily return
print(returns.std())            # daily volatility
print(returns.min(), returns.max())
print(np.percentile(returns, [5, 95]))  # 5th/95th percentile = VaR-ish

# ── Cumulative operations ──
cum_returns = np.cumprod(1 + returns)   # wealth index
cum_sum = np.cumsum(returns)              # cumulative P&L

# ── Linear Algebra ──
cov_matrix = np.cov(returns.reshape(1, -1))  # covariance
A = np.array([[2,1],[1,3]])
eigenvalues, eigenvectors = np.linalg.eig(A)  # eigendecomposition (PCA)
inv_A = np.linalg.inv(A)                       # matrix inverse

# ── Sorting & Ranking ──
sorted_r = np.sort(returns)[::-1]  # descending sort
ranks = np.argsort(np.argsort(returns))  # rank each return

# ── Where (conditional assignment — no loops!) ──
signal = np.where(returns > 0, 1, -1)  # 1 if positive day, -1 if negative

2.2 — scipy.stats in Depth

from scipy import stats
import numpy as np

# ── Work with any distribution ──
norm = stats.norm(loc=0.001, scale=0.02)  # N(mu=0.1%, sigma=2%)

# PDF, CDF — building blocks of option pricing
norm.pdf(0.0)      # probability density at 0
norm.cdf(-0.05)   # P(return < -5%) = tail probability
norm.ppf(0.05)    # 5th percentile = daily VaR at 95% confidence

# ── Value at Risk (VaR) ──
confidence = 0.95
mu, sigma = 0.001, 0.02
VaR_95 = -stats.norm.ppf(1 - confidence, mu, sigma)
print(f"1-day 95% VaR: {VaR_95:.4f} = {VaR_95*100:.2f}%")
# Interpretation: 95% of the time, daily loss ≤ this amount

# ── Fit a distribution to real data ──
rng = np.random.default_rng(42)
real_returns = rng.normal(0.001, 0.02, 1000)
mu_hat, sigma_hat = stats.norm.fit(real_returns)
print(f"Fitted mu={mu_hat:.4f}, sigma={sigma_hat:.4f}")

# ── Fat tails? Test for normality ──
stat, p_value = stats.shapiro(real_returns[:50])  # Shapiro-Wilk (small samples)
stat2, p2 = stats.normaltest(real_returns)  # D'Agostino (large samples)
print(f"Returns are normal? p={p2:.3f} ({'YES' if p2>0.05 else 'NO - fat tails!'})")

# ── Student's t distribution (better for finance) ──
df, loc, scale = stats.t.fit(real_returns)  # fit t-distribution
t_VaR = -stats.t.ppf(0.05, df, loc, scale)
print(f"t-dist 95% VaR: {t_VaR:.4f} (larger due to fat tails)")

2.3 — scipy.optimize — Portfolio Optimization

🧠 What is Optimization?

Given a function f(x), find the x that minimizes f. In finance: find portfolio weights that minimize risk for a given return. scipy.optimize.minimize does this numerically.

from scipy.optimize import minimize
import numpy as np

# Simulate 3-asset return data
rng = np.random.default_rng(42)
returns_data = rng.multivariate_normal(
    mean=[0.001, 0.0008, 0.0012],
    cov=[[0.0004, 0.0001, 0.0002],
         [0.0001, 0.0003, 0.00015],
         [0.0002, 0.00015, 0.0005]],
    size=252
)  # shape (252, 3): 252 days, 3 assets

cov = np.cov(returns_data.T)  # (3,3) covariance matrix
mu = returns_data.mean(axis=0)  # (3,) mean returns

# Objective: minimize portfolio variance = w^T Σ w
def portfolio_variance(w):
    return w @ cov @ w  # dot(w, cov @ w)

# Constraints: weights sum to 1
constraints = {'type': 'eq', 'fun': lambda w: np.sum(w) - 1}
# Bounds: each weight between 0 and 1 (long-only)
bounds = [(0, 1)] * 3
w0 = np.array([1/3, 1/3, 1/3])  # equal-weight starting point

result = minimize(portfolio_variance, w0,
                  method='SLSQP',
                  bounds=bounds,
                  constraints=constraints)

opt_weights = result.x
print(f"Optimal weights: {opt_weights.round(4)}")
print(f"Min variance: {result.fun:.6f}")
print(f"Expected return: {(opt_weights @ mu):.4f}")

2.4 — Cholesky Decomposition — Correlated Simulations

from scipy import linalg
import numpy as np

# Why: simulating correlated assets (e.g., AAPL and NASDAQ move together)
# Cholesky: decompose Σ into L*L^T, so Z*L gives correlated samples

corr = np.array([[1.0, 0.7, 0.3],
                 [0.7, 1.0, 0.2],
                 [0.3, 0.2, 1.0]])  # correlation matrix

L = linalg.cholesky(corr, lower=True)  # lower triangular

rng = np.random.default_rng(42)
Z = rng.standard_normal((10000, 3))  # independent standard normals
corr_samples = Z @ L.T  # now correlated!

# Verify: should match original corr matrix
print(np.corrcoef(corr_samples.T).round(2))

COLABExperiment 2 — Efficient Frontier Visualization
import numpy as np
from scipy.optimize import minimize
import matplotlib.pyplot as plt

# Random 3-asset data
rng = np.random.default_rng(42)
R = rng.multivariate_normal([0.001,0.0015,0.0008],
    [[4e-4,1e-4,2e-4],[1e-4,6e-4,1e-4],[2e-4,1e-4,3e-4]], 252)
cov = np.cov(R.T); mu = R.mean(0)

results = []
for _ in range(5000):
    w = np.random.dirichlet(np.ones(3))  # random weights summing to 1
    ret = w @ mu * 252
    vol = np.sqrt(w @ cov @ w * 252)
    results.append((vol, ret))

vols, rets = zip(*results)
plt.scatter(vols, rets, c=np.array(rets)/np.array(vols), cmap='viridis', s=3, alpha=.6)
plt.colorbar(label='Sharpe Ratio')
plt.xlabel('Annualised Volatility'); plt.ylabel('Annualised Return')
plt.title('Efficient Frontier — Monte Carlo'); plt.show()

3.3 — Indexing: loc vs iloc

⚠️ Critical Distinction

.loc uses labels (column names, date strings). .iloc uses integer positions (0-based). Getting this wrong is the #1 source of bugs in quant pandas code.

import pandas as pd

df = pd.DataFrame({'AAPL':[150,152,149], 'GOOG':[140,141,139]},
    index=pd.date_range('2024-01-02', periods=3, freq='B'))

# ── .loc — label-based ──
df.loc['2024-01-02']                   # single row by date label
df.loc['2024-01-02':'2024-01-04']    # date range (INCLUSIVE on both ends!)
df.loc[:, 'AAPL']                       # all rows, AAPL column
df.loc['2024-01-02', 'AAPL']           # specific cell: 150

# ── .iloc — position-based ──
df.iloc[0]           # first row (regardless of label)
df.iloc[1:3]         # rows 1 and 2 (3 is EXCLUSIVE, like Python slices)
df.iloc[:, 0]        # first column
df.iloc[-1, -1]      # bottom-right corner

# ── Boolean indexing ──
df[df['AAPL'] > 150]       # days where AAPL > 150
df[(df['AAPL'] > 150) & (df['GOOG'] < 141)]  # compound condition
df.query('AAPL > 150 and GOOG < 141')  # cleaner syntax for same thing

3.4 — Groupby & Aggregation

import pandas as pd
import numpy as np

# Imagine tick-by-tick data with a 'sector' column
rng = np.random.default_rng(42)
n = 1000
df = pd.DataFrame({
    'ticker': np.random.choice(['AAPL','GOOG','JPM','BAC'], n),
    'sector': np.random.choice(['Tech','Finance'], n),
    'return': rng.normal(0.001, 0.02, n),
    'volume': rng.integers(1000, 100000, n)
})

# Mean return by sector
df.groupby('sector')['return'].mean()

# Multiple aggregations
df.groupby('ticker').agg({
    'return': ['mean', 'std', 'count'],
    'volume': 'sum'
})

# Custom aggregation: Sharpe ratio by sector
def sharpe(r): return r.mean() / r.std() * np.sqrt(252)
df.groupby('sector')['return'].apply(sharpe)

3.5 — Missing Data (Critical in Finance)

import pandas as pd
import numpy as np

prices = pd.Series([100, np.nan, 102, np.nan, 105],
    index=pd.date_range('2024-01-01', periods=5))

# Detection
prices.isna()        # Boolean mask
prices.isna().sum()  # count of NaNs

# Fill strategies
prices.ffill()   # Forward fill — use last known price (most common in finance)
prices.bfill()   # Backward fill
prices.fillna(prices.mean())   # Fill with mean
prices.interpolate()           # Linear interpolation

# Drop
prices.dropna()  # Remove NaN rows entirely

# Tip: ALWAYS check for NaNs before computing returns!
returns = prices.pct_change().dropna()

import pandas as pd
import numpy as np

returns = prices.pct_change().dropna()
ann_return = returns.mean() * 252
calmar = ann_return / abs(max_drawdown)
print(f"Calmar Ratio: {calmar:.2f}")

Vectorization TOPIC 5
The single most important performance skill. Replace every Python loop with array operations.
🧠 The Factory Analogy

A Python loop is like a single worker assembling cars one at a time. Vectorization is like an assembly line — 10,000 cars built simultaneously in parallel. The same work, done 100× faster, because modern CPUs have SIMD (Single Instruction, Multiple Data) units that operate on entire arrays in one clock cycle.

5.1 — The Core Mental Model

❌ Think: "For each element..."
  • Leads to Python loops
  • Interpreted one element at a time
  • Function call overhead × N
  • Slow: 100–1000ms for 1M elements
✅ Think: "Transform the whole array"
  • Array operations on all elements at once
  • Runs in optimized C/Fortran
  • SIMD: process 8+ floats per clock
  • Fast: 1–5ms for 1M elements

5.2 — Replacing Loops: Pattern Catalog

import numpy as np

prices = np.array([100, 102, 98, 105, 103])

# ── Pattern 1: Element-wise transformation ──
# ❌ SLOW:
log_prices_slow = [np.log(p) for p in prices]

# ✅ FAST:
log_prices = np.log(prices)  # ufunc — operates on entire array

# ── Pattern 2: Pairwise differences (returns) ──
# ❌ SLOW:
returns_slow = [(prices[i]-prices[i-1])/prices[i-1] for i in range(1,len(prices))]

# ✅ FAST: use slicing to create "current" and "previous" arrays
returns = (prices[1:] - prices[:-1]) / prices[:-1]

# ── Pattern 3: Cumulative operations ──
# ❌ SLOW:
cum_ret = [1.0]
for r in returns: cum_ret.append(cum_ret[-1] * (1+r))

# ✅ FAST:
wealth = np.cumprod(1 + returns)

# ── Pattern 4: Conditional assignment ──
# ❌ SLOW:
signals_slow = []
for r in returns:
    if r > 0: signals_slow.append(1)
    else: signals_slow.append(-1)

# ✅ FAST:
signals = np.where(returns > 0, 1, -1)

# ── Pattern 5: Rolling sum without loop ──
# Equivalent to rolling(3).sum() — manual version
k = 3
rolling_sum = np.convolve(returns, np.ones(k), mode='valid')

5.3 — pandas Vectorization

import pandas as pd
import numpy as np

n = 100_000
rng = np.random.default_rng(42)
df = pd.DataFrame({
    'price': rng.uniform(50, 200, n),
    'volume': rng.integers(1000, 1_000_000, n),
    'side': rng.choice(['BUY', 'SELL'], n)
})

# ── ALWAYS prefer built-in vectorized methods ──
# ❌ Slow: .apply() on row-by-row basis
df['notional_slow'] = df.apply(lambda row: row['price'] * row['volume'], axis=1)

# ✅ Fast: direct column operations
df['notional'] = df['price'] * df['volume']  # element-wise vectorized

# ❌ Slow: .apply() for string comparison
df['signed_slow'] = df['side'].apply(lambda x: 1 if x=='BUY' else -1)

# ❌ Slow: .apply() for threshold logic
df['bucket_slow'] = df['price'].apply(lambda p: 'high' if p>150 else 'low')

# ✅ Fast: np.select for multi-condition logic
conditions = [df['price'] > 150, df['price'] > 100]
choices = ['high', 'mid']
df['bucket'] = np.select(conditions, choices, default='low')

5.5 — Advanced: Matrix Operations for Portfolio Work

import numpy as np

# Given: returns matrix R (T × N) — T days, N assets
T, N = 252, 10
rng = np.random.default_rng(42)
R = rng.normal(0.001, 0.02, (T, N))

# ── Covariance matrix — vectorized ──
# Manual (but correct) — all in one matrix op
R_demean = R - R.mean(axis=0)          # (T,N) - (N,) broadcast
Sigma = (R_demean.T @ R_demean) / (T-1)   # (N,N) covariance matrix

# ── Portfolio variance for ALL weight combos simultaneously ──
W = rng.dirichlet(np.ones(N), size=10000)  # (10000, N) — 10K portfolios
# w^T Σ w for all portfolios at once (matrix op, no loop!)
port_var = np.einsum('ij,jk,ik->i', W, Sigma, W)  # (10000,) — pure speed
port_std = np.sqrt(port_var)

# ── Cross-sectional z-score normalization (factor investing) ──
# Each row = one day, normalize across stocks (axis=1)
mu_cross = R.mean(axis=1, keepdims=True)    # (T,1)
sd_cross = R.std(axis=1, keepdims=True)     # (T,1)
R_normalized = (R - mu_cross) / sd_cross    # (T,N) — broadcasting!
Matrix Portfolio Ops
💡 The Einsum Trick

np.einsum is the most powerful vectorization tool. It expresses complex matrix operations in Einstein notation. 'ij,jk,ik->i' means: for each portfolio i, sum over j,k the product W[i,j] × Σ[j,k] × W[i,k]. This is w^T Σ w for all portfolios simultaneously.

Performance Optimization TOPIC 6
When vectorization isn't enough — profiling, memory layout, Numba JIT, and algorithmic tricks.
🧠 The Optimization Hierarchy

Always optimize in this order: (1) Algorithm — a O(n) algorithm beats O(n²) always. (2) Vectorization — remove Python loops. (3) Memory — keep data cache-friendly. (4) Parallelism — use all CPU cores. (5) JIT/C extensions — Numba as last resort. Never start at step 5.

6.1 — Profiling: Finding the Bottleneck

⚠️ Golden Rule

Never optimize without profiling first. 90% of execution time is in 10% of the code. Guess wrong and you waste hours. Profile to find the real bottleneck.

import cProfile
import pstats
import numpy as np

def slow_portfolio_calc(returns, weights):
    """Example with hidden bottleneck"""
    port_returns = []

        daily_port_return = 0
        for i, w in enumerate(weights):
            daily_port_return += w * returns[day, i]

6.4 — Numba: JIT Compilation

🧠 What is JIT?

Numba's @jit decorator compiles your Python function to native machine code the first time it runs. After that, it executes at C speed. Use it when a problem genuinely can't be vectorized (e.g., path-dependent algorithms like Asian option pricing).

from numba import jit, prange
import numpy as np

# Path-dependent: can't vectorize due to sequential dependency
# ❌ Pure Python — slow:
def asian_option_slow(S0, K, r, sigma, T, n_paths, n_steps):
    dt = T / n_steps
    payoffs = []
    for _ in range(n_paths):
        S = S0
        path = [S]
        for _ in range(n_steps):
            S *= np.exp((r - 0.5*sigma**2)*dt + sigma*np.sqrt(dt)*np.random.randn())
            path.append(S)
        payoffs.append(max(np.mean(path) - K, 0))
    return np.exp(-r*T) * np.mean(payoffs)

# ✅ Numba JIT — C-speed:
@jit(nopython=True, parallel=True)  # parallel=True uses all CPU cores!
def asian_option_fast(S0, K, r, sigma, T, n_paths, n_steps):
    dt = T / n_steps
    payoffs = np.zeros(n_paths)
    for i in prange(n_paths):  # prange = parallel range
        S = S0
        path_sum = S0
        for j in range(n_steps):
            S *= np.exp((r - 0.5*sigma**2)*dt + sigma*np.sqrt(dt)*np.random.randn())
            path_sum += S
        avg = path_sum / (n_steps + 1)
        payoffs[i] = max(avg - K, 0.0)
    return np.exp(-r*T) * payoffs.mean()

# First call: compiles (takes a moment). Subsequent calls: C-speed
price = asian_option_fast(100, 100, 0.05, 0.2, 1.0, 100_000, 252)
print(f"Asian option price: ${price:.4f}")
