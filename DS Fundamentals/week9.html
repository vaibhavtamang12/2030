Math You Need (Explained Simply)

Linear Algebra in 2 Minutes

Think of a vector as a list of numbers — like a stock's features: [price, volume, volatility]. A matrix is a table of such vectors — one row per stock, one column per feature. ML uses matrix math to find patterns across thousands of stocks simultaneously.

X = [[101, 5000, 0.2],
[98, 3200, 0.15],
[205, 8900, 0.31]]
3 stocks × 3 features matrix

Calculus in 2 Minutes

A derivative tells you how fast something changes. In ML, we use it to figure out: "if I tweak this parameter slightly, does my model get better or worse?" This is called gradient descent — the engine behind almost all ML learning.

slope = Δy / Δx
"How does my error change
as I change my weight?"

Statistics Foundations

The 5 Stats Concepts Every Quant Needs

1. Mean & Variance

Mean = average value of returns. Variance = how spread out those returns are (risk!). Standard deviation = √variance = the usual "volatility" number in finance.

μ = (1/n) Σxᵢ
σ² = (1/n) Σ(xᵢ - μ)²

2. Correlation

How much do two things move together? -1 = perfectly opposite, 0 = no relation, +1 = move in lockstep. Crucial for diversification and understanding feature relationships.

ρ = Cov(X,Y) / (σ_X × σ_Y)

3. Probability Distribution

A mathematical description of all possible outcomes and their likelihoods. Stock returns roughly follow a normal distribution — bell shaped, though real returns have "fat tails" (more extreme events than normal predicts).

4. p-value & Significance

Asks: "Could this pattern I found be just random noise?" A p-value of 0.05 means there's only a 5% chance the result happened by accident. Critical for validating trading signals.

5. Overfitting Intuition

Imagine memorising every question from a past exam vs. truly understanding the material. A model that overfits memorises the training data but fails on new unseen data. In trading, this is catastrophic — your strategy looks great in backtest but loses money live.

Financial Concepts You'll Encounter

Returns

The percentage change in price: (P_t - P_{t-1}) / P_{t-1}. This is the target variable most quant ML models predict.

Alpha

Return above the market benchmark. A trading strategy with "positive alpha" makes money beyond just riding the market up.

Signal

Any measurable indicator that predicts future price movement — momentum, earnings surprises, sentiment scores. ML finds signals automatically.

Sharpe Ratio

Return / Risk = (mean return - risk free rate) / std of returns. Higher = better risk-adjusted performance.

Look-ahead Bias

Using future information to make a decision in the past during backtesting. The cardinal sin of quant research — makes fake performance look real.

Time Series

Data indexed by time — e.g., daily prices for 5 years. Unlike random data, time series has serial correlation: yesterday's price affects today's.

Supervised Learning

The core paradigm: teaching a computer by showing it examples with correct answers.

What Is Supervised Learning? (The Big Idea)

The human analogy: Imagine you're training a new analyst. You show them 1,000 historical trades with outcomes — "this setup led to a +3% gain, this one led to a -5% loss." After seeing enough examples, they start to learn the patterns and can predict what will happen in new, unseen situations. That's exactly what supervised learning does — except the "analyst" is a mathematical function.

The Formal Definition

Given a dataset of inputs (features X) and known outputs (labels y), find a function f such that:

f(X) ≈ y

Where X = features (what you know) and y = target (what you want to predict)

After training, you use f to predict y for new data points where y is unknown.

The ML Workflow — Step by Step

Supervised Learning in Finance: Two Types

🔢 Regression — Predict Numbers

Output is a continuous number. You're predicting how much.

  • Predict next week's stock return: +1.3%
  • Predict portfolio volatility: 0.18
  • Forecast earnings per share: $4.22
  • Price an option contract: $12.50

Loss function: Mean Squared Error (MSE)

🏷️ Classification — Predict Categories

Output is a discrete category. You're predicting which bucket.

  • Will stock go up or down tomorrow?
  • Is this company likely to default?
  • Buy / Hold / Sell signal
  • Is market regime trending or mean-reverting?

Loss function: Cross-Entropy / Accuracy

The Train/Test Split — The Most Important Concept

Why You Must Never Train and Test on the Same Data

If you study the answers to a test, you'll score 100%. But you haven't actually learned anything. Same with ML — a model that has "seen" the test data will appear brilliant but fail in the real world.

⚠️
Finance-specific rule: Always split by time, never randomly. If you randomly mix training and test data in a time series, future information leaks into the past — that's look-ahead bias!

Bias vs. Variance — The Core Trade-off

Bias = Underfitting

Model too simple. Like a trader who only looks at price and ignores everything else. Makes systematic errors — even on training data.

Fix: More complex model, more features

Variance = Overfitting

Model too complex. Like a trader who creates a rule for every specific historical day. Works perfectly in backtest, fails live.

Fix: Regularisation, more data, cross-validation

🧪 Experiment #1 — Visualise Overfitting
import numpy as np
import matplotlib.pyplot as plt
from sklearn.preprocessing import PolynomialFeatures
from sklearn.linear_model import LinearRegression
from sklearn.pipeline import make_pipeline
from sklearn.metrics import mean_squared_error

# Simulate noisy stock return data
np.random.seed(42)
X = np.linspace(0, 10, 30).reshape(-1, 1)
y = np.sin(X.ravel()) + np.random.normal(0, 0.3, 30)

# Test data (unseen)
X_test = np.linspace(0, 10, 100).reshape(-1, 1)
y_test = np.sin(X_test.ravel())

fig, axes = plt.subplots(1, 3, figsize=(15, 4))

for i, degree in enumerate([1, 4, 15]):
    model = make_pipeline(PolynomialFeatures(degree), LinearRegression())
    model.fit(X, y)
    y_pred_train = model.predict(X)
    y_pred_test  = model.predict(X_test)
    
    train_mse = mean_squared_error(y, y_pred_train)
    test_mse  = mean_squared_error(y_test, y_pred_test)
    
    axes[i].scatter(X, y, color='#f59e0b', s=20, label='Train data')
    axes[i].plot(X_test, y_pred_test, color=['#00d4ff','#10b981','#ff6b35'][i], lw=2)
    labels = ['Underfitting (degree=1)', 'Good Fit (degree=4)', 'Overfitting (degree=15)']
    axes[i].set_title(f'{labels[i]}\nTrain MSE: {train_mse:.3f} | Test MSE: {test_mse:.3f}')
    axes[i].set_facecolor('#0a0e1a')

plt.tight_layout()
plt.show()

Regression

Predicting continuous numbers — the workhorse of quantitative return forecasting.

Linear Regression — The Foundation

The idea: Find the best straight line through your data. Given a stock's momentum score, what return can we expect? The line gives us our prediction. "Linear" means we assume the relationship between features and target is a straight line (or hyperplane in multiple dimensions).

Simple Linear Regression

One feature → one target. Like predicting next month's return from last month's return.

ŷ = β₀ + β₁x

β₀ = intercept (baseline prediction)
β₁ = coefficient (how much y changes per unit x)
ŷ = predicted value

Example: next_return = 0.001 + 0.15 × momentum

Multiple Linear Regression

Many features → one target. More realistic for finance.

ŷ = β₀ + β₁x₁ + β₂x₂ + ... + βₙxₙ

Example (3 features):
return = 0.001
+ 0.12 × momentum
- 0.08 × volatility
+ 0.05 × value_signal

How Does It "Learn"? Ordinary Least Squares

The Loss Function — What We're Minimising

Linear regression finds the β values that minimise the Mean Squared Error (MSE) — the average of all squared differences between predictions and actuals.

MSE = (1/n) × Σ(yᵢ - ŷᵢ)²

Residual = yᵢ - ŷᵢ (actual minus predicted)
We square it so: positives and negatives don't cancel,
large errors are penalised more

Evaluation Metrics for Regression

MetricFormulaRangeFinance Use
MSEmean((y - ŷ)²)0 to ∞ (lower=better)Training objective, penalises large errors
RMSE√MSE0 to ∞ (lower=better)Same units as target (e.g., return %)
MAEmean(|y - ŷ|)0 to ∞ (lower=better)Less sensitive to outlier returns
R²1 - SS_res/SS_tot-∞ to 1 (higher=better)% variance explained; >0.02 is good in finance!
IC (Info Coef)corr(predicted rank, actual rank)-1 to 1Industry standard for alpha signal quality
💡
Finance reality check: An R² of 0.02 (explaining 2% of variance) can be a profitable trading signal! Unlike physical sciences where R² > 0.9 is expected, financial markets are near-random — any predictability at all is valuable. Focus on the Information Coefficient (IC) instead.

Regularisation — Preventing Overfitting in Regression

The Problem: Too Many Features

With 100 technical indicators as features and only 200 data points, linear regression can overfit dramatically — finding spurious patterns. Regularisation adds a penalty for complexity to the loss function.

Ridge Regression (L2)

Adds the sum of squared coefficients as a penalty. Shrinks all coefficients toward zero but keeps all features. Good when you believe all features contribute a little.

Loss = MSE + λ × Σβᵢ²

λ controls strength (hyperparameter)
Higher λ = more shrinkage

Lasso Regression (L1)

Adds the sum of absolute coefficients as penalty. Can shrink some coefficients to exactly zero — automatic feature selection! Great in finance to identify the most important signals.

Loss = MSE + λ × Σ|βᵢ|

When λ is large enough,
weak features → coefficient = 0

Beyond Linear: Tree-Based Models for Return Prediction

Random Forest Regressor

An ensemble of decision trees. Each tree learns a different "view" of the data. The final prediction is the average of all trees. Handles non-linear relationships automatically — crucial in finance where patterns aren't linear.

Why it's good for finance

  • Captures non-linear interactions (e.g., momentum only works in high-volume regimes)
  • Robust to outliers in returns
  • Gives feature importance scores
  • Resistant to overfitting (due to averaging)

Key hyperparameters

  • n_estimators: number of trees (more = better, slower)
  • max_depth: how deep each tree can grow
  • max_features: features tried at each split
  • min_samples_leaf: minimum samples per leaf

🧪 Colab Experiment: Return Forecasting

🧪 Experiment #2 — Predict Stock Returns
import yfinance as yf
import pandas as pd
import numpy as np
from sklearn.linear_model import LinearRegression, Ridge, Lasso
from sklearn.ensemble import RandomForestRegressor
from sklearn.metrics import mean_squared_error
from scipy.stats import pearsonr
import matplotlib.pyplot as plt

# ── 1. Get data ──────────────────────────────────────────────────────
df = yf.download('SPY', start='2018-01-01', end='2024-01-01')
df.columns = [c.lower() for c in df.columns]

# ── 2. Create features ───────────────────────────────────────────────
df['ret']   = df['close'].pct_change()
df['mom5']  = df['ret'].rolling(5).mean()
df['mom20'] = df['ret'].rolling(20).mean()
df['vol20'] = df['ret'].rolling(20).std()
df['vol_ratio'] = df['volume'] / df['volume'].rolling(20).mean()

# Target: next day return
df['target'] = df['ret'].shift(-1)
df.dropna(inplace=True)

features = ['mom5', 'mom20', 'vol20', 'vol_ratio']
X = df[features].values
y = df['target'].values

# ── 3. Time-based split ──────────────────────────────────────────────
split = int(0.8 * len(X))
X_train, X_test = X[:split], X[split:]
y_train, y_test = y[:split], y[split:]

# ── 4. Train models ──────────────────────────────────────────────────
models = {
    'Linear': LinearRegression(),
    'Ridge':  Ridge(alpha=0.01),
    'Lasso':  Lasso(alpha=0.0001),
    'RandomForest': RandomForestRegressor(n_estimators=100, max_depth=5, random_state=42)
}

print(f"{'Model':<16} {'Test IC':>10} {'Test RMSE':>12}")
print("-" * 42)

for name, m in models.items():
    m.fit(X_train, y_train)
    preds = m.predict(X_test)
    ic = pearsonr(y_test, preds)[0]      # Information Coefficient
    rmse = np.sqrt(mean_squared_error(y_test, preds))
    print(f"{name:<16} {ic:>10.4f} {rmse:>12.6f}")

# ── 5. Feature importance (Random Forest) ────────────────────────────
rf = models['RandomForest']
importance = pd.Series(rf.feature_importances_, index=features).sort_values()
importance.plot(kind='barh', color='#00d4ff', title='Feature Importance')
plt.tight_layout(); plt.show()

Classification

Predicting discrete outcomes — buy/sell/hold, default/no-default, regime detection.

What Is Classification in Finance?

Instead of asking "how much will this stock return?", classification asks "which category does this observation belong to?" In finance, this maps naturally to trading signals: should we be long or short? Will this loan default? Are we in a bull or bear regime?

Binary Classification

Two outcomes: Up/Down, Default/No-Default, Buy/No-Buy. The most common in trading signals.

Multi-Class

Three+ outcomes: Buy/Hold/Sell, Regime (Bull/Bear/Sideways), Credit Rating (A/B/C/D).

Multi-Label

Each sample can belong to multiple classes simultaneously — e.g., a stock can be both "momentum" and "value."

Logistic Regression — The Gateway Classifier

From Regression to Classification

Despite the name, logistic regression is a classifier. It takes the linear regression output and squashes it through a sigmoid function to produce a probability between 0 and 1.

Evaluation Metrics — Beyond Accuracy

⚠️
Accuracy is misleading in finance! If stocks go up 55% of days, a model that always predicts "up" gets 55% accuracy — but it's useless. We need better metrics.

The Confusion Matrix

MetricFormulaFinance Use Case
Accuracy(TP+TN)/(TP+FP+FN+TN)Baseline — use carefully with imbalanced data
PrecisionTP/(TP+FP)"When we say buy, how often are we right?" — controls false signals

F1 Score2×(P×R)/(P+R)Balance of precision and recall

Key Classifiers for Quant Finance

Decision Tree

Creates a tree of if-then-else rules. Extremely interpretable — you can read the rules. A decision tree might learn: "If 5-day momentum > 1.5% AND volatility < 15%, predict UP." Great for understanding the model's reasoning.

Random Forest Classifier

Builds many decision trees with randomness (different subsets of data + features). Final prediction by majority vote. Key benefits:

  • Automatically handles non-linear relationships
  • Feature importance tells you which signals matter
  • Robust to overfitting via bagging
  • Handles missing data and mixed feature types

Gradient Boosting (XGBoost/LightGBM)

Builds trees sequentially, each one correcting the previous one's mistakes. Often best-in-class for tabular financial data:

  • Very high predictive accuracy
  • Built-in regularisation (L1/L2)
  • Handles missing values natively
  • Industry standard in quant competitions

Handling Class Imbalance in Finance

📊
Real problem: In credit default prediction, maybe only 2% of loans default. A model that predicts "no default" for everything gets 98% accuracy but is completely useless. Solutions include: SMOTE (synthetic oversampling), class_weight='balanced', adjusting prediction threshold, and using AUC-ROC instead of accuracy.

import yfinance as yf
import pandas as pd
import numpy as np
from sklearn.ensemble import RandomForestClassifier
from xgboost import XGBClassifier
from sklearn.metrics import classification_report, roc_auc_score
import matplotlib.pyplot as plt

# ── Data & Features ──────────────────────────────────────────────────
df = yf.download('SPY', start='2015-01-01', end='2024-01-01')
df.columns = [c.lower() for c in df.columns]
df['ret'] = df['close'].pct_change()

# Momentum features
for n in [5, 10, 20]:
    df[f'mom{n}'] = df['ret'].rolling(n).mean()
    df[f'vol{n}'] = df['ret'].rolling(n).std()

df['vol_ratio'] = df['volume'] / df['volume'].rolling(20).mean()

# Target: 1 if next day UP, 0 if DOWN
df['target'] = (df['ret'].shift(-1) > 0).astype(int)
df.dropna(inplace=True)

features = ['mom5','mom10','mom20','vol5','vol10','vol20','vol_ratio']
X, y = df[features].values, df['target'].values

split = int(0.8 * len(X))
X_tr, X_te = X[:split], X[split:]
y_tr, y_te = y[:split], y[split:]

# ── Train classifiers ────────────────────────────────────────────────
rf = RandomForestClassifier(n_estimators=200, max_depth=5, random_state=42)
xgb = XGBClassifier(n_estimators=200, max_depth=4, learning_rate=0.05,
                    use_label_encoder=False, eval_metric='logloss', random_state=42)

rf.fit(X_tr, y_tr)
xgb.fit(X_tr, y_tr)

for name, m in [('RandomForest', rf), ('XGBoost', xgb)]:
    preds = m.predict(X_te)
    probs = m.predict_proba(X_te)[:,1]
    print(f"\n── {name} ──")
    print(classification_report(y_te, preds, target_names=['DOWN','UP']))
    print(f"ROC-AUC: {roc_auc_score(y_te, probs):.4f}")

# ── Simple long/short backtest ────────────────────────────────────────
test_dates = df.index[split:]
test_ret   = df['ret'].iloc[split:].values

rf_signal  = np.where(rf.predict(X_te) == 1, 1, -1)  # +1 = long, -1 = short
strat_ret  = rf_signal * test_ret

cumulative = (pd.Series(strat_ret, index=test_dates) + 1).cumprod()
benchmark  = (pd.Series(test_ret, index=test_dates) + 1).cumprod()

plt.figure(figsize=(12, 4))
cumulative.plot(label='ML Strategy', color='#00d4ff')
benchmark.plot(label='Buy & Hold', color='#f59e0b')
plt.title('ML Signal vs Buy & Hold'); plt.legend(); plt.show()

Feature Engineering

Turning raw market data into meaningful signals — where domain expertise and ML intersect.

💡
"In God we trust, all others bring data — and better features." Feature engineering is the most impactful thing you can do for a quant ML model. A simple logistic regression with great features will beat a deep neural network with raw data every time in finance.

What Is a Feature?

A feature (also called a factor or signal in finance) is any measurable quantity that might help predict your target. Good features are:

Predictive

Actually correlates with future returns (not just past returns). Has an economic reason to work.

Non-redundant

Adds information not already captured by other features. Avoid using both "5-day return" and "5-day cumulative return" — they're the same thing.

Stable

Works across different market regimes and time periods, not just during a specific historical window.

The Feature Engineering Pipeline

Raw Data Collection

OHLCV (Open, High, Low, Close, Volume), fundamental data (earnings, PE ratios), alternative data (sentiment, satellite), macroeconomic data (rates, GDP).

Stationarity Transformation

Raw prices are non-stationary (they trend). Convert to returns, log-returns, or z-scores. Stationary features are required for most ML models to work correctly.

Signal Construction

Build momentum, mean-reversion, volatility, volume, and fundamental features from the transformed data.

Normalisation / Scaling

Scale features to a common range. Critical for linear models and gradient-based algorithms. Use rolling z-score for time series to prevent look-ahead bias.

Feature Selection

Remove noisy, redundant, or irrelevant features. Use correlation analysis, Lasso, or tree-based importance scores.

The Major Feature Categories in Finance

📈 Momentum Features

Momentum is the tendency for recent winners to keep winning. One of the most robust factors in finance.

# Return-based momentum
df['mom_1m']  = df['close'].pct_change(21)    # 1-month return
df['mom_3m']  = df['close'].pct_change(63)    # 3-month return
df['mom_12m'] = df['close'].pct_change(252)   # 12-month return

# Fama-French style: skip-1-month (avoids short-term reversal)
df['mom_ff']  = df['close'].pct_change(252) / df['close'].pct_change(21)

# Moving average crossover signal
df['ma_cross'] = df['close'].rolling(20).mean() / df['close'].rolling(50).mean() - 1

# RSI (Relative Strength Index)
delta = df['close'].diff()
gain  = delta.clip(lower=0).rolling(14).mean()
loss  = (-delta).clip(lower=0).rolling(14).mean()
df['rsi'] = 100 - (100 / (1 + gain / loss))

🌊 Volatility Features

    mean = series.rolling(window).mean()
    std  = series.rolling(window).std()
    return (series - mean) / std

Critical: The Look-Ahead Bias Trap

The #1 Mistake in Quant Feature Engineering

Using information that wouldn't have been available at the time of prediction is called look-ahead bias. It's subtle and destroys live performance.

❌ Bad — Look-ahead bias

# Using today's close to compute "today's" feature
# — but today's close isn't known until EOD
df['mom'] = df['close'].pct_change(5)
df['target'] = df['ret'].shift(-1)
# If you use same-day close for feature AND
# shift(-1) for target, you're using the
# closing price in two conflicting ways!

✅ Good — Proper alignment

# Features computed from data up to time t
# Target is return from t to t+1
# Predict at market open using prev day data
df['mom'] = df['close'].pct_change(5).shift(1)
df['vol'] = df['ret'].rolling(20).std().shift(1)
df['target'] = df['ret'].shift(-1)
# shift(1) delays features by 1 day — safe!

Standard Scaling

Subtract mean, divide by std. Good for normally distributed features. Use rolling version for time series!

z = (x - μ) / σ

# GOOD for time series (rolling)

Feature Selection — Quality over Quantity

Information Coefficient (IC) Based Selection

The IC is the correlation between your predicted feature values and the actual next-period returns. It's the primary metric quant researchers use to assess signal quality before including it in a model.

from scipy.stats import spearmanr
import pandas as pd

def compute_ic(feature_series, forward_return, window=252):
    """Rolling Information Coefficient"""
    combined = pd.DataFrame({
        'feature': feature_series,
        'return': forward_return
    }).dropna()
    
    def rolling_ic(x):
        return spearmanr(x['feature'], x['return'])[0]
    
    return combined.rolling(window).apply(rolling_ic, raw=False)

# Keep features with |IC| > 0.02 and t-stat > 2
# IC > 0.05 is excellent in practice
IC RangeSignal QualityAction
|IC| < 0.02Weak / noiseDiscard
0.02 – 0.05MarginalUse with caution, combine with others
0.05 – 0.10GoodInclude in model
> 0.10Excellent (rare)Prioritise — check for errors first!

Purged Cross-Validation

The right way to validate models on financial time series — avoiding the serial correlation trap.

Why Standard Cross-Validation Fails in Finance

The Serial Correlation Problem

Standard k-fold cross-validation randomly shuffles data and splits it into folds. This works for IID (independent and identically distributed) data like image classification. Financial time series data is NOT IID.

Why this is dangerous

  • Adjacent data points are correlated (Monday's return affects Tuesday's)
  • Features like "20-day momentum" overlap across time — today's feature shares 19 days with yesterday's
  • If fold 3 (e.g., March 2020) is adjacent to fold 2 (Feb 2020), the model has "seen" the volatility regime in advance
  • Results in massively optimistic out-of-sample metrics

Walk-Forward Cross-Validation

The simplest fix: always train on the past and test on the future. Split the timeline into rolling windows. This mirrors how a strategy would actually be deployed.

The Purged Cross-Validation Framework

The Problem Walk-Forward Doesn't Fully Solve: Label Leakage

Even with walk-forward validation, there's a subtle issue: if your labels (forward returns) overlap with training data, information leaks. Example:

  • You're predicting 5-day forward return
  • Training ends on Day 100
  • Test starts on Day 101
  • But the label for Day 98 uses price on Day 103! Day 103 is in the test set but was used to create a training label.

This is called label overlap and causes information leakage across the train/test boundary.

Purged K-Fold: The Solution (López de Prado)

Developed by Marcos López de Prado in "Advances in Financial Machine Learning," purged cross-validation removes (purges) any training observations whose labels overlap with the test set's time range. An optional embargo period is also added after each test fold to prevent leakage from serial correlations in features.

Implementing Purged CV from Scratch

import numpy as np
import pandas as pd
from sklearn.model_selection import KFold

class PurgedKFold:
    """
    Purged K-Fold Cross-Validation for financial time series.
    Based on López de Prado (2018) Advances in Financial ML.
    
    Removes training samples whose labels overlap with test fold.
    """
    
    def __init__(self, n_splits=5, purge_gap=0, embargo_pct=0.01):
        self.n_splits   = n_splits
        self.purge_gap  = purge_gap    # days to purge before test fold
        self.embargo_pct = embargo_pct # % of total data as embargo
    
    def split(self, X, y=None, groups=None):
        """
        X must be a DataFrame with DatetimeIndex.
        Returns (train_idx, test_idx) for each fold.
        """
        n = len(X)
        embargo_size = int(n * self.embargo_pct)
        
        # Create k evenly-spaced folds along timeline
        fold_boundaries = np.array_split(np.arange(n), self.n_splits)
        
        for fold_idx in range(self.n_splits):
            test_idx   = fold_boundaries[fold_idx]
            test_start = test_idx[0]
            test_end   = test_idx[-1]
            
            # Embargo: exclude data immediately after test fold
            embargo_end = min(n - 1, test_end + embargo_size)
            
            # Train: everything outside [test_start - purge_gap, embargo_end]
            train_idx = np.concatenate([
                np.arange(0, max(0, test_start - self.purge_gap)),
                np.arange(embargo_end + 1, n)
            ])
            
            if len(train_idx) > 0 and len(test_idx) > 0:
                yield train_idx.astype(int), test_idx.astype(int)

# ── Usage ─────────────────────────────────────────────────────────────
cv = PurgedKFold(n_splits=5, purge_gap=5, embargo_pct=0.01)

scores = []
for train_idx, test_idx in cv.split(X_df):
    X_tr, X_te = X_df.iloc[train_idx], X_df.iloc[test_idx]
    y_tr, y_te = y_series.iloc[train_idx], y_series.iloc[test_idx]
    
    model.fit(X_tr, y_tr)
    score = model.score(X_te, y_te)
    scores.append(score)

print(f"Purged CV Score: {np.mean(scores):.4f} ± {np.std(scores):.4f}")

Combinatorial Purged Cross-Validation (CPCV)

Going Further: When You Have Limited Data

Regular purged CV can only test on each time period once — wasting backtesting information. CPCV, also from López de Prado, generates multiple train/test splits from all possible combinations of folds, providing a more robust estimate of performance distribution. It allows estimation of strategy performance path (not just point estimate), enabling Sharpe ratio confidence intervals.

CPCV(N, k): N = total folds, k = number of test folds
Number of paths = C(N, k) = N! / (k! × (N-k)!)

Example: N=6, k=2 → C(6,2) = 15 paths

Putting It All Together

Build a complete ML-powered trading strategy from raw data to backtest — the full quant pipeline.

The Complete Quant ML Pipeline

Data Collection & Quality Check

Download price/volume data. Check for: gaps, splits, dividend adjustments, survivorship bias. Use adjusted close prices. For cross-section strategies, collect a consistent universe (e.g., S&P 500 constituents over time, not just current members).

Label Construction

Define your prediction target carefully. In finance, instead of just "next-day return", consider: (a) triple barrier method (label +1 if upper barrier hit first, -1 if lower hit first, 0 if time expires), (b) meta-labeling (label 1 if a side-bet on an existing strategy would have been profitable).

Feature Engineering (from Tab 4)

Build momentum, volatility, volume, and cross-sectional features. Always shift by at least 1 to avoid look-ahead. Winsorise outliers. Use rolling z-scores for normalisation.

Feature Selection via IC Analysis

Compute IC for every feature. Keep only features with |IC| > 0.02 and p-value < 0.05 (over training period). Cluster correlated features and keep one per cluster to avoid redundancy.

Model Training with Purged CV

Use purged k-fold CV for hyperparameter tuning. Sample weights proportional to label uniqueness (avoid giving equal weight to overlapping samples). Use class_weight='balanced' for imbalanced labels.

Position Sizing

Convert model output (probability or regression score) to position size. Scale position proportionally to signal strength AND inversely to volatility (Kelly criterion, volatility targeting). Never go all-in on a signal.

Backtest & Performance Attribution

Run walk-forward backtest. Compute: Sharpe ratio, Maximum Drawdown, Calmar ratio (return/MaxDD), Hit Rate, Profit Factor. Decompose performance by year, market regime, and feature contribution.

TEST_MONTHS = 6
EMBARGO_DAYS = 5

PitfallWhat HappensPrevention
Look-ahead biasUnrealistically high backtest returns; strategy fails liveAlways shift features, use only lagged data
Survivorship biasOverestimate performance — you only test on stocks that survivedUse point-in-time universe (companies that existed then, not now)
