Prerequisites

// FOUNDATION LAYER

Before diving into portfolio theory, you need to be comfortable with these foundational math concepts. Don't skip this — they appear everywhere. Each concept is explained from scratch.

1. Summation Notation (Σ)

The Greek letter Sigma (Σ) is just a shorthand for "add things up." Instead of writing:

x₁ + x₂ + x₃ + x₄ + x₅

We write:

Σ(i=1 to 5) xᵢ
💡 Real World Analogy
Think of Σ as a FOR loop in Python: sum([x1, x2, x3, x4, x5]). You're iterating through items and adding them up. Finance is full of sums — summing returns, summing weighted positions, summing errors.
Python 
# Summation in Python
returns = [0.05, 0.02, -0.03, 0.07, 0.01]

# Manual sum
total = 0
for r in returns:
    total += r

# Or simply:
total = sum(returns)  # = 0.12
print(total)

2. Logarithms (log)

A logarithm answers the question: "What power do I raise the base to, to get this number?"

log₁₀(100) = 2 because 10² = 100

In finance we use the natural logarithm (ln, base e ≈ 2.718). The key property we need:

ln(A × B) = ln(A) + ln(B)
💡 Why logs matter in finance
When a stock price goes from $100 to $110 to $121, it grew by the same percentage each step (10%). Log returns let us add these percentage changes instead of multiplying them. This makes math much easier and is used in almost every financial model.

3. Weighted Averages

A plain average treats all items equally. A weighted average lets some items matter more:

Weighted Mean = Σ(wᵢ × xᵢ) where Σwᵢ = 1
💡 Portfolio Analogy
If you invest 70% in Stock A (return 5%) and 30% in Stock B (return 10%), your portfolio return is: 0.70 × 5% + 0.30 × 10% = 3.5% + 3% = 6.5%. This is a weighted average — used constantly in portfolio theory.

4. Variance vs. Standard Deviation

These measure how "spread out" data is. Variance is the average of squared differences from the mean. Standard deviation is its square root (more interpretable because it's in the same units as the data).

Variance = Σ(xᵢ - mean)² / n
Std Dev (σ) = √Variance

5. Basic Probability

Probability measures likelihood on a scale of 0 (impossible) to 1 (certain). In finance, we use probability to model how likely different returns are.

ConceptDefinitionFinance Example
P(event)Probability of eventP(stock drops 5%) = 0.10
Expected valueΣ P(x) × xExpected daily return
PercentileValue below which X% of data falls5th percentile → used in VaR

Statistics Core

// PREREQUISITE — READ BEFORE TABS 3-10

Statistical concepts are the DNA of every risk model. This tab gives you everything you need before touching any financial formula.

Mean (Expected Value)

The mean is the "center of gravity" of a dataset. In finance, the mean of returns is the expected return — our best guess of what the investment will produce.

μ = (1/n) × Σ xᵢ
⚠️ Common Pitfall
A high expected return alone tells you nothing useful. A stock that returns on average +10% but swings wildly between +80% and -60% is very different from one that steadily returns +10%. You need to measure the variability too.

Variance & Standard Deviation

Variance measures how far returns are from their average, on average. Standard deviation (σ) is the square root — it gives us "typical swing" in the same units as returns.

σ² = (1/n) Σ (xᵢ - μ)²
σ = √σ²
// Analogy

Imagine two archers. Archer A hits exactly the bullseye every shot. Archer B's shots are scattered wildly — sometimes very high, sometimes very low, but averaging the bullseye. They have the same mean but very different standard deviations. Standard deviation in finance = how scattered your outcomes are.

Covariance & Correlation

Covariance tells us whether two assets tend to move together (positive covariance) or in opposite directions (negative covariance). Correlation normalizes this into a [-1, 1] range.

Cov(A,B) = (1/n) Σ (Aᵢ - μ_A)(Bᵢ - μ_B)
ρ(A,B) = Cov(A,B) / (σ_A × σ_B)
// Correlation Spectrum

The Normal Distribution (Bell Curve)

The normal distribution describes data that clusters around a mean with symmetric tails. It's defined entirely by two parameters: mean (μ) and standard deviation (σ).

// Normal Distribution — The Foundation of VaR
📐 The 68-95-99.7 Rule (Empirical Rule)

For any normal distribution:
• 68% of outcomes fall within 1 standard deviation of the mean
• 95% fall within 2 standard deviations
• 99.7% fall within 3 standard deviations

Example: If daily return μ = 0% and σ = 1%, there's a 95% chance the daily return falls between -2% and +2%.

Z-Scores

A Z-score tells you how many standard deviations a value is from the mean. This lets you compare values across different distributions.

Z = (x - μ) / σ

Key Z-scores used in finance:

Confidence LevelZ-ScoreUsed for
90%1.282Less conservative VaR
95%1.645Standard VaR
99%2.326Strict risk limits
99.9%3.090Extreme risk (bank regulation)

# EXPERIMENT: Normal Distribution in Finance
import numpy as np
import matplotlib.pyplot as plt
from scipy import stats
import yfinance as yf

# Download real stock data
ticker = yf.Ticker("SPY")  # S&P 500 ETF
hist = ticker.history(period="2y")

prices = hist['Close']
returns = np.log(prices / prices.shift(1)).dropna()

mu = returns.mean()
sigma = returns.std()

print(f"Mean daily return: {mu:.4f} ({mu*252*100:.1f}% annual)")
print(f"Daily volatility: {sigma:.4f} ({sigma*np.sqrt(252)*100:.1f}% annual)")

# Plot distribution
fig, ax = plt.subplots(1, 1, figsize=(10, 5))
ax.hist(returns, bins=60, density=True, alpha=0.6,
       color='#00d4ff', label='Actual Returns')

# Overlay normal distribution curve
x = np.linspace(returns.min(), returns.max(), 200)
ax.plot(x, stats.norm.pdf(x, mu, sigma), color='#ff6b6b',
       linewidth=2, label='Normal Fit')

ax.axvline(mu - 2*sigma, color='#f0c040', linestyle='--',
          label=f'μ ± 2σ zone')
ax.axvline(mu + 2*sigma, color='#f0c040', linestyle='--')

ax.set_title('SPY Daily Return Distribution', fontsize=14)
ax.legend()
plt.tight_layout()
plt.show()

Returns & Risk

// TOPIC 1 — FOUNDATION

Before measuring risk, we need to understand how we measure gains and losses from investments. This is the fundamental language of finance.

Simple Returns

The simplest measure of gain or loss over one period:

R = (P_end - P_start) / P_start = (P_end / P_start) - 1
💡 Example
You buy a stock at $100. Next day it closes at $103. Simple return = (103 - 100) / 100 = 0.03 = 3%.

Log Returns (Continuously Compounded Returns)

Log returns are the natural logarithm of the price ratio. They are preferred in quantitative finance because they are additive over time and have nicer mathematical properties.

r = ln(P_t / P_{t-1})
// Simple vs Log Returns

Simple returns are what you'd tell a friend: "I made 5% today!"
Log returns are what you use in models: they can be summed across time, and their distribution is closer to normal — which is why every risk formula in this guide uses them.

For small returns (under ±10%), simple ≈ log. For larger moves, they diverge. Always use log returns in quant work.

Why Volatility = Risk?

In finance, risk is uncertainty. Volatility (standard deviation of returns) measures how uncertain future returns are. Higher volatility = wider range of possible outcomes = more risk.

// Low vs High Volatility — Same Mean, Different Risk

Annualizing Volatility

Daily volatility is hard to interpret. We annualize it using the square root of time rule:

σ_annual = σ_daily × √252

Why 252? That's the number of trading days in a year (not 365 — markets are closed on weekends and holidays).

📐 Key Formulas Reference

Simple Return: R = (P_t - P_{t-1}) / P_{t-1}

Log Return: r = ln(P_t / P_{t-1})

Multi-period (log): r_{total} = r_1 + r_2 + ... + r_n (just add them!)

Multi-period (simple): R_{total} = (1+R_1)(1+R_2)...(1+R_n) - 1

Annual Vol: σ_annual = σ_daily × √252

Value at Risk (VaR)

// TOPIC 2

VaR is the most widely used risk metric in finance. Every major bank, hedge fund, and regulator uses it. Once you understand it, you'll see it everywhere.

Requires: Normal Distribution
Requires: Standard Deviation
Requires: Returns & Volatility

What is VaR?

Value at Risk answers one specific question:

📐 The VaR Question

"What is the maximum loss I can expect over the next X days, with Y% confidence?"

Example: "My 1-day 95% VaR is $50,000" means: "There is only a 5% chance I will lose MORE than $50,000 in the next trading day."

// Intuition: The Weather Forecast Analogy

Think of VaR like a weather forecast. A meteorologist might say "There's a 95% chance it won't rain more than 2 inches tomorrow." They're not saying it CAN'T rain more — they're saying the probability of that happening is only 5%.

VaR is the same: "There's a 95% chance our loss won't exceed $X." That remaining 5% — the tail — is what CVaR (the next topic) handles.

The Three VaR Methods

Method 1: Parametric VaR (Variance-Covariance)

Assumes returns follow a normal distribution. Uses mean and standard deviation directly.

VaR = -(μ - Z_α × σ) × Portfolio Value

Where Z_α is the Z-score for confidence level α (e.g., Z = 1.645 for 95% confidence).

// VaR Visualized on Normal Distribution

Method 2: Historical Simulation

No distribution assumption needed. Take actual historical returns, sort them worst to best, and find the return at the Xth percentile.

💡 Example Step-by-Step

You have 500 days of historical returns. For 95% VaR:

1. Sort all 500 returns from worst to best
2. The 5th percentile = 500 × 0.05 = 25th worst day
3. Look at the 25th worst return — that's your 95% VaR
4. If that return was -3.2%, your VaR is 3.2% of portfolio value

Method 3: Monte Carlo Simulation

Simulate thousands of possible future scenarios using random number generation based on statistical parameters. Take the percentile of simulated outcomes.

Google Colab — All 3 VaR Methods 
import numpy as np
import pandas as pd
import yfinance as yf
import matplotlib.pyplot as plt
from scipy import stats

# ── Data ──────────────────────────────────────
data = yf.download("SPY", start="2020-01-01", end="2025-01-01")
returns = np.log(data['Close'] / data['Close'].shift(1)).dropna()

portfolio_value = 1_000_000  # $1 million
confidence = 0.95
alpha = 1 - confidence  # 0.05

mu = returns.mean()
sigma = returns.std()

# ── Method 1: Parametric VaR ──────────────────
z = stats.norm.ppf(alpha)  # Z-score for 5th percentile
parametric_var = -(mu + z * sigma) * portfolio_value
print(f"Parametric VaR (95%): ${parametric_var:,.0f}")

# ── Method 2: Historical Simulation ───────────
historical_var = -np.percentile(returns, alpha*100) * portfolio_value
print(f"Historical VaR (95%): ${historical_var:,.0f}")

# ── Method 3: Monte Carlo ─────────────────────
np.random.seed(42)
simulated = np.random.normal(mu, sigma, 100_000)
monte_carlo_var = -np.percentile(simulated, alpha*100) * portfolio_value
print(f"Monte Carlo VaR (95%): ${monte_carlo_var:,.0f}")

# ── Visualization ─────────────────────────────
fig, ax = plt.subplots(figsize=(11, 5))
ax.hist(returns, bins=80, density=True, color='#00d4ff',
       alpha=0.5, label='SPY Daily Returns')
ax.axvline(-parametric_var/portfolio_value, color='#ff6b6b',
          lw=2, label=f'Parametric VaR (${parametric_var:,.0f})')
ax.axvline(-historical_var/portfolio_value, color='#f0c040',
          lw=2, linestyle='--', label=f'Historical VaR')
ax.axvline(-monte_carlo_var/portfolio_value, color='#00e5a0',
          lw=2, linestyle=':', label=f'Monte Carlo VaR')
ax.legend()
ax.set_title('SPY — Three VaR Methods Compared ($1M Portfolio)')
plt.show()

VaR Limitations

⚠️ Critical Flaws to Know

1. VaR says nothing about how bad "bad" gets. If VaR is $50K, it only says losses beyond $50K happen 5% of the time. It doesn't say whether those losses are $51K or $5 million.

2. Not sub-additive. Portfolio VaR can exceed the sum of individual VaRs under certain conditions — violating intuition.

3. Normal distribution assumption is wrong. Real returns have "fat tails" — extreme events happen more than normal distribution predicts. This is why CVaR (next tab) was invented.

4. It's backward-looking. Historical data doesn't predict future crises.

CVaR — Conditional Value at Risk

// TOPIC 3 — ALSO CALLED: Expected Shortfall (ES)

CVaR fixes VaR's biggest flaw. While VaR says "how bad can it get at the 5% threshold," CVaR asks "if things do go into that 5% zone, how bad is it on average?"

Requires: VaR (Tab 04)

The Core Problem with VaR

// The Insurance Analogy

Imagine two insurance policies. Both have a 95% VaR of $10,000 (meaning 5% of the time, losses exceed $10K).

Policy A: In the bad 5%, losses are always exactly $10,001.
Policy B: In the bad 5%, losses range from $10,001 to $1,000,000.

VaR says they're identical. CVaR would immediately show Policy B is far riskier. CVaR = average loss in that 5% tail.

CVaR Formula

CVaR_α = E[Loss | Loss > VaR_α]

Read as: "The expected (average) loss, given that the loss is worse than VaR."

For a normal distribution, CVaR has a clean formula:

CVaR_α = μ + σ × φ(Z_α) / α

Where φ(Z_α) is the standard normal PDF at the Z-score, and α is the tail probability (e.g., 0.05).

// VaR vs CVaR on the Loss Distribution

Why CVaR is Better (Coherent Risk Measure)

✅
Sub-additive
Portfolio CVaR ≤ sum of individual CVaRs. Diversification always helps. VaR doesn't guarantee this.
✅
Tells you the worst
Quantifies the average loss in bad scenarios — critical for stress testing and portfolio insurance.
✅
Convex optimization
CVaR can be minimized directly using linear programming — making it practical for portfolio optimization.
Google Colab — VaR vs CVaR Comparison 
import numpy as np
import pandas as pd
import yfinance as yf
import matplotlib.pyplot as plt
from scipy import stats

data = yf.download("SPY", start="2018-01-01", end="2025-01-01")
returns = np.log(data['Close'] / data['Close'].shift(1)).dropna()

portfolio = 1_000_000
alpha = 0.05

# ── Historical VaR ─────────────────────────────
var_95 = -np.percentile(returns, alpha * 100)

# ── Historical CVaR (Expected Shortfall) ───────
# Average of all returns WORSE than VaR
tail_returns = returns[returns < -var_95]
cvar_95 = -tail_returns.mean()

print(f"=== $1M SPY Portfolio ===")
print(f"95% Historical VaR:  ${var_95 * portfolio:,.0f}")
print(f"95% Historical CVaR: ${cvar_95 * portfolio:,.0f}")
print(f"CVaR is {cvar_95/var_95:.1f}x worse than VaR")
print(f"Number of tail days: {len(tail_returns)}")
print(f"Worst single day: ${returns.min() * portfolio:,.0f}")

# ── Visualization ─────────────────────────────
fig, ax = plt.subplots(figsize=(11, 5))

# Returns histogram
ax.hist(returns, bins=80, density=True, color='#00d4ff',
       alpha=0.4, label='Daily Returns')

# Highlight tail
tail_vals = np.linspace(returns.min(), -var_95, 100)
mu, sigma = returns.mean(), returns.std()
ax.fill_between(tail_vals, stats.norm.pdf(tail_vals, mu, sigma),
                alpha=0.6, color='#ff6b6b', label='5% Tail')

ax.axvline(-var_95, color='#ff6b6b', lw=2,
          label=f'VaR = ${var_95*portfolio:,.0f}')
ax.axvline(-cvar_95, color='#ff3a3a', lw=2, linestyle='--',
          label=f'CVaR = ${cvar_95*portfolio:,.0f}')

ax.set_xlabel('Daily Log Return')
ax.set_title('VaR vs CVaR: SPY 2018–2025')
ax.legend()
plt.tight_layout()
plt.show()

The Sharpe Ratio

// TOPIC 4 — RISK-ADJUSTED PERFORMANCE

The Sharpe Ratio is arguably the most important metric for comparing investments. It measures how much return you earn per unit of risk taken. Created by Nobel Prize winner William Sharpe in 1966.

Requires: Returns & Volatility (Tab 03)
Requires: Standard Deviation

The Core Insight

// The Investment Efficiency Analogy

Imagine two investment strategies:

Strategy A: Returns 12% per year with 20% volatility
Strategy B: Returns 8% per year with 5% volatility

Strategy A looks better (higher return). But is it really? Strategy B is giving you solid, predictable returns with far less risk. The Sharpe Ratio quantifies this trade-off by asking: "How much return am I getting per unit of risk I'm taking?"

If Strategy A has Sharpe = 0.5 and Strategy B has Sharpe = 1.2 — B is the better investment per unit of risk.

The Formula

Sharpe = (R_p - R_f) / σ_p

Where:

SymbolMeaningExample
R_pPortfolio return (annualized)12%
R_fRisk-free rate (e.g., US Treasury)4.5%
R_p - R_fExcess return (the "reward")7.5%
σ_pPortfolio volatility (annualized)15%
SharpeReturn per unit of risk0.50

Why Subtract the Risk-Free Rate?

💡 The Free Lunch Baseline

The risk-free rate (e.g., 3-month US Treasury yield) is what you earn by doing absolutely nothing risky — just lending money to the US government, which essentially can't default in its own currency.

If your stock portfolio returns 5% and the risk-free rate is 5%, you haven't done anything useful — you took on risk but earned no extra return. The Sharpe Ratio measures the excess return per unit of risk, ensuring we only credit returns that exceed this "free" baseline.

// Sharpe Ratio Benchmark Guide

Sharpe Ratio Variants

MetricFormulaWhen to Use
Sharpe(R_p - R_f) / σ_totalGeneral performance comparison
Sortino(R_p - R_f) / σ_downsideWhen you only care about downside risk
CalmarAnnual Return / Max DrawdownFor trading strategies
Information Ratio(R_p - R_benchmark) / σ_trackingActive management vs benchmark
⚠️ Sharpe Ratio Limitations

Assumes symmetry: Penalizes upside and downside equally. A portfolio with big gains would be "punished" for high volatility even if the swings are positive.

Can be gamed: Smooth strategies (like selling options) can have artificially high Sharpe ratios that hide tail risk.

Not comparable across periods: Daily, monthly, and annual Sharpe ratios aren't directly comparable without proper annualization.

Diversification Math

// TOPIC 5 — WHY NOT TO PUT ALL EGGS IN ONE BASKET

Diversification is the only "free lunch" in finance. By combining assets intelligently, you can reduce risk without necessarily reducing return. Let's understand the math behind it.

Requires: Variance & Standard Deviation
Requires: Correlation (Tab 02)

Portfolio Variance Formula (2 Assets)

This is where diversification lives. For two assets A and B with weights w_A and w_B:

σ²_portfolio = w²_A σ²_A + w²_B σ²_B + 2·w_A·w_B·Cov(A,B)

Substituting Cov(A,B) = ρ_{AB} · σ_A · σ_B:

σ²_portfolio = w²_A σ²_A + w²_B σ²_B + 2·w_A·w_B·ρ_{AB}·σ_A·σ_B
💡 The Magic of Low Correlation

Look at the formula. The last term has ρ in it. When ρ is low (or negative), that last term shrinks — meaning portfolio variance is less than the weighted average of individual variances. That's diversification in action.

When ρ = 1: No benefit — portfolio vol = weighted average of individual vols
When ρ = 0: Portfolio vol is meaningfully reduced
When ρ = -1: Perfect diversification — you can theoretically eliminate all risk!

Visualizing Diversification

// How Correlation Affects Portfolio Volatility

N-Asset Portfolio Variance

For a portfolio of N assets with weight vector w and covariance matrix Σ:

σ²_p = wᵀ Σ w

The covariance matrix Σ is an N×N matrix where diagonal elements are variances and off-diagonal elements are covariances. This is the core of Modern Portfolio Theory (Markowitz 1952).

The Efficient Frontier

For any given set of assets, there's a set of "optimal" portfolios that give the highest return for each level of risk. This is the Efficient Frontier.

// The Efficient Frontier — Return vs Risk Space

How Many Assets to Diversify?

For a portfolio of N equal-weight uncorrelated assets, portfolio variance is:

σ²_p = σ² / N

As N increases, idiosyncratic (firm-specific) risk approaches zero. But systematic (market) risk cannot be diversified away — it's the floor.

⚠️ The Diversification Trap

After about 30–40 well-chosen, low-correlation stocks, additional diversification gives diminishing returns. Beyond 50–100 stocks, you're essentially replicating an index — you might as well just buy an index fund.

More importantly: correlation between assets rises during crises. "Diversification fails when you need it most" is a well-known phenomenon — during market crashes, correlations spike toward 1.

import numpy as np
import pandas as pd
import yfinance as yf
import matplotlib.pyplot as plt

# Download data for 5 assets
tickers = ["AAPL", "MSFT", "GLD", "TLT", "SPY"]
data = yf.download(tickers, start="2020-01-01", end="2025-01-01")['Close']
log_ret = np.log(data / data.shift(1)).dropna()

mu = log_ret.mean() * 252          # Annual expected returns
cov = log_ret.cov() * 252           # Annual covariance matrix
rf = 0.045

# Monte Carlo: generate random portfolios
np.random.seed(42)
n_sims = 10_000
n_assets = len(tickers)

port_returns, port_vols, port_sharpes = [], [], []
all_weights = []

for _ in range(n_sims):
    w = np.random.random(n_assets)
    w /= w.sum()                     # Normalize to sum to 1
    all_weights.append(w)

    ret = np.dot(w, mu)             # Portfolio return
    vol = np.sqrt(np.dot(w, np.dot(cov, w)))  # Portfolio vol
    sharpe = (ret - rf) / vol

    port_returns.append(ret)
    port_vols.append(vol)
    port_sharpes.append(sharpe)

port_returns = np.array(port_returns)
port_vols = np.array(port_vols)
port_sharpes = np.array(port_sharpes)

# Find max Sharpe portfolio
best_idx = port_sharpes.argmax()

print("=== Max Sharpe Portfolio ===")
for t, w in zip(tickers, all_weights[best_idx]):
    print(f"  {t}: {w*100:.1f}%")
print(f"  Return: {port_returns[best_idx]*100:.1f}%")
print(f"  Vol: {port_vols[best_idx]*100:.1f}%")
print(f"  Sharpe: {port_sharpes[best_idx]:.2f}")

# Plot efficient frontier
plt.figure(figsize=(10, 6))
sc = plt.scatter(port_vols*100, port_returns*100,
                 c=port_sharpes, cmap='viridis', alpha=0.4, s=8)
plt.colorbar(sc, label='Sharpe Ratio')
plt.scatter(port_vols[best_idx]*100, port_returns[best_idx]*100,
           color='red', s=200, marker='*', zorder=5, label='Max Sharpe')
plt.xlabel('Annual Volatility (%)')
plt.ylabel('Annual Return (%)')
plt.title('Efficient Frontier — 10,000 Random Portfolios')
plt.legend()

CAPM — Capital Asset Pricing Model

// TOPIC 6 — THE MOST FAMOUS MODEL IN FINANCE

CAPM (developed by Sharpe, Lintner, and Mossin in the 1960s) tells us how much return you should expect from a risky asset, given its relationship to the overall market. It introduced the concept of Beta — one of the most-used words in finance.

Requires: Diversification Math (Tab 07)
Requires: Correlation & Covariance

The Big Idea: Systematic vs Idiosyncratic Risk

// The Two Types of Risk

Idiosyncratic Risk (unsystematic): Risk specific to one company. Apple might have a bad product launch. This risk can be diversified away by holding many stocks.

Systematic Risk (market risk): Risk that affects all stocks — recessions, interest rate changes, wars. No amount of diversification eliminates this.

CAPM says: Investors are only rewarded for taking systematic risk. Why? Because idiosyncratic risk is free to eliminate through diversification, so the market won't pay you for bearing unnecessary risk.

CAPM Formula

E[R_i] = R_f + β_i × (E[R_m] - R_f)
SymbolMeaningTypical Value
E[R_i]Expected return of asset iWhat we're solving for
R_fRisk-free rate~4–5% (current US Treasury)
β_iBeta of asset i0 to 2+ (see below)
E[R_m]Expected market return~8–10% historically
E[R_m] - R_fMarket Risk Premium (MRP)~5–6%

Beta (β) — The Key Parameter

β_i = Cov(R_i, R_m) / Var(R_m)
// Beta Interpretation Guide

The Security Market Line (SML)

CAPM defines a linear relationship between Beta and expected return. This line is the Security Market Line (SML). Every asset should lie on this line if CAPM holds perfectly.

💡 Alpha — Deviation from CAPM

In practice, stocks don't perfectly sit on the SML. The deviation is called Alpha (α):

α = Actual Return - CAPM Expected Return

Positive alpha = outperformed CAPM → stock delivered more than its systematic risk predicted.
Negative alpha = underperformed CAPM → bad risk-adjusted performance.
Finding positive alpha is literally what active portfolio managers are paid to do.

import numpy as np

from scipy import stats

# Market proxy: SPY; assets to analyze
tickers = ["AAPL", "TSLA", "WMT", "GLD"]
market = "SPY"

all_tickers = tickers + [market]
data = yf.download(all_tickers, start="2021-01-01", end="2025-01-01")['Close']
ret = np.log(data / data.shift(1)).dropna()

market_excess = ret[market] - rf  # Excess market returns

print("="*55)
print(f"{'Asset':8} {'Beta':8} {'Alpha(ann)':12} {'R²':8} {'p-val β'}")
print("="*55)

axes = axes.flatten()

    # OLS regression: R_i - Rf = α + β(R_m - Rf)

        market_excess, asset_excess)

    alpha_annual = intercept * 252
    r_sq = r_val**2

    print(f"{ticker:8} {beta:8.2f} {alpha_annual*100:10.2f}% {r_sq:8.3f} {p_val:.4f}")

    ax = axes[i]
    ax.scatter(market_excess*100, asset_excess*100,
               alpha=0.3, s=8, color='#00d4ff')
    x_range = np.array([market_excess.min(), market_excess.max()]) * 100
    ax.plot(x_range, slope * x_range/(100) * 100 + intercept*100,
           color='#ff6b6b', lw=2)
    ax.set_title(f"{ticker} | β={beta:.2f}, α={alpha_annual*100:.1f}%/yr, R²={r_sq:.2f}")
    ax.set_xlabel('Market Excess Return (%)')
    ax.set_ylabel(f'{ticker} Excess Return (%)')

plt.tight_layout()

Factor Models

// TOPIC 7 — BEYOND CAPM

CAPM uses one factor (market beta) to explain returns. But decades of research showed this isn't enough. Factor models use multiple systematic drivers to better explain why stocks earn the returns they do.

Requires: CAPM & Beta (Tab 08)
Requires: Regression concepts

Why CAPM Fails (Empirically)

In the 1990s, Eugene Fama and Kenneth French documented that CAPM left systematic patterns unexplained:

📐
Size Effect (SMB)
Small-cap stocks historically outperform large-cap stocks over time, even after adjusting for beta. CAPM cannot explain this.
💰
Value Effect (HML)
Value stocks (high book-to-market ratio) outperform growth stocks historically. CAPM says no — it should only care about beta.
📈
Momentum (MOM)
Stocks that performed well in the past 6–12 months tend to continue outperforming. This is a fourth factor added by Carhart.

Fama-French 3-Factor Model

R_i - R_f = α + β_mkt(R_m - R_f) + β_smb(SMB) + β_hml(HML) + ε
FactorWhat it capturesHow constructed
MKT (Market)Overall market risk (same as CAPM beta)R_market - R_f
SMB (Small Minus Big)Size premiumReturn of small stocks minus return of large stocks
HML (High Minus Low)Value premiumReturn of high book/market minus low book/market stocks

Carhart 4-Factor Model

R_i - R_f = α + β_mkt·MKT + β_smb·SMB + β_hml·HML + β_mom·MOM + ε

Fama-French 5-Factor Model (2015)

R_i - R_f = α + β_mkt·MKT + β_smb·SMB + β_hml·HML + β_rmw·RMW + β_cma·CMA + ε

Adding RMW (Robust Minus Weak, profitability factor) and CMA (Conservative Minus Aggressive, investment factor).

// The Recipe Analogy for Factor Models

Imagine the return of any stock is like a recipe made from standard ingredients. Instead of buying each exotic ingredient separately, you buy "factor ETFs" that provide exposure to each ingredient.

A tech growth stock might have: heavy market exposure (β_mkt = 1.4), negative value loading (β_hml = -0.5), positive momentum (β_mom = 0.3), small size (β_smb = 0.2). The factor model lets you decompose exactly what drives that stock's returns.

import statsmodels.api as sm
import matplotlib.pyplot as plt
from io import BytesIO
import zipfile
import requests

# ── Download Fama-French factors from Ken French's website ──
url = ("https://mba.tuck.dartmouth.edu/pages/faculty/ken.french"
       "/ftp/F-F_Research_Data_Factors_daily_CSV.zip")
response = requests.get(url)
z = zipfile.ZipFile(BytesIO(response.content))
fname = [f for f in z.namelist() if '.CSV' in f][0]

with z.open(fname) as f:
    ff = pd.read_csv(f, skiprows=3, index_col=0)

# Clean up
ff = ff[ff.index.astype(str).str.match(r'^\d{8}$')]
ff.index = pd.to_datetime(ff.index, format='%Y%m%d')
ff = ff.astype(float) / 100  # Convert from % to decimal
ff = ff["2020-01-01":"2024-12-31"]
ff.rename(columns={'Mkt-RF': 'MKT'}, inplace=True)

aapl = yf.download("AAPL", start="2020-01-01", end="2025-01-01",

df = pd.concat([aapl_ret, ff], axis=1).dropna()

df['AAPL_excess'] = df['AAPL'] - df['RF']

# ── CAPM regression (1 factor) ─────────────────
X1 = sm.add_constant(df['MKT'])

ff3 = sm.OLS(df['AAPL_excess'], X3).fit()

print("=== CAPM Results for AAPL ===")
print(capm.summary().tables[1])

print("\n=== Fama-French 3-Factor Results for AAPL ===")
print(ff3.summary().tables[1])

print(f"\nCAPM R² = {capm.rsquared:.3f}")
print(f"FF3 R²  = {ff3.rsquared:.3f}")
print("A higher R² means more return is explained by the model")

Here's the complete workflow a quant engineer follows:

1

Universe Selection

Choose your investable universe (e.g., S&P 500 stocks). Filter by liquidity, market cap, data availability.

2

Return & Risk Estimation

Compute log returns. Estimate μ (expected returns) via CAPM or factor models. Estimate Σ (covariance matrix) — often with shrinkage to improve stability.

3

Risk Assessment

Compute individual VaR, CVaR for each asset. Compute correlation structure. Identify concentration risks.

4

Portfolio Optimization

Maximize Sharpe Ratio (or minimize CVaR) subject to: position limits, sector limits, VaR limits, turnover constraints.

5

Factor Analysis

Decompose portfolio into factor exposures using FF model. Ensure no unintended factor tilts. Hedge unwanted exposures if needed.

6

Backtesting & Monitoring

Backtest strategy on historical data. Compute realized Sharpe, max drawdown, Calmar ratio. Set up live risk monitoring with daily VaR/CVaR reporting.
