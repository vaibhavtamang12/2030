Before diving into probability and statistics, make sure you're comfortable with these building blocks. Don't skip this section — the notation and concepts here appear everywhere in the rest of the guide.

1.1 Sets & Set Notation

A set is just a collection of distinct objects. Think of it as a bag of unique items.

📦 Intuition
Imagine you have a bag of outcomes from rolling a die: {1, 2, 3, 4, 5, 6}. This is a set. Sets are the foundation of probability spaces.

1.2 Sigma (Σ) Notation

Σ (capital sigma) is shorthand for "add up a sequence of terms". You'll see it constantly.

Sigma Notation Σᵢ₌₁ⁿ xᵢ = x₁ + x₂ + x₃ + ... + xₙ Example: Σᵢ₌₁⁵ i = 1 + 2 + 3 + 4 + 5 = 15

Similarly, Π (capital pi) means multiply:

Product Notation Πᵢ₌₁ⁿ xᵢ = x₁ × x₂ × x₃ × ... × xₙ

1.3 Basic Calculus You'll Need

We need two calculus concepts: derivatives (rates of change) and integrals (areas under curves). Here's the minimum you need:

Derivative — rate of change d/dx [xⁿ] = n·xⁿ⁻¹ d/dx [eˣ] = eˣ d/dx [ln(x)] = 1/x Integral — area under curve ∫ xⁿ dx = xⁿ⁺¹/(n+1) + C ∫₋∞^∞ f(x) dx = 1 (for any probability density)
🎯 Why integrals matter in stats
For continuous random variables, probability IS an integral. The probability that X falls between a and b is the area under the curve from a to b: P(a ≤ X ≤ b) = ∫ₐᵇ f(x) dx.

1.4 Logarithms & Exponentials

Log and exp appear in almost every formula in this guide. Here are the rules you must know:

Log Rules ln(a·b) = ln(a) + ln(b) ← Products become sums ln(a/b) = ln(a) - ln(b) ln(aᵏ) = k·ln(a) ln(eˣ) = x and e^(ln x) = x
⭐ Most important log property for quants
ln(a·b·c) = ln(a) + ln(b) + ln(c)

This converts products into sums, which is why MLE uses log-likelihood — products of small probabilities become manageable sums!

Probability Foundations

Probability is the formal language of uncertainty. Before understanding distributions or hypothesis tests, you must speak this language fluently.

2.1 The Three Axioms of Probability

All of probability theory rests on three simple rules (Kolmogorov axioms):

Kolmogorov Axioms Axiom 1: P(A) ≥ 0 (probabilities are non-negative) Axiom 2: P(Ω) = 1 (something must happen) Axiom 3: P(A ∪ B) = P(A) + P(B) if A ∩ B = ∅ (additivity)
🎲 Real-world analogy
Think of a weather forecast. "30% chance of rain" means if you simulated tomorrow 1000 times, about 300 would have rain. Probability measures the long-run frequency of outcomes.

Useful derived rules

Rules derived from axioms P(Aᶜ) = 1 - P(A) (complement) P(A ∪ B) = P(A) + P(B) - P(A ∩ B) (inclusion-exclusion) P(∅) = 0

2.2 Conditional Probability

Key insight: When you learn that event B happened, it changes the probability of A. This is called conditional probability.

Conditional Probability P(A | B) = P(A ∩ B) / P(B) (read: "probability of A given B")
📊 Quant context
P(stock rises | earnings beat) — the probability a stock rises given that earnings beat expectations. Conditioning on information is the core of quantitative finance.

2.3 Independence

Two events A and B are independent if knowing B happened gives you zero information about A.

Independence A ⊥ B ⟺ P(A ∩ B) = P(A) · P(B) ⟺ P(A | B) = P(A)

2.4 Bayes' Theorem (Intro)

Bayes' theorem lets you reverse conditional probabilities. Given P(B|A), find P(A|B).

Bayes' Theorem P(A | B) = P(B | A) · P(A) ───────────────── P(B) where: P(B) = Σᵢ P(B|Aᵢ)·P(Aᵢ) (law of total probability)
⭐ Classic example: Medical Test
Disease prevalence: 1%. Test accuracy: 99%. You test positive. What's P(you have disease)?

P(disease|+) = P(+|disease)·P(disease) / P(+)
= 0.99 × 0.01 / (0.99×0.01 + 0.01×0.99)
= 0.0099 / 0.0198 ≈ 50%!

This surprises everyone. Even with a 99% accurate test, a rare disease means a positive test is only 50% reliable. This is why we'll spend a whole tab on Bayesian reasoning.

2.5 Random Variables

A random variable (RV) X is a function that maps outcomes of a random experiment to numbers. There are two types:

TypeValuesDescribed byExample
Discrete RVCountable (0,1,2,…)PMF: P(X=k)Number of heads in 10 flips
Continuous RVAny real intervalPDF: f(x), CDF: F(x)Stock return tomorrow
CDF — Cumulative Distribution Function F(x) = P(X ≤ x) Properties: · F is non-decreasing · lim F(x) = 0 as x→-∞ · lim F(x) = 1 as x→+∞ · For continuous: f(x) = dF/dx (PDF is derivative of CDF)

Probability Distributions

A distribution is a complete description of a random variable — it tells us all possible values and their probabilities. Think of it as the "personality profile" of randomness.

3.1 The Normal (Gaussian) Distribution

The most important distribution in statistics. Bell-shaped, symmetric, and appears everywhere due to the Central Limit Theorem (next section!).

Normal Distribution N(μ, σ²) f(x) = 1/(σ√(2π)) · exp(-(x-μ)²/(2σ²)) where: μ = mean (center of bell) σ = standard deviation (width of bell) σ² = variance E[X] = μ Var(X) = σ²
🔔 68–95–99.7 Rule
For N(μ,σ²):
  • ~68% of data falls within μ ± 1σ
  • ~95% of data falls within μ ± 2σ
  • ~99.7% of data falls within μ ± 3σ
In quant finance: if daily returns are N(0%, 1%), only 0.3% of days should be "3-sigma events."

Standard Normal Z-Score

Standardization If X ~ N(μ, σ²), then Z = (X - μ)/σ ~ N(0, 1) Z-score tells you: "how many standard deviations is X from the mean?" In quant finance: Z = (daily return - mean return) / volatility

3.2 The Binomial Distribution

Models the number of "successes" in n independent trials, each with probability p of success.

Binomial Distribution Bin(n, p) P(X = k) = C(n,k) · pᵏ · (1-p)^(n-k) where C(n,k) = n! / (k!(n-k)!) (n choose k) E[X] = np Var(X) = np(1-p)
📈 Quant context
If a strategy wins 55% of trades (p=0.55), over 100 trades (n=100), the expected number of wins = 100×0.55 = 55, with σ = √(100×0.55×0.45) ≈ 4.97. This lets you assess if a backtest result of "63 wins" is statistically significant.

3.3 The Poisson Distribution

Models the number of events in a fixed time period when events happen at a constant average rate λ (lambda) and independently.

Poisson Distribution Pois(λ) P(X = k) = e^(-λ) · λᵏ / k! E[X] = λ Var(X) = λ (mean equals variance — distinctive property!)
💡 When to use Poisson
Count of events per unit time/space where: events are rare, independent, at constant rate. Examples: number of trades in a minute, number of order book cancellations per second, number of defaults in a bond portfolio.

3.4 The Exponential Distribution

If Poisson counts events, Exponential models the time between events. If events follow Poisson(λ), time between events follows Exp(λ).

Exponential Distribution Exp(λ) f(x) = λ · e^(-λx) for x ≥ 0 F(x) = 1 - e^(-λx) (CDF) E[X] = 1/λ Var(X) = 1/λ² KEY PROPERTY — Memorylessness: P(X > s+t | X > s) = P(X > t) "How long you've waited doesn't change how long you'll still wait"

3.5 Lognormal Distribution

If X = e^Y where Y ~ N(μ,σ²), then X is lognormal. Stock prices are often modeled as lognormal because prices can't go negative, and log-returns are approximately normal.

Lognormal Distribution If X ~ LogNormal(μ, σ²): ln(X) ~ N(μ, σ²) E[X] = e^(μ + σ²/2) Var(X) = (e^σ² - 1) · e^(2μ+σ²) Stock price model: S(t) = S(0) · e^((μ - σ²/2)t + σW(t)) where W(t) is Brownian motion

Central Limit Theorem (CLT)

The Central Limit Theorem is one of the most astonishing facts in all of mathematics. It explains why the normal distribution appears so often in nature and in finance.

⭐ The CLT — Plain English
Take ANY distribution (uniform, exponential, binomial, whatever). Take the average of n samples from it. As n grows, that average will approach a normal distribution — no matter what the original distribution looked like.

4.1 Formal Statement

Central Limit Theorem Let X₁, X₂, ..., Xₙ be i.i.d. with mean μ and variance σ². Define the sample mean: X̄ₙ = (X₁ + X₂ + ... + Xₙ) / n Then as n → ∞: √n · (X̄ₙ - μ) / σ → N(0, 1) Equivalently: X̄ₙ ~ N(μ, σ²/n) approximately, for large n The standard error of the mean: SE = σ/√n
📌 Key insight: Standard Error
The std deviation of X̄ is σ/√n, not σ. This means: as you average more samples, your estimate of the mean becomes more precise. Doubling the sample size makes your estimate √2 ≈ 1.41 times more precise.

4.3 Why CLT Matters for Quants

ApplicationCLT in action
Portfolio returnsSum of many asset returns → approximately normal, even if individuals aren't
Monte Carlo simulationAverage of simulated outcomes has N(μ, σ²/n) distribution, letting us build confidence intervals
BacktestingAverage strategy P&L over many trades → normal, enabling significance testing
Risk aggregationVaR of diversified portfolio can use normal approximation

4.4 Conditions and Caveats

⚠️ When CLT can fail
CLT requires finite mean and variance. Be careful when:
  • Fat tails: Returns from Cauchy distribution have infinite variance — CLT doesn't apply
  • Small n: CLT is approximate; for n<30 with non-normal data, be cautious
  • Dependence: CLT assumes i.i.d. — correlated returns (time series) require corrections
  • Extreme events: CLT doesn't help predict tail risk / black swans

4.5 Confidence Intervals from CLT

CLT directly gives us confidence intervals for the population mean:

95% Confidence Interval for Mean X̄ ± z₀.₀₂₅ · (σ/√n) where z₀.₀₂₅ = 1.96 Example: X̄ = 5.2, σ = 2.1, n = 100 CI = 5.2 ± 1.96 × (2.1/√100) = 5.2 ± 1.96 × 0.21 = 5.2 ± 0.412 = [4.788, 5.612] Interpretation: If you repeated this experiment many times, 95% of such intervals would contain the true mean μ.

Maximum Likelihood Estimation (MLE)

You have data. You want to fit a distribution to it. MLE is the principled method: find the parameters that make your observed data most probable.

⭐ Core Idea
Imagine you measured 5 people's heights: [168, 172, 175, 169, 174]. You want to fit a Normal distribution to this. MLE asks: "What values of μ and σ make these 5 numbers most likely?" The answer is μ=sample mean, σ=sample std. But MLE generalizes this to ANY distribution.

5.1 The Likelihood Function

Likelihood Function Given data x₁, x₂, ..., xₙ and parameters θ: L(θ) = P(data | θ) = Πᵢ f(xᵢ; θ) (Assumes data points are independent — joint probability = product) Log-Likelihood (always use this!): ℓ(θ) = ln L(θ) = Σᵢ ln f(xᵢ; θ) MLE estimate: θ̂ = argmax_θ ℓ(θ)
📌 Why log-likelihood?
Multiplying 1000 probabilities (all between 0-1) causes numerical underflow — the product becomes effectively zero. Taking log converts the product to a sum, which is numerically stable and mathematically easier to optimize (derivatives of sums are simple).

5.2 MLE for Normal Distribution — Full Derivation

Step-by-step MLE for Normal N(μ, σ²) 1. Write log-likelihood: ℓ(μ,σ) = Σᵢ [-½ ln(2π) - ln(σ) - (xᵢ-μ)²/(2σ²)] = -n/2·ln(2π) - n·ln(σ) - 1/(2σ²)·Σᵢ(xᵢ-μ)² 2. Maximize over μ: take ∂ℓ/∂μ = 0 ∂ℓ/∂μ = Σᵢ (xᵢ-μ)/σ² = 0 ⟹ μ̂ = (1/n)·Σᵢ xᵢ = x̄ ✓ (sample mean!) 3. Maximize over σ: take ∂ℓ/∂σ = 0 ∂ℓ/∂σ = -n/σ + Σᵢ(xᵢ-μ)²/σ³ = 0 ⟹ σ̂² = (1/n)·Σᵢ(xᵢ-x̄)² (biased sample variance!) Note: MLE gives biased σ̂. Unbiased uses (n-1) denominator.

5.3 MLE for Exponential Distribution

MLE for Exp(λ) f(x; λ) = λe^(-λx) Log-likelihood: ℓ(λ) = n·ln(λ) - λ·Σᵢ xᵢ Take ∂ℓ/∂λ = 0: n/λ - Σᵢ xᵢ = 0 ⟹ λ̂ = n / Σᵢ xᵢ = 1/x̄ The MLE rate is 1 over the sample mean. Intuitive!

5.5 Properties of MLE

PropertyMeaning
Consistencyθ̂ → θ as n → ∞ (converges to truth with enough data)

5.6 Fisher Information & Cramér-Rao Bound

Fisher Information I(θ) = E[(∂/∂θ ln f(X;θ))²] = -E[∂²/∂θ² ln f(X;θ)] Cramér-Rao Bound: Var(θ̂) ≥ 1/I(θ) for any unbiased estimator θ̂ This is the theoretical minimum variance any estimator can achieve. MLE achieves this bound asymptotically — it's "as good as it gets."

Hypothesis Testing

Hypothesis testing is the formal framework for making decisions from data under uncertainty. In quant finance, this is how you decide if a strategy is genuinely profitable or just got lucky.

⭐ Core Question
"My strategy made $500 over 100 trades. Is this real signal or just luck?" Hypothesis testing gives a rigorous, probability-based answer.

6.1 The Framework

Hypothesis Testing Framework H₀ (Null hypothesis): The "boring" default (no effect, no difference) H₁ (Alternative hypothesis): What you're trying to show Example: H₀: Strategy has zero mean return (μ = 0) H₁: Strategy has positive mean return (μ > 0) Steps: 1. State H₀ and H₁ 2. Choose significance level α (typically 0.05 or 0.01) 3. Compute test statistic from data 4. Find p-value 5. Reject H₀ if p-value < α

6.2 The p-value — Most Misunderstood Concept

📌 What a p-value IS and IS NOT
p-value = P(observing data as extreme as yours | H₀ is true)

✅ It IS: The probability of seeing results this extreme if H₀ is true
❌ It is NOT: The probability that H₀ is true
❌ It is NOT: The probability that your result is due to chance
❌ It is NOT: A measure of effect size or practical importance

Small p-value → data is surprising under H₀ → evidence against H₀

6.3 Z-Test (known σ)

One-Sample Z-Test H₀: μ = μ₀ H₁: μ ≠ μ₀ (two-tailed) Test statistic: Z = (X̄ - μ₀) / (σ/√n) ~ N(0,1) under H₀ Reject H₀ if |Z| > z_{α/2} For α=0.05: z_{0.025} = 1.96 p-value = 2·P(Z > |z_obs|) (two-tailed)

6.4 T-Test (unknown σ)

In practice, you rarely know σ. The t-test uses the sample standard deviation instead:

One-Sample t-Test Test statistic: t = (X̄ - μ₀) / (s/√n) ~ t(n-1) under H₀ where s = sample standard deviation t(n-1) = Student's t with (n-1) degrees of freedom Key difference from z-test: - t-distribution has heavier tails than Normal - Heavier tails = more conservative (harder to reject H₀) - As n → ∞, t(n-1) → N(0,1)

6.5 Type I and Type II Errors

H₀ is TRUEH₀ is FALSE
Reject H₀❌ Type I Error (False Positive) — rate = α✅ Correct (True Positive) — Power = 1-β
Fail to Reject H₀✅ Correct (True Negative)❌ Type II Error (False Negative) — rate = β
📊 Quant context: Strategy backtesting
Type I Error (α=0.05): You think the strategy is profitable, but it's actually random luck. You trade it live and lose money. In quant finance, setting α too high leads to overfitted strategies.

Type II Error: You miss a genuinely profitable strategy. You incorrectly think it's luck and don't trade it.

6.6 Two-Sample Tests & ANOVA

Two-Sample t-Test (independent samples) H₀: μ₁ = μ₂ (two populations have same mean) t = (X̄₁ - X̄₂) / √(s₁²/n₁ + s₂²/n₂) Used when: comparing returns of two strategies, two stocks, two time periods. Paired t-Test: Use when observations are paired (same stock, before/after). t = d̄ / (s_d/√n) where d = x₁ - x₂ (differences)

6.7 Multiple Testing Problem

⚠️ Critical for Quants!
If you test 100 strategies with α=0.05, you expect ~5 false positives just by chance! This is the "strategy overfitting" problem in quant finance.

Bonferroni correction: Use α/m as threshold if testing m hypotheses simultaneously.
If testing 100 strategies with target α=0.05: use 0.0005 per test.

Bayesian Basics

Bayesian statistics is a completely different philosophy from frequentist (classical) hypothesis testing. Instead of fixed unknown parameters, Bayesian statistics treats parameters as random variables with distributions — and updates them when new data arrives.

⭐ Frequentist vs Bayesian Philosophy
Frequentist: "μ is a fixed unknown value. Data is random. I make decisions based on P(data | μ)."

Bayesian: "μ is a random variable with a distribution. I start with a prior belief about μ, then update it with data to get a posterior belief."

7.1 The Bayesian Framework

Bayes' Theorem for Inference P(θ | data) ∝ P(data | θ) · P(θ) Posterior ∝ Likelihood × Prior Full form: P(θ | data) = P(data | θ) · P(θ) / P(data) where P(data) = ∫ P(data|θ)·P(θ) dθ (normalizing constant)

7.2 Prior, Likelihood, Posterior — Explained

TermMeaningExample (coin fairness)
Prior P(θ)Your belief about θ before seeing dataP(p=0.5) is high — you think coin is fair
Likelihood P(data|θ)Probability of data given a parameter valueP(8 heads in 10 flips | p=0.5)
Posterior P(θ|data)Updated belief after seeing dataNew distribution over p after 10 flips
Normalizing constant P(data)Makes posterior integrate to 1A constant — often computed numerically

7.3 Conjugate Priors — When Math Is Elegant

A conjugate prior is a prior that, when combined with a specific likelihood, gives a posterior in the same distributional family. This makes computation exact and tractable.

Beta-Binomial Conjugacy (most common example) Likelihood: X | p ~ Binomial(n, p) (n trials, k successes) Prior: p ~ Beta(α, β) Posterior: p | X=k ~ Beta(α + k, β + n - k) Beta(α, β) has mean = α/(α+β) So prior mean = α/(α+β), posterior mean = (α+k)/(α+β+n) As n → ∞, posterior mean → k/n (MLE estimate) Data eventually overwhelms the prior!

7.4 Credible Intervals vs Confidence Intervals

📌 Critical distinction
Frequentist Confidence Interval: "If I repeated this experiment many times, 95% of my computed intervals would contain the true μ." The parameter is fixed; the interval is random.

Bayesian Credible Interval: "Given the data, there is 95% probability that θ lies in this interval." The interval is fixed; the parameter is random (has a distribution).

The Bayesian version says what most people think confidence intervals mean!

7.5 Bayesian Updating — Sequential Learning

One of Bayesian inference's greatest strengths: you can update continuously as new data arrives. Each posterior becomes the next prior.

Sequential Bayesian Updating Start: P(θ) = prior See data batch 1: P(θ|data₁) ∝ P(data₁|θ) · P(θ) See data batch 2: (use posterior as new prior!) P(θ|data₁,data₂) ∝ P(data₂|θ) · P(θ|data₁) Order doesn't matter — final posterior is the same! This is perfect for streaming financial data.

7.6 Bayesian vs Frequentist — Practical Comparison

AspectFrequentistBayesian
What varies?Data (parameter is fixed)Parameter (data is fixed)
Prior knowledgeNot used formallyExplicitly encoded as prior
OutputPoint estimate + confidence intervalFull posterior distribution
InterpretationStatements about long-run frequenciesProbability statements about parameter
ComputationUsually simpler formulasOften needs MCMC/numerical methods
Quant useBacktesting, significance testingRegime detection, signal combination

Lab 2 — Central Limit Theorem Simulation

# Lab 2: CLT Simulation — Watch the magic happen

import numpy as np
import matplotlib.pyplot as plt
from scipy import stats

def simulate_clt(dist_name, n_samples_per_mean, n_experiments=3000):
    """Sample means from different distributions"""
    if dist_name == 'uniform':
        samples = np.random.uniform(0, 1, (n_experiments, n_samples_per_mean))
        true_mu, true_sigma = 0.5, np.sqrt(1/12)
    elif dist_name == 'exponential':
        samples = np.random.exponential(1, (n_experiments, n_samples_per_mean))
        true_mu, true_sigma = 1, 1
    elif dist_name == 'bimodal':
        s1 = np.random.normal(-2, 0.5, (n_experiments, n_samples_per_mean//2))
        s2 = np.random.normal(2, 0.5, (n_experiments, n_samples_per_mean//2))
        samples = np.hstack([s1, s2])
        true_mu, true_sigma = 0, np.sqrt(4 + 0.25)
    else:  # chi-squared
        k = 3
        samples = np.random.chisquare(k, (n_experiments, n_samples_per_mean))
        true_mu, true_sigma = k, np.sqrt(2*k)
    
    return samples.mean(axis=1), true_mu, true_sigma

fig, axes = plt.subplots(4, 4, figsize=(16, 14))
fig.suptitle('Central Limit Theorem: Any Distribution → Normal Means', 
             fontsize=14, fontweight='bold')

distributions = [('uniform', 'Uniform'), ('exponential', 'Exponential'),
                 ('bimodal', 'Bimodal'), ('chisq', 'Chi-Squared')]
sample_sizes = [1, 5, 30, 100]
colors = ['#2c3e7a', '#c0392b', '#1a7a4a', '#b8860b']

for row, (dist, name) in enumerate(distributions):
    for col, n in enumerate(sample_sizes):
        ax = axes[row, col]
        means, mu, sigma = simulate_clt(dist, n)
        se = sigma / np.sqrt(n)
        
        ax.hist(means, bins=50, density=True, color=colors[row], alpha=0.7, 
                edgecolor='white', linewidth=0.3)
        
        # Overlay theoretical normal curve
        x = np.linspace(means.min(), means.max(), 200)
        ax.plot(x, stats.norm.pdf(x, mu, se), 'k-', linewidth=2)
        
        # Normality test
        _, p_val = stats.shapiro(means[:200])
        
        if row == 0: ax.set_title(f'n = {n}', fontsize=12, fontweight='bold')
        if col == 0: ax.set_ylabel(name, fontsize=10, fontweight='bold')
        ax.text(0.05, 0.92, f'Shapiro p={p_val:.3f}', transform=ax.transAxes, 
                fontsize=7, color='green' if p_val > 0.05 else 'red')
        ax.set_yticks([])

plt.tight_layout()
plt.show()
print("Lab 2 complete! Notice how all distributions converge to bell curve.")

Lab 3 — Maximum Likelihood Estimation

# Lab 3: MLE — Find the best-fitting distribution

import numpy as np
import matplotlib.pyplot as plt
from scipy import stats, optimize

np.random.seed(42)
true_mu, true_sigma = 2.5, 1.3
data = np.random.normal(true_mu, true_sigma, 100)

# Manual log-likelihood function
def log_likelihood(params, data):
    mu, sigma = params
    if sigma <= 0: return -np.inf
    return np.sum(stats.norm.logpdf(data, mu, sigma))

# Grid search: visualize likelihood landscape
mu_range = np.linspace(1.5, 3.5, 100)
sig_range = np.linspace(0.5, 2.5, 100)
LL = np.array([[log_likelihood([m, s], data) for s in sig_range] for m in mu_range])

fig, axes = plt.subplots(1, 3, figsize=(16, 5))
fig.suptitle('Maximum Likelihood Estimation for Normal Distribution')

# 1. Log-likelihood surface
ax = axes[0]
im = ax.contourf(sig_range, mu_range, LL, levels=30, cmap='viridis')
plt.colorbar(im, ax=ax)
ax.set_xlabel('σ'); ax.set_ylabel('μ')
ax.set_title('Log-Likelihood Surface')
# Mark MLE and true values
ax.axvline(true_sigma, color='white', linestyle='--', label='True')
ax.axhline(true_mu, color='white', linestyle='--')
mle_mu, mle_sig = data.mean(), data.std()
ax.plot(mle_sig, mle_mu, 'r*', markersize=15, label='MLE')
ax.legend()

# 2. Data histogram with MLE fit
ax = axes[1]
ax.hist(data, bins=20, density=True, color='steelblue', alpha=0.7, label='Data')
x = np.linspace(data.min()-1, data.max()+1, 200)
ax.plot(x, stats.norm.pdf(x, true_mu, true_sigma), 'r--', lw=2, label=f'True N({true_mu},{true_sigma})')
ax.plot(x, stats.norm.pdf(x, mle_mu, mle_sig), 'b-', lw=2.5, label=f'MLE N({mle_mu:.2f},{mle_sig:.2f})')
ax.legend(); ax.set_title('MLE Fit to Data')

# 3. MLE as function of n (convergence)
ax = axes[2]
ns = [5, 10, 20, 50, 100, 200, 500, 1000]
mu_estimates, sig_estimates = [], []
for n in ns:
    d = data[:n] if n <= len(data) else np.random.normal(true_mu, true_sigma, n)
    mu_estimates.append(d.mean())
    sig_estimates.append(d.std())

ax.axhline(true_mu, color='r', linestyle='--', label=f'True μ={true_mu}')
ax.plot(ns, mu_estimates, 'bo-', label='MLE μ̂')
ax.set_xscale('log'); ax.set_xlabel('Sample size n')
ax.set_title('MLE Consistency: μ̂ → μ'); ax.legend()

plt.tight_layout()
plt.show()

print(f"True: μ={true_mu}, σ={true_sigma}")
print(f"MLE:  μ̂={mle_mu:.4f}, σ̂={mle_sig:.4f}")
print(f"MLE is consistent: estimates converge to truth as n increases!")

Lab 5 — Bayesian Updating

# Lab 5: Bayesian Updating — Coin fairness estimation

import numpy as np
import matplotlib.pyplot as plt
from scipy import stats

def beta_posterior(alpha_prior, beta_prior, n_heads, n_tails):
    return alpha_prior + n_heads, beta_prior + n_tails

# Simulate a biased coin (true p=0.7)
np.random.seed(42)
true_p = 0.7
n_flips_total = 100
flips = np.random.binomial(1, true_p, n_flips_total)

# Start with uninformative prior Beta(1,1) = Uniform
alpha_prior, beta_prior = 1, 1
checkpoints = [1, 5, 10, 25, 50, 100]

fig, axes = plt.subplots(2, 3, figsize=(15, 9))
fig.suptitle('Bayesian Updating: Learning Coin Fairness', fontsize=14)
axes = axes.flatten()

p_range = np.linspace(0, 1, 300)

for i, n in enumerate(checkpoints):
    ax = axes[i]
    heads = flips[:n].sum()
    tails = n - heads
    
    # Compute posterior
    alpha_post = alpha_prior + heads
    beta_post = beta_prior + tails
    
    # Plot prior, likelihood, posterior
    prior = stats.beta.pdf(p_range, alpha_prior, beta_prior)
    posterior = stats.beta.pdf(p_range, alpha_post, beta_post)
    likelihood = stats.binom.pmf(heads, n, p_range)
    likelihood = likelihood / likelihood.max() * posterior.max()
    
    ax.plot(p_range, prior, 'b--', label='Prior', alpha=0.5)
    ax.plot(p_range, likelihood, 'g:', label='Likelihood', alpha=0.6)
    ax.plot(p_range, posterior, 'r-', lw=2.5, label='Posterior')
    ax.axvline(true_p, color='black', linestyle='--', alpha=0.5, label=f'True p={true_p}')
    ax.axvline(alpha_post/(alpha_post+beta_post), color='red', linewidth=1.5, linestyle='dotted')
    
    post_mean = alpha_post / (alpha_post + beta_post)
    ax.set_title(f'n={n}: {heads}H/{tails}T | Post. mean={post_mean:.3f}')
    ax.set_xlabel('p (coin bias)'); ax.legend(fontsize=7)
    ax.set_xlim(0, 1)

plt.tight_layout()
plt.show()
print("\nObserve: As n increases, posterior peaks at true p=0.7 and gets narrower!")
print("This is Bayesian learning in action — data overwhelms prior uncertainty.")
