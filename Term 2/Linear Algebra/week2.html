Linear Systems & Gaussian Elimination

A Self-Contained Study Guide for Aspiring Quant Engineers

Module 1: Prerequisites & Foundations

Before we solve complex pricing models, we need a shared language. This module builds the absolute basics—vectors, matrices, and operations—using only high-school algebra as a starting point.

1.1 What is a Vector?

A vector is simply an ordered list of numbers. Think of it as a point in space or an arrow from the origin to that point.

Example: A 2-dimensional vector v = (3, 2) means "go 3 units right and 2 units up." In quant finance, vectors store things like portfolio weights, daily returns, or cash flows.

Vector Operations

  • Addition: Add corresponding entries. (1, 2) + (3, 4) = (4, 6). Geometrically, place the tail of one at the head of the other.
  • Scalar Multiplication: Multiply every entry by a number (scalar). 2 * (1, 3) = (2, 6). This stretches or shrinks the arrow.

1.2 What is a Matrix?

A matrix is a rectangular array of numbers. We describe it by its rows (horizontal) and columns (vertical). An m x n matrix has m rows and n columns.

A = a_11a_12a_13 a_21a_22a_23 is a 2 x 3 matrix

The entry a_23 lives in row 2, column 3. In quant work, matrices store correlations, Jacobians of pricing functions, or transition probabilities.

Matrix-Vector Multiplication

This is the most important operation for linear systems. When we multiply a matrix A by a vector x, we take the dot product of each row of A with x.

21-13 x x_1x_2 = 2x_1 + 1x_2-1x_1 + 3x_2

Each row of the matrix defines a linear equation. The result is a new vector.

1.3 Linear Combinations

A linear combination of vectors is formed by multiplying each vector by a scalar and adding them. If we can reach every point in space through linear combinations of a set of vectors, those vectors are called a basis.

Quant intuition: A replicating portfolio is a linear combination of underlying assets. If you have stocks A and B, any portfolio (w_1, w_2) is a linear combination of the individual asset payoff vectors.

1.4 The Identity Matrix & Inverses

The identity matrix I is the "do nothing" matrix—multiplying by I leaves a vector unchanged. It has 1s on the diagonal and 0s elsewhere.

I_2 = 1001

If there exists a matrix B such that AB = BA = I, then B is the inverse of A, written A^-1. Not all matrices have inverses—only square matrices with linearly independent rows/columns.

Checkpoint: Before moving on, ensure you can multiply a 2x2 matrix by a 2-vector, and that you understand why matrix-vector multiplication represents a system of linear equations.

Module 2: Systems of Linear Equations

2.1 The Core Problem

A system of linear equations asks: "What inputs (x, y, z...) simultaneously satisfy all of these linear relationships?" Each equation represents a line, plane, or hyperplane. The solution is where they all intersect.

2x + y = 5
x - y = 1

Here we have 2 equations and 2 unknowns. The solution x = 2, y = 1 satisfies both.

2.2 The Three Possible Outcomes

For any linear system, exactly one of three things happens:

Case A: Unique Solution

The equations intersect at exactly one point. Geometrically, two non-parallel lines cross once. Algebraically, the coefficient matrix has full rank.

Case B: No Solution (Inconsistent)

The equations describe parallel lines or planes that never meet. In quant finance, this often signals an arbitrage opportunity or a mis-specified model.

Case C: Infinitely Many Solutions

The equations describe the same line or plane, or they intersect along a common line (in higher dimensions). One variable is "free" to be anything, and the others follow.

2.3 Matrix Form: Ax = b

Every linear system can be written compactly as Ax = b, where:

  • A is the coefficient matrix (contains the numbers multiplying the variables)
  • x is the unknown vector (what we want to find)
  • b is the constant vector (the right-hand side numbers)
211-1 xy = 51

2.4 The Augmented Matrix

To solve a system, we often combine A and b into one array called the augmented matrix [A | b]. This lets us manipulate the entire system at once.

[ A | b ] = 21|51-1|1

The vertical bar is just a visual reminder of the equals sign.

Quant intuition: In yield curve fitting, A might contain bond cash flow timings, x contains the unknown zero rates, and b contains observed market prices. Solving Ax = b finds the rates consistent with market data.

Module 3: Gaussian Elimination

Gaussian elimination is the algorithmic heart of linear algebra. It systematically transforms any system into an easy-to-solve triangular form using three simple moves.

3.1 The Three Elementary Row Operations

These operations change the system's appearance without changing its solution set:

OperationSymbolWhat it doesQuant analogy
SwapR_i <-> R_jInterchange two rowsReordering pricing equations by maturity
ScalekR_i -> R_iMultiply a row by a non-zero constantConverting prices to percentages
ReplaceR_i + kR_j -> R_iAdd a multiple of one row to anotherNetting out common risk factors
Critical rule: You can multiply a row by any number except zero. Multiplying by zero destroys information and changes the solution set.

3.2 The Goal: Upper Triangular Form

We want to create zeros below the main diagonal. Once achieved, we use back substitution (solving from the bottom up).

3.3 Worked Example: Step-by-Step

Solve the system:

2x + y - z = 8
-3x - y + 2z = -11
-2x + y + 2z = -3

Step 0 — Augmented matrix:

21-18 -3-12-11 -212-3
1
Make the first pivot 1. Divide Row 1 by 2 (scale operation):
10.5-0.54 -3-12-11 -212-3
2
Eliminate below the pivot. Add 3*R1 to R2, and add 2*R1 to R3:
10.5-0.54 00.50.51 0215
3
Second pivot. Scale R2 by 2, then eliminate below using R3 - 4*R2:
10.5-0.54 0112 00-3-3
4
Third pivot. Scale R3 by -1/3:
10.5-0.54 0112 0011

Back substitution:

  • From R3: z = 1
  • From R2: y + z = 2 -> y = 1
  • From R1: x + 0.5y - 0.5z = 4 -> x + 0.5 - 0.5 = 4 -> x = 4
Solution: x = 4, y = 1, z = 1. Verify by plugging back into the original equations.

3.4 Pivoting & Partial Pivoting

If a pivot position contains 0 (or a very small number), we swap rows to move a non-zero entry into that spot. In quant computing, we always use partial pivoting (swap with the row having the largest absolute value in that column) to minimize floating-point errors.

Why this matters for quants: When calibrating a model to 100+ market prices, small numerical errors from bad pivoting can propagate and produce arbitrage-violating prices. Stable elimination is not optional—it is a risk management tool.

Module 4: Echelon & Reduced Echelon Forms

4.1 Row Echelon Form (REF)

A matrix is in Row Echelon Form when it satisfies three conditions:

  1. All nonzero rows are above any rows of all zeros.
  2. Each leading entry (pivot) of a row is in a column to the right of the leading entry above it.
  3. All entries in a column below a leading entry are zeros.
Example of REF:
12-14 0132 001-1

4.2 Reduced Row Echelon Form (RREF)

RREF adds two more rules, making the solution completely obvious:

  1. The leading entry in each nonzero row is 1.
  2. Each leading 1 is the only nonzero entry in its column.
Same matrix in RREF:
100-8 0105 001-1

In RREF, the solution reads directly: x_1 = -8, x_2 = 5, x_3 = -1. No back substitution needed.

4.3 Rank and Free Variables

The rank of a matrix is the number of pivots in its REF. It tells us the number of "independent" equations.

  • If rank = number of variables -> unique solution.
  • If rank < number of variables -> infinitely many solutions (free variables exist).
  • If rank < rank of augmented matrix -> no solution (inconsistent).
Free variables are columns without pivots. They act as parameters. If x_3 is free, we write x_3 = t (any real number), then express the other variables in terms of t.

4.4 Parametric Solutions

When a system has infinitely many solutions, we describe the solution set using parametric form:

x = p + t*v_1 + s*v_2 + ...

where p is a particular solution, and v_1, v_2 span the space of homogeneous solutions (solutions to Ax = 0).

Key theorem: The general solution to Ax = b equals one particular solution plus the entire solution space of Ax = 0. This decomposition is fundamental to understanding no-arbitrage pricing (the homogeneous space represents zero-cost portfolios).

Module 5: Quantitative Finance Applications

5.1 Solving Calibration Equations

Calibration means tuning model parameters so that model outputs match observed market prices. This almost always reduces to solving a linear system.

Example: Calibrating a Local Volatility Surface

Suppose we observe call option prices C_1, C_2, ..., C_n for different strikes and maturities. A finite-difference discretization of the Dupire PDE yields one linear equation per market quote:

M * sigma = C

where sigma is the vector of unknown local volatilities at grid points, C is the vector of observed prices, and M is a matrix of differential operators and boundary conditions. Gaussian elimination (or its sparse variants) solves for sigma.

5.2 Arbitrage-Free Pricing Systems

The Fundamental Theorem of Asset Pricing states: a market is arbitrage-free if and only if there exists a risk-neutral measure under which discounted prices are martingales. In finite-state models, this becomes a linear system.

The Replicating Portfolio as a Linear System

Consider a one-period model with 3 states and 2 assets (stock + bond). A derivative with payoff D in each state can be replicated by holding Delta shares and B bonds if:

Delta*S_1(up) + B*(1+r) = D(up)
Delta*S_1(down) + B*(1+r) = D(down)

This is Ax = b. If a solution exists for every derivative, the market is complete. If no solution exists for some payoff, that payoff cannot be perfectly hedged.

Arbitrage detection: If the system for a "zero-cost" portfolio (b = 0) has a nonzero solution, you have found an arbitrage: a portfolio that costs nothing today but pays something positive in some state tomorrow.

5.3 Yield Curve Fitting

The yield curve shows interest rates across maturities. We construct it by solving for zero-coupon rates that reproduce observed bond prices.

Bootstrapping as Sequential Elimination

Given bond prices P_1, P_2, ..., P_n with known cash flows, the price of bond i is the discounted sum of its cash flows:

P_i = CF_i1*d(1) + CF_i2*d(2) + ... + CF_in*d(n)

where d(t) = e^(-r(t)*t) is the discount factor. Collecting all bonds gives a lower-triangular system:

CF_11000 CF_21CF_2200 CF_31CF_32CF_330 CF_41CF_42CF_43CF_44 d_1d_2d_3d_4 = P_1P_2P_3P_4

Because it is lower-triangular, forward substitution (the mirror of back substitution) solves it instantly: find d_1 from bond 1, plug into bond 2 to find d_2, and so on.

Spline Fitting: Over-Determined Systems

When we have more bonds than time points, the system becomes over-determined (more equations than unknowns). We solve the least-squares problem:

min_x ||Ax - b||^2

whose solution satisfies the normal equations: A^T A x = A^T b. Gaussian elimination on A^T A finds the best-fit yield curve.

Experiment 2: Manual Gaussian Elimination

Implement elimination step-by-step to see the algorithm, not just the answer.

import numpy as np def gauss_elimination(A, b): """Perform Gaussian elimination with partial pivoting.""" n = len(b) Ab = np.hstack([A, b.reshape(-1,1)]).astype(float) for col in range(n): # Partial pivot: find row with max absolute value in current column pivot_row = np.argmax(np.abs(Ab[col:, col])) + col Ab[[col, pivot_row]] = Ab[[pivot_row, col]] pivot = Ab[col, col] if abs(pivot) < 1e-12: raise ValueError("Matrix is singular or nearly singular") # Eliminate rows below for row in range(col+1, n): factor = Ab[row, col] / pivot Ab[row, col:] -= factor * Ab[col, col:] print(f"Eliminating row {row} using factor {factor:.3f}") print(f"After column {col}: {Ab} ") # Back substitution x = np.zeros(n) for i in range(n-1, -1, -1): x[i] = (Ab[i, -1] - np.dot(Ab[i, i+1:n], x[i+1:n])) / Ab[i, i] return x A = np.array([[2,1,-1],[-3,-1,2],[-2,1,2]], dtype=float) b = np.array([8, -11, -3], dtype=float) x = gauss_elimination(A, b) print("Solution:", x) print("Verification A@x:", A @ x)

Experiment 5: Detecting Arbitrage

Set up a state-price system and check for arbitrage via rank.

import numpy as np # 3 assets, 3 states (payoff matrix) D = np.array([ [1.1, 1.1, 1.1], # Risk-free bond, price=1 [1.2, 1.0, 0.8], # Stock, price=1 [0.0, 0.0, 1.1], # Call option, price=? ], dtype=float) prices = np.array([1.0, 1.0, 0.15]) # Check if state prices exist: D.T * psi = prices # If no solution, arbitrage exists try: psi = np.linalg.solve(D.T, prices) print("State prices:", psi) if np.any(psi <= 0): print("WARNING: Negative state price detected -> Arbitrage opportunity!") else: print("No arbitrage: all state prices positive.") except np.linalg.LinAlgError: print("Market is incomplete or arbitrage exists (singular matrix).")

Experiment 6: Ill-Conditioned Calibration

Demonstrate why partial pivoting and condition numbers matter in quant work.

import numpy as np # Hilbert matrix: notoriously ill-conditioned n = 5 A = np.array([[1/(i+j+1) for j in range(n)] for i in range(n)]) b = np.ones(n) print("Condition number:", np.linalg.cond(A)) # Solve with default (partial pivoting) x1 = np.linalg.solve(A, b) # Solve with a tiny perturbation b_perturbed = b + 1e-10 * np.random.randn(n) x2 = np.linalg.solve(A, b_perturbed) print("Change in solution under tiny perturbation:") print(np.linalg.norm(x1 - x2)) print(" Moral: In quant calibration, ill-conditioned systems amplify data errors.")

7.5 Quantitative Applications

Tier: Easy

E5.1 A 1-year zero-coupon bond trades at 95.23. What is the continuously compounded zero rate r? Set up the equation 100*e^(-r*1) = 95.23 and solve.
e^(-r) = 0.9523 -> -r = ln(0.9523) -> r = 0.0489 or 4.89%.
E5.2 In a 2-state world, a stock costs 100 and pays 110 (up) or 90 (down). The risk-free bond pays 102 in both states. Set up the 2x2 linear system to find the replicating portfolio (Delta shares, B bonds) for a call option with payoff 10 (up) and 0 (down).
110*Delta + 102*B = 10; 90*Delta + 102*B = 0. Subtract: 20*Delta = 10 -> Delta = 0.5. Then B = -90*0.5/102 = -0.4412. Option price = 100*Delta + 1*B = 50 - 0.4412 = 49.56. (Note: This is a simplified one-period binomial model.)

Tier: Medium

M5.1 You observe three bonds:
• 1Y zero: price 96.12, face 100
• 2Y coupon 5%: price 99.50, pays 5 at Y1 and 105 at Y2
• 3Y coupon 6%: price 101.25, pays 6, 6, 106
Bootstrap the discount factors d(1), d(2), d(3).
d(1) = 96.12/100 = 0.9612.
99.50 = 5*d(1) + 105*d(2) -> d(2) = (99.50 - 4.806)/105 = 0.9028.
101.25 = 6*d(1) + 6*d(2) + 106*d(3) -> d(3) = (101.25 - 5.767 - 5.417)/106 = 0.8492.
M5.2 A quant calibrates a model to 5 market prices using 4 parameters. The normal equations A^T A x = A^T b yield a 4x4 system. After elimination, the RREF shows a free variable. What does this imply about the calibration, and what should the quant do?
A free variable means the calibration problem is under-determined: infinitely many parameter sets produce the same market fit. The quant should add a regularization constraint (e.g., minimum norm, smoothness penalty) or collect an additional independent market quote to make the system full-rank.

Tier: Hard

H5.1 (Arbitrage matrix) In a 3-state market, assets have payoff matrix D = [[1.05,1.05,1.05],[1.2,1.0,0.8],[1.0,1.1,1.2]] and prices p = (1, 1, 1.02). Find whether an arbitrage exists by attempting to solve D^T psi = p for state prices psi. If no exact solution exists, compute the least-squares residual and discuss whether approximate state prices are sufficient for risk management.
D^T is 3x3. Compute det(D^T) or attempt direct solve. If D is invertible, psi = (D^T)^-1 p. If any psi_i <= 0, arbitrage exists. If D is singular and p is not in the column space, no exact state prices exist—the market admits arbitrage or is incomplete. Least-squares gives the best-fit psi but does NOT guarantee no-arbitrage; the residual represents mispricing that a trader could exploit.
H5.2 (Yield curve spline) Fit a cubic spline r(T) = a + bT + cT^2 + dT^3 to the zero rates bootstrapped at T = 1, 2, 3, 4 years. The spline must be continuous and have continuous first derivative at interior knots. Set up the 8x8 linear system for the coefficients of two cubic pieces (one for [1,2], one for [2,3], one for [3,4]) and explain why the tridiagonal structure makes Gaussian elimination O(n) instead of O(n^3).
For each interior knot, continuity of r and r' gives 2 equations. Endpoint values give 2 more. Natural spline conditions (r"=0 at ends) give the final 2. The resulting matrix has nonzeros only on the main diagonal and the two adjacent diagonals. Gaussian elimination only needs to eliminate one element per column, so each step is O(1) and total cost is O(n) instead of O(n^3). This is the Thomas algorithm, critical for real-time curve updates.
