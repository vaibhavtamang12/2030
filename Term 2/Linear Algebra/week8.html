🧱 Prerequisites

Before touching SVD, you need solid footing in four areas. Each is explained below in "first-principles" terms — no prior knowledge assumed.

Prereq 1 Vectors & Matrices (The LEGO Blocks)

A vector is just a list of numbers. Think of it as an arrow pointing from the origin to a point in space.

Column Vector

v = [3, 4]ᵀ

This arrow goes 3 units right,
4 units up.
Matrix

A = [[1, 2], [3, 4], [5, 6]]

A collection of vectors stacked together.
3 rows × 2 columns = 3 data points in 2D space.
🎯 Key Intuition for Quants

In finance, a matrix often represents returns data: rows = days, columns = assets. Each column is a time series (a vector of daily returns for one stock). Each row is a snapshot of all assets on one day.

Matrix Multiplication as Transformation

When you multiply a matrix A by a vector x, you are transforming x into a new vector Ax.

Matrix × Vector = Transformed Vector. The matrix "does something" to the vector.

Prereq 2 Eigenvalues & Eigenvectors (The Special Directions)

Most vectors get both stretched and rotated when multiplied by a matrix. But some special vectors only get stretched — they keep their direction. These are eigenvectors.

Eigenvalue Equation

A·v = λ·v

The matrix A stretches eigenvector v by factor λ (the eigenvalue), without rotating it.
Eigenvectors are the "special directions" that survive matrix multiplication unchanged in direction.
🧠 Why This Matters for SVD

SVD is essentially a generalized eigen-decomposition that works for any matrix (even non-square ones). Eigen-decomposition only works for square matrices. SVD finds the "special directions" of any data matrix.

Prereq 3 Covariance (How Things Move Together)

Covariance measures how two variables change together. In finance, we care about how stock returns move together.

Cov(X, Y) = E[(X − μₓ)(Y − μᵧ)]

Positive covariance → they tend to rise and fall together.
Negative covariance → when one rises, the other tends to fall.
Near zero → they move independently.

The covariance matrix Σ for n assets is an n×n matrix where entry Σᵢⱼ is the covariance between asset i and asset j.

Prereq 4 Orthogonality (Perpendicularity in High Dimensions)

Two vectors are orthogonal if they are perpendicular — their dot product is zero.

u ⊥ v ⟺ u·v = 0

Orthogonal vectors carry independent information.
This is the foundation of "decoupling" signals in quant finance.
🔗 Connection to SVD

SVD decomposes any matrix into orthogonal components. This means it finds a set of completely independent "directions" in your data — no redundancy, no overlap. This is incredibly powerful for removing noise and finding true signals.

🔬 SVD Theory

The Singular Value Decomposition (SVD) is one of the most important matrix factorizations in all of mathematics. It works for any matrix — square, rectangular, full rank, rank-deficient, symmetric, non-symmetric. This universality is why it dominates quant finance.

The Fundamental Theorem

Any real m × n matrix A can be decomposed as:

A = U Σ Vᵀ

Where:
• U is m × m orthogonal matrix (left singular vectors)
• Σ is m × n diagonal matrix (singular values, σ₁ ≥ σ₂ ≥ ... ≥ 0)
• Vᵀ is n × n orthogonal matrix (right singular vectors, transposed)
🎯 The Big Picture

SVD says: Any linear transformation can be broken down into three simple steps: (1) rotate the input space (Vᵀ), (2) stretch along axes (Σ), (3) rotate the output space (U). That's it. Every matrix operation is just "rotate → stretch → rotate."

Visual Decomposition

SVD factors any matrix into rotation → scaling → rotation. The expanded form shows A as a sum of rank-1 "layers."

Understanding Each Component

U — The Left Singular Vectors (Output Directions)

U is an orthogonal matrix. Its columns u₁, u₂, ..., uₘ form an orthonormal basis for the column space of A.

💡 Quant Interpretation

If A is a returns matrix (days × assets), the columns of U represent orthogonal time patterns (e.g., market-wide trends, sector rotations, idiosyncratic shocks). Each uᵢ is a "pure" temporal signal with no overlap with others.

Σ — The Singular Values (Importance Weights)

Σ is a diagonal matrix (rectangular, padded with zeros) containing the singular values σ₁ ≥ σ₂ ≥ ... ≥ σᵣ ≥ 0, where r is the rank of A.

Typically, only the first few singular values are large (signal). The rest are small (noise). This is the foundation of dimensionality reduction.

Vᵀ — The Right Singular Vectors (Input Directions)

V is also orthogonal. Its columns v₁, v₂, ..., vₙ form an orthonormal basis for the row space of A.

💡 Quant Interpretation

The columns of V represent orthogonal portfolios or factor loadings. If v₁ = [0.5, 0.5, 0.5, 0.5], it means the first principal component is an equal-weighted portfolio of all four assets. The singular value σ₁ tells you how much variance this portfolio explains.

The Sum-of-Rank-1 View

This is the most powerful way to understand SVD:

A = σ₁u₁v₁ᵀ + σ₂u₂v₂ᵀ + ... + σᵣuᵣvᵣᵀ

Each term σᵢuᵢvᵢᵀ is a rank-1 matrix.
It is the outer product of a time pattern (uᵢ) and a portfolio loading (vᵢ), scaled by importance (σᵢ).
The full matrix is just the sum of these "layers."
⚠️ Critical Insight

The singular values σᵢ are like "volume knobs" for each layer. If σ₅ is tiny, then layer 5 contributes almost nothing to A. You can turn that knob down to zero and barely change the matrix. This is low-rank approximation — the core idea of dimensionality reduction.

Connection to PCA

For a centered data matrix X (mean subtracted), the right singular vectors V are exactly the principal components of the data. The singular values squared, divided by (n−1), are the eigenvalues of the covariance matrix.

If X = UΣVᵀ, then:

Cov(X) = (1/(n−1)) XᵀX = V (Σ²/(n−1)) Vᵀ

So V diagonalizes the covariance matrix. The columns of V are the eigenvectors of XᵀX.
🎯 Why Quants Prefer SVD over Direct PCA

1. Numerical stability: SVD is more stable than computing XᵀX directly (which squares condition numbers).
2. Works with missing data: SVD can be extended to handle incomplete matrices.
3. Direct factor extraction: U gives you the time series of factors directly.

⚙️ SVD Computation

Understanding how SVD is computed deepens your intuition and helps you debug issues in production quant code. We'll cover the conceptual algorithm and practical implementation.

The Power Iteration Intuition

The simplest conceptual method to find the top singular vector is power iteration:

1. Start with random vector x
2. Repeat: x ← AᵀAx / ||AᵀAx||
3. x converges to the top right singular vector v₁

Why? AᵀA has eigenvectors = right singular vectors, eigenvalues = σ².
Power iteration amplifies the largest eigenvalue direction.
Power iteration repeatedly applies AᵀA and normalizes. The random vector "aligns" with the dominant singular vector.

Modern Algorithms: Golub-Kahan Bidiagonalization

In practice, libraries like NumPy, LAPACK, and Intel MKL use sophisticated algorithms:

1. Bidiagonalization
→
2. QR Iteration
→
3. Singular Values
→
4. Back-substitution
📐 Step 1: Bidiagonalization

Reduce A to bidiagonal form B (non-zero only on main diagonal and first super-diagonal) using Householder reflections. This is O(mn²) for m ≥ n.

📐 Step 2: QR Iteration on BᵀB

Apply implicit QR algorithm to find eigenvalues of BᵀB, which are σ². This is numerically stable and converges rapidly.

Practical Implementation

💻 NumPy SVD

import numpy as np

# Create a sample returns matrix: 252 days × 10 assets
np.random.seed(42)
returns = np.random.randn(252, 10) * 0.02  # 2% daily vol

# Full SVD
U, s, Vt = np.linalg.svd(returns, full_matrices=False)

print(f"U shape: {U.shape}")      # (252, 10)
print(f"s shape: {s.shape}")      # (10,) — singular values
print(f"Vt shape: {Vt.shape}")    # (10, 10)

# Reconstruct
Sigma = np.diag(s)
reconstructed = U @ Sigma @ Vt
print(f"Reconstruction error: {np.linalg.norm(returns - reconstructed):.2e}")

# Explained variance ratio
explained_var = s**2 / np.sum(s**2)
print(f"Explained variance by PC1: {explained_var[0]:.1%}")
print(f"Explained variance by PC2: {explained_var[1]:.1%}")

from sklearn.decomposition import TruncatedSVD

# If you only need top-k components (much faster!)

print(f"Components shape: {svd.components_.shape}")  # (5, 10)
print(f"Explained variance: {svd.explained_variance_ratio_}")

# components_[i] is the i-th right singular vector (portfolio loading)
# factors[:, i] is the i-th left singular vector (time series)

💻 Randomized SVD (for massive matrices)

from sklearn.utils.extmath import randomized_svd

# For very large matrices (e.g., 10,000 assets)

# Much faster than full SVD for large n, with provable accuracy bounds

📉 Low-Rank Approximation

Low-rank approximation is the core mechanism of dimensionality reduction. It says: "Your data matrix looks complicated, but it's mostly made of a few simple patterns. Let's keep only those."

The Core Idea

From the SVD sum-of-rank-1 form:

A = σ₁u₁v₁ᵀ + σ₂u₂v₂ᵀ + ... + σᵣuᵣvᵣᵀ

If σₖ₊₁, σₖ₊₂, ... are small, then:

A ≈ σ₁u₁v₁ᵀ + ... + σₖuₖvₖᵀ = Aₖ

Aₖ is a rank-k matrix that approximates A.
Low-rank approximation keeps the "big layers" and discards the "small layers." The error is the sum of discarded singular values.

Measuring Approximation Quality

How good is Aₖ? We use matrix norms:

Frobenius Norm

||A − Aₖ||F² = Σi=k+1r σᵢ²

Sum of squared discarded singular values.
Good for overall energy.
Spectral (Operator) Norm

||A − Aₖ||2 = σₖ₊₁

Largest singular value of the error.
Good for worst-case error.
📊 Choosing k in Practice

Scree plot: Plot σᵢ vs. i. Look for the "elbow" where values drop off.
Variance threshold: Choose smallest k such that Σᵢ₌₁ᵏ σᵢ² / Σᵢ₌₁ʳ σᵢ² ≥ 0.95 (95% variance explained).
Cross-validation: Hold out data, choose k that minimizes prediction error.
Domain knowledge: In equities, k=3-5 factors often capture 60-80% of variance.

Geometric Interpretation

Low-rank approximation projects your data onto a k-dimensional subspace spanned by the top k right singular vectors. This is the "best" k-dimensional subspace in the least-squares sense.

Rank-1 approximation finds the line that minimizes the sum of squared perpendicular distances to all data points.

🏛️ Eckart-Young Theorem

The Eckart-Young-Mirsky theorem is the mathematical guarantee that makes SVD-based dimensionality reduction rigorous. It tells us that truncating SVD is not just a heuristic — it is optimal.

The Theorem

Eckart-Young-Mirsky Theorem

Let A be an m×n matrix with SVD A = UΣVᵀ.
Let Aₖ = UₖΣₖVₖᵀ be the rank-k truncation (top k singular values).

Then for any matrix B with rank(B) ≤ k:

||A − Aₖ|| ≤ ||A − B||

This holds for both the Frobenius norm and the spectral (operator) norm.
🎯 What This Means in Plain English

If you want the best rank-k approximation of any matrix, just keep the top k singular values. There is no better way. No other algorithm, no neural network, no optimization — nothing can beat SVD truncation for this specific problem.

Visual Proof Sketch

The set of rank-k matrices forms a curved manifold in matrix space. Aₖ is the orthogonal projection of A onto this manifold — the closest point.

Step-by-Step Proof (Frobenius Norm)

Step 1: Express the error in the SVD basis

Since U and V are orthogonal, they preserve norms:

||A − B||F² = ||Uᵀ(A − B)V||F² = ||Σ − UᵀBV||F²

Let C = UᵀBV. We need to minimize ||Σ − C||F subject to rank(C) ≤ k.

Step 2: Rank constraint on C

Since rank(B) ≤ k and U, V are full-rank, rank(C) ≤ k.

Step 3: Best rank-k approximation of a diagonal matrix

For a diagonal matrix Σ, the best rank-k approximation is clearly obtained by keeping the k largest diagonal entries and zeroing the rest. Any other choice either keeps smaller values (worse fit) or changes off-diagonal entries (adds error without reducing rank).

Step 4: Conclude

The minimum is achieved when C has the same top k diagonal entries as Σ and zeros elsewhere. This means B = UₖΣₖVₖᵀ = Aₖ.

⚠️ Important Caveats

1. Uniqueness: If σₖ = σₖ₊₁, the optimal approximation is not unique. Any combination of the tied singular vectors works.
2. Only for unitarily invariant norms: The theorem holds for norms that don't change under orthogonal transformations (Frobenius, spectral, Ky Fan). It does NOT hold for arbitrary norms like the L1 norm.
3. Fixed rank: The theorem assumes you fix the rank k in advance. In practice, you often choose k based on data.

Quantitative Implications

📐 Frobenius Error

||A − Aₖ||F² = Σi=k+1r σᵢ²

The error is exactly the "energy" you threw away. You can compute it before approximating!

📐 Spectral Error

||A − Aₖ||2 = σₖ₊₁

The worst-case error is exactly the first singular value you discarded. Simple and tight.

Python Verification

💻 Verify Eckart-Young

import numpy as np

np.random.seed(42)
A = np.random.randn(50, 30)
U, s, Vt = np.linalg.svd(A, full_matrices=False)

k = 5
# Optimal rank-k approximation
A_k = U[:, :k] @ np.diag(s[:k]) @ Vt[:k, :]

# Random rank-k approximation (for comparison)
B = np.random.randn(50, k) @ np.random.randn(k, 30)

error_optimal = np.linalg.norm(A - A_k, 'fro')
error_random = np.linalg.norm(A - B, 'fro')
error_spectral = np.linalg.norm(A - A_k, 2)

print(f"Optimal Frobenius error: {error_optimal:.4f}")
print(f"Random rank-k error:     {error_random:.4f}")
print(f"Optimal spectral error:  {error_spectral:.4f}")
print(f"Expected spectral error (σₖ₊₁): {s[k]:.4f}")
print(f"
Random error / Optimal error: {error_random/error_optimal:.2f}x worse")

📊 Factor Reduction

In quant finance, we often observe thousands of assets but suspect only a handful of latent factors drive most returns. SVD provides the rigorous framework for extracting these factors and reducing dimensionality.

The Factor Model Setup

We assume returns are generated by:

r = Bf + ε

Where:
• r = asset returns vector (n×1)
• B = factor loadings matrix (n×k)
• f = factor returns vector (k×1)
• ε = idiosyncratic noise (n×1), independent of f

The goal: Given historical returns matrix R (T×n), estimate B and f.
🔗 How SVD Solves This

If R = UΣVᵀ, then:
• V gives the factor loadings B (columns are portfolio weights for each factor)
• UΣ gives the factor returns F (time series of each factor)
• The rank-k truncation gives the best rank-k factor model by Eckart-Young

Step-by-Step: SVD Factor Extraction

1. Center returns
→
2. Run SVD
→
3. Choose k
→
4. Extract B = Vₖ
→
5. Extract F = UₖΣₖ
The returns matrix is approximated as factor returns × factor loadings + noise. SVD finds the optimal factors by Eckart-Young.

💡 Practical Tip: Factor Stability

Real factors are slowly varying over time. If your PC2 flips sign every month, it's probably noise, not a real factor. Use rolling window SVD and track how singular vectors evolve. Stable vectors = real factors.

Python: Factor Extraction from Stock Returns

💻 Extract & Interpret Factors

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt

# Simulate returns for 20 stocks, 252 days, with 3 true factors
np.random.seed(42)

returns_centered = returns - returns.mean(axis=0)

# Plot singular value spectrum

plt.bar(range(1, len(s)+1), s**2 / np.sum(s**2))
plt.axvline(x=k_true+0.5, color='r', linestyle='--', label='True rank')
plt.xlabel('Component'); plt.ylabel('Explained Variance')
plt.title('Scree Plot — Factor Detection')
plt.legend()
plt.show()

# Plot first loading (should be all positive = market)
plt.figure(figsize=(10, 4))
plt.bar(range(n), loadings[:, 0])

Risk parity sets each factor's contribution to total risk equal.

💹 Statistical Arbitrage Signals

Statistical arbitrage (stat arb) is a class of trading strategies that exploit statistical relationships between assets rather than fundamental valuations. SVD is the engine that finds these relationships.

The Core Idea: Mean Reversion in Residual Space

Stat arb assumes that while asset prices may wander randomly, linear combinations of assets (portfolios) mean-revert. SVD finds the optimal mean-reverting portfolios.

The Stat Arb Pipeline

1. Model returns as: r = Bf + ε
2. The systematic part Bf is hard to predict (efficient market)
3. The residual ε = r − Bf may be predictable (mean-reverting)
4. SVD gives us the best low-rank estimate of Bf
5. Trade the residuals: buy when ε << 0, sell when ε >> 0
🎯 Why SVD is Perfect for Stat Arb

Eckart-Young guarantees that the SVD truncation gives the best possible systematic factor estimate. The residuals are therefore the "cleanest" possible mean-reverting signals — you have removed the maximum amount of non-predictable variance.

The SVD Residual Strategy

1. Build returns matrix R
→
2. SVD: R ≈ UₖΣₖVₖᵀ
→
3. Systematic: Ŕ = UₖΣₖVₖᵀ
→
4. Residual: E = R − Ŕ
→
5. Z-score residuals
→
6. Trade extremes
SVD separates systematic (unpredictable) from residual (mean-reverting). Z-scores of residuals generate trading signals.

Why Residuals Mean-Revert

The systematic factors (market, sector, style) represent shared risk that is efficiently priced. The residual represents asset-specific deviations. These deviations are:

  • Temporary: Caused by liquidity shocks, index rebalancing, or short-term sentiment
  • Bounded: A stock can't deviate from its fair value forever (arbitrage forces)
  • Cross-sectional: When one asset is "too high" and another "too low," pair trading works

Cross-Sectional vs. Time-Series Approaches

📊 Cross-Sectional (PCA-based)

At each time t:
1. Run SVD on recent returns window
2. Compute residuals for all assets
3. Rank assets by residual Z-score
4. Go long bottom decile, short top decile

Pros: Diversified, market-neutral
Cons: High turnover, transaction costs

📈 Time-Series (Single-name)

For each asset i:
1. Model rᵢ = βᵢ·f + εᵢ using SVD factors
2. Track εᵢ over time
3. When |εᵢ| > 2σ, take contrarian position
4. Hold until εᵢ reverts to 0

Pros: Lower turnover per asset
Cons: Less diversified, factor exposure

import pandas as pd
import matplotlib.pyplot as plt

np.random.seed(42)

# Simulate 50 stocks, 500 days, with 5 factors
T, n = 500, 50
k_factors = 5

f = np.random.randn(T, k_factors) * 0.015
B = np.random.randn(n, k_factors)
returns = f @ B.T + np.random.randn(T, n) * 0.008

# Parameters
lookback = 60      # Days for SVD window
k = 3              # Number of factors to remove
z_threshold = 2.0  # Z-score threshold

pnl = []
positions = np.zeros(n)

for t in range(lookback, T-1):
    # Recent returns window
    window = returns[t-lookback:t, :]

    # Center
    window_centered = window - window.mean(axis=0)

    # SVD
    U, s, Vt = np.linalg.svd(window_centered, full_matrices=False)

    # Systematic component (today's returns projected onto factors)
    today = returns[t, :] - returns[t-lookback:t, :].mean(axis=0)
    systematic = today @ Vt[:k, :].T @ Vt[:k, :]
    residual = today - systematic

    # Z-score using historical residual volatility
    residuals_hist = window_centered - (window_centered @ Vt[:k, :].T @ Vt[:k, :])
    resid_vol = residuals_hist.std(axis=0) + 1e-8
    z_scores = residual / resid_vol

    # Generate signals
    signals = np.zeros(n)
    signals[z_scores < -z_threshold] = 1   # Long
    signals[z_scores > z_threshold] = -1   # Short

    # Next day P&L
    next_day_return = returns[t+1, :]
    daily_pnl = np.sum(signals * next_day_return)
    pnl.append(daily_pnl)

pnl = np.array(pnl)
cumulative = np.cumsum(pnl)
sharpe = np.mean(pnl) / np.std(pnl) * np.sqrt(252)

print(f"Total Return: {cumulative[-1]:.2%}")
print(f"Sharpe Ratio: {sharpe:.2f}")
print(f"Max Drawdown: {(cumulative - np.maximum.accumulate(cumulative)).min():.2%}")

🛡️ Covariance Regularization

Portfolio optimization requires estimating the covariance matrix of asset returns. But when you have many assets and limited history, the sample covariance matrix is noisy, ill-conditioned, and often singular. SVD-based regularization fixes this.

The Problem: Curse of Dimensionality

Sample Covariance Matrix

Σ̂ = (1/(T−1)) RᵀR

Where R is T×n (T observations, n assets).

If n > T, Σ̂ is guaranteed to be singular.
Even if n < T, if n/T is large (e.g., 0.5), Σ̂ is extremely noisy.
The condition number explodes: σmax/σmin → ∞
When you have more assets than observations, the sample covariance matrix is mathematically broken. Regularization is mandatory.

SVD-Based Shrinkage: The Spectral Filter

The simplest SVD regularization is spectral truncation — keep only the top k eigencomponents of the covariance matrix:

Spectral Filter

Σ̂ = V Λ Vᵀ (eigen-decomposition of sample covariance)

Σ̃ = Vₖ Λₖ Vₖᵀ + δI

Where Λₖ keeps only top k eigenvalues, and δI adds a small diagonal (ridge) to ensure positive definiteness.
🔗 Connection to SVD

Since Σ̂ = (1/(T−1)) RᵀR, and R = UΣVᵀ, the eigenvectors of Σ̂ are exactly the right singular vectors V of R. The eigenvalues of Σ̂ are σᵢ²/(T−1). So SVD of returns directly gives the spectral decomposition of covariance.

Ledoit-Wolf Shrinkage (Advanced)

The Ledoit-Wolf estimator is the gold standard in quant finance. It shrinks the sample covariance toward a structured target (e.g., identity or single-factor model):

Ledoit-Wolf Shrinkage

Σ* = δ·F + (1−δ)·Σ̂

Where F is a shrinkage target (e.g., F = tr(Σ̂)/n · I) and δ is the optimal shrinkage intensity.

SVD helps compute δ efficiently and apply the shrinkage in factor space.
🎯 Why This Matters for Quants

Markowitz portfolio optimization: w* = Σ⁻¹μ. If Σ is singular, Σ⁻¹ doesn't exist. If Σ is ill-conditioned, tiny changes in μ cause huge changes in w (the "error maximization" problem). Regularized covariance gives stable, investable portfolios.

Visual: Effect of Regularization on Portfolio Weights

Sample covariance produces extreme long/short positions. SVD regularization smooths weights and reduces turnover.

Out-of-Sample Risk Forecasting

The real test of a covariance estimator is out-of-sample risk prediction:

Risk Forecast Error

Predicted risk: σ̂p² = wᵀ Σ̃ w
Realized risk: σp² = wᵀ Σtrue w

We want |σ̂p² − σp²| to be small.
Sample covariance typically underestimates risk (overfitting).
SVD regularization provides more honest risk estimates.
📐 The Marchenko-Pastur Law

For random matrices, there is a theoretical threshold below which sample eigenvalues are pure noise. The Marchenko-Pastur distribution gives the noise spectrum. In practice, eigenvalues below λ₊ = σ²(1+√(n/T))² are noise and should be shrunk or removed.

import pandas as pd
import matplotlib.pyplot as plt

np.random.seed(42)

# Simulate realistic returns with regime changes
T, n = 1000, 100
regime_change = 500

returns = np.zeros((T, n))
for t in range(T):
    if t < regime_change:
        # Normal regime: 5 factors
        f = np.random.randn(5) * 0.01
        B = np.random.randn(n, 5)
        returns[t] = B @ f + np.random.randn(n) * 0.005
    else:
        # Crisis regime: 1 dominant factor, high correlation
        f = np.random.randn(1) * 0.03
        B = np.ones((n, 1)) * 0.8 + np.random.randn(n, 1) * 0.2
        returns[t] = B @ f + np.random.randn(n) * 0.008

# Stat arb parameters
lookback = 60
k = 5
z_thresh = 1.5

pnls = []
for t in range(lookback, T-1):
    window = returns[t-lookback:t]
    wc = window - window.mean(axis=0)

    try:
        U, s, Vt = np.linalg.svd(wc, full_matrices=False)
        today = returns[t] - window.mean(axis=0)
        sys = today @ Vt[:k].T @ Vt[:k]
        resid = today - sys

        # Robust z-score using MAD
        med = np.median(resid)
        mad = np.median(np.abs(resid - med))
        z = 0.6745 * (resid - med) / (mad + 1e-8)

        signals = np.zeros(n)
        signals[z < -z_thresh] = 1
        signals[z > z_thresh] = -1

        # Equal weight within long/short legs
        if signals.sum() != 0 and signals.sum() != n:
            long_mask = signals > 0
            short_mask = signals < 0
            signals[long_mask] = 1 / long_mask.sum() if long_mask.sum() > 0 else 0
            signals[short_mask] = -1 / short_mask.sum() if short_mask.sum() > 0 else 0

        pnl = signals @ returns[t+1]
        pnls.append(pnl)
    except:
        pnls.append(0)

pnls = np.array(pnls)
cum = np.cumsum(pnls)

# Plot with regime change marked
fig, axes = plt.subplots(2, 1, figsize=(12, 8))
axes[0].plot(cum, linewidth=1.5)
axes[0].axvline(x=regime_change-lookback, color='r', linestyle='--', alpha=0.7, label='Regime Change')
axes[0].set_title('Cumulative P&L')
axes[0].legend()
axes[0].grid(True, alpha=0.3)

axes[1].plot(pnls, alpha=0.5, linewidth=0.5)
axes[1].axvline(x=regime_change-lookback, color='r', linestyle='--', alpha=0.7)
axes[1].set_title('Daily P&L')
axes[1].grid(True, alpha=0.3)

plt.tight_layout()
plt.show()

print(f"Pre-regime Sharpe: {np.mean(pnls[:regime_change-lookback])/np.std(pnls[:regime_change-lookback])*np.sqrt(252):.2f}")
print(f"Post-regime Sharpe: {np.mean(pnls[regime_change-lookback:])/np.std(pnls[regime_change-lookback:])*np.sqrt(252):.2f}")
print(f"Overall Sharpe: {np.mean(pnls)/np.std(pnls)*np.sqrt(252):.2f}")

💻 Lab 6 — Noise Filtering

import numpy as np
import matplotlib.pyplot as plt

np.random.seed(42)

# Pure noise matrix
T, n = 500, 200
noise = np.random.randn(T, n) * 0.02

# Sample covariance eigenvalues
cov = np.cov(noise.T)
eigenvalues = np.sort(np.linalg.eigvalsh(cov))[::-1]

# Marchenko-Pastur bounds
sigma2 = 0.02**2
q = n / T
lambda_plus = sigma2 * (1 + np.sqrt(q))**2
lambda_minus = sigma2 * (1 - np.sqrt(q))**2

plt.figure(figsize=(10, 6))
plt.hist(eigenvalues, bins=50, density=True, alpha=0.7, label='Empirical eigenvalues')
plt.axvline(x=lambda_plus, color='r', linestyle='--', linewidth=2, label=f'λ+ = {lambda_plus:.4f}')
plt.axvline(x=lambda_minus, color='r', linestyle='--', linewidth=2, label=f'λ- = {lambda_minus:.4f}')
plt.xlabel('Eigenvalue')
plt.ylabel('Density')
plt.title('Marchenko-Pastur: Pure Noise Eigenvalue Distribution')
plt.legend()
plt.yscale('log')
plt.grid(True, alpha=0.3)
plt.show()

# Now add signal
k_signal = 5
signal_eigen = [0.5, 0.3, 0.2, 0.15, 0.1]
factors = np.random.randn(T, k_signal)
for i, se in enumerate(signal_eigen):
    factors[:, i] *= np.sqrt(se)
loadings = np.random.randn(n, k_signal)
returns = factors @ loadings.T + noise

cov2 = np.cov(returns.T)
eigenvalues2 = np.sort(np.linalg.eigvalsh(cov2))[::-1]

plt.figure(figsize=(10, 6))
plt.bar(range(1, 21), eigenvalues2[:20], alpha=0.7, label='Top 20 eigenvalues')
plt.axhline(y=lambda_plus, color='r', linestyle='--', label='MP threshold')
plt.xlabel('Eigenvalue Index')
plt.ylabel('Eigenvalue')
plt.title('Eigenvalues with Signal + Noise')
plt.legend()
plt.grid(True, alpha=0.3)
plt.show()

print(f"Eigenvalues above MP threshold: {np.sum(eigenvalues2 > lambda_plus)}")
print(f"True signal rank: {k_signal}")
