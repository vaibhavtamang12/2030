Quadratic Forms & Optimization | From First Principles to Portfolio Applications

🎯 Prerequisites: The Foundation

Before diving into quadratic forms and optimization, you need to understand a few building blocks. Think of these as the "alphabet" before we write "sentences" in quant finance.

1. Vectors & Matrices

What is it? A vector is just a list of numbers (like [2, 3, 5]). A matrix is a grid of numbers (like a spreadsheet).

Vector x = [x₁, x₂, ..., xₙ]ᵀ (column vector) Matrix A = | a₁₁ a₁₂ | | a₂₁ a₂₂ |

Why it matters: In quant finance, a vector might represent portfolio weights, and a matrix might represent covariances between assets.

A vector is like a shopping list (quantities of items). A matrix is like a price table showing how every item correlates with every other item.

2. Matrix Transpose & Multiplication

Transpose (Aᵀ): Flip rows and columns. If A is 2×3, Aᵀ is 3×2.

Multiplication: Multiply rows by columns. The result tells us how quantities interact.

If x = [w₁, w₂]ᵀ (portfolio weights) and A = covariance matrix Then xᵀAx = quadratic form (portfolio variance!)

Key rule: (AB)ᵀ = BᵀAᵀ. This matters when we derive optimization formulas.

3. Gradients (First Derivatives)

What is it? The gradient is just a vector of partial derivatives. It points in the direction of steepest ascent.

For f(x,y) = x² + 3xy ∇f = [∂f/∂x, ∂f/∂y] = [2x + 3y, 3x]
Imagine hiking a mountain. The gradient is the compass direction that takes you uphill fastest. In optimization, we often want to go downhill (negative gradient).

4. Hessian (Second Derivatives)

What is it? The Hessian matrix contains all second partial derivatives. It tells us about the "curvature" of a function.

H = | ∂²f/∂x² ∂²f/∂x∂y | | ∂²f/∂y∂x ∂²f/∂y² |

Why it matters: The Hessian determines whether a critical point is a minimum, maximum, or saddle point. In finance, it tells us if our portfolio optimization is stable.

5. Eigenvalues & Eigenvectors

What is it? For a matrix A, an eigenvector v is a special direction that only gets stretched (not rotated) when multiplied by A. The stretch factor is the eigenvalue λ.

A·v = λ·v Example: If A represents a transformation, eigenvectors are the "natural axes" of that transformation.
Think of a rubber sheet being stretched. Most points move in complex ways, but some special points (eigenvectors) just move straight away from the center. The amount they stretch (eigenvalue) tells you how "strong" the transformation is in that direction.

6. Covariance & Correlation

Covariance: Measures how two assets move together. Positive = tend to rise together. Negative = tend to move opposite.

Correlation: Normalized covariance (between -1 and +1).

Cov(X,Y) = E[(X-μₓ)(Y-μᵧ)] Σ (Sigma) = covariance matrix where Σᵢⱼ = Cov(assetᵢ, assetⱼ)

Why it matters: The covariance matrix Σ is the heart of portfolio optimization. It appears in every quant application we'll study.

Before moving on, ensure you can answer:

  1. What does xᵀΣx represent in portfolio theory? (Answer: Portfolio variance)
  2. Why is the Hessian symmetric? (Answer: ∂²f/∂x∂y = ∂²f/∂y∂x for smooth functions)
  3. What does a negative eigenvalue imply about a direction? (Answer: The transformation flips/reverses that direction)

📐 Quadratic Forms

What is a Quadratic Form?

A quadratic form is a scalar function of a vector x that looks like:

Q(x) = xᵀAx = Σᵢ Σⱼ Aᵢⱼ xᵢ xⱼ

Where A is a square matrix (usually symmetric) and x is a vector. It is called "quadratic" because every term involves exactly two variables multiplied together.

Imagine you have a landscape (like a valley or mountain). A quadratic form is the simplest mathematical shape that landscape can have near any point. It is the "Taylor series approximation" of degree 2. Just like a parabola approximates any curve near its vertex, a quadratic form approximates any multi-dimensional function near a point.

Simple 2D Example

Let x = [x₁, x₂]ᵀ and A = [[2, 1], [1, 3]]. Then:

Q(x) = [x₁, x₂] · | 2 1 | · | x₁ | | 1 3 | | x₂ | = 2x₁² + x₁x₂ + x₂x₁ + 3x₂² = 2x₁² + 2x₁x₂ + 3x₂²

Notice: The diagonal elements (2 and 3) multiply squared terms, while off-diagonal elements (both 1) multiply cross-terms.

Visualizing Quadratic Forms: Three Fundamental Shapes

Why Quadratic Forms Matter in Quant Finance

Portfolio Variance is a Quadratic Form!

If w = portfolio weights and Σ = covariance matrix:

σ²ₚ = wᵀΣw = Σᵢ Σⱼ wᵢ wⱼ σᵢⱼ

This is exactly a quadratic form. The matrix A is the covariance matrix Σ. This is why understanding quadratic forms is non-negotiable for quants.

Key Properties

Property Explanation Finance Meaning
Homogeneity Q(αx) = α²Q(x) Doubling position size quadruples variance
Symmetry We can always assume A is symmetric Covariance matrices are symmetric: Cov(i,j) = Cov(j,i)
Definiteness Determines if Q(x) is always positive, negative, or mixed Determines if portfolio variance is always positive

✅ Positive Definiteness

What Does "Definite" Mean?

When we say a matrix is positive definite, we mean its quadratic form is always positive (except at zero). It is like asking: "Is this landscape a bowl, a dome, or a saddle?"

A is Positive Definite (PD) ⇔ xᵀAx > 0 for all x ≠ 0 A is Positive Semi-Definite (PSD) ⇔ xᵀAx ≥ 0 for all x A is Negative Definite (ND) ⇔ xᵀAx < 0 for all x ≠ 0 A is Indefinite ⇔ xᵀAx takes both positive and negative values
Think of a matrix as a "personality test" for landscapes: - Positive Definite = The "Optimist" — everything is uphill from the bottom (one global minimum) - Indefinite = The "Dramatic" — has both mountains and valleys (saddle points) - Negative Definite = The "Pessimist" — everything is downhill from the top (one global maximum)

Testing for Positive Definiteness: Five Methods

Method 1: Eigenvalue Test (Most Intuitive)

A is PD ⇔ ALL eigenvalues of A are > 0 A is PSD ⇔ ALL eigenvalues of A are ≥ 0 A is ND ⇔ ALL eigenvalues of A are < 0

Why this works: Eigenvalues tell you how much the matrix "stretches" space in its natural directions. If all stretches are positive, every direction goes "uphill" from the origin.

Method 2: Principal Minors (Sylvester's Criterion)

For a 3×3 matrix A: Δ₁ = det([a₁₁]) > 0 Δ₂ = det(| a₁₁ a₁₂ |) > 0 | a₂₁ a₂₂ | Δ₃ = det(A) > 0 If ALL leading principal minors > 0 → A is PD
Sylvester's Criterion Visualization

Method 3: Cholesky Decomposition

If A = L·Lᵀ where L is lower triangular with positive diagonals, then A is PD. This is how computers actually check in practice!

Method 4: Quadratic Form Definition

Try to write xᵀAx and complete the square. If it reduces to a sum of squares with positive coefficients, it is PD.

Method 5: Convexity Connection

A function f(x) = xᵀAx is convex if and only if A is PSD. (We will explore this deeply in the Convexity tab.)

Finance Application: Why Covariance Matrices Must be PSD

Critical Insight: A covariance matrix Σ must be positive semi-definite. Why? Because portfolio variance wᵀΣw must never be negative — you cannot have negative risk!

If a covariance matrix has negative eigenvalues, it implies the existence of a portfolio with negative variance, which is impossible. This often happens due to:

  • Noisy data with more assets than observations
  • Rounding errors
  • Asynchronous data

Quant Fix: Use "shrinkage estimators" or "nearest correlation matrix" algorithms to force PSD.

🔺 Convexity

What is Convexity?

A convex function is one where the line segment between any two points on the graph lies above the graph. A convex set is one where the line segment between any two points in the set stays in the set.

Convex vs Non-Convex Functions
Imagine you drop a ball on a landscape: - Convex landscape = The ball always rolls to the ONE bottom of the bowl. No matter where you drop it, it finds the global minimum. - Non-convex landscape = The ball might get stuck in a small dip (local minimum) and never reach the true bottom.

Mathematical Definition

A function f is convex if for all x, y and λ ∈ [0,1]: f(λx + (1-λ)y) ≤ λf(x) + (1-λ)f(y) For twice-differentiable functions: f is convex ⇔ ∇²f(x) is Positive Semi-Definite everywhere

Why Convexity is the Holy Grail of Optimization

Property Convex Problem Non-Convex Problem
Local vs Global Min Any local minimum = global minimum Local minima can be terrible
Algorithms Gradient descent guaranteed to converge Need sophisticated global optimizers
Complexity Polynomial time Often NP-hard
Quant Finance Mean-variance optimization Drawdown-constrained optimization

Convexity of Quadratic Forms

For f(x) = xᵀAx + bᵀx + c:

  • If A is PD → f is strictly convex (unique global minimum)
  • If A is PSD → f is convex (global minimum exists, may not be unique)
  • If A has negative eigenvalues → f is non-convex (saddle points, local maxima)

Convex Sets in Finance

Common convex constraints in portfolio optimization:

  • Budget constraint: Σwᵢ = 1 (hyperplane — convex)
  • No-shorting: wᵢ ≥ 0 (positive orthant — convex)
  • Box constraints: lᵢ ≤ wᵢ ≤ uᵢ (box — convex)
  • Risk constraint: wᵀΣw ≤ σ²_max (ellipsoid — convex)
Key Theorem: If you minimize a convex function over a convex set, the problem is convex. This is why mean-variance optimization (convex objective + convex constraints) is "easy" relative to drawdown-constrained problems.

🔍 Unconstrained Optimization

The Problem

Given a function f(x), find x* such that f(x*) is minimized:

x* = argminₓ f(x)
You are blindfolded in a valley and want to find the lowest point. You can feel the slope under your feet. How do you walk to the bottom?

First-Order Condition (Necessary)

At a minimum, the gradient must be zero (flat ground):

∇f(x*) = 0

Points where ∇f = 0 are called critical points or stationary points.

Second-Order Condition (Sufficient)

To ensure it is a minimum (not maximum or saddle), check the Hessian:

∇²f(x*) is Positive Definite → Local minimum ∇²f(x*) is Negative Definite → Local maximum ∇²f(x*) is Indefinite → Saddle point
Optimization Landscape: Critical Points

Algorithm 1: Gradient Descent

Repeat until convergence: xₖ₊₁ = xₖ - α·∇f(xₖ) Where α = step size (learning rate)

Intuition: Walk downhill in the direction of steepest descent. The step size α controls how big your steps are.

Algorithm 2: Newton's Method

xₖ₊₁ = xₖ - [∇²f(xₖ)]⁻¹ · ∇f(xₖ)

Intuition: Instead of just looking at slope, look at curvature too. If the ground is flattening out, take a bigger step. This converges much faster but requires computing and inverting the Hessian.

Closed-Form Solution for Quadratic Functions

For f(x) = ½xᵀAx - bᵀx where A is PD:

∇f = Ax - b = 0 → x* = A⁻¹b
Finance Connection: This is exactly the form of unconstrained mean-variance optimization! If A = Σ (covariance) and b = expected returns, x* = Σ⁻¹μ gives the optimal portfolio weights (up to scaling).

Convergence & Condition Number

The condition number κ(A) = λ_max / λ_min (ratio of largest to smallest eigenvalue) determines how hard optimization is:

  • κ ≈ 1 (circular bowl) → Easy, fast convergence
  • κ >> 1 (elongated valley) → Hard, zig-zagging descent

⛓️ Constrained Optimization & Lagrange Multipliers

The Problem

Real-world problems have constraints. In portfolio optimization, you cannot invest negative money (no shorting), weights must sum to 100%, risk must be below a limit, etc.

minimize f(x) subject to gᵢ(x) ≤ 0 (inequality constraints) hⱼ(x) = 0 (equality constraints)
Imagine you are hiking in a national park and want to reach the lowest point, but: - You must stay on the trail (equality constraint: h(x) = 0) - You cannot enter the restricted area (inequality constraint: g(x) ≤ 0) Lagrange multipliers tell you how "annoyed" you are by each constraint.

Lagrange Multipliers: Equality Constraints

For minimize f(x) subject to h(x) = 0:

Form the Lagrangian: L(x, λ) = f(x) + λ·h(x) At the optimal point: ∇ₓL = 0 → ∇f(x*) + λ∇h(x*) = 0 ∇λL = 0 → h(x*) = 0

Geometric Interpretation: At the constrained minimum, the gradient of the objective ∇f must be parallel to the gradient of the constraint ∇h. They point in the same (or opposite) direction, just scaled by λ.

λ is the Lagrange multiplier — it measures how much the objective would improve if the constraint were relaxed slightly.

Lagrange Multiplier Geometry

KKT Conditions: The Full Picture (Inequality Constraints)

For minimize f(x) subject to g(x) ≤ 0, the Karush-Kuhn-Tucker (KKT) conditions generalize Lagrange multipliers:

L(x, λ) = f(x) + λ·g(x) 1. Stationarity: ∇f(x*) + λ∇g(x*) = 0 2. Primal Feasibility: g(x*) ≤ 0 3. Dual Feasibility: λ ≥ 0 4. Complementary Slackness: λ·g(x*) = 0

Complementary Slackness Explained:

  • If λ > 0 (constraint is "active"), then g(x*) = 0 (you are right on the boundary)
  • If g(x*) < 0 (constraint is "inactive"), then λ = 0 (constraint doesn't matter)

Think of it as: "Either the constraint is binding, or it is irrelevant."

Portfolio Application: Budget Constraint

minimize ½ wᵀΣw - γ·μᵀw (risk-adjusted return) subject to 1ᵀw = 1 (fully invested) w ≥ 0 (no shorting) Lagrangian: L = ½wᵀΣw - γμᵀw + λ(1 - 1ᵀw) - νᵀw

Where λ handles the equality constraint and ν (nu) handles the inequality constraints.

Dual Problem & Shadow Prices

The dual problem maximizes the Lagrangian with respect to the multipliers. In finance:

  • The Lagrange multiplier λ for the budget constraint tells you the "marginal utility of extra capital"
  • The multiplier for a risk constraint tells you how much return you sacrifice for safety

📊 Mean-Variance Optimization (MVO)

The Birth of Modern Portfolio Theory

In 1952, Harry Markowitz revolutionized finance by showing that investors should care about both return (mean) and risk (variance). This earned him the Nobel Prize.

You are at a buffet with many dishes (assets). Each dish has: - Expected taste (expected return μ) - Spiciness variance (risk σ²) - Flavor correlations (covariances Σ) Markowitz asked: "What is the best plate (portfolio) that gives me the most taste for the least spiciness?"

The Three Formulations

Formulation 1: Minimum Variance Portfolio

minimize wᵀΣw (portfolio variance) subject to 1ᵀw = 1 (fully invested)

Goal: Find the portfolio with the lowest possible risk, regardless of return.

Formulation 2: Maximum Return for Target Risk

maximize μᵀw (expected return) subject to wᵀΣw = σ²ₜ (target risk) 1ᵀw = 1 (fully invested)

Formulation 3: Risk-Adjusted Return (Most Common)

maximize μᵀw - (γ/2)·wᵀΣw Where γ = risk aversion coefficient

This is an unconstrained quadratic optimization! The solution is:

w* = (1/γ) · Σ⁻¹μ
The Efficient Frontier

The Efficient Frontier: Key Insights

  1. Diversification Magic: Portfolios on the frontier have lower risk than individual assets with the same return. This is because covariances allow risk cancellation.
  2. Tangency Portfolio: The point where the Capital Market Line (from risk-free rate) touches the frontier. This is the optimal risky portfolio.
  3. Two-Fund Theorem: Any efficient portfolio can be constructed as a combination of two frontier portfolios.
  4. Shorting: Without short constraints, frontier portfolios can have extreme weights. With constraints, the frontier shrinks.

Practical Challenges in MVO

Challenge Problem Solution
Estimation Error Sample covariance Σ is noisy; inverting it amplifies errors Shrinkage estimators, factor models, ridge regularization
Ill-conditioning Σ is nearly singular (assets highly correlated) Add small constant to diagonal (Tikhonov), use PCA
Concentration Optimal weights concentrate in few assets Add diversification constraints, use risk parity
Instability Small changes in inputs → large weight changes Robust optimization, Bayesian approaches

Closed-Form Solutions

Global Minimum Variance (GMV) portfolio: w_GMV = Σ⁻¹1 / (1ᵀΣ⁻¹1) Tangency Portfolio (max Sharpe): w_tan = Σ⁻¹μ / (1ᵀΣ⁻¹μ) Optimal risk-adjusted: w* = (1/γ) · Σ⁻¹μ

⚖️ Risk Parity

The Problem with MVO

Traditional MVO often produces portfolios where a few assets dominate risk. If those assets crash, the portfolio crashes. Risk Parity asks: "What if every asset contributed equally to total portfolio risk?"

Imagine a team of 10 people carrying a heavy table: - MVO = 2 strong people carry 80% of the weight, 8 people carry 20% - Risk Parity = Everyone carries exactly 10% of the weight If one person trips in the MVO case, the table might fall. In risk parity, the table is stable.

Marginal Risk Contribution

First, define the risk contribution of asset i:

RCᵢ = wᵢ · (∂σₚ/∂wᵢ) = wᵢ · (Σw)ᵢ / σₚ Where σₚ = √(wᵀΣw) is portfolio volatility

This decomposes total risk into contributions from each asset. Note: Σᵢ RCᵢ = σₚ (they sum to total risk).

The Risk Parity Condition

RCᵢ = RCⱼ for all i, j Or equivalently: wᵢ · (Σw)ᵢ = wⱼ · (Σw)ⱼ for all i, j With budget constraint Σwᵢ = 1: RCᵢ = σₚ / n (each asset contributes 1/n of total risk)
Risk Contribution: MVO vs Risk Parity

Special Case: Inverse Volatility

When correlations are zero (or equal), risk parity simplifies beautifully:

wᵢ ∝ 1/σᵢ Where σᵢ = standard deviation of asset i

This is called Inverse Volatility Weighting and is extremely popular because it requires no covariance estimation!

General Risk Parity: Iterative Solution

When correlations exist, there is no closed form. We solve iteratively:

Newton-Raphson style iteration: wᵢ^(k+1) = wᵢ^(k) · [target_RCᵢ / RCᵢ^(k)]^α Where α is a damping factor (e.g., 0.5)

Or formulate as convex optimization:

minimize ½ wᵀΣw - Σᵢ bᵢ ln(wᵢ) Where bᵢ = risk budget (equal for risk parity)

Risk Budgeting (Generalization)

Risk Parity is a special case of Risk Budgeting where we assign specific risk budgets bᵢ:

RCᵢ / σₚ = bᵢ Example: 60/40 stock/bond risk budget → b_stocks = 0.6, b_bonds = 0.4

Pros and Cons

Pros Cons
True diversification (by risk, not capital) Can overweight low-return assets (bonds)
More stable than MVO Leverage often needed to hit return targets
Less sensitive to estimation error Requires iterative optimization
Performs well in crises "Risk parity unwind" can cause contagion

📉 Drawdown Constraints

What is Drawdown?

Drawdown measures the decline from a historical peak. If your portfolio reaches $100, then falls to $80, you are in a 20% drawdown.

DD(t) = [Peak(t) - CurrentValue(t)] / Peak(t) Where Peak(t) = max_{s≤t} Value(s)
Imagine climbing a mountain range: - Your altitude = Portfolio value - Highest point reached so far = Peak - How far down you are from that highest point = Drawdown Investors hate drawdowns because: 1. It takes exponentially more return to recover (50% loss needs 100% gain) 2. Behavioral pain of losing money > pleasure of gaining 3. May force liquidation at the worst time
Portfolio Value vs Drawdown Over Time

Why Drawdown Constraints Are Hard

Drawdown is NOT convex! The maximum drawdown constraint makes the optimization problem non-convex. This is because the "running peak" function is non-linear and path-dependent.

Mathematically, max drawdown depends on the entire path of returns, not just the final distribution. This makes it much harder than variance constraints.

Approaches to Drawdown-Constrained Optimization

Approach 1: Convex Approximations

Replace drawdown with a convex proxy:

  • CVaR (Conditional Value at Risk): Average of worst α% losses. Convex!
  • Semi-variance: Variance of returns below mean. Convex!
  • Weighted variance: Penalize downside more than upside

Approach 2: Scenario-Based Optimization

Given historical return scenarios r₁, r₂, ..., r_T: For each scenario path, compute cumulative returns: V_t = Πₛ₌₁ᵗ (1 + rₛ) Constraint: max_t [1 - V_t/max_s≤t(V_s)] ≤ D_max

Approach 3: Dynamic Programming / Stochastic Control

For continuous-time models, use Hamilton-Jacobi-Bellman equations. This is advanced but gives the theoretically optimal policy.

Approach 4: Heuristic Methods

  • Stop-loss rules: Hard exits at certain loss levels
  • Volatility targeting: Reduce exposure when volatility spikes
  • Trend following overlays: Move to cash during drawdowns

Calmar Ratio: The Drawdown-Adjusted Metric

Calmar Ratio = CAGR / Max Drawdown Where CAGR = Compound Annual Growth Rate

Unlike Sharpe ratio (which uses volatility), Calmar directly measures return per unit of worst-case pain.

Connection to All Previous Topics

Drawdown constraints tie together everything we have learned:

  • Quadratic forms: Variance is a quadratic form, but drawdown is not
  • Convexity: MVO is convex; drawdown-constrained MVO is not
  • Lagrange multipliers: Can still be used for convex approximations
  • Risk parity: Often produces better drawdown profiles naturally

# Experiment 2: Implement Gradient Descent import numpy as np import matplotlib.pyplot as plt # Objective: f(x) = x^2 + 2y^2 (elliptical bowl) # Gradient: [2x, 4y] # True minimum: [0, 0] def f(x): return x[0]**2 + 2*x[1]**2 def grad_f(x): return np.array([2*x[0], 4*x[1]]) # Gradient descent x = np.array([3.0, 3.0]) # Starting point alpha = 0.1 # Step size history = [x.copy()] for i in range(20): x = x - alpha * grad_f(x) history.append(x.copy()) history = np.array(history) # Plot convergence plt.figure(figsize=(10, 5)) # Contour plot x_grid = np.linspace(-4, 4, 100) y_grid = np.linspace(-4, 4, 100) X, Y = np.meshgrid(x_grid, y_grid) Z = X**2 + 2*Y**2 plt.subplot(1, 2, 1) plt.contour(X, Y, Z, levels=20, cmap='viridis') plt.plot(history[:,0], history[:,1], 'r.-', markersize=10) plt.plot(0, 0, 'g*', markersize=15, label='Minimum') plt.title('Gradient Descent Path') plt.legend() # Function value over iterations plt.subplot(1, 2, 2) values = [f(h) for h in history] plt.semilogy(values, 'b-o') plt.xlabel('Iteration') plt.ylabel('f(x) (log scale)') plt.title('Convergence') plt.grid(True) plt.tight_layout() plt.show() print(f"Final x: {history[-1]}")

Global Minimum Variance Portfolio ones = np.ones(n_assets) Sigma_inv = np.linalg.inv(Sigma) w_gmv = Sigma_inv @ ones / (ones @ Sigma_inv @ ones) # 2. Tangency Portfolio (max Sharpe, assuming rf=0.02) rf = 0.02 excess_returns = returns - rf w_tan = Sigma_inv @ excess_returns / (ones @ Sigma_inv @ excess_returns) # 3. Risk-adjusted optimal (gamma = 2) gamma = 2 w_opt = (1/gamma) * Sigma_inv @ excess_returns # Compute portfolio stats def portfolio_stats(w, returns, Sigma): ret = w @ returns vol = np.sqrt(w @ Sigma @ w) sharpe = (ret - rf) / vol return ret, vol, sharpe for name, w in [("GMV", w_gmv), ("Tangency", w_tan), ("Risk-Adj", w_opt)]: ret, vol, sharpe = portfolio_stats(w, returns, Sigma) print(f" {name}: Return={ret:.3f}, Vol={vol:.3f}, Sharpe={sharpe:.3f}") print(f"Weights: {w}") # Plot Efficient Frontier def efficient_frontier(returns, Sigma, n_points=100): Sigma_inv = np.linalg.inv(Sigma) ones = np.ones(len(returns)) # Two mutual funds w1 = Sigma_inv @ ones / (ones @ Sigma_inv @ ones) # GMV w2 = Sigma_inv @ returns # Max return portfolio rets, vols = [], [] for alpha in np.linspace(-0.5, 1.5, n_points): w = alpha * w1 + (1-alpha) * w2 w = w / np.sum(w) # Normalize ret, vol, _ = portfolio_stats(w, returns, Sigma) rets.append(ret) vols.append(vol) return rets, vols rets, vols = efficient_frontier(returns, Sigma) plt.figure(figsize=(10, 6)) plt.plot(vols, rets, 'b-', linewidth=2, label='Efficient Frontier') # Plot individual assets for i in range(n_assets): vol = np.sqrt(Sigma[i,i]) plt.plot(vol, returns[i], 'ro', markersize=8) plt.annotate(f'Asset {i+1}', (vol, returns[i]), textcoords="offset points", xytext=(5,5)) # Plot optimal portfolios for name, w, color in [('GMV', w_gmv, 'green'), ('Tangency', w_tan, 'purple')]: ret, vol, _ = portfolio_stats(w, returns, Sigma) plt.plot(vol, ret, 'o', color=color, markersize=10, label=name) plt.xlabel('Volatility') plt.ylabel('Expected Return') plt.title('Efficient Frontier') plt.legend() plt.grid(True) plt.show()

# Experiment 4: Risk Parity Portfolio import numpy as np def risk_parity(Sigma, max_iter=100, tol=1e-8): """ Iterative risk parity using Newton-style updates """ n = Sigma.shape[0] w = np.ones(n) / n # Equal start for _ in range(max_iter): vol = np.sqrt(w @ Sigma @ w) marginal = (Sigma @ w) / vol rc = w * marginal # Update: move towards equal risk contribution target_rc = vol / n w_new = w * (target_rc / rc) ** 0.5 w_new = w_new / np.sum(w_new) # Budget constraint if np.max(np.abs(w_new - w)) < tol: break w = w_new return w # Test w_rp = risk_parity(Sigma) vol_rp = np.sqrt(w_rp @ Sigma @ w_rp) rc_rp = w_rp * (Sigma @ w_rp) / vol_rp print("Risk Parity Weights:", w_rp) print("Risk Contributions:", rc_rp) print("Equal target:", vol_rp / n_assets) print("Sum of RCs:", np.sum(rc_rp), "= Volatility:", vol_rp) # Compare with inverse volatility inv_vol = 1 / np.sqrt(np.diag(Sigma)) w_inv_vol = inv_vol / np.sum(inv_vol) print(" Inverse Vol Weights:", w_inv_vol)

# Experiment 5: Drawdown Calculation import numpy as np import matplotlib.pyplot as plt # Simulate cumulative portfolio value np.random.seed(42) returns = np.random.normal(0.001, 0.02, 252) # Daily returns cumulative = np.cumprod(1 + returns) # Calculate running peak and drawdown running_peak = np.maximum.accumulate(cumulative) drawdown = (running_peak - cumulative) / running_peak max_dd = np.max(drawdown) plt.figure(figsize=(12, 6)) plt.subplot(2, 1, 1) plt.plot(cumulative, label='Portfolio Value') plt.plot(running_peak, 'r--', label='Running Peak') plt.legend() plt.title('Portfolio Value vs Running Peak') plt.subplot(2, 1, 2) plt.fill_between(range(len(drawdown)), drawdown, alpha=0.5, color='red') plt.axhline(y=max_dd, color='darkred', linestyle='--', label=f'Max DD: {max_dd:.2%}') plt.legend() plt.title('Drawdown Over Time') plt.ylabel('Drawdown') plt.tight_layout() plt.show() print(f"Maximum Drawdown: {max_dd:.2%}") # Calmar ratio cagr = cumulative[-1] ** (252/len(cumulative)) - 1 calmar = cagr / max_dd print(f"CAGR: {cagr:.2%}") print(f"Calmar Ratio: {calmar:.2f}")
