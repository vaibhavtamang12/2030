Python Programming Basics

Python is the standard language of quantitative finance. You need variables, loops, functions, and basic object-oriented programming (classes). You do not need to be a software engineer, but you must be comfortable reading and writing code.

Key Concepts

  • Variables & Types: price = 100.5 (float), symbol = "AAPL" (string), is_long = True (boolean).
  • Lists & Dictionaries: Lists store ordered data. Dictionaries store key-value pairs (e.g., {"AAPL": 150.0, "TSLA": 200.0}).
  • Functions: Reusable blocks of code. def calculate_return(p1, p2): return (p2-p1)/p1
  • Classes: Templates for objects. In event-driven backtesting, everything is an object (Strategy, Portfolio, Execution).
Analogy: A Python class is like a blueprint for a house. The object is the actual house built from that blueprint. You build many houses (objects) from one blueprint (class).
2

NumPy & Pandas

These libraries are the Excel of Python. NumPy handles fast numerical arrays. Pandas handles tabular data with row labels (dates) and column labels (Open, High, Low, Close).

What You Must Know

  • NumPy Array: A grid of numbers. Operations happen on the entire grid at once (vectorization). prices = np.array([100, 101, 102]); returns = prices[1:] / prices[:-1] - 1
  • Pandas DataFrame: A table. Each column is a Series. df["Close"] gives you the closing prices.
  • Indexing: .loc["2023-01-01"] selects by label. .iloc[0] selects by position.
  • Time Index: Financial data is always ordered by time. pd.to_datetime() converts strings to dates.
Pro Tip: In quant finance, vectorization means performing an operation on an entire array at once instead of looping row-by-row. It is 10-100x faster and essential for backtesting.
3

Basic Statistics & Probability

Trading is about making decisions under uncertainty. Statistics gives you the language to describe uncertainty.

Core Statistical Ideas

  • Mean (Average): The central tendency. mean_return = returns.mean()
  • Standard Deviation (Volatility): How much values spread around the mean. This is the most common measure of risk.
  • Normal Distribution: The bell curve. Many models assume returns are normally distributed, though real returns have "fat tails" (extreme events are more likely than the bell curve predicts).
  • Correlation: A number between -1 and 1 describing how two assets move together. Diversification works because correlations are less than 1.
  • Percentiles: The 95th percentile of losses is used in Value-at-Risk (VaR).
Standard Deviation: σ = sqrt( Σ(xᵢ - μ)² / (N-1) )
Analogy: If the mean return is your "expected destination," standard deviation is the "average wrongness" of your GPS. A high standard deviation means your actual path wanders far from the expected path.

Financial Market Basics

You cannot backtest what you do not understand. Learn how markets are structured and what data looks like.

Market Structure & Data

  • Stock / Equity: A share of ownership in a company.
  • Long Position: You profit if the price goes up. You buy first, sell later.
  • Short Position: You profit if the price goes down. You borrow and sell first, buy back later.
  • OHLCV: Every bar of market data contains Open (first trade price), High (highest price), Low (lowest price), Close (last trade price), and Volume (number of shares traded).
  • Bid-Ask Spread: The gap between what buyers are willing to pay (bid) and what sellers demand (ask). This is a hidden transaction cost.
  • Adjusted Close: The closing price corrected for stock splits and dividends. Always use Adjusted Close for total return calculations.
5

Returns & Time Value

The building block of every strategy is the return—how much money you made or lost.

Simple vs. Log Returns

Simple Return: The percentage change in price. This is intuitive.

Rₜ = (Pₜ − Pₜ₋₁) / Pₜ₋₁

Log Return (Continuously Compounded): The natural log of the price ratio. This is mathematically convenient.

rₜ = ln(Pₜ / Pₜ₋₁)
Why Log Returns? Log returns are time-additive. If you know 10 daily log returns, you can sum them to get the 10-day log return. You cannot do this with simple returns. This makes portfolio math and backtesting much cleaner.

Annualization: To compare strategies, we annualize metrics.

  • Annualized Return = Daily Return × 252 (trading days per year)
  • Annualized Volatility = Daily Volatility × √252

📊 Financial Data & Returns

Every backtest begins with data. Garbage in, garbage out. This section teaches you how financial data is structured, how to clean it, and how to compute the returns that every strategy depends on.

The Structure of Market Data

Market data comes in bars (also called candles or OHLCV). A daily bar summarizes one trading day. A 1-minute bar summarizes one minute.

OHLCV Deep Dive

FieldMeaningWhy It Matters
OpenFirst traded price of the periodOften used as the execution price in backtests because you cannot trade at the prior close without lookahead bias.
HighHighest price during the periodUsed for stop-loss and limit order simulations.
LowLowest price during the periodUsed for stop-loss and limit order simulations.
CloseLast traded price of the periodMost commonly used for signal generation.
VolumeNumber of shares/contracts tradedLiquidity proxy. Low volume means higher slippage.
Adjusted Close vs. Close: Close is the raw last price. Adjusted Close corrects for stock splits and dividends. If a stock splits 2-for-1, the price drops by half, but you now own twice as many shares. Using unadjusted Close will show a fake 50% crash. Always use Adjusted Close for total return calculations.

Computing Returns

Returns are the raw material of every backtest. There are two types you must master.

Simple Returns

Rₜ = (Pₜ − Pₜ₋₁) / Pₜ₋₁

Intuitive and easy to understand. If you bought at $100 and sold at $105, your simple return is 5%.

Use case: Reporting performance to humans.

# Simple returns in Pandas
returns = df["Adj Close"].pct_change()

Log Returns

rₜ = ln(Pₜ / Pₜ₋₁)

Less intuitive but mathematically superior. Log returns assume continuous compounding.

Use case: Statistical modeling, portfolio math, and backtesting engines.

# Log returns in Pandas
import numpy as np
log_returns = np.log(df["Adj Close"] / df["Adj Close"].shift(1))
Key Property: Log returns are additive across time. The log return over 10 days is the sum of the 10 daily log returns. Simple returns are not additive—you must multiply (1+R) terms.

Volatility: The Measure of Risk

Volatility is simply the standard deviation of returns. It measures how much the price "bounces around."

σ = std(returns)   →   Annualized σ = σ_daily × √252

We use 252 because there are approximately 252 trading days in a year. We use the square root because volatility scales with the square root of time (this comes from the mathematics of random walks).

Risk-Free Rate

The return you could earn with zero risk, usually proxied by the 3-month U.S. Treasury Bill. It is essential for computing the Sharpe Ratio later.

In backtesting, we often assume a constant risk-free rate (e.g., 2% per year, or 0.02/252 per day) for simplicity.

Data Quality Issues

  • Missing Data: Markets are closed on weekends and holidays. Some stocks do not trade every day. Use df.fillna(method="ffill") with caution—forward-filling is standard, but never backward-fill (that is lookahead bias).
  • Survivorship Bias: If you only test on companies that are alive today, you ignore all the bankrupt companies. Your backtest will be artificially optimistic. (Covered in depth later.)
  • Corporate Actions: Splits, dividends, and spin-offs change the price. Use Adjusted Close to account for these.
  • Look-Ahead Bias: Using information you would not have had at the time. Example: using quarterly earnings released after market close to trade at that day's close. (Covered in depth later.)

📈 Technical Indicators

Indicators are mathematical transformations of price and volume. They reduce noise and highlight patterns. A strategy is simply a set of rules applied to indicators.

Moving Averages: Smoothing Price

Prices are noisy. A moving average smooths price to reveal the underlying trend.

Simple Moving Average (SMA)

The arithmetic mean of the last N prices. Every day is weighted equally.

SMA₂₀ = (Pₜ + Pₜ₋₁ + ... + Pₜ₋₁₉) / 20
df["SMA_20"] = df["Adj Close"].rolling(20).mean()

Trait: Slow to react. Lots of lag.

Exponential Moving Average (EMA)

Weights recent prices more heavily. Reacts faster to new information.

EMAₜ = α × Pₜ + (1−α) × EMAₜ₋₁   where α = 2/(N+1)
df["EMA_20"] = df["Adj Close"].ewm(span=20, adjust=False).mean()

Trait: Less lag, but more whipsaws (false signals).

Momentum Indicators

Relative Strength Index (RSI)

RSI measures the speed and magnitude of recent price changes. It oscillates between 0 and 100.

RSI = 100 − [100 / (1 + RS)]   where RS = Avg Gain / Avg Loss over N periods
  • RSI > 70: Potentially overbought (price may fall).
  • RSI < 30: Potentially oversold (price may rise).
delta = df["Adj Close"].diff()
gain = (delta.where(delta > 0, 0)).rolling(14).mean()
loss = (-delta.where(delta < 0, 0)).rolling(14).mean()
rs = gain / loss
df["RSI"] = 100 - (100 / (1 + rs))
Trap: In a strong uptrend, RSI can stay above 70 for weeks. It is not a pure "sell" signal—it is a warning that the move is extended.

MACD (Moving Average Convergence Divergence)

MACD tracks the relationship between two EMAs of a price.

  • MACD Line: EMA(12) − EMA(26)
  • Signal Line: EMA(9) of the MACD Line
  • Histogram: MACD Line − Signal Line

A bullish crossover occurs when MACD crosses above Signal. A bearish crossover occurs when MACD crosses below Signal.

ema_12 = df["Adj Close"].ewm(span=12, adjust=False).mean()
ema_26 = df["Adj Close"].ewm(span=26, adjust=False).mean()
df["MACD"] = ema_12 - ema_26
df["Signal"] = df["MACD"].ewm(span=9, adjust=False).mean()
df["Histogram"] = df["MACD"] - df["Signal"]

Bollinger Bands

A volatility envelope around a moving average.

  • Middle Band: SMA(20)
  • Upper Band: SMA(20) + 2 × StdDev(20)
  • Lower Band: SMA(20) − 2 × StdDev(20)

The logic: Prices tend to stay within 2 standard deviations of the mean. If price touches the upper band, it may be overextended. If it touches the lower band, it may be cheap.

df["BB_Middle"] = df["Adj Close"].rolling(20).mean()
df["BB_Std"] = df["Adj Close"].rolling(20).std()
df["BB_Upper"] = df["BB_Middle"] + 2 * df["BB_Std"]
df["BB_Lower"] = df["BB_Middle"] - 2 * df["BB_Std"]

From Indicators to Signals

An indicator is just a number. A signal is a trading decision derived from that number.

Signal Mapping: You define a rule that converts the indicator value into an action: +1 (Buy/Long), 0 (Hold/Neutral), or -1 (Sell/Short).

Example rules:

  • SMA Crossover: Signal = +1 if Price > SMA(50), else 0.
  • RSI Reversal: Signal = +1 if RSI < 30, Signal = -1 if RSI > 70.
  • Bollinger Bounce: Signal = +1 if Price < Lower Band, Signal = -1 if Price > Upper Band.

🎯 Strategy Design

A trading strategy is a rule-based system that converts market data into trading decisions. Without strict rules, you are not a quant—you are a gambler with a spreadsheet.

The Anatomy of a Strategy

Every strategy, no matter how complex, answers three questions:

  1. When do I enter? (The Entry Rule)
  2. When do I exit? (The Exit Rule)
  3. How much do I trade? (Position Sizing — covered later)

Signal Generation

A signal is a time series of integers representing your desired market position:

  • +1 = Long (you want to own the asset)
  • 0 = Flat/Neutral (no position)
  • -1 = Short (you want to bet against the asset)
Signal vs. Position: A signal is what your brain (the strategy) wants to do. A position is what your portfolio actually holds. They differ because of lag, fills, and risk limits.

Example: SMA Crossover Strategy

Rule: Go long when the fast SMA crosses above the slow SMA. Go short when it crosses below.

# Generate raw signal
df["Signal"] = 0
df.loc[df["SMA_Fast"] > df["SMA_Slow"], "Signal"] = 1
df.loc[df["SMA_Fast"] < df["SMA_Slow"], "Signal"] = -1

# Detect changes (crossovers)
df["Position"] = df["Signal"].diff().fillna(0)
entry_dates = df[df["Position"] != 0].index
Look-Ahead Bias Alert: If you use .loc[df["SMA_Fast"] > df["SMA_Slow"]] and trade at the same day's close, you are cheating. You do not know the closing price until the market closes. We will fix this in the Backtesting section.

Example: Mean Reversion Strategy

Rule: Buy when price deviates significantly below its moving average (it will "revert" to the mean). Sell when it deviates above.

df["Z_Score"] = (df["Adj Close"] - df["SMA_20"]) / df["Adj Close"].rolling(20).std()
df["Signal"] = 0
df.loc[df["Z_Score"] < -2.0, "Signal"] = 1   # Far below mean → Buy
df.loc[df["Z_Score"] > +2.0, "Signal"] = -1  # Far above mean → Sell

This is the core idea behind Bollinger Band strategies.

Example: Momentum Strategy

Rule: Buy when the trend is strong and rising. The trend is your friend.

df["Return_20d"] = df["Adj Close"].pct_change(20)
df["Signal"] = 0
df.loc[df["Return_20d"] > 0.05, "Signal"] = 1   # Up >5% in 20 days
df.loc[df["Return_20d"] < -0.05, "Signal"] = -1 # Down >5% in 20 days

🔄 Vectorized Backtesting

Vectorized backtesting is the fastest way to test a strategy. Instead of simulating day-by-day, you compute everything at once using Pandas/NumPy. It is ideal for prototyping but makes simplifying assumptions about execution.

The Core Concept

In a vectorized backtest, you assume you can trade at the closing price of the day after the signal is generated. This avoids look-ahead bias.

Step-by-Step Implementation

Step 1: Load Data & Calculate Indicators

import yfinance as yf
import pandas as pd
import numpy as np

df = yf.download("SPY", start="2018-01-01", end="2024-01-01")
df["SMA_50"] = df["Adj Close"].rolling(50).mean()
df["SMA_200"] = df["Adj Close"].rolling(200).mean()

Step 2: Generate Raw Signal

# +1 when fast > slow, -1 otherwise
df["Signal"] = np.where(df["SMA_50"] > df["SMA_200"], 1, -1)

Step 3: Eliminate Look-Ahead Bias

The signal at time t is computed using the close at time t. You cannot trade at time t's close using time t's close. You must trade at the next available price.

# Shift signal by 1 day so we trade on the next day's return
df["Position"] = df["Signal"].shift(1)
Critical: Forgetting .shift(1) is the #1 mistake beginners make. Your backtest will show impossible returns because you are trading with future knowledge.

Step 4: Calculate Strategy Returns

# Asset returns
df["Asset_Return"] = df["Adj Close"].pct_change()

Handling Transaction Costs

Real trading is not free. You pay commissions and suffer slippage. A realistic backtest must subtract these.

# Assume 0.1% commission per trade (round-trip = 0.2%)
commission = 0.001

# Detect trades (position changes)
df["Trades"] = df["Position"].diff().abs()

# Subtract commission from returns on trade days
df["Strategy_Return_Net"] = df["Strategy_Return"] - (df["Trades"] * commission)

# Recalculate equity curve
df["Cumulative_Net"] = (1 + df["Strategy_Return_Net"]).cumprod()
Why multiply by abs(diff)? If your position changes from 0 to 1, or 1 to -1, or 1 to 0, you traded. The absolute difference captures all these cases. A position change of 2 (long to short) costs twice as much as a change of 1.

Limitations of Vectorized Backtesting

  • Execution Assumption: Assumes you get the closing price. In reality, you get a price near the close, and large orders move the market.
  • No Intraday Detail: Cannot model stop-losses that trigger intraday based on the High or Low.
  • No Complex Logic: Hard to implement rules like "cancel order if not filled in 5 minutes."
  • All-at-Once: You must have all data in memory. Cannot stream from a live feed.

For these reasons, vectorized backtesting is for research and prototyping. Event-driven backtesting is for realism and production.

def vectorized_backtest(ticker, start, end, fast=50, slow=200, commission=0.001):
    df = yf.download(ticker, start=start, end=end)

    # Indicators
    df["SMA_Fast"] = df["Adj Close"].rolling(fast).mean()
    df["SMA_Slow"] = df["Adj Close"].rolling(slow).mean()

    # Signal and Position (shifted!)
    df["Signal"] = np.where(df["SMA_Fast"] > df["SMA_Slow"], 1, -1)
    df["Position"] = df["Signal"].shift(1)

    # Returns
    df["Asset_Return"] = df["Adj Close"].pct_change()
    df["Strategy_Return"] = df["Position"] * df["Asset_Return"]

    # Costs
    df["Trades"] = df["Position"].diff().abs()
    df["Strategy_Net"] = df["Strategy_Return"] - (df["Trades"] * commission)

    # Equity curves
    df["Cum_BuyHold"] = (1 + df["Asset_Return"]).cumprod()
    df["Cum_Strategy"] = (1 + df["Strategy_Net"]).cumprod()

    # Metrics
    total_return = df["Cum_Strategy"].iloc[-1] - 1
    volatility = df["Strategy_Net"].std() * np.sqrt(252)
    sharpe = (df["Strategy_Net"].mean() * 252) / (df["Strategy_Net"].std() * np.sqrt(252))

    # Max Drawdown
    rolling_max = df["Cum_Strategy"].cummax()
    drawdown = (df["Cum_Strategy"] - rolling_max) / rolling_max
    max_dd = drawdown.min()

    print(f"Total Return: {total_return:.2%}")
    print(f"Annualized Vol: {volatility:.2%}")
    print(f"Sharpe Ratio: {sharpe:.3f}")
    print(f"Max Drawdown: {max_dd:.2%}")

    # Plot
    fig, axes = plt.subplots(2, 1, figsize=(11, 8), gridspec_kw={"height_ratios": [3,1]})
    df[["Cum_BuyHold", "Cum_Strategy"]].plot(ax=axes[0], title=f"{ticker} Backtest")
    axes[0].set_ylabel("Growth of $1")
    drawdown.plot(ax=axes[1], title="Drawdown", color="red")
    axes[1].fill_between(drawdown.index, drawdown, 0, color="red", alpha=0.3)
    axes[1].set_ylabel("Drawdown")
    plt.tight_layout()
    plt.show()

    return df

⚡ Event-Driven Backtesting

Event-driven backtesting simulates the live trading loop. It is slower than vectorized backtesting but far more realistic. It is the bridge between research and production.

Why Event-Driven?

  • Realism: Models order types (market, limit), partial fills, and slippage precisely.
  • Intraday: Can process tick-by-tick or minute-by-minute data.
  • Complex Logic: Supports bracket orders, stop-losses, and dynamic position sizing that depends on portfolio state.
  • Production Similarity: The same code can be reused for live trading with minimal changes.

The Architecture

An event-driven backtester is composed of independent modules that communicate through an event queue.

The Event Loop

The heart of the system is a while loop that processes events until the data stream ends.

while True:
    if event_queue.is_empty():
        # Get next market data bar
        if data_handler.continue_backtest:
            data_handler.update_bars()
        else:
            break  # End of data

                symbol=event.symbol,
                signal=-1

class NaivePortfolio:
    def __init__(self, events, initial_capital=100000.0):
        self.events = events
        self.initial_capital = initial_capital
        self.cash = initial_capital
        self.positions = {}  # symbol -> quantity
        self.current_holdings = {}  # symbol -> market value

    def update_timeindex(self, event):
        # Update current market value of holdings
        if event.symbol in self.positions:
            self.current_holdings[event.symbol] =                 self.positions[event.symbol] * event.close

    def generate_order(self, signal_event):
        symbol = signal_event.symbol
        direction = signal_event.signal

        # Naive sizing: invest 10% of cash per signal
        target_value = self.cash * 0.10
        quantity = int(target_value / signal_event.strength)

        if direction == 1:
            order = OrderEvent(
                timestamp=signal_event.timestamp,
                symbol=symbol,
                quantity=quantity,
                direction="BUY"
            )
        elif direction == -1:
            order = OrderEvent(
                timestamp=signal_event.timestamp,
                symbol=symbol,
                quantity=quantity,
                direction="SELL"
            )
        else:
            return

        self.events.put(order)

    def update_fill(self, fill_event):
        if fill_event.direction == "BUY":
            self.positions[fill_event.symbol] =                 self.positions.get(fill_event.symbol, 0) + fill_event.quantity
            self.cash -= (fill_event.fill_price * fill_event.quantity + fill_event.commission)
        else:
            self.positions[fill_event.symbol] =                 self.positions.get(fill_event.symbol, 0) - fill_event.quantity
            self.cash += (fill_event.fill_price * fill_event.quantity - fill_event.commission)

                quantity=order_event.quantity,
                direction=order_event.direction,
                fill_price=fill_price,
                commission=commission
            )
            self.events.put(fill)

📉 Performance Metrics

A strategy that returns 50% is not necessarily better than one that returns 10%. You must measure return per unit of risk and worst-case pain. This section covers the essential metrics every quant must know.

Return Metrics

Total Return

Total Return = (Final Value / Initial Value) − 1

The simplest measure. Tells you how much money you made overall.

CAGR (Compound Annual Growth Rate)

CAGR = (Final / Initial)^(1/Years) − 1

Smooths returns into an annual figure. Allows comparison across different time horizons.

Risk-Adjusted Return Metrics

Sharpe Ratio

The most famous metric in finance. Measures excess return per unit of total volatility.

Sharpe = (Rₚ − Rբ) / σₚ

Where Rₚ is strategy return, Rբ is risk-free rate, and σₚ is strategy volatility.

  • Sharpe > 1.0: Good
  • Sharpe > 2.0: Excellent
  • Sharpe < 0: Worse than risk-free rate
Limitation: Sharpe penalizes upside volatility equally with downside volatility. A strategy with massive positive jumps will have a lower Sharpe than a boring strategy with the same return. 
def sharpe_ratio(returns, risk_free=0.02, periods=252):
    excess = returns - risk_free/periods
    return excess.mean() / excess.std() * np.sqrt(periods)

Sortino Ratio

Like Sharpe, but only penalizes downside volatility.

Sortino = (Rₚ − Rբ) / σ₋

Where σ₋ is the standard deviation of negative returns only.

def sortino_ratio(returns, risk_free=0.02, periods=252):
    excess = returns - risk_free/periods
    downside = returns[returns < 0].std() * np.sqrt(periods)
    return excess.mean() * periods / downside

Drawdown Metrics

A drawdown is the decline from a peak to a trough. It measures the pain of holding the strategy.

Maximum Drawdown (MDD)

MDD = min( (Peak − Trough) / Peak )

The worst peak-to-trough decline during the period. A strategy with 50% return but 40% max drawdown is very painful to hold.

def max_drawdown(equity_curve):
    rolling_max = equity_curve.cummax()
    drawdown = (equity_curve - rolling_max) / rolling_max
    return drawdown.min()

Calmar Ratio

CAGR divided by Max Drawdown. Measures return per unit of worst-case pain.

Calmar = CAGR / |MDD|

A Calmar > 1.0 is considered excellent. It means your annual return exceeds your worst drawdown.

Trade-Level Metrics

Win Rate

Win Rate = Winning Trades / Total Trades

How often you are right. A high win rate (>60%) is common in mean-reversion. Trend-following often has win rates <40% but wins are large.

Profit Factor

Profit Factor = Gross Profit / Gross Loss

How many dollars you make for every dollar you lose. >1.5 is good. >2.0 is excellent.

Benchmark Relative Metrics

Alpha & Beta

These measure performance relative to a benchmark (usually the S&P 500).

  • Beta (β): Measures market sensitivity. β=1 means you move with the market. β=0 means market-neutral.
  • Alpha (α): Excess return after accounting for market exposure. α > 0 means you beat the market on a risk-adjusted basis.
Strategy_Return = α + β × Benchmark_Return + ε
import numpy as np

def alpha_beta(strategy_returns, benchmark_returns):
    covariance = np.cov(strategy_returns, benchmark_returns)[0,1]
    benchmark_var = np.var(benchmark_returns)
    beta = covariance / benchmark_var
    alpha = strategy_returns.mean() - beta * benchmark_returns.mean()
    return alpha, beta

🛡️ Risk Management

A good strategy without risk management will eventually blow up. Risk management is not about avoiding losses—it is about ensuring that no single loss (or string of losses) can destroy your capital.

Position Sizing

Position sizing answers: "How many shares do I buy?" It is more important than entry timing.

Fixed Dollar Sizing

Invest a fixed amount (e.g., $10,000) per trade. Simple but does not scale with account growth.

Shares = FixedDollar / Price

Fixed Fractional Sizing

Risk a fixed percentage of your current equity on each trade. This compounds gains and reduces risk after losses.

Shares = (Equity × Fraction) / Price
position_value = portfolio.equity * 0.10  # Risk 10% of equity
shares = int(position_value / current_price)

Volatility Targeting

Scale your position so that the expected volatility of your portfolio matches a target (e.g., 10% annualized).

Position Size ∝ Target Volatility / Asset Volatility
target_vol = 0.10  # 10% annualized
asset_vol = returns.rolling(20).std().iloc[-1] * np.sqrt(252)
weight = target_vol / asset_vol  # Leverage factor
shares = int((equity * weight) / price)
Insight: In high-volatility periods (like March 2020), volatility targeting automatically reduces your exposure. In calm periods, it increases it. This is a form of dynamic risk management.

Kelly Criterion (Simplified)

The Kelly Criterion tells you the optimal fraction of your bankroll to bet given your edge.

f* = (bp − q) / b

Where b is average win/average loss, p is win probability, q = 1−p.

Warning: Full Kelly is extremely aggressive and leads to massive drawdowns. Professional quants use Half Kelly or Quarter Kelly (divide f* by 2 or 4). 
def kelly_fraction(win_rate, avg_win, avg_loss):
    b = avg_win / avg_loss
    p = win_rate
    q = 1 - p
    return (b * p - q) / b

Stop Losses

A stop loss is a pre-defined price at which you exit a losing trade. It limits the damage from being wrong.

Fixed Stop Loss

Exit if the price moves against you by X%.

stop_price = entry_price * (1 - 0.05)  # 5% stop

Trailing Stop

The stop price moves up with the asset. Locks in profits while limiting downside.

trailing_stop = highest_price_since_entry * (1 - 0.05)

Drawdown Control

Equity Curve Trading

Reduce position size (or stop trading entirely) when your account is in drawdown beyond a threshold.

current_dd = (equity - equity_peak) / equity_peak
if current_dd < -0.10:  # In 10% drawdown
    position_size *= 0.5  # Halve size
if current_dd < -0.20:  # In 20% drawdown
    position_size = 0  # Stop trading
Analogy: Drawdown control is like a circuit breaker in your house. When the current (risk) gets too high, the breaker trips to prevent a fire (ruin).

Correlation & Diversification

Risk is not just about one asset—it is about how assets move together.

  • Diversification: Spread capital across uncorrelated strategies or assets.
  • Correlation Breakdown: During crises (2008, 2020), correlations often spike to +1.0. Your diversification disappears when you need it most.
  • Risk Parity: Allocate capital so that each asset contributes equally to portfolio risk (not equal dollar amounts).

🧪 Bias, Overfitting & Robustness

The difference between a good quant and a bad quant is not finding strategies—it is avoiding false strategies that look good due to luck or bias. This section teaches you to be skeptical of your own results.

The Four Horsemen of Bad Backtests

1. Look-Ahead Bias

Using information that was not available at the time of the trade.

  • Example: Trading at today's close based on today's close. You only know the close after the market closes.
  • Example: Using quarterly earnings announced after market close to trade at that day's close.
  • Fix: Always shift signals by at least 1 period. Use .shift(1) or trade at next open.
Test: If your backtest returns 500% per year, you probably have look-ahead bias. No real strategy does that.

2. Survivorship Bias

Testing only on companies that are still alive today. Dead companies (delisted, bankrupt) are excluded.

  • Effect: Your backtest only sees winners. It ignores all the stocks that went to zero.
  • Fix: Use datasets that include delisted stocks (e.g., CRSP, Norgate Data). If using free data, be aware your results are optimistic.
Analogy: Survivorship bias is like interviewing millionaires and asking their secrets, while ignoring all the people who tried the same things and went bankrupt. You only see the survivors.

3. Data Snooping (Overfitting)

Testing so many strategies, parameters, or indicators that one is bound to look good by random chance.

  • Example: Testing 1000 parameter combinations and picking the best one. The best one is optimized for noise, not signal.
  • Example: Adding rules until the backtest looks good. "If it rains on Tuesday and the RSI is prime, buy."
  • Fix: Out-of-sample testing, walk-forward analysis, and adjusting for multiple comparisons (Bonferroni, False Discovery Rate).

4. Overfitting

The strategy fits the noise in the historical data rather than the underlying signal.

  • Symptoms: Too many parameters, perfect in-sample performance, terrible out-of-sample performance.
  • Rule of Thumb: You need at least 100 observations per parameter. A strategy with 5 parameters needs 500+ trades to be trustworthy.

Robustness Checks

Monte Carlo Simulation

Shuffle the order of your returns 1000 times and recompute metrics. If your strategy relies on a specific sequence of trades, it is fragile.

def monte_carlo(returns, n_sims=1000):
    final_values = []
    for _ in range(n_sims):
        shuffled = np.random.permutation(returns)
        equity = (1 + shuffled).cumprod()
        final_values.append(equity[-1])
    return np.percentile(final_values, [5, 50, 95])

🔬 Optimization & Walk-Forward Analysis

Finding good parameters is necessary, but optimizing them naively leads to overfitting. Walk-Forward Analysis (WFA) is the gold standard for robust parameter selection.

Naive Optimization: Grid Search

Test every combination of parameters and pick the best. This is fast but dangerous.

best_sharpe = -np.inf
best_params = None

for fast in range(10, 101, 10):
    for slow in range(fast+20, 301, 20):
        sharpe = backtest(fast, slow)
        if sharpe > best_sharpe:
            best_sharpe = sharpe
            best_params = (fast, slow)

print(f"Best: {best_params} with Sharpe {best_sharpe}")
Problem: With 10×10 = 100 combinations, you have a high chance that the "best" one is lucky. This is data snooping.

Walk-Forward Analysis (WFA)

WFA simulates how you would actually trade: optimize on recent history, trade for a short period, then repeat.

WFA Algorithm

  1. Split data into N windows.
  2. For each window i:
    • Optimize parameters on the Train portion.
    • Run the strategy with those parameters on the Test portion. Do not re-optimize during the test.
  3. Concatenate all Test portions to form the final out-of-sample equity curve.
Key Insight: WFA mimics reality. You optimize on the past, trade for a while, then re-optimize. If the strategy fails WFA, it will fail in live trading.

Implementation

def walk_forward(df, train_size=252*3, test_size=252, fast_range=range(10,101,10), slow_range=range(100,301,20)):
    n = len(df)
    test_returns = []

    for start in range(0, n - train_size - test_size, test_size):
        train = df.iloc[start : start+train_size]
        test = df.iloc[start+train_size : start+train_size+test_size]

        # Optimize on train
        best_sharpe = -np.inf
        best_fast, best_slow = 50, 200
        for fast in fast_range:
            for slow in slow_range:
                if fast >= slow: continue
                sharpe = backtest(train, fast, slow)
                if sharpe > best_sharpe:
                    best_sharpe = sharpe
                    best_fast, best_slow = fast, slow

        # Test with best params (fixed!)
        test_ret = backtest_returns(test, best_fast, best_slow)
        test_returns.extend(test_ret)

    return pd.Series(test_returns)

Parameter Stability

A robust strategy should have optimal parameters that are stable across windows. If the best fast SMA jumps from 10 to 90 between windows, the strategy is fragile.

⚙️ Execution, Slippage & Costs

A backtest that ignores costs is a fantasy. In reality, every trade costs you money. This section teaches you to model the difference between theoretical P&L and realized P&L.

Types of Orders

Market Order

Execute immediately at the best available price. Guaranteed fill, uncertain price.

  • Pros: Always filled.
  • Cons: Subject to slippage, especially in low-volume periods.

Limit Order

Execute only at a specified price or better. Guaranteed price, uncertain fill.

  • Pros: No slippage (if filled).
  • Cons: May never be filled. In fast markets, price may run away from you.

The Cost Trinity

Every trade has three costs:

  1. Commission: The broker's fee. Often fixed per trade or per share.
  2. Slippage: The difference between expected fill price and actual fill price.
  3. Market Impact: Your own order moves the price. Significant for large orders.

Modeling Slippage

Fixed Slippage

Add a fixed percentage to every trade. Simple but unrealistic.

fill_price = close_price * (1 + 0.001)  # 0.1% slippage

Volume-Weighted Slippage

More realistic. Slippage increases when your order size is large relative to market volume.

volume_ratio = order_size / daily_volume
slippage = 0.0001 + 0.001 * volume_ratio  # Base + variable
fill_price = close_price * (1 + slippage)

Market Impact (Square-Root Law)

For large institutional orders, market impact follows a square-root relationship with order size.

Impact = σ × √(Order Size / Daily Volume)

Where σ is daily volatility. This is used by execution algorithms (TWAP, VWAP) to minimize impact.

Commission Models

ModelStructureBest For
Fixed per trade$5 per trade regardless of sizeRetail investors, large trades
Per share$0.005 per shareSmall trades, high frequency

Execution Algorithms

  • TWAP (Time-Weighted Average Price): Slice the order evenly over time. Simple, predictable.
  • VWAP (Volume-Weighted Average Price): Slice the order proportionally to historical volume profile. Harder to detect.
  • Implementation Shortfall: Balance market impact vs. timing risk. More aggressive early, then passive.

def backtest(ticker, start, end, fast=50, slow=200, commission=0.001):
    df = yf.download(ticker, start=start, end=end)
    df["SMA_F"] = df["Adj Close"].rolling(fast).mean()
    df["SMA_S"] = df["Adj Close"].rolling(slow).mean()
    df["Signal"] = np.where(df["SMA_F"] > df["SMA_S"], 1, -1)
    df["Position"] = df["Signal"].shift(1)
    df["AssetRet"] = df["Adj Close"].pct_change()
    df["StratRet"] = df["Position"] * df["AssetRet"]
    df["Trades"] = df["Position"].diff().abs()
    df["NetRet"] = df["StratRet"] - df["Trades"] * commission
    df["Cum_BH"] = (1 + df["AssetRet"]).cumprod()
    df["Cum_Strat"] = (1 + df["NetRet"]).cumprod()

df = yf.download("SPY", start="2010-01-01", end="2024-01-01")
df["Returns"] = df["Adj Close"].pct_change()

def bt_sharpe(data, f, s):
    data = data.copy()
    data["SF"] = data["Adj Close"].rolling(f).mean()
    data["SS"] = data["Adj Close"].rolling(s).mean()
    data["Sig"] = np.where(data["SF"] > data["SS"], 1, -1)
    data["Pos"] = data["Sig"].shift(1)
    data["SR"] = data["Pos"] * data["Returns"]
    data = data.dropna()
    if data["SR"].std() == 0: return -999
    return (data["SR"].mean() / data["SR"].std()) * np.sqrt(252)

train_sz, test_sz = 252*3, 252
n = len(df)
test_rets = []

for start in range(0, n - train_sz - test_sz, test_sz):
    train = df.iloc[start:start+train_sz]
    test = df.iloc[start+train_sz:start+train_sz+test_sz]
    best_s, best_f = -999, (50,200)
    for f in range(20, 101, 20):
        for s in range(150, 301, 50):
            if f >= s: continue
            sh = bt_sharpe(train, f, s)
            if sh > best_s: best_s, best_f = sh, (f,s)
    test["SF"] = test["Adj Close"].rolling(best_f[0]).mean()
    test["SS"] = test["Adj Close"].rolling(best_f[1]).mean()
    test["Sig"] = np.where(test["SF"] > test["SS"], 1, -1)
    test["Pos"] = test["Sig"].shift(1)
    test_rets.extend((test["Pos"] * test["Returns"]).dropna().tolist())

wfa = (1 + pd.Series(test_rets)).cumprod()
plt.figure(figsize=(10,5))
plt.plot(wfa.values, label="Walk-Forward OOS")
plt.title("Walk-Forward Equity Curve"); plt.legend(); plt.show()
