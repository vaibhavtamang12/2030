Prerequisites & Foundation
Before diving in — concepts every Python beginner needs to understand first.

⚠️ Read This First

These topics are not Python-specific, but they are the foundation upon which everything else is built. If you understand these well, Python will feel intuitive rather than confusing.

💻 What is a Program?

A program is a list of instructions you give to a computer. Computers are incredibly fast but not smart — they do exactly what you tell them, nothing more, nothing less. A programming language like Python lets you write those instructions in a way that's readable by humans and translatable by the computer.

Think of it like a recipe: the recipe is the program, you are the programmer, and the oven is the computer. The oven doesn't "understand" the recipe — it just follows the steps you set on its dials.

🧠 What is Memory (RAM)?

Your computer has two kinds of storage: RAM (Random Access Memory) — fast, temporary, used while a program runs — and a hard drive/SSD — slow, permanent, used for files. When Python runs your program, it stores all variables and data in RAM.

📝 What is a Variable?

A variable is a named container that holds a value in memory. You can think of it as a labelled sticky note. The label is the variable name, the sticky note is the memory slot, and what's written on it is the value.

Python — Variables
# Create a variable called "city" holding the string "Paris"
city = "Paris"

# You can change what a variable holds at any time
city = "Tokyo"

# You can use the variable's name anywhere you need its value
print(city)   # Output: Tokyo

⚙️ Interpreted vs Compiled

Python is an interpreted language. This means Python reads and runs your code line-by-line, at the moment you run it. Languages like C or Java compile the whole program into machine code first, then run it. Interpretation makes Python slower to run but faster to write and test.

🐍
Python (Interpreted)

Write code → Run immediately → Python reads it line-by-line → Output. Great for learning and data work.

⚙️
C/Java (Compiled)

Write code → Compiler translates entire file → Machine code binary → Run binary. Faster at runtime but more steps.

🖥️ The Terminal / Command Line

The terminal (also called command line, shell, or console) is a text-based way to talk to your computer. Instead of clicking icons, you type commands. Python developers use the terminal constantly — to run scripts, install packages, use git, and manage environments.

🔢 Numbers: int & float

int stands for integer — any whole number. float stands for floating-point — any number with a decimal point. Python automatically handles very large integers (no overflow). Division always returns a float.

Python — Numbers
# Integer arithmetic
a = 10
b = 3
print(a + b)    # 13   — addition
print(a - b)    # 7    — subtraction
print(a * b)    # 30   — multiplication
print(a / b)    # 3.333... — TRUE division → always float
print(a // b)   # 3   — FLOOR division → drops decimal
print(a % b)    # 1   — MODULUS (remainder)
print(a ** b)   # 1000 — EXPONENT (10 to the power 3)

# Float gotcha — this is a famous one!
print(0.1 + 0.2)   # 0.30000000000000004  (floating-point precision!)

# Type conversion
x = int(3.9)   # x = 3  (truncates, not rounds)
y = float(5)   # y = 5.0
⚠️
The 0.1 + 0.2 Surprise

Computers store floats in binary and can't represent 0.1 exactly — it's like trying to write 1/3 in decimal. This is why 0.1 + 0.2 ≠ 0.3 exactly. Use the round() function or Python's decimal module when precision matters.

📝 Strings: str

A string is a sequence of characters — text. You wrap it in single quotes '...' or double quotes "...". They behave identically; convention is to pick one and be consistent.

Python — Strings
name = "Alice"
greeting = 'Hello'

# Concatenation — gluing strings together
message = greeting + " " + name    # "Hello Alice"

# f-strings — the modern, best way to embed variables in strings
age = 30
intro = f"My name is {name} and I am {age} years old."
print(intro)   # My name is Alice and I am 30 years old.

# String methods — built-in operations on strings
print(name.upper())        # "ALICE"
print(name.lower())        # "alice"
print(name.replace("A","E")) # "Elice"
print(len(name))           # 5  (number of characters)
print(name[0])             # "A"  (indexing starts at 0!)
print(name[1:3])            # "li" (slicing — characters 1 up to but not including 3)
print(name[-1])             # "e"  (last character)

# Multi-line strings with triple quotes
poem = """Roses are red,
Violets are blue,
Python is fun."""

✅ Booleans: bool

A boolean is the simplest type: it's either True or False. Booleans are what comparisons produce, and they control which branches of code run.

Python — Booleans
# Comparison operators → always return bool
print(5 > 3)       # True
print(5 == 5)      # True  (== checks equality, = assigns!)
print(5 != 3)      # True  (not equal)
print(5 < 3)       # False

# Logical operators — combine conditions
print(True and True)   # True  (both must be True)
print(True or False)  # True  (at least one must be True)
print(not True)        # False (flips the boolean)

# Truthy and Falsy — important Python concept!
# These are all "falsy" (behave like False in conditions):
# False, 0, 0.0, "", [], {}, set(), None
# Everything else is "truthy"
bool(0)    # False
bool("")   # False
bool(42)   # True
bool("hi") # True

📋 Lists

A list is an ordered, changeable (mutable) collection of items. Items can be of any type, even mixed. Lists are defined with square brackets [ ].

Python — Lists
fruits = ["apple", "banana", "cherry"]

# Accessing items
print(fruits[0])       # "apple"  (first item)
print(fruits[-1])      # "cherry" (last item)
print(fruits[1:3])     # ["banana", "cherry"] (slicing)

# Modifying lists — lists ARE mutable (changeable)
fruits[0] = "mango"    # Replace first item
fruits.append("grape") # Add to end
fruits.insert(1, "kiwi") # Insert at index 1
fruits.remove("banana") # Remove first occurrence
popped = fruits.pop()    # Remove & return last item

# Useful list operations
print(len(fruits))     # Number of items
print("mango" in fruits) # True — membership check
fruits.sort()           # Sort in place (alphabetically)
print(fruits.count("mango")) # Count occurrences

🔒 Tuples

A tuple is exactly like a list but immutable — you cannot change it after creation. Use tuples when data should not change (like coordinates, RGB colors, database records). Defined with parentheses ( ).

🔒 Immutable — Tuple
point = (3, 7)
point[0] = 5 ← ERROR!
Once created, cannot be changed. Data is safe.
✏️ Mutable — List
coords = [3, 7]
coords[0] = 5 ← Works!
Contents can be modified freely at any time.

🗂️ Dictionaries: dict

A dictionary stores data as key → value pairs. Think of a real dictionary: you look up a word (key) to get its definition (value). Dictionaries are incredibly useful and very common in Python. Defined with curly braces { }.

Python — Dictionaries
person = {
    "name": "Alice",
    "age": 30,
    "city": "Paris"
}

# Access by key
print(person["name"])          # "Alice"
print(person.get("job", "N/A")) # "N/A" — safe access with default

# Add / update
person["email"] = "alice@ex.com"
person["age"] = 31

# Delete
del person["city"]

# Iterate over key-value pairs
for key, value in person.items():
    print(f"{key}: {value}")

# Check if key exists
print("name" in person)   # True

# All keys, all values
print(person.keys())     # dict_keys(['name', 'age', 'email'])
print(person.values())   # dict_values(['Alice', 31, 'alice@ex.com'])

🔵 Sets

A set is an unordered collection of unique items — duplicates are automatically removed. Sets support fast membership testing and mathematical set operations like union, intersection, and difference.

Python — Sets
numbers = {1, 2, 3, 2, 1}   # duplicates removed!
print(numbers)  # {1, 2, 3}

# Classic use: remove duplicates from a list
my_list = [1, 2, 2, 3, 4, 3]
unique = list(set(my_list))   # [1, 2, 3, 4]

# Set operations
a = {1, 2, 3, 4}
b = {3, 4, 5, 6}
print(a | b)    # Union: {1,2,3,4,5,6}
print(a & b)    # Intersection: {3,4}
print(a - b)    # Difference: {1,2}

Control Flow
Control flow decides which lines of code run, when, and how many times.

🔀 What is Control Flow?

Without control flow, a program would just run every line from top to bottom, once, and stop. Control flow gives you decisions (if/else) and repetition (loops). It's what makes programs actually useful and intelligent.

🤔 Conditionals: if / elif / else

An if statement says: "run this code block only IF this condition is True." You can chain conditions with elif ("else if") and handle every other case with else.

Python uses indentation (spaces) to define blocks of code — this is mandatory, not stylistic. Convention is 4 spaces per level.

if / elif / else — Decision Flow
Start: grade = 75
grade >= 90?
No →
Check next: grade >= 75?
← Yes → print "A"
grade >= 75?
No → print "C" Yes → print "B"
Python — if / elif / else
grade = 75

if grade >= 90:
    print("A — Excellent!")
elif grade >= 75:
    print("B — Good job")         # ← This runs (75 >= 75 is True)
elif grade >= 60:
    print("C — Passing")
else:
    print("F — Please retry")
# Output: B — Good job

# One-liner: ternary expression (use sparingly)
status = "pass" if grade >= 60 else "fail"

# Nested if
x = 5
if x > 0:
    if x > 10:
        print("Big positive")
    else:
        print("Small positive")   # ← This runs
🚨
= vs == — The Most Common Beginner Bug

= is assignment (set a value). == is comparison (check equality). Writing if x = 5: is a syntax error. Always use == inside conditions.

🔁 for Loops

A for loop repeats a block of code for each item in a sequence (list, string, range, etc.). The word "for" in English maps perfectly: "for each fruit in fruits, do this."

Python — for loops
# Loop over a list
fruits = ["apple", "banana", "cherry"]
for fruit in fruits:
    print(f"I like {fruit}")

# Loop a specific number of times with range()
for i in range(5):        # i goes 0, 1, 2, 3, 4
    print(i)

for i in range(2, 10, 2):  # start=2, stop=10, step=2 → 2,4,6,8
    print(i)

# Loop with index AND value using enumerate()
for index, fruit in enumerate(fruits):
    print(f"{index}: {fruit}")

# Loop over a string (character by character)
for char in "Python":
    print(char)

# Loop over dict items
person = {"name": "Bob", "age": 25}
for key, val in person.items():
    print(f"{key} = {val}")

🔄 while Loops

A while loop keeps running as long as a condition is True. You use it when you don't know in advance how many times the loop should run.

Python — while loops
# Count down
count = 5
while count > 0:
    print(f"Countdown: {count}")
    count -= 1   # ALWAYS update the condition variable or you get infinite loop!
print("Blast off!")

# while loop for user input (common pattern)
# password = ""
# while password != "secret":
#     password = input("Enter password: ")
# print("Access granted!")
⚠️
Infinite Loops

If a while loop's condition never becomes False, it runs forever. Always make sure you're updating the variable that controls the condition. Press Ctrl+C to stop a runaway program.

🛑 break, continue, pass

Python — Loop Control
# break — exit the loop immediately
for num in [1, 2, 3, 4, 5]:
    if num == 3:
        break
    print(num)
# Output: 1, 2

# continue — skip rest of this iteration, continue loop
for num in [1, 2, 3, 4, 5]:
    if num == 3:
        continue
    print(num)
# Output: 1, 2, 4, 5  (3 is skipped)

# pass — do nothing placeholder (for empty blocks)
for i in range(5):
    pass   # useful while building code structure

Functions
Functions are reusable blocks of code. Define once, use anywhere, avoid repetition.

📦 What is a Function?

Imagine you write the same 10 lines of code in 5 different places. If you need to fix a bug, you need to fix it in 5 places. A function solves this: write the 10 lines once, give them a name, and call that name whenever you need them.

Functions also make code easier to read. calculate_total_price(items) is far more readable than the 10 lines of arithmetic it might contain.

📐 Anatomy of a Function

Function anatomy — labeled breakdown
def greet(name, greeting="Hello"): ← keyword · name · parameters · default value
"""Returns a personalised greeting.""" ← docstring (documentation)
result = greeting + " " + name ← function body (logic)
return result ← return statement (sends value back)
msg = greet("Alice") ← function call with argument
Python — Defining & Calling Functions
# Simplest function — no parameters, no return
def say_hello():
    print("Hello, World!")

say_hello()   # Call the function

# Function with parameters and return value
def add(a, b):
    return a + b

result = add(3, 5)
print(result)   # 8

# Function with a default parameter
def power(base, exponent=2):   # exponent defaults to 2
    return base ** exponent

print(power(3))      # 9  (uses default exponent=2)
print(power(2, 10))   # 1024

📥 Parameters vs Arguments

These words are often used interchangeably, but technically: parameters are the variable names in the function definition. Arguments are the actual values you pass in when you call the function.

📌
Parameter

def greet(name): — name is the parameter. It's a placeholder that lives inside the function definition.

📌
Argument

greet("Alice") — "Alice" is the argument. It's the actual value passed at call time.

🔑 Keyword Arguments & *args / **kwargs

Python — Advanced Parameters
# Keyword arguments — pass by name (order doesn't matter)
def describe(name, age, city):
    print(f"{name}, {age}, from {city}")

describe(age=30, city="Paris", name="Alice")  # Works!

# *args — accept any number of positional arguments as a tuple
def sum_all(*numbers):
    return sum(numbers)

print(sum_all(1, 2, 3))        # 6
print(sum_all(10, 20, 30, 40)) # 100

# **kwargs — accept any number of keyword arguments as a dict
def print_info(**details):
    for key, val in details.items():
        print(f"  {key}: {val}")

print_info(name="Bob", job="Dev", country="India")

🗺️ Scope: Where Variables Live

Scope determines where a variable is accessible. Python uses the LEGB rule: when you use a variable name, Python searches: Local → Enclosing → Global → Built-in.

LEGB Scope — nested zones
🌍 GLOBAL — module level
x = 10 # global variable
📦 LOCAL — inside outer_func
y = 20 # local to outer_func
🔮 ENCLOSING — inside inner_func (can see y)
z = 30 # local to inner_func
# Can access x, y, z from here
Python — Scope Demo
x = "global"

def outer():
    x = "enclosing"
    def inner():
        x = "local"
        print(x)   # "local" — finds local x first
    inner()
    print(x)       # "enclosing"

outer()
print(x)           # "global"

# Use global keyword to modify a global variable inside a function
counter = 0
def increment():
    global counter
    counter += 1
increment()
print(counter)   # 1

⚡ Lambda Functions

A lambda is a tiny, anonymous (nameless) function defined on a single line. They're useful when you need a simple function in a specific place and don't want to write a full def.

Python — Lambda
# Regular function
def double(x):
    return x * 2

# Equivalent lambda
double = lambda x: x * 2

# Most common use: as key function for sorting
words = ["banana", "apple", "kiwi", "cherry"]
words.sort(key=lambda word: len(word))  # sort by length
print(words)  # ['kiwi', 'apple', 'banana', 'cherry']

# Lambda with multiple params
multiply = lambda x, y: x * y
print(multiply(3, 4))   # 12

Comprehensions
Write powerful data transformations in one clean, readable line — a distinctly Pythonic feature.

✨ What is a Comprehension?

A comprehension is a concise way to create a new collection (list, set, or dictionary) by transforming or filtering an existing one — in a single expression. They replace the common pattern of: create empty list → loop → append item → done.

// The Pattern — Deconstructed

Expression
x * 2
[
for clause
for x
iterable
in numbers
optional filter
if x > 0
]

Read as: "give me x times 2, for each x in numbers, but only if x is positive"

📋 List Comprehensions

Python — List Comprehensions
# Traditional loop approach:
squares = []
for n in range(1, 6):
    squares.append(n ** 2)
# squares = [1, 4, 9, 16, 25]

# ✅ Comprehension — same result, one line
squares = [n ** 2 for n in range(1, 6)]
# [1, 4, 9, 16, 25]

# With condition (filter): only even squares
even_squares = [n ** 2 for n in range(1, 11) if n % 2 == 0]
# [4, 16, 36, 64, 100]

# String manipulation: uppercase words from a sentence
words = ["hello", "world", "python"]
upper = [w.upper() for w in words]
# ['HELLO', 'WORLD', 'PYTHON']

# Filter: only words longer than 4 characters
long_words = [w for w in words if len(w) > 4]
# ['hello', 'world', 'python']

# Nested comprehension (creates pairs)
pairs = [(x, y) for x in [1,2,3] for y in ["a","b"]]
# [(1,'a'), (1,'b'), (2,'a'), (2,'b'), (3,'a'), (3,'b')]

🗂️ Dictionary Comprehensions

Python — Dict Comprehensions
# Pattern: {key_expr: value_expr for item in iterable}

# Word → length mapping
words = ["python", "is", "awesome"]
lengths = {w: len(w) for w in words}
# {'python': 6, 'is': 2, 'awesome': 7}

# Square numbers dict: {n: n²}
sq_dict = {n: n**2 for n in range(1, 6)}
# {1: 1, 2: 4, 3: 9, 4: 16, 5: 25}

# Swap keys and values
original = {"a": 1, "b": 2, "c": 3}
swapped = {v: k for k, v in original.items()}
# {1: 'a', 2: 'b', 3: 'c'}

🔵 Set Comprehensions

Python — Set Comprehensions
# Like list comprehension but with { } → gives unique values
numbers = [1, 2, 2, 3, 3, 4]
unique_squares = {n**2 for n in numbers}
# {1, 4, 9, 16}  ← duplicates from n=2 and n=3 removed

# First letters (unique)
words = ["apple", "apricot", "banana", "blueberry"]
first_letters = {w[0] for w in words}
# {'a', 'b'}

🌀 Generator Expressions

A generator expression looks like a list comprehension but uses ( ) instead of [ ]. The key difference: it doesn't build the entire list in memory — it generates items one at a time, lazily. This is memory-efficient for large datasets.

Python — Generator Expressions
# List comprehension — ALL items stored in memory immediately
big_list = [x**2 for x in range(1000000)]  # uses lots of RAM

# Generator expression — generates one item at a time
big_gen = (x**2 for x in range(1000000))    # uses tiny RAM

# sum() works great with generators
total = sum(x**2 for x in range(100))
print(total)  # 328350
💡
Comprehension Readability Rule

If your comprehension is longer than one line or hard to read — just use a regular for loop. Readability > cleverness. The best code is the code others (and future-you) can understand.

Modules & Packages
Don't reinvent the wheel — import code that already exists. Python's superpower is its ecosystem.

📁 What is a Module?

A module is simply a Python file (.py) that contains functions, variables, and classes you can reuse in other Python files. When you create a file called math_utils.py, that file IS a module.

A package is a folder containing multiple modules (and a special __init__.py file that tells Python "this folder is a package").

Project structure — modules & packages
📁 my_project/
📁 utils/              ← a package
🐍 __init__.py    ← marks folder as package
🐍 math_utils.py ← module you wrote
🐍 string_utils.py
🐍 main.py             ← your main script
📄 requirements.txt   ← lists dependencies

📥 The import Statement

Python — Importing
# 1. Import the whole module
import math
print(math.pi)         # 3.141592653589793
print(math.sqrt(16))   # 4.0

# 2. Import specific names FROM a module
from math import pi, sqrt, ceil
print(pi)           # 3.14... (no "math." prefix needed)
print(sqrt(25))     # 5.0

# 3. Import with an alias (rename)
import numpy as np        # Very common in data science
import pandas as pd
from datetime import datetime as dt

# 4. Import from your OWN module (same folder)
# Suppose you have a file "math_utils.py" with a function "double()":
from math_utils import double
print(double(5))   # 10

📝 Creating Your Own Module

Python — my_utils.py (your module file)
# File: my_utils.py

def greet(name):
    return f"Hello, {name}!"

def is_palindrome(word):
    return word == word[::-1]

PI = 3.14159

# This block runs ONLY when you run this file directly
# NOT when it's imported by another file
if __name__ == "__main__":
    print(greet("World"))     # Only runs when: python my_utils.py
Python — main.py (imports your module)
# File: main.py
from my_utils import greet, is_palindrome

print(greet("Alice"))           # "Hello, Alice!"
print(is_palindrome("racecar")) # True
print(is_palindrome("python"))  # False
🔑
The __name__ == "__main__" Pattern

When Python runs a file directly, it sets __name__ to "__main__". When imported, __name__ is the module's filename. This pattern lets you have test code that only runs when you execute the file directly, not when it's imported.

Imagine Project A needs numpy 1.20 and Project B needs numpy 2.0. If you install globally, you can only have one version — breaking one project when the other needs a different one. Virtual environments solve this by giving each project its own isolated box of packages.

venv comes with Python 3.3+ — no installation needed. It creates a lightweight isolated environment in a folder within your project.

💡
Add .venv to .gitignore

Never commit your .venv folder to git — it's large and machine-specific. Instead, commit requirements.txt. Add .venv/ to your .gitignore file.

Git Basics
Git is time travel for your code — track every change, collaborate with others, recover anything.

🤔 Why Git?

Imagine writing a long essay and saving it as essay_final.docx, then essay_final_v2.docx, then essay_REALLY_final.docx. That's version control without a tool. Git does this properly: it takes snapshots of your project at every point you tell it to, and lets you revisit, compare, or restore any point in history.

🗺️ Core Concepts

Repository (repo)
Your project folder that Git is tracking. Contains all files + the full history.
git init / git clone
Commit
A snapshot of your project at a moment in time. Like a save point.
git commit -m "message"
Staging Area
A waiting room. You add files here before committing them.
git add filename
Branch
A parallel version of the code. Work on features without breaking main code.
git branch feature-x
Remote
A copy of your repo on a server (e.g., GitHub). Enables collaboration.
git push / git pull
Merge
Combine changes from one branch into another.
git merge feature-x

PEP 8 — Python Style Guide
Code is read far more often than it is written. PEP 8 makes Python code consistent and readable.

📖 What is PEP 8?

PEP stands for Python Enhancement Proposal. PEP 8 is the official style guide written by Python's creator Guido van Rossum. It defines conventions for formatting Python code so that all Python code looks consistent, regardless of who wrote it.

Following PEP 8 matters because: it makes your code easier to read, it shows professional maturity, and most companies require it for contributions.

🔑
The Zen of Python

Run import this in Python to see the guiding philosophy. Key lines: "Readability counts." · "Beautiful is better than ugly." · "Explicit is better than implicit."

💬 Comments & Docstrings

Python — Comments & Docstrings
# Inline comment — space after #, explains WHY not WHAT

# Block comment — explains a section of code

# Docstring — documents a function (triple quotes)
def calculate_area(radius):
    """
    Calculate the area of a circle.

    Args:
        radius (float): The radius of the circle.

    Returns:
        float: The area (π × r²).

    Example:
        >>> calculate_area(5)
        78.53981633974483
    """
    return math.pi * radius ** 2

📏 Spacing Rules

Python — Spacing Rules
# ✅ Space around operators (except in defaults)
x = y + z
a, b = b, a

# ✅ No space before colon in slices
items[1:3]
items[::2]

# ✅ No space around = for default params
def func(x, y=10): ...   # NOT y = 10

# ✅ Two blank lines between top-level functions

def function_one():
    pass


def function_two():   # ← two blank lines above
    pass

# ✅ One blank line between methods in a class
class MyClass:
    def method_one(self):
        pass

    def method_two(self):   # ← one blank line
        pass

# ✅ Imports at the top, one per line, grouped
import os
import sys
# blank line
import numpy as np
import pandas as pd
# blank line
from my_module import helper

Type Hints
Annotate your code with expected types — for documentation, editor support, and catching bugs early.

🏷️ What are Type Hints?

Python is dynamically typed — you don't have to declare types, and variables can hold any type. But type hints (introduced in Python 3.5, improved each version) let you optionally annotate what types functions expect and return. They are purely informational at runtime — Python doesn't enforce them — but editors and tools use them to catch bugs before you run the code.

Think of them as a contract: "this function takes a str and returns an int" — clear communication for yourself and anyone reading your code.

📐 Basic Syntax

Variable annotation
name: str = "Alice"
Function parameter
def greet(name: str):
Return type
def add() -> int:
Python — Type Hints: Basic
# Without type hints (valid Python, but opaque)
def add(a, b):
    return a + b

# With type hints (much clearer!)
def add(a: int, b: int) -> int:
    return a + b

# Variable annotations
name: str = "Alice"
age: int = 30
height: float = 5.7
active: bool = True

# Function with multiple params and return type
def create_greeting(name: str, times: int = 1) -> str:
    """Return a greeting repeated 'times' times."""
    return (f"Hello, {name}! " * times).strip()

# Function that returns nothing
def log_message(msg: str) -> None:
    print(f"[LOG] {msg}")

📋 Collection Types

Python 3.9+ — Collection Type Hints
# Python 3.9+ — use built-in names directly (no imports!)
def process_names(names: list[str]) -> list[str]:
    return [name.upper() for name in names]

def count_words(text: str) -> dict[str, int]:
    # Returns {word: count}
    words = text.split()
    return {word: words.count(word) for word in set(words)}

def get_dimensions() -> tuple[int, int]:
    return (1920, 1080)

def unique_tags(posts: list[str]) -> set[str]:
    return set(posts)

# Nested types
matrix: list[list[int]] = [[1, 2], [3, 4]]

❓ Optional & Union Types

Python — Optional, Union, Any
from typing import Optional, Union, Any

# Optional[X] means X or None
def find_user(user_id: int) -> Optional[str]:
    # Returns the name or None if not found
    ...

# Python 3.10+ shorthand: X | None (cleaner!)
def find_user(user_id: int) -> str | None:
    ...

# Union — can be one of multiple types
def process(value: Union[int, str]) -> str:
    return str(value)

# Python 3.10+ shorthand for Union
def process(value: int | str) -> str:
    return str(value)

# Any — opts out of type checking (use sparingly)
def flexible(data: Any) -> Any:
    return data

🎓 Real-World Complete Example

Python — Type Hints in a Real Context
from typing import Optional

def calculate_bmi(weight_kg: float, height_m: float) -> float:
    """Calculate Body Mass Index."""
    return weight_kg / (height_m ** 2)

def bmi_category(bmi: float) -> str:
    """Return BMI category as a string."""
    if bmi < 18.5:
        return "Underweight"
    elif bmi < 25:
        return "Normal"
    elif bmi < 30:
        return "Overweight"
    else:
        return "Obese"

def full_health_report(
    name: str,
    weight_kg: float,
    height_m: float,
    age: Optional[int] = None
) -> dict[str, str | float]:
    """Generate a health report dictionary."""
    bmi = calculate_bmi(weight_kg, height_m)
    report: dict[str, str | float] = {
        "name": name,
        "bmi": round(bmi, 2),
        "category": bmi_category(bmi),
    }
    if age is not None:
        report["age"] = age
    return report

# Usage
report = full_health_report("Alice", 65, 1.70, age=30)
print(report)
# {'name': 'Alice', 'bmi': 22.49, 'category': 'Normal', 'age': 30}
💡
When to use type hints?

Always annotate function signatures (parameters + return type). Variable annotations are optional — add them when the type is not obvious. You don't need to annotate every line, just enough for clarity and tool support.
