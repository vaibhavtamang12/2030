Prerequisites — The Foundation Before SciPy & Quant Stats

Goal: You do not need to be an expert in these, but you must be comfortable with the ideas below. We explain each in plain language.

1. Python & NumPy Comfort

SciPy sits on top of NumPy. NumPy gives us fast arrays (ndarray) and vectorized math. If you can create arrays, slice them, and do element-wise operations, you are ready.

2. Basic Calculus (Derivatives & Integrals)

Derivative = slope of a curve at a point. In quant work, derivatives tell us how sensitive a price is to a small change (the "Greeks").
Integral = area under a curve. In probability, the area under a density curve between two points is the probability of landing in that range.

3. Linear Algebra Basics

Vectors are lists of numbers. Matrices are tables of numbers. In quant finance, a portfolio is a vector of weights; a covariance matrix tells us how assets move together. You need to know:

  • Dot product (weighted sum)
  • Matrix multiplication
  • Transpose and inverse
  • Eigenvalues (measure of "stretching" in data directions)

4. Probability & Statistics Intuition

Before using scipy.stats, know these ideas in words:

Random Variable

A number that comes from a random process. Example: tomorrow's stock return.

PDF vs CDF

PDF = density at a point (height). CDF = cumulative probability up to that point (area).

Expected Value

The long-run average. Weight each outcome by its probability.

Variance & Std Dev

How spread out outcomes are. Std dev is in the same units as the original data.

5. Basic Finance Vocabulary

  • Return: (Pt - Pt-1) / Pt-1
  • Volatility: Standard deviation of returns
  • Correlation: How two assets move together (-1 to +1)
  • Sharpe Ratio: Return per unit of risk

scipy.special & scipy.integrate

scipy.special — Mathematical Functions

Quant finance uses functions that are not in the standard library: the error function (used in Black-Scholes), gamma function (used in skewed distributions), and Bessel functions (used in interest rate models).

Colab Experiment — Error Function & Black-Scholes
import numpy as np
from scipy import special

# The standard normal CDF is (1 + erf(x/sqrt(2))) / 2
def norm_cdf(x):
    return 0.5 * (1 + special.erf(x / np.sqrt(2)))

print("N(0):", norm_cdf(0.0))
print("N(1.96):", norm_cdf(1.96))

# Gamma function generalizes factorial: gamma(n) = (n-1)!
print("Gamma(5) = 4! =", special.gamma(5))

scipy.integrate — Numerical Integration

Many quant problems have no closed-form solution. We approximate integrals numerically.

quad

General-purpose single integration. Use for expected values and option pricing.

dblquad / tplquad

Double and triple integration. Used in multi-asset option pricing.

odeint / solve_ivp

Solve ordinary differential equations. Used in stochastic differential equation approximations.

simps / trapz

Integrate sampled data. Use when you only have discrete prices/rates.

Colab Experiment — Integrate a PDF to get Probability
from scipy import integrate, stats

# Integrate the standard normal PDF from -1 to +1
# This should equal approx 0.6827 (68-95-99.7 rule)
result, error = integrate.quad(stats.norm.pdf, -1, 1)
print("Probability within 1 std:", result)
print("Integration error estimate:", error)

# Expected value of x^2 under standard normal
# Theory says E[x^2] = Var(x) = 1
result2, _ = integrate.quad(lambda x: x**2 * stats.norm.pdf(x), -np.inf, np.inf)
print("E[x^2] =", result2)
Tip: quad accepts np.inf as limits. It adaptively chooses points to minimize error.

scipy.optimize — Finding the Best Solution

Optimization is the engine of quantitative finance. We use it to:

  • Fit models: Find parameters that make a model closest to data (Least Squares, MLE).
  • Build portfolios: Maximize return for a given risk, or minimize risk for a given return.
  • Calibrate curves: Find parameters that make a yield curve model match market prices.

Core Concepts

ConceptMeaningQuant Example
Objective FunctionThe number you want to minimize or maximizePortfolio variance
ConstraintsRules the solution must obeyWeights must sum to 1
BoundsHard limits on each variableNo short selling: 0 ≤ w ≤ 1
GradientDirection of steepest increaseUsed in gradient descent
HessianMatrix of second derivativesCurvature of objective function

Key Functions

minimize

General multivariate minimization. Methods: 'Nelder-Mead', 'BFGS', 'L-BFGS-B', 'SLSQP', 'trust-constr'.

minimize_scalar

For one-variable problems. Fast and robust.

curve_fit

Non-linear least squares. Fits a function to (x,y) data.

fsolve / root

Find roots of equations. Used to solve for implied volatility.

linprog

Linear programming. Used in transaction cost optimization.

differential_evolution

Global optimizer. Good for non-convex, noisy surfaces.

Colab Experiment — Minimize Portfolio Variance
import numpy as np
from scipy.optimize import minimize

# Two assets with covariance matrix
Sigma = np.array([[0.04, 0.02], [0.02, 0.03]])

# Objective: portfolio variance w^T Σ w
def variance(w):
    return w @ Sigma @ w

# Constraints: weights sum to 1
cons = {'type': 'eq', 'fun': lambda w: np.sum(w) - 1}
# Bounds: allow short selling for demo
bounds = [(-1, 1), (-1, 1)]

result = minimize(variance, x0=[0.5, 0.5], bounds=bounds, constraints=cons)
print("Optimal weights:", result.x)
print("Minimum variance:", result.fun)
Colab Experiment — Implied Volatility via root
from scipy.optimize import brentq

# Simplified Black-Scholes for call (assumes r=0, T=1)
def bs_call(S, K, sigma):
    d1 = (np.log(S/K) + 0.5*sigma**2) / sigma
    d2 = d1 - sigma
    return S*norm_cdf(d1) - K*norm_cdf(d2)

S, K, market_price = 100, 100, 10
# Find sigma such that BS price = market price
objective = lambda sigma: bs_call(S, K, sigma) - market_price
iv = brentq(objective, 1e-6, 5.0)
print("Implied Volatility:", iv)

scipy.linalg — Fast & Stable Matrix Math

While NumPy has linalg, SciPy's version is more comprehensive and often faster because it wraps more advanced LAPACK routines. In quant work, speed and numerical stability matter.

Why SciPy linalg Over NumPy linalg?

  • More decomposition options: LU, QR, Schur, Cholesky
  • Better handling of banded, triangular, and structured matrices
  • Matrix functions: expm, logm, sqrtm (used in stochastic calculus)
  • Often faster for large matrices due to optimized BLAS

Key Decompositions

DecompositionWhat It DoesQuant Use
LUA = L·U (lower × upper triangular)Solving linear systems efficiently
CholeskyA = L·LT (for positive-definite matrices)Generating correlated random variables
QRA = Q·R (orthogonal × upper triangular)Least squares regression
SVDA = U·Σ·VTPCA, rank reduction, pseudo-inverse
EigendecompositionA = V·Λ·V-1Stability analysis, covariance structure

import numpy as np

# Target covariance
Sigma = np.array([[1.0, 0.8], [0.8, 1.0]])
L = linalg.cholesky(Sigma, lower=True)

# Independent standard normals

# Correlated returns

", np.cov(R))
print("Target covariance:
", Sigma)

Hypothesis Testing — Making Decisions with Data

A quant does not guess. A quant tests. Hypothesis testing gives us a disciplined way to ask: "Is this pattern real, or did it happen by chance?"

The Framework

  1. Null Hypothesis (H₀): The default belief (e.g., "this strategy has zero alpha").
  2. Alternative Hypothesis (H₁): What we suspect (e.g., "this strategy has positive alpha").
  3. Test Statistic: A number summarizing the evidence.
  4. p-value: Probability of seeing this evidence if H₀ were true.
  5. Decision: If p-value < α (usually 0.05), reject H₀.

Common Tests in Quant Finance

TestQuestion It AnswersSciPy Function
One-sample t-testIs the mean equal to a value?ttest_1samp
Two-sample t-testDo two groups have the same mean?ttest_ind
KS testDo samples follow a distribution?kstest
Jarque-BeraIs the data normal?jarque_bera
Shapiro-WilkIs the data normal? (small samples)shapiro
LeveneDo groups have equal variance?levene
Anderson-DarlingFit to a distribution?anderson
Multiple Testing Problem: If you run 100 tests at α=0.05, you expect ~5 false positives by chance. Use Bonferroni correction or False Discovery Rate (FDR) when testing many strategies.

Regression & Maximum Likelihood Estimation

Regression finds relationships. MLE finds the distribution that most likely generated the data. Both are parameter estimation techniques, and both rely heavily on SciPy optimization.

Linear Regression (OLS)

We model y = Xβ + ε, where ε ~ N(0, σ²). The goal is to find β that minimizes the sum of squared errors.

Colab Experiment — OLS from Scratch with SciPy
import numpy as np
from scipy import linalg

# Synthetic data: y = 2 + 3*x + noise
np.random.seed(0)
x = np.random.randn(100)
y = 2 + 3*x + np.random.randn(100)*0.5

# Design matrix with intercept
X = np.column_stack([np.ones_like(x), x])

# Normal equation: beta = (X^T X)^{-1} X^T y
beta = linalg.inv(X.T @ X) @ X.T @ y
print("Intercept, Slope:", beta)

# Predictions and R^2
y_pred = X @ beta
ss_res = np.sum((y - y_pred)**2)
ss_tot = np.sum((y - y.mean())**2)
r2 = 1 - ss_res/ss_tot
print("R-squared:", r2)

Maximum Likelihood Estimation (MLE)

Instead of minimizing squared error, MLE asks: "What parameters make the observed data most probable?" We maximize the likelihood function (or minimize the negative log-likelihood).

Colab Experiment — MLE for Normal Parameters
from scipy.optimize import minimize
import numpy as np

# True parameters
true_mu, true_sigma = 5, 2
data = np.random.normal(true_mu, true_sigma, size=500)

# Negative log-likelihood for normal
def neg_log_lik(params):
    mu, sigma = params
    n = len(data)
    return 0.5*n*np.log(2*np.pi*sigma**2) + np.sum((data - mu)**2)/(2*sigma**2)

result = minimize(neg_log_lik, x0=[0, 1], bounds=[(None, None), (1e-6, None)])
print("MLE mu, sigma:", result.x)
print("Sample mean, std:", data.mean(), data.std(ddof=1))
Key Insight: For a normal distribution, MLE for μ is the sample mean. For σ², MLE divides by n (not n-1), making it slightly biased. The unbiased estimator uses Bessel's correction (n-1).

Time Series Analysis

Financial data is ordered in time. Prices today depend on prices yesterday. Time series analysis helps us model these dependencies, forecast, and extract signals.

Key Concepts

Stationarity

A stationary series has constant mean, variance, and autocorrelation over time. Most models require it. Returns are often stationary; prices are not.

Autocorrelation (ACF)

Correlation of a series with its lagged self. Significant ACF at lag 1 suggests momentum; at lag >1 suggests seasonality or mean reversion.

Partial ACF (PACF)

Correlation at lag k after removing effects of lags 1..k-1. Helps identify the order of an AR model.

White Noise

A series with zero mean, constant variance, and no autocorrelation. The "ideal" residual of a good model.

ARMA Models

ARMA(p,q) combines autoregressive (AR) and moving average (MA) components:

  • AR(p): Today's value = weighted sum of past p values + noise
  • MA(q): Today's value = weighted sum of past q noise shocks

Monte Carlo Methods

Monte Carlo uses random sampling to solve problems. In quant finance, we simulate thousands of possible future paths for asset prices to estimate option prices, risk metrics, and strategy performance.

The Monte Carlo Recipe

  1. Define a model for how variables evolve (e.g., geometric Brownian motion).
  2. Generate many random paths.
  3. Compute the payoff or metric for each path.
  4. Average the results and discount if needed.

Geometric Brownian Motion (GBM)

The classic stock price model:

dS = μS·dt + σS·dW
ST = S0 · exp[(μ - ½σ²)T + σ√T·Z]
Colab Experiment — Monte Carlo Option Pricing
import numpy as np
from scipy import stats

# Parameters
S0, K, T, r, sigma = 100, 105, 1, 0.05, 0.2
n_paths = 100000

# Antithetic variates for variance reduction
Z = np.random.randn(n_paths)
# Two paths per draw
ST1 = S0 * np.exp((r - 0.5*sigma**2)*T + sigma*np.sqrt(T)*Z)
ST2 = S0 * np.exp((r - 0.5*sigma**2)*T - sigma*np.sqrt(T)*Z)

# Call payoff
payoff1 = np.maximum(ST1 - K, 0)
payoff2 = np.maximum(ST2 - K, 0)
price = np.exp(-r*T) * np.mean((payoff1 + payoff2)/2)

print("Monte Carlo Call Price:", price)

# Black-Scholes analytic price for comparison
d1 = (np.log(S0/K) + (r + 0.5*sigma**2)*T) / (sigma*np.sqrt(T))
d2 = d1 - sigma*np.sqrt(T)
bs_price = S0*stats.norm.cdf(d1) - K*np.exp(-r*T)*stats.norm.cdf(d2)
print("Black-Scholes Price:", bs_price)
Variance Reduction: Antithetic variates (using -Z) cancel some random noise. Other techniques: control variates, importance sampling, stratified sampling.

Risk Metrics — VaR & CVaR

Risk management separates professionals from gamblers. Value at Risk (VaR) and Conditional Value at Risk (CVaR) are the industry standard metrics for quantifying tail risk.

Value at Risk (VaR)

Definition: VaR at confidence level α is the maximum loss not exceeded with probability α.

Example: 1-day 95% VaR = $1M means there is a 5% chance of losing more than $1M tomorrow.

Conditional VaR (CVaR) / Expected Shortfall

CVaR is the average loss in the worst (1-α)% of cases. It is coherent, sub-additive, and regulators prefer it.

Methods to Compute VaR

MethodApproachSciPy Tools
Parametric (Variance-Covariance)Assume normal distributionnorm.ppf
HistoricalUse empirical quantile of past returnsnp.percentile
Monte CarloSimulate future paths, take quantilestats + random
Colab Experiment — Parametric & Historical VaR/CVaR
import numpy as np
from scipy import stats

# Synthetic daily returns
np.random.seed(42)
returns = np.random.normal(loc=0.0005, scale=0.02, size=1000)

# Portfolio value
portfolio = 1_000_000
alpha = 0.95

# Parametric VaR (assume normal)
mu, sigma = returns.mean(), returns.std()
var_param = portfolio * (mu - sigma*stats.norm.ppf(alpha))
cvar_param = portfolio * (mu - sigma*stats.norm.pdf(stats.norm.ppf(alpha))/(1-alpha))

# Historical VaR
var_hist = portfolio * np.percentile(returns, 100*(1-alpha))
# Historical CVaR
tail_returns = returns[returns <= np.percentile(returns, 100*(1-alpha))]
cvar_hist = portfolio * tail_returns.mean()

print(f"Parametric VaR (95%): ${var_param:,.0f}")
print(f"Historical VaR (95%): ${var_hist:,.0f}")
print(f"Historical CVaR (95%): ${cvar_hist:,.0f}")
Colab Experiment — Cornish-Fisher VaR (Adjusts for Skew & Kurt)
from scipy.stats import skew, kurtosis

s = skew(returns)
k = kurtosis(returns)  # excess kurtosis
z = stats.norm.ppf(alpha)

# Cornish-Fisher adjusted quantile
z_cf = (z + (z**2 - 1)*s/6 + (z**3 - 3*z)*k/24 - (2*z**3 - 5*z)*s**2/36)

var_cf = portfolio * (mu - sigma*z_cf)
print("Cornish-Fisher VaR:", var_cf)

Portfolio Mathematics

A portfolio is more than a collection of assets. It is a weighted combination where diversification can reduce risk. Portfolio math is the algebra of weights, returns, and covariance.

Core Formulas

Portfolio Return: Rp = wT · R
Portfolio Variance: σp² = wT Σ w
Sharpe Ratio: (E[Rp] - Rf) / σp

Optimization Problems

ProblemObjectiveConstraints
Min VarianceMinimize wTΣwΣw = 1
Max SharpeMaximize (wTμ - rf) / √(wTΣw)Σw = 1
Risk ParityEqual risk contributionΣw = 1, w ≥ 0
Max Return (target risk)Maximize wTμwTΣw = σtarget², Σw = 1
Colab Experiment — Efficient Frontier with SciPy
import numpy as np
from scipy.optimize import minimize
import matplotlib.pyplot as plt

# 3 assets
mu = np.array([0.10, 0.12, 0.08])
Sigma = np.array([[0.04, 0.02, 0.01],
                  [0.02, 0.05, 0.015],
                  [0.01, 0.015, 0.03]])

n = len(mu)

def portfolio_return(w): return mu @ w
def portfolio_variance(w): return w @ Sigma @ w

# Generate frontier by varying target returns
target_returns = np.linspace(mu.min(), mu.max(), 50)
frontier_risk = []

for target in target_returns:
    cons = [
        {'type': 'eq', 'fun': lambda w: np.sum(w) - 1},
        {'type': 'eq', 'fun': lambda w: portfolio_return(w) - target}
    ]
    bounds = [(0, 1) for _ in range(n)]
    res = minimize(portfolio_variance, x0=np.ones(n)/n, bounds=bounds, constraints=cons)
    frontier_risk.append(np.sqrt(res.fun))

plt.figure(figsize=(7,5))
plt.plot(frontier_risk, target_returns, color='#34d399', linewidth=2, label='Efficient Frontier')
plt.xlabel('Risk (σ)'); plt.ylabel('Expected Return')
plt.title('Efficient Frontier'); plt.legend(); plt.grid(True, alpha=0.3)
plt.show()

This section ties everything together. We go from raw data to a risk-managed portfolio using SciPy and statistical methods.

Step 1: Data & Returns

import numpy as np
import pandas as pd

# Simulate 3 years of daily data for 4 assets
np.random.seed(7)
dates = pd.date_range('2021-01-01', periods=756, freq='B')
returns = np.random.multivariate_normal(
    mean=[0.0004, 0.0005, 0.0003, 0.0006],
    cov=[[0.0004, 0.0001, 0.00005, 0.00008],
         [0.0001, 0.0005, 0.0001, 0.00012],
         [0.00005, 0.0001, 0.0003, 0.00006],
         [0.00008, 0.00012, 0.00006, 0.0006]],
    size=len(dates)
)
assets = ['AAPL', 'MSFT', 'GOOGL', 'TSLA']
df = pd.DataFrame(returns, index=dates, columns=assets)

Step 2: Distribution Fitting & Testing

from scipy import stats

# Fit t-distribution to AAPL
params = stats.t.fit(df['AAPL'])
print("Fitted t params (df, loc, scale):", params)

# Jarque-Bera test
jb, p = stats.jarque_bera(df['AAPL'])
print("Normality p-value:", p)

Step 3: Covariance & Cholesky

from scipy import linalg

Sigma = df.cov().values
L = linalg.cholesky(Sigma, lower=True)
print("Cholesky factor computed for simulation.")

Step 4: Monte Carlo Simulation

n_sims = 50000
Z = np.random.randn(len(assets), n_sims)
future_returns = L @ Z  # correlated daily returns

Step 5: Portfolio Optimization

from scipy.optimize import minimize

mu = df.mean().values * 252  # annualized
Sigma_ann = df.cov().values * 252

def neg_sharpe(w):
    ret = mu @ w
    vol = np.sqrt(w @ Sigma_ann @ w)
    return -(ret - 0.03) / vol

cons = {'type': 'eq', 'fun': lambda w: np.sum(w) - 1}
bounds = [(0, 0.4) for _ in assets]  # max 40% in any asset
res = minimize(neg_sharpe, x0=np.ones(len(assets))/len(assets), bounds=bounds, constraints=cons)
optimal_weights = res.x
print("Optimal weights:", dict(zip(assets, np.round(optimal_weights, 3))))

Step 6: Risk Metrics

portfolio_returns = optimal_weights @ future_returns
var_95 = np.percentile(portfolio_returns, 5)
cvar_95 = portfolio_returns[portfolio_returns <= var_95].mean()
print(f"1-day 95% VaR: {var_95:.4f}")
print(f"1-day 95% CVaR: {cvar_95:.4f}")
