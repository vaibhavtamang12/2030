0.1 Python Fundamentals (The Language)

Python is the lingua franca of quantitative finance. Think of it as the instruction set you use to tell the computer what to do with data.

Variables & Data Types

A variable is a labeled box that stores a value. In Python, you do not need to declare the type explicitly.

# Primitive data types
price = 150.75           # float (decimal number)
shares = 100             # int (whole number)
ticker = "AAPL"          # str (text)
is_active = True         # bool (True/False)

Data Structures

Before Pandas, Python provides native containers:

  • List [1, 2, 3]: Ordered, mutable collection. Like a row in a spreadsheet.
  • Dictionary {"key": "value"}: Key-value pairs. Like a lookup table for stock metadata.
  • Tuple (1, 2, 3): Immutable ordered collection. Use for fixed records (e.g., OHLC as a tuple).
# List of closing prices
closes = [150.0, 152.5, 151.0, 153.25]

# Dictionary mapping ticker to sector
sector_map = {"AAPL": "Technology", "JPM": "Finance"}

# Accessing data
print(closes[0])        # First element: 150.0
print(sector_map["AAPL"]) # Lookup: Technology

Control Flow & Functions

Quant strategies require logic. Use if/else for decisions and for loops for iteration. However, in Pandas, we replace slow Python loops with vectorized operations (covered later).

# Function to calculate simple return
def simple_return(p_today, p_yesterday):
    return (p_today - p_yesterday) / p_yesterday

# List comprehension (Pythonic and faster than loops)
squared = [x**2 for x in range(10)]
Quant Warning: Native Python loops are slow for large datasets. A quant processing tick data (millions of rows) must use vectorized NumPy/Pandas operations, not for loops.

0.2 NumPy Essentials (The Engine)

Pandas is built directly on top of NumPy. Understanding NumPy is non-negotiable. NumPy provides the ndarray: a fast, homogeneous, multi-dimensional array.

import numpy as np

# Create an array of prices
prices = np.array([150.25, 152.00, 151.75, 153.50, 155.00])

# Vectorized operation: calculate returns for ALL elements at once
returns = np.diff(prices) / prices[:-1]
print(returns)  # [ 0.01164725 -0.00164474  0.01151393  0.00977199]

# Broadcasting: add 1 to every element (no loop needed)
prices_plus_1 = prices + 1

# Statistical aggregates
print(np.mean(prices), np.std(prices))

Key NumPy Concepts for Pandas

  • Shape & Dimensions: prices.shape tells you the array dimensions. A DataFrame is essentially a 2D NumPy array with labels.
  • Broadcasting: NumPy automatically aligns arrays of different shapes for arithmetic. This is the magic behind Pandas operations.
  • Vectorization: Operations apply to entire arrays element-wise in compiled C code.
  • Boolean Indexing: prices[prices > 152] selects elements where the condition is True. Pandas extends this powerfully.

0.3 Statistics & Probability Basics

Quantitative engineering is applied statistics. You do not need a PhD, but you must internalize these concepts:

Measures of Central Tendency

  • Mean (Average): sum(x) / len(x). The "center of gravity" of returns.
  • Median: The middle value when sorted. Robust to outliers (e.g., one flash crash won't skew it).
  • Mode: Most frequent value. Less common in continuous price data.

Measures of Dispersion (Risk!)

  • Variance: Average squared deviation from the mean. Units are squared dollars (hard to interpret).
  • Standard Deviation (Volatility): Square root of variance. Measured in same units as data. This is the most common measure of risk in finance.
  • Percentiles/Quantiles: The 5th percentile of returns tells you: "On the worst 5% of days, I lost at least X%." Essential for Value-at-Risk (VaR).

Correlation

Correlation measures how two assets move together. Range: -1 (perfect inverse) to +1 (perfect sync). Quants use correlation matrices to build diversified portfolios.

Core 1. Introduction to Pandas

What is Pandas?

Pandas = Panel Data. It is an open-source library that provides high-performance, easy-to-use data structures for data analysis. For a quant engineer, Pandas is the bridge between raw market data and actionable signals.

Analogy: If NumPy is the engine of a race car, Pandas is the entire vehicle with a dashboard, GPS, and labels. NumPy gives you speed; Pandas gives you labeled, tabular data with time-series superpowers.

The Two Core Data Structures

Series Deep Dive

A Series is a 1D labeled array. It wraps a NumPy array and adds an index. Every quant calculation starts here: a single time-series of prices, returns, or volumes.

import pandas as pd
import numpy as np

# Creating a Series from a list
prices = pd.Series([150.25, 152.00, 151.75, 153.50])
print(prices)
# Output shows default integer index: 0, 1, 2, 3

# Creating a Series with a custom index (dates as strings for simplicity)
prices_idx = pd.Series(
    [150.25, 152.00, 151.75, 153.50],
    index=["2024-01-01", "2024-01-02", "2024-01-03", "2024-01-04"],
    name="AAPL_Close"
)
print(prices_idx)

# A Series behaves like a NumPy array + dict hybrid
print(prices_idx["2024-01-02"])   # Dict-like label access
print(prices_idx[1])             # Array-like position access (avoid this; ambiguous)
Critical Distinction: A Series has both position (0, 1, 2...) and label (your custom index). Accessing by [1] uses position if the index is not integer-based, but this behavior changes if your index is integers. This ambiguity is why we later use .loc[] and .iloc[].

DataFrame Deep Dive

A DataFrame is a dictionary of Series sharing the same index. It is the primary structure for quant work: each column is a variable (Open, High, Low, Close, Volume), and each row is an observation (a timestamp).

}
index = ["2024-01-01", "2024-01-02", "2024-01-03"]
df = pd.DataFrame(data, index=index)
print(df)

# Each column IS a Series
print(type(df["Close"]))  # <class 'pandas.core.series.Series'>

# Shape gives (rows, columns) — your first sanity check
print(df.shape)  # (3, 4)

Understanding the Index

The index is the row label. In quant finance, this is almost always a datetime index. The index is not just a column; it is the "anchor" for alignment, joining, and time-series operations.

Quant Insight: In Pandas, when you add two Series together, they align by index label, not by position. This is incredibly powerful for finance: you can add the returns of two stocks on the same date even if the rows are in different orders. 
# The index object
print(ohlc.index)       # DatetimeIndex
print(ohlc.columns)     # Index of column names

# Resetting vs Setting index
df_reset = ohlc.reset_index()   # Moves index to a column named 'index'
df_indexed = df_reset.set_index("index")  # Sets a column as the new index

# For time series, ensure the index is sorted
print(ohlc.index.is_monotonic_increasing)  # Should be True

Column Operations

# Selecting a single column → Series
closes = ohlc["Close"]

# Selecting multiple columns → DataFrame (pass a LIST!)
ohlc_only = ohlc[["Open", "High", "Low", "Close"]]

# Adding a new column
ohlc["Range"] = ohlc["High"] - ohlc["Low"]

# Dropping columns
ohlc = ohlc.drop(columns=["Range"])

# Renaming columns
ohlc = ohlc.rename(columns={"Volume": "Vol"})

Core 3. Indexing & Selection

The Indexing Problem

Pandas offers multiple ways to select data. This flexibility is powerful but confusing for beginners. In quantitative finance, selecting the right data quickly and unambiguously is critical—your trading signal depends on it.

Golden Rule: Use .loc[] for labels and .iloc[] for positions. Avoid plain [] for row selection because its behavior changes based on context.

.loc[] — Label-Based Selection

# Setup
dates = pd.date_range("2024-01-01", periods=5, freq="B")
df = pd.DataFrame({
    "Open": [100, 102, 101, 103, 105],
    "Close": [102, 101, 103, 105, 104]
}, index=dates)

# Single value
print(df.loc["2024-01-02", "Close"])   # 101

# Entire row
row = df.loc["2024-01-02"]   # Returns a Series

# Entire column
col = df.loc[:, "Close"]     # The colon means "all rows"

# Slice (INCLUSIVE of end point!)
slice_df = df.loc["2024-01-02":"2024-01-04", "Open":"Close"]

# List of labels
df.loc[["2024-01-02", "2024-01-04"], ["Open", "Close"]]

.iloc[] — Position-Based Selection

# Same rules as Python/NumPy indexing
print(df.iloc[0, 1])        # Row 0, Column 1 → 102
print(df.iloc[1:3, 0])      # Rows 1,2 (3 is EXCLUDED), Column 0
print(df.iloc[-1])          # Last row
print(df.iloc[:, -1])       # Last column

Boolean Indexing (Masking)

This is the quant's best friend. You select rows based on conditions.

# Select days where Close > 102
mask = df["Close"] > 102
print(df[mask])   # or directly: df[df["Close"] > 102]

# Multiple conditions
# Note: use & (and), | (or), ~ (not) with parentheses!
condition = (df["Close"] > 102) & (df["Open"] < 103)
print(df[condition])

# .loc with boolean mask
print(df.loc[df["Close"] > 102, ["Open", "Close"]])
Operator Trap: You cannot use Python's and, or, not with Pandas boolean Series. You must use &, |, ~. Also, each condition must be wrapped in parentheses due to operator precedence.

Setting Values

# Setting with .loc
df.loc["2024-01-02", "Close"] = 999

# Setting a whole column
df.loc[:, "Signal"] = 0   # Initializes a new column with zeros

# Setting based on condition (e.g., generate a buy signal)
df.loc[df["Close"] > df["Open"], "Signal"] = 1   # 1 = Buy

Core 4. Data Cleaning

The Quant's Cleaning Mandate

Real financial data is messy. Missing ticks, stock splits recorded as outliers, delistings causing NaNs, and timezone mismatches are daily realities. A quant who does not clean data builds models on illusions.

Principle: Always ask: Why is the data missing? In finance, missing data is not random—it often contains information (e.g., halted trading, delisting). Blindly filling NaNs with the mean can destroy your strategy.

Missing Data (NaN)

Pandas uses np.nan (Not a Number) to represent missing floating-point data. For integers, Pandas uses nullable Int64 (capital I) or converts to float.

# Detecting missing data
df = pd.DataFrame({
    "A": [1.0, 2.0, np.nan, 4.0],
    "B": [5, np.nan, np.nan, 8]
})

print(df.isna())        # Boolean mask: True where NaN
print(df.isna().sum())  # Count NaNs per column
print(df.isna().mean()) # Proportion missing (critical check!)

# Dropping missing data
df_drop_rows = df.dropna()             # Drop ANY row with a NaN
df_drop_cols = df.dropna(axis=1)       # Drop columns with ANY NaN
df_drop_thresh = df.dropna(thresh=2)   # Keep rows with at least 2 non-NaN values

# Filling missing data
df_fill_zero = df.fillna(0)          # Rarely correct for prices
df_fill_mean = df.fillna(df.mean())    # Forward-looking bias danger!

# Forward fill (ffill) — CRITICAL for time series
# Use previous valid observation to fill gap
df_ffill = df.ffill()

# Backward fill (bfill) — Use next valid observation
df_bfill = df.bfill()

# Limit the fill (e.g., only fill 1 consecutive NaN)
df_ffill_limited = df.ffill(limit=1)
Quant Danger: Filling price data with the mean implies future knowledge (look-ahead bias). For time-series, ffill() is usually safer because it only uses past data. Never use df.mean() fill on a training set without understanding the leakage implications.

Handling Duplicates

# Detect duplicates
print(df.duplicated())           # True for duplicated rows
print(df.duplicated().sum())     # Count duplicates

# Drop duplicates
df_unique = df.drop_duplicates()

# Drop duplicates based on specific columns (e.g., same timestamp)
df.drop_duplicates(subset=["timestamp"], keep="last", inplace=True)
# keep="first" or "last" determines which duplicate survives

Type Conversion

Data types determine memory usage and computational speed. Quants often work with millions of rows—optimizing types matters.

# Common conversions
df["price"] = pd.to_numeric(df["price"], errors="coerce")
# errors="coerce" converts bad values (like "N/A") to NaN instead of crashing

df["date"] = pd.to_datetime(df["date"])
# Converts strings to datetime64[ns] — unlocks .dt accessor and resampling

df["category"] = df["category"].astype("category")
# Converts string column to categorical — massive memory savings for repeated strings

# Downcasting numerics (memory optimization)
df["int_col"] = pd.to_numeric(df["int_col"], downcast="integer")
df["float_col"] = pd.to_numeric(df["float_col"], downcast="float")

Outlier Detection (Basic)

Outliers in finance can be legitimate (market crashes) or errors (bad ticks). You must distinguish them.

# Z-score method (|z| > 3 is roughly 99.7% of normal data)
from scipy import stats
z_scores = np.abs(stats.zscore(df["returns"].dropna()))
outliers = df[z_scores > 3]

# IQR method (Interquartile Range)
Q1 = df["returns"].quantile(0.25)
Q3 = df["returns"].quantile(0.75)
IQR = Q3 - Q1
outliers_iqr = df[(df["returns"] < (Q1 - 1.5 * IQR)) | (df["returns"] > (Q3 + 1.5 * IQR))]

# Winsorization: cap outliers instead of removing
df["returns_capped"] = df["returns"].clip(lower=Q1 - 1.5*IQR, upper=Q3 + 1.5*IQR)

Core 5. Data Transformation

From Raw Data to Signals

Raw prices are not tradable signals. Transformation converts data into features: returns, moving averages, volatility regimes, technical indicators. This section covers the mechanics of changing data shape and values.

apply(), map(), applymap()

These are often confused. Here is the definitive quant guide:

Method Operates On Use Case
series.map(func) Each element of a Series Map tickers to sectors; encode labels
series.apply(func) Each element of a Series Custom element-wise calculation
df.apply(func, axis=0/1) Entire column (axis=0) or row (axis=1) Row-wise portfolio calculation 
# map(): element-wise on Series
tickers = pd.Series(["AAPL", "MSFT", "JPM"])
sector_map = {"AAPL": "Tech", "MSFT": "Tech", "JPM": "Finance"}
sectors = tickers.map(sector_map)

# apply() with lambda on Series
prices = pd.Series([150.25, 200.50, 75.00])
# Classify by price level
labels = prices.apply(lambda x: "Large" if x > 100 else "Small")

# apply() on DataFrame rows (axis=1)
df = pd.DataFrame({
    "Open": [100, 102],
    "Close": [102, 101],
    "Volume": [1000, 2000]
})
# Calculate a custom VWAP approximation per row
df["VWAP_Proxy"] = df.apply(lambda row: (row["Open"] + row["Close"]) / 2 * row["Volume"], axis=1)
Performance Warning: apply() is a Python loop in disguise. It is convenient but slow. For quant work, prefer vectorized operations. If you find yourself using apply on >100,000 rows, stop and look for a vectorized alternative.

Vectorized Operations (The Quant Way)

# BAD: Looping through rows
for i in range(len(df)):
    df.loc[i, "Return"] = df.loc[i, "Close"] / df.loc[i, "Open"] - 1

# GOOD: Vectorized (100x+ faster)
df["Return"] = df["Close"] / df["Open"] - 1

# Vectorized with conditions (no apply needed)
df["Signal"] = np.where(df["Close"] > df["Open"], 1, -1)
# np.where(condition, value_if_true, value_if_false)

# Vectorized clip (Winsorize)
df["Return_Capped"] = df["Return"].clip(-0.05, 0.05)

String Operations (.str accessor)

# When you have string data (e.g., news sentiment, tickers)

Core 6. GroupBy & Aggregation

Split-Apply-Combine

The GroupBy paradigm is the SQL GROUP BY equivalent in Pandas. For quants, it is essential for analyzing sectors, calculating monthly statistics, or backtesting strategies across regimes.

Basic GroupBy Mechanics

# Sample data: multi-stock panel
df = pd.DataFrame({
    "Ticker": ["AAPL", "AAPL", "MSFT", "MSFT", "JPM", "JPM"],
    "Sector": ["Tech", "Tech", "Tech", "Tech", "Finance", "Finance"],
    "Return": [0.01, -0.02, 0.015, 0.005, 0.008, -0.004],
    "Volume": [1e6, 1.2e6, 0.8e6, 0.9e6, 2e6, 1.8e6]
})

# Group by one column
sector_returns = df.groupby("Sector")["Return"].mean()
print(sector_returns)

# Group by multiple columns
# Not applicable here but syntax is:
# df.groupby(["Sector", "Industry"])["Return"].mean()

# Multiple aggregations at once
agg_stats = df.groupby("Sector")["Return"].agg(["mean", "std", "min", "max", "count"])
print(agg_stats)

# Named aggregations (clean column names)
agg_named = df.groupby("Sector").agg(
    avg_return=("Return", "mean"),
    vol_return=("Return", "std"),
    total_volume=("Volume", "sum")
)
print(agg_named)

transform() vs agg() vs filter()

# agg(): Reduces each group to a single value
# Result has length = number of groups

# transform(): Returns a Series/DataFrame with SAME length as input
# Useful for demeaning (removing group average from each observation)
df["Sector_Mean"] = df.groupby("Sector")["Return"].transform("mean")
df["Demeaned_Return"] = df["Return"] - df["Sector_Mean"]

# filter(): Keeps or drops entire groups based on a condition
# Keep only sectors with average return > 0
high_perf = df.groupby("Sector").filter(lambda x: x["Return"].mean() > 0)
print(high_perf)

Pivot Tables

Pivot tables are GroupBy operations arranged in a spreadsheet format. Essential for quant reporting.

# Pivot table: Sector vs mean Return and Volume
pivot = df.pivot_table(
    values=["Return", "Volume"],
    index="Sector",
    aggfunc={"Return": ["mean", "std"], "Volume": "sum"}
)
print(pivot)

# Cross-tabulation (frequency table)
# Useful for regime analysis: e.g., count of up/down days per sector
df["Direction"] = np.where(df["Return"] > 0, "Up", "Down")
crosstab = pd.crosstab(df["Sector"], df["Direction"])
print(crosstab)

Core 7. Merging, Joining & Concatenation

Why Quants Need Joins

You rarely have all data in one file. Prices come from one vendor, fundamentals from another, and sentiment from a third. Merging aligns these datasets by a common key (usually date + ticker).

pd.merge()

# Two DataFrames to merge
prices = pd.DataFrame({
    "Date": ["2024-01-01", "2024-01-02", "2024-01-03"],
    "Ticker": ["AAPL", "AAPL", "AAPL"],
    "Close": [150, 152, 151]
})

earnings = pd.DataFrame({
    "Date": ["2024-01-02", "2024-01-04"],
    "Ticker": ["AAPL", "AAPL"],
    "EPS": [2.5, 2.8]
})

# Inner join: only dates present in BOTH
inner = pd.merge(prices, earnings, on=["Date", "Ticker"], how="inner")

# Left join: keep all prices, attach EPS where available
left = pd.merge(prices, earnings, on=["Date", "Ticker"], how="left")

# If column names differ, use left_on and right_on
# pd.merge(df1, df2, left_on="date", right_on="Date", how="outer")

# Indicator column shows source of each row
merged = pd.merge(prices, earnings, on=["Date", "Ticker"], how="outer", indicator=True)
print(merged["_merge"].value_counts())

pd.concat()

Use concat to stack DataFrames vertically (more rows) or horizontally (more columns).

# Vertical concat: append more time periods
q1 = pd.DataFrame({"Close": [100, 101]}, index=["Jan", "Feb"])
q2 = pd.DataFrame({"Close": [102, 103]}, index=["Mar", "Apr"])
annual = pd.concat([q1, q2], axis=0)   # axis=0 is default (stack rows)

# Horizontal concat: add more columns side-by-side
prices_df = pd.DataFrame({"Close": [100, 101]})
volumes_df = pd.DataFrame({"Volume": [1e6, 1.1e6]})
combined = pd.concat([prices_df, volumes_df], axis=1)

# Keys create a MultiIndex (hierarchical)
combined = pd.concat([q1, q2], keys=["Q1", "Q2"])
print(combined)
# Access: combined.loc["Q1"]

join() — Index-Based

# join() merges on indices by default (convenient for time series)
df1 = pd.DataFrame({"A": [1, 2]}, index=["x", "y"])
df2 = pd.DataFrame({"B": [3, 4]}, index=["x", "z"])
print(df1.join(df2, how="outer"))
Merge Checklist for Quants:
  1. Check for duplicate keys before merging—they cause a Cartesian explosion (memory blowup).
  2. Always verify the indicator column or row counts post-merge.
  3. For time-series, ensure both sides are sorted by date before merging.
  4. Be aware of timezone differences if merging intraday data.

Quant Essential 8. Time Series Essentials

Time is the Independent Variable

For a quant, time is not just another column—it is the dimension along which all analysis happens. Pandas has world-class time-series capabilities built on the DatetimeIndex.

Why this matters: A regular DataFrame with a string column "2024-01-01" cannot be resampled, shifted, or aligned. Converting to DatetimeIndex unlocks the full quant toolkit.

Datetime Fundamentals

# Creating datetime objects
ts = pd.Timestamp("2024-03-15 09:30:00")
print(ts.dayofweek)   # 4 = Friday
print(ts.quarter)     # 1
print(ts.is_month_end) # False

# DatetimeIndex: the heart of quant time series
dates = pd.date_range(start="2024-01-01", periods=10, freq="B")  # Business days
print(dates)

# Common frequencies
# "D" = daily, "B" = business day, "W" = weekly
# "M" = month end, "Q" = quarter end, "A" = year end
# "H" = hourly, "T" or "min" = minute

# Parsing strings to datetime
raw_dates = ["2024-01-01", "01/02/2024", "2024-03-15 14:30:00"]
parsed = pd.to_datetime(raw_dates, dayfirst=False)
print(parsed)

Resampling

Resampling changes the frequency of your data. Upsampling (low freq → high freq) introduces NaNs; downsampling (high freq → low freq) requires an aggregation rule.

# Generate intraday-like data
minutes = pd.date_range("2024-01-01 09:30", periods=390, freq="T")  # One trading day of minutes
prices = 100 + np.cumsum(np.random.normal(0, 0.01, len(minutes)))
intRADAY = pd.DataFrame({"Price": prices}, index=minutes)

# Downsampling: Minute → Hourly (take the last price of each hour)
hourly = intraday.resample("H").last()

# OHLC resampling (classic quant operation)
ohlc = intraday["Price"].resample("H").ohlc()
print(ohlc.head())

# Upsampling: Daily → Hourly (introduces NaNs)
daily = pd.DataFrame({"Close": [100, 101, 102]},
                     index=pd.date_range("2024-01-01", periods=3, freq="D"))
up = daily.resample("6H").asfreq()   # asfreq = just introduce NaNs
up_filled = daily.resample("6H").ffill()  # Forward fill

Shifting & Lagging

Calculating returns requires aligning today's price with yesterday's price. .shift(n) moves data by n periods.

# Daily price data
df = pd.DataFrame({"Close": [100, 102, 101, 105, 104]},
                  index=pd.date_range("2024-01-01", periods=5))

# Previous day's close
df["Prev_Close"] = df["Close"].shift(1)

# Daily return
df["Return"] = df["Close"] / df["Close"].shift(1) - 1

# Future return (lead) — useful for labeling in machine learning
df["Future_1d_Return"] = df["Close"].shift(-1) / df["Close"] - 1

# Percent change (convenience method)
df["Pct_Change"] = df["Close"].pct_change()

print(df)

Datetime Accessor (.dt)

# Extract features from datetime index
df["Year"] = df.index.year
df["Month"] = df.index.month
df["DayOfWeek"] = df.index.dayofweek   # Monday=0, Sunday=6
df["Quarter"] = df.index.quarter

# If you have a datetime column instead of index
# df["Date"].dt.year, df["Date"].dt.is_month_end, etc.

Quant Essential 9. Window & Rolling Functions

The Moving Window Concept

Markets are non-stationary. A 20-day volatility estimate from 2020 is useless in 2024. Rolling windows allow quants to compute statistics on a local, evolving subset of data.

Rolling Statistics

# Setup: 1 year of daily returns
np.random.seed(42)
dates = pd.date_range("2023-01-01", periods=252, freq="B")
returns = pd.Series(np.random.normal(0.0005, 0.02, len(dates)), index=dates)
price = 100 * (1 + returns).cumprod()

# Rolling mean (Moving Average)
price_sma_20 = price.rolling(window=20).mean()
price_sma_50 = price.rolling(window=50).mean()

# Rolling standard deviation (Realized Volatility)
vol_20d = returns.rolling(window=20).std() * np.sqrt(252)  # Annualized

# Rolling min/max (support/resistance proxies)
rolling_high = price.rolling(window=20).max()
rolling_low = price.rolling(window=20).min()

# Rolling sum (e.g., cumulative volume over window)
volume = pd.Series(np.random.randint(1e6, 5e6, len(dates)), index=dates)
vol_sum_5d = volume.rolling(window=5).sum()

Expanding Windows

Unlike rolling (fixed window), expanding uses all data from the start up to the current point. Useful for backtests that must not use future data.

# Expanding mean: uses all history up to date t
expanding_mean = price.expanding().mean()

# Expanding max drawdown calculation
rolling_max = price.expanding().max()
drawdown = (price - rolling_max) / rolling_max

# Cumulative product (buy-and-hold return)
cum_return = (1 + returns).cumprod()

Exponentially Weighted Windows (EWM)

EWM gives more weight to recent observations. This is how quants build adaptive indicators that react faster to new information than simple moving averages.

# Exponentially weighted moving average
# span=20 means approximately equivalent to 20-day SMA in terms of center of mass
ewma_20 = price.ewm(span=20, adjust=False).mean()

# Exponentially weighted moving standard deviation
ewmstd_20 = returns.ewm(span=20).std()

# Halflife: the period of time for the weight to decay to half
# ewm(halflife=10) is common in risk management
risk_est = returns.ewm(halflife=10).std()

Custom Rolling Apply

# Apply a custom function to each window
# Example: rolling Sharpe ratio
def sharpe(window_returns):
    return window_returns.mean() / window_returns.std()

rolling_sharpe = returns.rolling(63).apply(sharpe, raw=True)
# raw=True passes NumPy array (faster)
# 63 days ≈ 3 months

Quant Essential 10. Financial Computations

The Quant's Formula Sheet

This section ties together all previous Pandas skills into actual quantitative finance calculations. These are the building blocks of strategies, risk models, and portfolio analytics.

Returns

# Simple returns
returns = df["Close"].pct_change()
# Equivalent to: df["Close"] / df["Close"].shift(1) - 1

# Log returns (time-additive, preferred for statistical analysis)
log_returns = np.log(df["Close"] / df["Close"].shift(1))

# Why log returns? log(P_t / P_{t-1}) + log(P_{t-1} / P_{t-2}) = log(P_t / P_{t-2})
# Simple returns do not compound additively.

# Cumulative returns
cum_simple = (1 + returns).cumprod() - 1
cum_log = log_returns.cumsum().apply(np.exp) - 1   # Convert back to simple

# Multi-period returns (e.g., monthly from daily)
monthly_returns = returns.resample("M").apply(lambda x: (1+x).prod() - 1)

Volatility & Risk

# Realized volatility (annualized)
realized_vol = returns.rolling(21).std() * np.sqrt(252)
# 21 trading days ≈ 1 month; 252 ≈ trading days per year

# Value at Risk (VaR) — Historical simulation
var_95 = returns.quantile(0.05)   # 5th percentile of historical returns
# Interpretation: We expect to lose more than var_95 on 5% of days

# Conditional VaR (Expected Shortfall)
cvar_95 = returns[returns <= var_95].mean()

# Maximum Drawdown
wealth_index = (1 + returns).cumprod()
peak = wealth_index.expanding().max()
drawdown = (wealth_index - peak) / peak
max_dd = drawdown.min()

Portfolio Math

# Correlation matrix (diversification analysis)
returns_df = pd.DataFrame({
    "AAPL": np.random.normal(0.001, 0.02, 252),
    "MSFT": np.random.normal(0.001, 0.018, 252),
    "JPM": np.random.normal(0.0005, 0.015, 252)
}, index=pd.date_range("2023-01-01", periods=252))

corr_matrix = returns_df.corr()
print(corr_matrix)

# Portfolio return (weighted sum)
weights = np.array([0.4, 0.4, 0.2])
portfolio_return = returns_df @ weights   # Matrix multiplication

# Portfolio volatility (accounting for correlation)
cov_matrix = returns_df.cov() * 252   # Annualized covariance

# Sharpe Ratio (assuming risk-free rate = 0 for simplicity)
sharpe = portfolio_return.mean() / portfolio_return.std() * np.sqrt(252)

Simple Technical Indicators

# RSI (Relative Strength Index) — simplified
delta = df["Close"].diff()
gain = delta.clip(lower=0)
loss = -delta.clip(upper=0)
avg_gain = gain.rolling(14).mean()
avg_loss = loss.rolling(14).mean()
rs = avg_gain / avg_loss
rsi = 100 - (100 / (1 + rs))

# MACD (Moving Average Convergence Divergence)
ema_12 = df["Close"].ewm(span=12).mean()
ema_26 = df["Close"].ewm(span=26).mean()
macd = ema_12 - ema_26
signal_line = macd.ewm(span=9).mean()

Best Practices for Quant Workflows

  1. Never modify raw data in-place. Always create new columns (df["new"]) rather than overwriting source data.
  2. Avoid loops. If your code has a for loop iterating over DataFrame rows, there is almost certainly a vectorized solution.
  3. Check for look-ahead bias. Any transformation using future data (e.g., mean-centering with global mean) invalidates a backtest.
  4. Use inplace=False (default) to maintain a functional pipeline where each step returns a new object.
  5. Version your data. Save intermediate Parquet files with dates in filenames: prices_2024_cleaned.parquet.
  6. Document units. Is your return 0.01 (1%) or 1.0 (100%)? Consistency prevents catastrophic errors.

Project 1: The Cross-Sectional Momentum Screener

Scenario: You are a quant analyst at a long/short equity fund. Every month, you must rank 100 stocks by their 12-month past return (excluding the most recent month to avoid reversal), and select the top 10 and bottom 10.

Requirements:

  1. Generate synthetic daily price data for 100 stocks over 2 years.
  2. Calculate monthly returns for each stock.
  3. For each month-end, compute the 11-month cumulative return (t-12 to t-1).
  4. Rank stocks within that month. Select decile 10 (winners) and decile 1 (losers).
  5. Calculate the equal-weighted return of the long-short portfolio for the subsequent month.
  6. Compute the Sharpe ratio and max drawdown of this strategy.

Skills Tested:

Time-series, GroupBy, shift, ranking, merge, portfolio math.

Project 2: The Risk Parity Allocator

Scenario: Instead of equal weights, allocate capital inversely proportional to each asset's volatility. Higher volatility assets get smaller weights.

Requirements:

  1. Use 5 asset return series (synthetic or real).
  2. Calculate rolling 60-day volatility for each asset.
  3. At each month-end, set weights = (1/vol_i) / sum(1/vol_j).
  4. Compute the portfolio return for the next month using these weights.
  5. Compare cumulative returns of Risk Parity vs Equal Weight vs 60/40 benchmark.

Skills Tested:

Rolling windows, GroupBy, transform, vectorization, visualization.

Project 3: The Earnings Surprise Analyzer

Scenario: Merge quarterly earnings data with daily price data. Measure the price drift before and after the announcement.

Requirements:

  1. Create a daily price DataFrame for 20 stocks over 3 years.
  2. Create a sparse earnings DataFrame: Date, Ticker, EPS_Actual, EPS_Estimate.
  3. Calculate Surprise = (Actual - Estimate) / |Estimate|.
  4. Merge earnings into prices using pd.merge_asof (backward, by ticker).
  5. For each earnings event, extract the return from t-5 to t+5 (event study).
  6. Group by surprise quintile and plot the average cumulative return path.

Skills Tested:

Merge_asof, time alignment, shift, GroupBy, visualization.

Project 4: The Intraday Microstructure Cleaner

Scenario: Clean real-world tick data with bad prints, duplicates, and missing timestamps.

Requirements:

  1. Generate synthetic 1-minute bars with occasional bad ticks (negative prices, zero volume).
  2. Remove ticks where High < Low or Volume = 0.
  3. Remove duplicate timestamps keeping the last.
  4. Reindex to a complete 1-minute grid (9:30-16:00) and forward-fill missing bars up to 5 minutes.
  5. Resample to 5-minute OHLC bars.
  6. Calculate the realized variance for the day: sum of squared 5-min returns.

Skills Tested:

Cleaning, resample, reindex, interpolation, time-series math.
