Topic 0.1: Python for Data — NumPy & Pandas

NumPy — The Engine

Finance is numerical. NumPy provides arrays — fast, memory-efficient grids of numbers. Think of an array as an Excel column that can do math on every cell instantly.

Key concept: Vectorization. Instead of looping through prices to calculate returns, you do: returns = prices[1:] / prices[:-1] - 1

Pandas — The Organizer

Pandas adds DataFrames (labeled tables) and Series (labeled columns). It connects raw numbers to meaning — dates, stock tickers, prices.

Key concept: Indexing by time. Financial data is a time series, and Pandas makes df['2020':'2023'] slice effortlessly.

Topic 0.3: Returns — The Universal Language

Prices vary across stocks ($5 vs $5000). Returns normalize everything. A 2% gain is a 2% gain whether the stock is $10 or $1000.

Simple Return: Rt = (Pt - Pt-1) / Pt-1 = Pt/Pt-1 - 1
Log Return: rt = ln(Pt / Pt-1)

Why log returns? They are time-additiveLog returns over multiple periods simply sum up: r_total = r_1 + r_2 + r_3 and mathematically cleaner for modeling. For small daily moves, simple and log returns are nearly identical.

Topic 0.4: Basic Statistics for Finance

Mean (Average Return)

The center of gravity. Tells you the expected drift of an asset. In Python: returns.mean()

Standard Deviation (Volatility)

The "wobble." How far returns typically deviate from the mean. This is the most common risk measure. In Python: returns.std()

Variance

Standard deviation squared. Used in portfolio math because variances add (if uncorrelated), but standard deviations don't.

Percentiles / Quantiles

The 5th percentile of returns tells you: "5% of the time, I lost more than X." Critical for risk management.

    'Open': prices + np.random.randn(252) * 0.3,

1. Time Series Visualization Core

Time series is the DNA of quantitative finance. Every price, every return, every signal exists in time. Visualizing it well separates amateur charts from professional analytics.

Prerequisites Needed

OHLCV data structure, Pandas time-indexing, basic Matplotlib/Seaborn, simple returns calculation.

1.1 The Line Chart — Foundation of Everything

A line chart connects closing prices over time. It's simple, but dangerous — it hides intraday volatility. Always ask: "What time granularity am I looking at?"

Interactive Concept: Time Granularity Changes Meaning

The same data looks calm on a monthly chart but chaotic on an hourly chart. Zoom level changes narrative.

1.2 Candlestick Charts — Seeing the Battle

Candlesticks encode 4 prices in one shape. The body (open-close) shows who won the period. The wicks (high-low) show how far the battle extended.

  • Green/White candle: Close > Open (bulls won)
  • Red/Black candle: Close < Open (bears won)
  • Long wick + small body: Indecision (reversal potential)
  • No wick + long body: Strong conviction

Diagram: Candlestick Anatomy

1.3 Volume — The Lie Detector

Price moves backed by high volume are more trustworthy. A price breakout on low volume is like a party with no music — suspicious.

Common visualizations:

  • Volume bars below price: Classic, simple, effective
  • Volume Profile: Histogram rotated sideways — shows where most trading happened (Value Area)
  • On-Balance Volume (OBV): Cumulative volume flow to predict price

1.4 Moving Averages — Smoothing the Noise

A Moving AverageThe average price over the last N periods. As time moves forward, the window "moves" with it. filters noise and reveals trend. Two main types:

SMA (Simple Moving Average)

Equal weight to all N periods. Slower to react. Formula: SMAt = (Pt + Pt-1 + ... + Pt-N+1) / N

EMA (Exponential Moving Average)

More weight to recent prices. Faster, more jittery. Preferred for signals. Formula involves a smoothing factor α = 2/(N+1).

import matplotlib.pyplot as plt
import pandas as pd

# 1. Line chart with dual EMAs

plt.plot(df.index, df['Close'].ewm(span=20).mean(), label='EMA 20', color='#38bdf8')
plt.plot(df.index, df['Close'].ewm(span=50).mean(), label='EMA 50', color='#fbbf24')
plt.fill_between(df.index, df['Close'], alpha=0.1, color='white')
plt.legend(); plt.title('SPY Price with EMA Cloud')
plt.show()

# 2. Matplotlib candlestick (simplified)
def plot_candles(df, n=30):
    df = df.tail(n)
    fig, ax = plt.subplots(figsize=(10, 5))
    width = 0.6
    for i, (idx, row) in enumerate(df.iterrows()):
        color = '#34d399' if row['Close'] >= row['Open'] else '#f87171'
        ax.plot([i, i], [row['Low'], row['High']], color='white', linewidth=1)
        ax.add_patch(Rectangle((i-width/2, min(row['Open'], row['Close'])), 
                               width, abs(row['Close']-row['Open']), 
                               facecolor=color, edgecolor=color))
    ax.set_xticks(range(0, n, 5))
    ax.set_xticklabels([d.strftime('%m-%d') for d in df.index[::5]])
    ax.set_title('Candlestick Chart (Last 30 Days)')
    plt.show()

plot_candles(df)

# 3. Volume profile (price-level histogram)
plt.figure(figsize=(8, 5))
plt.hist2d(df['Close'], df['Volume'], bins=30, cmap='Blues')
plt.colorbar(label='Density')
plt.xlabel('Price'); plt.ylabel('Volume')
plt.title('Volume Profile Heatmap')
plt.show()

2. Statistical Distributions Core

Returns are random variables. To model them, we must see their distribution — the shape of uncertainty. This is where quant finance meets statistics.

Prerequisites Needed

Returns calculation, mean, standard deviation, percentiles. Basic Matplotlib histograms.

2.1 Histograms — The First Look

A histogram bins returns and counts frequency. It answers: "How often do I get a return of roughly X%?"

Key parameters: Number of bins (too few hides detail, too many adds noise). Use rules like Sturges or Freedman-Diaconis.

Diagram: Histogram Construction

2.2 KDE — Smoothing the Histogram

Kernel Density Estimation draws a smooth curve through the histogram. It's a probability density function (PDF) estimate. The area under the curve between two points gives the probability of returns falling in that range.

2.3 The Normal Distribution Myth

Finance textbooks love the Normal (Gaussian) distribution because it's mathematically convenient. Reality is different:

  • Fat tails (Leptokurtosis): Extreme moves happen more often than normal predicts
  • Skewness: Crashes are sudden (negative skew); rallies are gradual
  • Volatility clustering: Big moves cluster together; normal assumes independence

Diagram: Normal vs. Real Returns

2.4 Box Plots & Violin Plots — Summary Statistics Visualized

A box plot shows: median (line), interquartile range (box), and whiskers (typically 1.5×IQR). Points beyond are outliers.

Violin plots combine box plots with KDE — showing both summary and full distribution shape.

2.5 Q-Q Plots — Testing Normality

A Quantile-Quantile plot compares your data's quantiles against a theoretical normal distribution. If points fall on the diagonal line, your data is normal. Curved ends = fat tails. S-shaped = skewness.

🧪 Colab Experiment: Distribution Analysis

import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns
from scipy import stats
import yfinance as yf

# Get real data
returns = yf.download("SPY", start="2010-01-01")['Adj Close'].pct_change().dropna()

# 1. Histogram + KDE + Normal overlay
plt.figure(figsize=(10, 6))
sns.histplot(returns, bins=60, kde=True, stat='density', color='#38bdf8', alpha=0.6)
xmin, xmax = plt.xlim()
x = np.linspace(xmin, xmax, 100)
p = stats.norm.pdf(x, returns.mean(), returns.std())
plt.plot(x, p, '#f87171', linewidth=2, label='Normal Fit')
plt.title('SPY Returns: Real vs Normal')
plt.legend()
plt.show()

# 2. Box plot by year
returns_df = returns.to_frame(name='Returns')
returns_df['Year'] = returns_df.index.year
plt.figure(figsize=(12, 5))
sns.boxplot(data=returns_df, x='Year', y='Returns', palette='viridis')
plt.xticks(rotation=45)
plt.title('Annual Return Distributions (Box Plots)')
plt.show()

# 3. Q-Q Plot
fig, ax = plt.subplots(figsize=(8, 8))
stats.probplot(returns, dist="norm", plot=ax)
ax.get_lines()[0].set_markerfacecolor('#38bdf8')
ax.get_lines()[0].set_markersize(4)
ax.get_lines()[1].set_color('#f87171')
ax.set_title('Q-Q Plot: Are SPY Returns Normal?')
plt.show()

# 4. Print moments
print(f"Skewness: {returns.skew():.3f} (0 = symmetric)")
print(f"Kurtosis: {returns.kurtosis():.3f} (0 = normal tails)")

3. Correlation & Relationships Core

No asset trades in isolation. Correlation tells us how assets move together — the foundation of diversification, pairs trading, and risk aggregation.

Prerequisites Needed

Returns calculation, basic statistics (mean, std), scatter plots, time series visualization.

3.1 Correlation Coefficient — The Magic Number

Pearson's r ranges from -1 to +1:

  • +1: Perfect positive (A goes up, B goes up proportionally)
  • 0: No linear relationship
  • -1: Perfect negative (A goes up, B goes down proportionally)
r = Cov(X,Y) / (σX × σY)

Warning: Correlation measures linear relationships only. Two assets can be perfectly related in a non-linear way (e.g., circular) and have r ≈ 0.

Diagram: Correlation Visual Spectrum

3.2 Scatter Plots — Seeing the Relationship

Every point is one day. X-axis = Asset A return, Y-axis = Asset B return. The cloud's tilt reveals correlation. The spread reveals noise.

Pro tip: Color points by time to see if correlation is stable. Early years blue, recent years red. If colors separate, correlation is changing.

3.3 Correlation Matrix & Heatmaps

With many assets, pairwise scatter plots become overwhelming. A correlation matrix compresses everything into a grid. Heatmaps color-code the numbers for instant pattern recognition.

Diagram: Heatmap Structure

3.4 Rolling Correlation — Nothing Lasts Forever

Correlation is not static. During crises, correlations tend toward +1 (everything falls together). A rolling 60-day correlation plot reveals this instability.

3.5 Cointegration vs. Correlation

Correlation measures directional movement. Cointegration measures a long-term equilibrium relationship. Two assets can be uncorrelated in returns but cointegrated in price (they don't drift apart forever). This is the statistical basis for pairs trading.

🧪 Colab Experiment: Correlation Deep Dive

import yfinance as yf
import matplotlib.pyplot as plt
import seaborn as sns
import pandas as pd

# Download multiple assets
tickers = ['SPY', 'QQQ', 'IWM', 'GLD', 'TLT', 'VIXY']

sns.heatmap(returns.corr(), mask=mask, annot=True, cmap='RdYlGn', 
            center=0, vmin=-1, vmax=1, square=True, linewidths=1,
            cbar_kws={"shrink": .8})
plt.title('Cross-Asset Correlation Matrix', fontsize=16)
plt.show()

# 2. Scatter with regression
fig, axes = plt.subplots(2, 3, figsize=(15, 10))
axes = axes.flatten()
for i, t in enumerate(tickers[1:]):
    sns.regplot(x=returns['SPY'], y=returns[t], ax=axes[i], 
                scatter_kws={'alpha':0.3, 's':10}, line_kws={'color':'red'})
    axes[i].set_title(f'SPY vs {t} (r={returns.corr().loc["SPY", t]:.2f})')
plt.tight_layout()
plt.show()

# 3. Rolling correlation (crisis detector)
plt.figure(figsize=(12, 5))
rolling_corr = returns['SPY'].rolling(60).corr(returns['QQQ'])
plt.plot(rolling_corr.index, rolling_corr, color='#38bdf8', linewidth=2)
plt.axhline(y=returns.corr().loc['SPY', 'QQQ'], color='#f87171', linestyle='--', label='Full-period correlation')
plt.fill_between(rolling_corr.index, rolling_corr, alpha=0.3, color='#38bdf8')
plt.title('60-Day Rolling Correlation: SPY vs QQQ')
plt.legend()
plt.show()

4. Risk Metrics Visualization Core

Returns get the glory, but risk pays the bills. Quants visualize risk to answer: "How much can I lose, and when?"

Prerequisites Needed

Returns, distributions (percentiles, fat tails), time series, correlation basics.

4.1 Drawdown — The Investor's Nightmare

A drawdown measures the decline from a peak to a trough. Even if your strategy makes 10% annually, a 50% drawdown can ruin you (psychologically or via margin calls).

DDt = (Peakt - Currentt) / Peakt

The underwater curve plots drawdown over time. Every quant stares at this before deploying capital.

Diagram: Drawdown Mechanics

4.2 Value at Risk (VaR) — The Industry Standard

VaR asks: "What is my worst loss over the next N days, with X% confidence?"

  • Parametric VaR: Assumes normal distribution. Fast, flawed.
  • Historical VaR: Uses actual past returns. No distribution assumption.
  • Monte Carlo VaR: Simulates thousands of future paths. Computationally intensive.
VaRα = μ - zα × σ (parametric, 1-day)

4.3 Conditional VaR (CVaR) / Expected Shortfall

VaR tells you the threshold. CVaR tells you the average loss given you exceed that threshold. It's the "what happens in the tail" metric. Regulators prefer CVaR because it's coherentSatisfies mathematical properties like subadditivity: the risk of a portfolio should be less than or equal to the sum of individual risks..

4.4 Volatility Clustering

Markets have memory in variance. High-volatility days cluster together (ARCH/GARCH effects). Visualizing squared returns or rolling volatility reveals this clustering.

🧪 Colab Experiment: Risk Dashboard

import yfinance as yf
import numpy as np
import matplotlib.pyplot as plt
from scipy import stats

# Data
prices = yf.download("SPY", start="2018-01-01")['Adj Close']
returns = prices.pct_change().dropna()

# 1. Underwater curve (Drawdown)
running_max = prices.cummax()
drawdown = (prices - running_max) / running_max

fig, axes = plt.subplots(3, 1, figsize=(12, 10), sharex=True)
axes[0].plot(prices.index, prices, color='#34d399', linewidth=1.5)
axes[0].set_title('Price & Drawdown Analysis', fontsize=14)
axes[0].set_ylabel('Price')

axes[1].fill_between(drawdown.index, drawdown, 0, color='#f87171', alpha=0.7)
axes[1].set_ylabel('Drawdown %')
axes[1].axhline(y=-0.2, color='#fbbf24', linestyle='--', label='-20% line')
axes[1].legend()

# 2. Rolling volatility (30-day)
rolling_vol = returns.rolling(30).std() * np.sqrt(252)  # Annualized
axes[2].plot(rolling_vol.index, rolling_vol, color='#38bdf8', linewidth=1.5)
axes[2].fill_between(rolling_vol.index, rolling_vol, alpha=0.3, color='#38bdf8')
axes[2].set_ylabel('Annualized Volatility')
axes[2].set_xlabel('Date')

plt.tight_layout()
plt.show()

# 3. VaR visualization
mean = returns.mean()
std = returns.std()
var_95_param = mean - 1.645 * std
var_99_param = mean - 2.33 * std
var_95_hist = returns.quantile(0.05)
var_99_hist = returns.quantile(0.01)

plt.figure(figsize=(10, 6))
sns.histplot(returns, bins=80, stat='density', color='#1e2129', edgecolor='#38bdf8')
plt.axvline(var_95_param, color='#fbbf24', linestyle='--', linewidth=2, label=f'Parametric VaR 95%: {var_95_param:.4f}')
plt.axvline(var_95_hist, color='#f87171', linestyle='-', linewidth=2, label=f'Historical VaR 95%: {var_95_hist:.4f}')
plt.axvline(var_99_hist, color='#f87171', linestyle=':', linewidth=2, label=f'Historical VaR 99%: {var_99_hist:.4f}')
plt.legend()
plt.title('Value at Risk Visualization')
plt.show()

# 4. CVaR / Expected Shortfall
cvar_95 = returns[returns <= var_95_hist].mean()
print(f"CVaR 95% (Expected Shortfall): {cvar_95:.4f}")
print(f"Meaning: When SPY has a bad day (worst 5%), the average loss is {abs(cvar_95)*100:.2f}%")

5. Portfolio Theory Visualization Core

This is where quant engineering shines. You don't just analyze one asset — you construct, optimize, and visualize entire portfolios.

Prerequisites Needed

Returns, correlation, risk metrics (volatility, VaR), basic linear algebra (vectors, matrices), Matplotlib scatter plots.

5.1 The Risk-Return Plane

Every asset is a point: X-axis = risk (volatility), Y-axis = expected return. Investors want to be up and to the left (high return, low risk).

Diagram: The Efficient Frontier

5.2 Modern Portfolio Theory (Markowitz)

Given expected returns (μ) and covariance matrix (Σ), we solve:

Minimize: wTΣw subject to: wTμ = target_return, Σwi = 1

Where w is the weight vector. The solution traces the efficient frontier.

5.3 The Tangency Portfolio & Sharpe Ratio

The Sharpe Ratio measures excess return per unit of risk:

Sharpe = (Rp - Rf) / σp

The portfolio with the highest Sharpe ratio is the tangency portfolio — where the Capital Market Line touches the efficient frontier. This is the optimal risky portfolio.

5.4 Asset Allocation Visuals

Beyond scatter plots, we visualize weights:

  • Pie charts: Simple, but misleading (correlation matters more than weights)
  • Treemaps: Show hierarchy (asset class → sector → stock)
  • Stacked area charts: Show how weights drift over time
  • Risk parity waterfal: Equal risk contribution, not equal capital

5.5 Cumulative Returns — The Bottom Line

While the efficient frontier is theoretical, the cumulative return chart shows what actually matters: growth of $1 over time. Plot multiple strategies together to compare wealth generation.

🧪 Colab Experiment: Portfolio Optimization

import numpy as np
import pandas as pd
import yfinance as yf
import matplotlib.pyplot as plt
from scipy.optimize import minimize

# Download data
tickers = ['SPY', 'QQQ', 'IWM', 'GLD', 'TLT']
data = yf.download(tickers, start='2018-01-01')['Adj Close']
returns = data.pct_change().dropna()

# Expected returns and covariance
mu = returns.mean() * 252  # Annualized
Sigma = returns.cov() * 252

# Generate random portfolios
n_portfolios = 5000
n_assets = len(tickers)
results = np.zeros((3, n_portfolios))
weights_record = []

for i in range(n_portfolios):
    w = np.random.random(n_assets)
    w /= np.sum(w)
    weights_record.append(w)
    port_return = np.dot(w, mu)
    port_vol = np.sqrt(np.dot(w.T, np.dot(Sigma, w)))
    sharpe = (port_return - 0.02) / port_vol  # Assuming 2% risk-free
    results[0,i] = port_vol
    results[1,i] = port_return
    results[2,i] = sharpe

# Plot efficient frontier
plt.figure(figsize=(10, 7))
scatter = plt.scatter(results[0,:], results[1,:], c=results[2,:], cmap='viridis', alpha=0.6, s=20)
plt.colorbar(scatter, label='Sharpe Ratio')

# Mark max Sharpe
max_sharpe_idx = np.argmax(results[2,:])
plt.scatter(results[0,max_sharpe_idx], results[1,max_sharpe_idx], c='red', s=200, marker='*', label='Max Sharpe')

# Mark min volatility
min_vol_idx = np.argmin(results[0,:])
plt.scatter(results[0,min_vol_idx], results[1,min_vol_idx], c='blue', s=200, marker='*', label='Min Volatility')

plt.xlabel('Volatility (Risk)')
plt.ylabel('Expected Return')
plt.title('Monte Carlo Simulation: Efficient Frontier')
plt.legend()
plt.show()

# Print optimal weights
print("Max Sharpe Portfolio Weights:")
for t, w in zip(tickers, weights_record[max_sharpe_idx]):
    print(f"  {t}: {w:.2%}")

6. Advanced Quant Visualizations Advanced

Professional quant platforms visualize data that retail charts never show: microstructure, regime changes, factor exposures, and backtest diagnostics.

Prerequisites Needed

All previous topics: time series, distributions, correlation, risk metrics, portfolio theory. Some machine learning basics helpful but not required.

6.1 Order Book Depth — Market Microstructure

The order book is a real-time list of buy (bid) and sell (ask) orders. Visualizing depth shows liquidity and support/resistance levels.

  • Depth chart: Cumulative volume at each price level. Steep walls = strong support/resistance.
  • Spread: Gap between best bid and best ask. Tight spread = liquid market.
  • Imbalance: More bid volume than ask = bullish pressure.

Diagram: Order Book Depth

6.2 Regime Detection — When the Rules Change

Markets switch between "calm" and "crisis" regimes. A Hidden Markov Model (HMM) can infer the current regime from price action. Visualizing regimes as background colors on price charts is powerful.

6.3 Factor Exposure — What's Really Driving Returns?

Is your portfolio up because you're smart, or because the market is up? Factor models decompose returns into:

  • Market (Beta): Sensitivity to overall market
  • Value (HML): Cheap vs expensive stocks
  • Size (SMB): Small vs large cap
  • Momentum (MOM): Trend-following effect
  • Quality: Profitable, stable companies

6.4 Backtest Visualization — Don't Fool Yourself

Every quant strategy must be backtested. Essential visualizations:

  • Equity curve: Cumulative P&L over time
  • Underwater curve: Drawdowns of the strategy
  • Monthly returns heatmap: Spot consistency
  • Trade P&L distribution: Win rate, skew, outliers
  • Rolling Sharpe: Is the edge decaying?

# Advanced Quant Visualization Lab
import numpy as np
import pandas as pd
import yfinance as yf
import matplotlib.pyplot as plt
from sklearn.mixture import GaussianMixture

# 1. Regime Detection with Gaussian Mixture
spy = yf.download('SPY', start='2000-01-01')['Adj Close']
returns = spy.pct_change().dropna()

# Fit 2-regime model on volatility features
vol_features = returns.rolling(20).std().dropna().values.reshape(-1, 1)
gmm = GaussianMixture(n_components=2, random_state=42)
regimes = gmm.fit_predict(vol_features)

# Align indices
aligned_returns = returns.iloc[19:]
aligned_spy = spy.iloc[19:]

# Plot regimes
fig, ax = plt.subplots(figsize=(14, 6))
colors = ['#34d399', '#f87171']  # low vol, high vol
for regime in [0, 1]:
    mask = regimes == regime
    ax.scatter(aligned_spy.index[mask], aligned_spy.values[mask], 
               c=colors[regime], s=1, alpha=0.6, label=f'Regime {regime}')
ax.plot(spy.index, spy.values, color='white', alpha=0.3, linewidth=0.5)
ax.set_title('SPY Price Colored by Volatility Regime')
ax.legend()
plt.show()

# 2. Simple Backtest Visualization Framework
def visualize_backtest(returns, title='Strategy'):
    cum_returns = (1 + returns).cumprod()
    running_max = cum_returns.cummax()
    drawdown = (cum_returns - running_max) / running_max
    
    fig, axes = plt.subplots(3, 1, figsize=(12, 10), sharex=True)
    
    # Equity curve
    axes[0].plot(cum_returns.index, cum_returns, color='#34d399', linewidth=1.5)
    axes[0].set_title(f'{title} - Equity Curve')
    axes[0].set_ylabel('Growth of $1')
    
    # Drawdown
    axes[1].fill_between(drawdown.index, drawdown, 0, color='#f87171', alpha=0.7)
    axes[1].set_ylabel('Drawdown')
    
    # Rolling Sharpe (252-day)
    rolling_sharpe = returns.rolling(252).mean() / returns.rolling(252).std() * np.sqrt(252)
    axes[2].plot(rolling_sharpe.index, rolling_sharpe, color='#38bdf8')
    axes[2].axhline(y=1, color='#fbbf24', linestyle='--')
    axes[2].set_ylabel('Rolling Sharpe')
    axes[2].set_xlabel('Date')
    
    plt.tight_layout()
    plt.show()

# Example: Simple momentum strategy
position = np.where(returns.rolling(60).mean() > returns.rolling(120).mean(), 1, -1)
strategy_returns = position[1:] * returns[1:]
visualize_backtest(strategy_returns, '60/120 Momentum Strategy')

Static plots are for research. Interactive dashboards are for trading desks. Learn to build tools that update, filter, and respond.

Prerequisites Needed

All visualization topics above. Basic HTML/JS understanding helpful but not required (we use Python abstractions).

7.1 Plotly — Interactive Python Charts

Plotly renders web-based visualizations with zoom, pan, hover tooltips, and toggles. It's the bridge between Python analysis and web interactivity.

Key features: Range sliders, dropdown menus, buttons, linked subplots, and 3D charts.

7.2 Dash — Full Applications

Dash (by Plotly) lets you build full analytical web apps in pure Python. Layouts are declarative, callbacks handle interactivity.

Components

dcc.Graph: Renders Plotly figures.
dcc.Dropdown: User selections.
dcc.Slider: Numeric inputs.
html.Div: Containers and layout.

Callbacks

Python functions that run automatically when inputs change. @app.callback(Output, Input) connects user actions to data updates.

7.3 Streamlit — Rapid Prototyping

Streamlit is even simpler than Dash. One script becomes a web app. Perfect for internal quant tools and quick prototypes.

7.4 Real-Time Considerations

Financial dashboards often need live data:

  • WebSockets: Push data from server to client (e.g., Webull, Binance streams)
  • Interval polling: Client asks for updates every N seconds
  • Redis/Queue: Buffer incoming tick data before visualization
