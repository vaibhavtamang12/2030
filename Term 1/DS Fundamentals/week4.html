Prerequisites: Building Your Foundation

Before diving into calculus and optimization, you need to feel comfortable with a handful of concepts from algebra and functions. This section is a friendly refresher — skip what you know, absorb what you don't.

1. Functions — What are they?

A function is simply a machine: you put something in, it gives something back. We write f(x) — "f of x". Whatever you put in place of x, the function processes it and returns a number.

🧠 Analogy
Think of a function as a vending machine. You press button x = 3, and the machine gives you f(3) = 9 (if it's squaring your input). Press x = 5, get 25. Always the same rule, different inputs.
examples of functions f(x) = x² ← "square x" g(x) = 2x + 5 ← "double x, then add 5" h(x) = eˣ ← "raise Euler's number to the power x" p(x) = ln(x) ← "logarithm of x"

2. Variables & Notation

Scalar

A single number. E.g., x = 3.14, or the price of a stock today. One-dimensional.

Vector

A list of numbers. E.g., x = [1, 2, 3]. Think of it as a point in multi-dimensional space.

Matrix

A grid of numbers. Multiple vectors stacked together. A 3×3 matrix has 9 numbers arranged in 3 rows and 3 columns.

3. Limits — The Concept of "Approaching"

A limit asks: what value does a function approach as x gets close to (but doesn't necessarily reach) some number? This is the conceptual engine behind calculus.

lim (x → 2) of f(x) = 4 Means: as x gets closer and closer to 2, f(x) gets closer and closer to 4. We don't care what happens exactly at x = 2, only near it.
💡

You don't need to compute limits rigorously for most ML/quant work. Just understand: a derivative IS a limit — specifically a limit of a very small ratio.

4. Slope of a Line

The slope of a straight line tells you how steep it is: for every 1 step you take to the right, how many steps do you go up (or down)?

slope = (rise) / (run) = Δy / Δx = (y₂ - y₁) / (x₂ - x₁) If slope = 2: go 1 right → go 2 up. Steep upward. If slope = 0: horizontal line. Flat. If slope = -3: go 1 right → go 3 down. Downward.

5. Exponentials and Logarithms

These two functions appear constantly in quant finance and ML. They are inverses of each other.

eˣ (exponential): grows extremely fast. e ≈ 2.718... ln(x) (natural log): inverse of eˣ. ln(e³) = 3. Key rules: ln(a·b) = ln(a) + ln(b) ln(aᵇ) = b·ln(a) d/dx [eˣ] = eˣ ← (the derivative of eˣ is itself — magic!)

Derivatives: The Calculus of Change

A derivative answers one question: "How fast is this function changing right now?" It's the instantaneous slope at any point on a curve. Everything in optimization — gradient descent, Lagrange multipliers, convex analysis — is built on derivatives.

What Is a Derivative, Really?

Imagine you're driving a car. Your speedometer shows 60 mph. That's a derivative — it tells you how fast your position is changing right now, not over the whole trip.

🚗 Analogy: Speed vs. Position
Position = f(t) = where you are at time t.
Velocity = f'(t) = how fast position is changing = the derivative.
Acceleration = f''(t) = how fast velocity is changing = the second derivative.

Mathematically, a derivative is computed by taking the slope of a line between two nearby points and letting those points get infinitely close:

formal definition of derivative f'(x) = lim(h → 0) [ f(x + h) - f(x) ] / h Translation: • Compute the slope between point x and a nearby point x+h • Make h smaller and smaller (approach zero) • The value that slope approaches = the derivative at x

Derivative Notation

There are three common ways to write the same thing — they all mean "derivative of f with respect to x":

f'(x) ← Lagrange notation (most common for functions) dy/dx ← Leibniz notation (most common in optimization) Df(x) ← Operator notation All three say: "differentiate f with respect to x"

Core Differentiation Rules — Your Toolkit

You don't need to re-derive from first principles every time. These rules let you differentiate almost anything:

Power Rule

d/dx [xⁿ] = n·xⁿ⁻¹

Example: d/dx [x³] = 3x²

Constant Rule

d/dx [c] = 0

Constants don't change, so their derivative is zero.

Sum Rule

d/dx [f + g] = f' + g'

Differentiate each term separately, then add.

Product Rule

d/dx [fg] = f'g + fg'

"First times deriv of second, plus second times deriv of first"

Chain Rule

d/dx [f(g(x))] = f'(g(x))·g'(x)

⚠️ The most important rule for ML/deep learning!

Exponential & Log

d/dx [eˣ] = eˣ d/dx [ln x] = 1/x

eˣ is its own derivative — incredibly special.

The Chain Rule — Why It's Everything

The chain rule handles composed functions — functions inside functions. Neural networks are, at their core, chains of composed functions. Backpropagation is literally just the chain rule applied recursively.

chain rule walkthrough Let h(x) = (3x² + 1)⁵ Step 1: Identify outer and inner function outer: f(u) = u⁵ → f'(u) = 5u⁴ inner: g(x) = 3x² + 1 → g'(x) = 6x Step 2: Apply chain rule h'(x) = f'(g(x)) · g'(x) = 5(3x² + 1)⁴ · 6x = 30x(3x² + 1)⁴

Second Derivatives — Concavity & Curvature

The first derivative tells you the slope. The second derivative tells you whether the slope is increasing or decreasing — in other words, whether the function is curving upward (concave up, like a bowl) or downward (concave down, like a hill).

f''(x) > 0 → concave up (bowl shape) → local minimum candidate f''(x) < 0 → concave down (hill shape) → local maximum candidate f''(x) = 0 → inflection point (changes concavity)

Critical Points — Finding Extrema

An extremum (minimum or maximum) of a function occurs where the slope is zero — the function is "flat" for an instant. These are called critical points.

Set f'(x) = 0 and solve for x. These are your critical point candidates.
Compute f''(x) at each critical point. If f''(x) > 0 → local min. If f''(x) < 0 → local max.
Check boundary conditions if the domain is bounded (e.g., x ∈ [0, 10]).
example: find the minimum of f(x) = x² - 4x + 7 Step 1: f'(x) = 2x - 4 = 0 → x = 2 Step 2: f''(x) = 2 > 0 → concave up → local minimum at x = 2 Step 3: f(2) = 4 - 8 + 7 = 3 → minimum value is 3 at x = 2

Partial Derivatives — First Glimpse

When a function takes multiple inputs, we can differentiate with respect to one variable at a time while treating all others as constants. This is called a partial derivative and it's the foundation of gradients (Tab 02).

partial derivatives of f(x,y) = x² + 3xy + y² ∂f/∂x = 2x + 3y ← differentiate w.r.t. x, treat y as constant ∂f/∂y = 3x + 2y ← differentiate w.r.t. y, treat x as constant
📈 Quant Finance Connection

Option Greeks are literally derivatives of option prices:

  • Delta (Δ) = ∂V/∂S — how option price changes with stock price
  • Gamma (Γ) = ∂²V/∂S² — second derivative; rate of change of Delta
  • Theta (Θ) = ∂V/∂t — how option price changes with time
  • Vega (ν) = ∂V/∂σ — sensitivity to volatility

Gradients: Direction of Steepest Ascent

A gradient is just a derivative for functions with multiple inputs — it's a vector of partial derivatives. While a derivative says "how steep is this 1D curve," a gradient says "which direction is steepest uphill in this multi-dimensional landscape." Gradient descent flips this: we walk downhill.

From 1D Derivatives to Multi-Dimensional Gradients

Think of a mountain range. With one variable, you could only walk north-south. A derivative told you whether north was uphill or downhill. Now with two or more variables, you can walk in any direction. The gradient is an arrow pointing in the steepest uphill direction.

🏔️ Analogy: Mountain Hiker
Imagine standing on a hilly terrain. Close your eyes and take one step in a completely random direction. Now imagine finding the direction where the slope is steepest — that's the direction of the gradient. If you want to reach the summit fastest, walk along the gradient. If you want to reach the valley fastest (minimize), walk against the gradient.

Definition of the Gradient

For a function f(x₁, x₂, ..., xₙ) of n variables, the gradient is a vector of all partial derivatives:

gradient definition ∇f = [∂f/∂x₁, ∂f/∂x₂, ..., ∂f/∂xₙ]ᵀ For f(x, y): ∇f = [∂f/∂x, ∂f/∂y]ᵀ Example: f(x,y) = x² + 3xy + y² ∂f/∂x = 2x + 3y ∂f/∂y = 3x + 2y ∇f = [2x + 3y, 3x + 2y]ᵀ At point (1, 1): ∇f(1,1) = [2(1) + 3(1), 3(1) + 2(1)] = [5, 5] This arrow [5,5] points in the steepest uphill direction from (1,1).

Gradient Descent — The Core of ML Training

If the gradient points uphill, then to find the minimum of a function (minimize a loss function in ML), we walk in the opposite direction of the gradient. This is gradient descent.

gradient descent update rule θ_new = θ_old - α · ∇f(θ_old) Where: θ = parameters we're optimizing (weights, prices, etc.) α = learning rate (step size — how far to step downhill) ∇f = gradient at current position Intuition: "Take a step in the downhill direction, scaled by α"

Learning Rate: The Critical Hyperparameter

Too Large (α too big)

You overshoot the minimum. Like jumping over the valley. The algorithm diverges and bounces around or explodes.

Just Right

Smooth, steady convergence to the minimum. Typical starting values: 0.01, 0.001, or 0.0001.

Too Small (α too small)

You converge, but extremely slowly. Takes forever. In practice, computationally wasteful.

The Jacobian Matrix — Gradients of Vector Functions

When your function outputs a vector (not just a scalar), the gradient generalizes to a Jacobian matrix. Each row is the gradient of one output with respect to all inputs.

jacobian matrix For f: Rⁿ → Rᵐ (takes n inputs, gives m outputs) [ ∂f₁/∂x₁ ∂f₁/∂x₂ ... ∂f₁/∂xₙ ] J = [ ∂f₂/∂x₁ ∂f₂/∂x₂ ... ∂f₂/∂xₙ ] [ ... ] [ ∂fₘ/∂x₁ ∂fₘ/∂x₂ ... ∂fₘ/∂xₙ ] This is an m×n matrix. Used extensively in neural network backpropagation.

The Hessian Matrix — Second-Order Information

The Hessian is the matrix of second partial derivatives. It generalizes the second derivative to multiple dimensions and tells us about curvature — critical for determining whether a critical point is a min, max, or saddle point.

hessian matrix for f(x,y) [ ∂²f/∂x² ∂²f/∂x∂y ] H = [ ∂²f/∂y∂x ∂²f/∂y² ] For f(x,y) = x² + 3xy + y²: ∂²f/∂x² = 2, ∂²f/∂y² = 2, ∂²f/∂x∂y = 3 [ 2 3 ] H = [ 3 2 ] Rules: • If det(H) > 0 and ∂²f/∂x² > 0 → local minimum • If det(H) > 0 and ∂²f/∂x² < 0 → local maximum • If det(H) < 0 → saddle point (min in one direction, max in another)

Stochastic Gradient Descent (SGD) & Variants

In practice, computing the full gradient over millions of data points is expensive. Variants of gradient descent compute approximate gradients:

Portfolio optimization minimizes a risk objective over many assets. The gradient of the portfolio variance with respect to asset weights tells you exactly how to adjust each holding to reduce risk. Modern portfolio theory (Markowitz) is gradient-based optimization at its core. Factor models, risk attribution, and hedging all use gradient concepts.

Lagrange Multipliers: Constrained Optimization

What if you can't just wander freely looking for the minimum? Real-world problems have constraints: "you only have $1,000 to invest," "portfolio weights must sum to 1," "this protein must fit in this binding pocket." Lagrange multipliers let you find the optimal solution while respecting constraints.

The Core Problem: Constrained Optimization

🎒 Analogy: Hiker with a Trail
Earlier (gradient descent) you were free to walk anywhere on the mountain. Now imagine there's a fence — a constraint — that forces you to walk along a specific trail. You want to find the lowest point on that trail. You can't just walk straight downhill; you must stay on the trail. Lagrange multipliers solve exactly this.
constrained optimization problem Minimize: f(x, y) ← objective function Subject to: g(x, y) = 0 ← equality constraint Examples: • Minimize portfolio variance, subject to: weights sum to 1 • Maximize utility, subject to: budget = $1000 • Minimize loss, subject to: model complexity ≤ threshold

The Key Insight: Gradient Alignment

At the optimal constrained solution, something beautiful happens: the gradients of f and g must be parallel (point in the same direction). If they weren't parallel, you could slide along the constraint and still decrease f — meaning you haven't found the minimum yet.

lagrange condition ∇f(x*, y*) = λ · ∇g(x*, y*) This says: "At the optimum, f's gradient equals λ times g's gradient" λ is the Lagrange multiplier — a scalar we also solve for. In component form: ∂f/∂x = λ · ∂g/∂x ∂f/∂y = λ · ∂g/∂y g(x, y) = 0 ← don't forget the constraint!

The Lagrangian Function

We combine the objective and constraint into one unconstrained function called the Lagrangian. We then find its critical points by setting all partial derivatives (including the derivative with respect to λ) to zero.

the lagrangian ℒ(x, y, λ) = f(x, y) - λ · g(x, y) Now take all partial derivatives and set = 0: ∂ℒ/∂x = ∂f/∂x - λ·∂g/∂x = 0 ∂ℒ/∂y = ∂f/∂y - λ·∂g/∂y = 0 ∂ℒ/∂λ = -g(x, y) = 0 ← this just enforces the constraint!

Complete Worked Example

Let's minimize f(x, y) = x² + y² (distance from origin) subject to g(x, y) = x + y - 1 = 0 (must lie on the line x + y = 1).

step-by-step lagrange multiplier solution Step 1: Write the Lagrangian ℒ = x² + y² - λ(x + y - 1) Step 2: Set all partial derivatives = 0 ∂ℒ/∂x = 2x - λ = 0 → λ = 2x ...(1) ∂ℒ/∂y = 2y - λ = 0 → λ = 2y ...(2) ∂ℒ/∂λ = -(x+y-1) = 0 → x + y = 1 ...(3) Step 3: Solve the system From (1) and (2): 2x = 2y → x = y Substituting into (3): x + x = 1 → x = 1/2, y = 1/2 λ = 2(1/2) = 1 Step 4: The answer Optimal point: (x*, y*) = (1/2, 1/2) Minimum value: f(1/2, 1/2) = (1/2)² + (1/2)² = 1/2 Interpretation of λ = 1: If we relaxed the constraint from x+y=1 to x+y=1+ε, the minimum would decrease by approximately λ·ε = 1·ε. λ measures the "value of relaxing the constraint."

Multiple Constraints

You can have multiple equality constraints. Each one gets its own Lagrange multiplier:

Minimize f(x) Subject to: g₁(x) = 0, g₂(x) = 0, ..., gₖ(x) = 0 Lagrangian: ℒ(x, λ₁,...,λₖ) = f(x) - λ₁g₁(x) - λ₂g₂(x) - ... - λₖgₖ(x)

KKT Conditions — Inequality Constraints

Lagrange multipliers handle equality constraints (g = 0). The KKT (Karush-Kuhn-Tucker) conditions extend this to inequality constraints (g ≤ 0), which are far more common in practice.

KKT conditions for: min f(x) s.t. gᵢ(x) ≤ 0 1. Stationarity: ∇f(x*) + Σ μᵢ·∇gᵢ(x*) = 0 2. Primal Feas: gᵢ(x*) ≤ 0 for all i 3. Dual Feas: μᵢ ≥ 0 for all i 4. Complementarity: μᵢ · gᵢ(x*) = 0 for all i Condition 4 is key: either the constraint is active (gᵢ = 0) OR its multiplier is zero (μᵢ = 0). They can't both be nonzero.
⚠️

KKT conditions are necessary for optimality (always), but only sufficient when the problem is convex. This is why convex optimization (Tab 04) is so powerful — KKT becomes a complete characterization.

📈 Quant Finance Connection

Markowitz Portfolio Optimization: Minimize portfolio variance = wᵀΣw, subject to wᵀ1 = 1 (weights sum to 1) and wᵀμ = r (achieve target return r). This is solved exactly with Lagrange multipliers with two constraints, yielding the famous efficient frontier. Every point on the frontier is a Lagrange solution with a different target return r.

Convex Optimization: When Local = Global

Most optimization problems are hard because every local minimum you find might not be the global minimum. Convex optimization is the golden class of problems where this ambiguity vanishes — any local minimum IS the global minimum. This property makes convex problems tractable, efficient, and reliable.

What Is a Convex Set?

🍇 Analogy: The Rubber Band Test
Take two points in a set. Draw the straight line connecting them. If the entire line segment stays inside the set, the set is convex. A circle is convex. A crescent moon is not — draw a line from tip to tip and part of it falls outside.

What Is a Convex Function?

A function is convex if the set of points above its graph is a convex set. Equivalently: for any two points on the function, the line segment between them lies above or on the function.

formal definition of convexity f is convex if for all x, y and t ∈ [0, 1]: f(tx + (1-t)y) ≤ t·f(x) + (1-t)·f(y) In English: "The function value at any weighted average of x and y is no more than the weighted average of the function values." This is the "chord lies above the curve" condition.

Why Convexity Is So Powerful

No Local Minima Traps

Every local minimum is the global minimum. Gradient descent always finds the true optimal solution, not a suboptimal local one.

KKT = Optimality

For convex problems, KKT conditions are both necessary AND sufficient. You can certify optimality algebraically.

Efficient Algorithms

Problems like Linear Programming, Quadratic Programming, and SOCP can be solved in polynomial time. Interior point methods are blazing fast.

Duality

Convex problems have a "dual problem" whose solution gives you a lower bound and often a certificate of optimality. No guessing.

Testing Convexity

how to verify convexity Method 1: Second Derivative Test (single variable) f''(x) ≥ 0 for all x → f is convex Method 2: Hessian Test (multiple variables) H(x) is positive semidefinite (PSD) for all x → f is convex H is PSD means: all eigenvalues of H are ≥ 0 Method 3: Known Convex Functions (memorize these!) • Linear: f(x) = aᵀx + b → convex (and concave) • Quadratic: f(x) = xᵀQx + bᵀx + c, Q PSD → convex • Exponential: eˣ → convex • Powers: |x|ᵖ for p ≥ 1 → convex • Log-sum-exp: ln(Σ eˣⁱ) → convex • Norm: ||x|| → convex Method 4: Composition Rules • Sum of convex functions → convex • Nonneg scaling of convex → convex • Affine composition: f(Ax+b) is convex if f is convex

Standard Convex Optimization Problem Forms

Linear Programming (LP)

min cᵀx s.t. Ax ≤ b Cx = d

Linear objective, linear constraints. Solved by Simplex or interior point methods. Used heavily in logistics, trading, resource allocation.

Quadratic Programming (QP)

min (1/2)xᵀQx + cᵀx s.t. Ax ≤ b

Quadratic objective (Q PSD), linear constraints. Markowitz portfolio optimization is a QP. SVMs are QPs.

Second-Order Cone Program (SOCP)

min fᵀx s.t. ||Aᵢx + bᵢ||₂ ≤ cᵢᵀx + dᵢ

More general than LP and QP. Used in robust optimization and risk-constrained portfolio design.

Semidefinite Program (SDP)

min Tr(CX) s.t. Tr(AᵢX) = bᵢ X ⪰ 0 (PSD)

Optimization over positive semidefinite matrices. Used in covariance estimation, control theory, and combinatorial relaxations.

Duality — The Mirror Problem

Every convex optimization problem has a dual problem. The dual provides a lower bound on the optimal value. For convex problems, strong duality holds — the dual optimal equals the primal optimal.

primal-dual relationship Primal: min f(x) s.t. g(x) ≤ 0 Lagrangian: ℒ(x, λ) = f(x) + λᵀg(x), λ ≥ 0 Dual function: q(λ) = min_x ℒ(x, λ) Dual problem: max q(λ) s.t. λ ≥ 0 Key facts: • q(λ) ≤ p* for all λ ≥ 0 (weak duality — always true) • For convex problems: q(λ*) = p* (strong duality) • λ* gives the "price" of each constraint

Gradient Descent for Convex Functions

When a function is convex, gradient descent is guaranteed to converge to the global minimum (given appropriate learning rate). The convergence rate depends on the problem structure:

convergence rates Convex + Lipschitz: O(1/√k) iterations to ε accuracy Convex + Smooth: O(1/k) iterations Strongly convex + smooth: O(log(1/ε)) iterations (linear convergence) Strong convexity: f(y) ≥ f(x) + ∇f(x)ᵀ(y-x) + (m/2)||y-x||² (function curves up at least as fast as a quadratic with curvature m)

Modern Portfolio Theory is a QP. Risk parity is an SOCP. Robust portfolio optimization under uncertainty is an SOCP or SDP. Solve a constrained optimization problem numerically and visualize the solution with Lagrange geometry.

📓 Colab Cell 3from scipy.optimize import minimize import numpy as np import matplotlib.pyplot as plt # Problem: minimize f(x,y) = (x-3)² + (y-2)² # Subject to: x + y = 3 (equality constraint) def f(xy): x, y = xy return (x-3)**2 + (y-2)**2 constraint = {'type': 'eq', 'fun': lambda xy: xy[0] + xy[1] - 3} result = minimize(f, [0, 0], constraints=constraint, method='SLSQP') x_opt, y_opt = result.x print(f"Numerical solution: x={x_opt:.4f}, y={y_opt:.4f}") print(f"Optimal value: f = {result.fun:.4f}") print(f"Analytical solution: x=2.5, y=0.5") # verify by hand! # Visualize x_range = np.linspace(-1, 5, 200) y_range = np.linspace(-1, 4, 200) X, Y = np.meshgrid(x_range, y_range) Z = (X-3)**2 + (Y-2)**2 plt.figure(figsize=(9, 7)) cs = plt.contourf(X, Y, Z, levels=25, cmap='plasma', alpha=0.7) plt.colorbar(cs, label='f(x,y)') plt.contour(X, Y, Z, levels=15, colors='white', alpha=0.4, lw=0.5) # Draw constraint line x+y=3 x_line = np.linspace(-1, 4, 100) plt.plot(x_line, 3-x_line, 'cyan', lw=3, label='Constraint: x+y=3') plt.scatter([x_opt], [y_opt], color='gold', s=250, zorder=10, label=f'Optimum ({x_opt:.2f},{y_opt:.2f})') plt.scatter([3], [2], color='white', s=100, marker='*', zorder=9, label='Unconstrained min (3,2)') plt.legend(); plt.title('Constrained Optimization via Lagrange Multipliers') plt.xlabel('x'); plt.ylabel('y'); plt.show()
Experiment 4 — Convex Optimization with CVXPY

Use the CVXPY library to solve a Markowitz-style portfolio optimization problem.

📓 Colab Cell 4 (run: !pip install cvxpy first)# !pip install cvxpy ← uncomment and run this first import cvxpy as cp import numpy as np import matplotlib.pyplot as plt np.random.seed(42) n_assets = 5 # number of assets in portfolio # Generate random expected returns and covariance matrix mu = np.random.uniform(0.05, 0.20, n_assets) # expected returns A = np.random.randn(n_assets, n_assets) Sigma = A.T @ A / n_assets + 0.1 * np.eye(n_assets) # PSD covariance # CVXPY problem: minimize variance, target return r_target w = cp.Variable(n_assets) # portfolio weights (our optimization variable) # Efficient frontier: sweep over target returns target_returns = np.linspace(mu.min(), mu.max(), 40) frontier_risk, frontier_ret = [], [] for r in target_returns: constraints = [ cp.sum(w) == 1, # weights sum to 1 mu @ w >= r, # achieve at least target return w >= 0 # no short selling (try removing this!) ] prob = cp.Problem(cp.Minimize(cp.quad_form(w, Sigma)), constraints) prob.solve(solver=cp.OSQP, verbose=False) if prob.status == 'optimal': frontier_risk.append(np.sqrt(prob.value)) frontier_ret.append(r) # Individual assets asset_risks = [np.sqrt(Sigma[i,i]) for i in range(n_assets)] plt.figure(figsize=(10, 6)) plt.plot(frontier_risk, frontier_ret, 'gold', lw=3, label='Efficient Frontier') plt.scatter(asset_risks, mu, s=100, c='cyan', zorder=5, label='Individual Assets') for i, (r, m) in enumerate(zip(asset_risks, mu)): plt.annotate(f'Asset {i+1}', (r, m), textcoords='offset points', xytext=(8,4), fontsize=9, color='white') plt.xlabel('Risk (std dev)'); plt.ylabel('Expected Return') plt.title('Markowitz Efficient Frontier (Convex QP)') plt.legend(); plt.grid(True, alpha=0.2); plt.show() print("\nOptimal weights for min-variance portfolio:") print(np.round(w.value, 3))
Experiment 5 — Automatic Differentiation with PyTorch

Use PyTorch autograd to automatically compute gradients — the foundation of all modern deep learning.

📓 Colab Cell 5import torch import torch.nn as nn import numpy as np import matplotlib.pyplot as plt # Create a tensor with gradient tracking x = torch.tensor(2.0, requires_grad=True) # Define a function y = x**4 - 3*x**3 + x**2 + 5 # Compute gradient automatically! y.backward() # dy/dx computed automatically via chain rule print(f"x = {x.item()}") print(f"y = {y.item():.4f}") print(f"dy/dx (autograd) = {x.grad.item():.4f}") print(f"dy/dx (manual: 4x³-9x²+2x = {4*8 - 9*4 + 4:.4f}) ✓") # Gradient descent to minimize y=f(x) x = torch.tensor([-0.5], requires_grad=True) optimizer = torch.optim.SGD([x], lr=0.005) history = [] for step in range(200): optimizer.zero_grad() loss = x**4 - 3*x**3 + x**2 + 5 loss.backward() optimizer.step() history.append(loss.item()) x_vals = np.linspace(-1, 3, 300) y_vals = x_vals**4 - 3*x_vals**3 + x_vals**2 + 5 fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(13, 5)) ax1.plot(x_vals, y_vals, 'royalblue', lw=2) ax1.axvline(x.item(), color='gold', lw=2, ls='--', label=f'Found min at x={x.item():.3f}') ax1.legend(); ax1.set_title('Function and Found Minimum') ax2.plot(history, 'tomato'); ax2.set_title('Loss over Gradient Descent Steps') ax2.set_xlabel('Step'); ax2.set_ylabel('f(x)') plt.tight_layout(); plt.show()
