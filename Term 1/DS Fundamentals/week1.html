Your Learning Roadmap

How all these topics connect — and why we study them in this order

This guide is designed for someone who wants to become a quantitative engineer (quant). Quants build mathematical models that price financial instruments, manage risk, and find alpha (profit opportunities). Every single topic in this guide is used in real trading systems, risk engines, and portfolio optimizers at hedge funds, banks, and asset managers.

The Big Picture: Why Linear Algebra?

Imagine you manage a portfolio of 500 stocks. Every day you need to know: How much risk am I taking? That question, at its core, is a linear algebra problem. Prices move together (correlations), the total risk of your portfolio is a quadratic form involving a covariance matrix, and you can compress 500 dimensions of risk into a handful of key drivers using PCA and eigenvectors. Without linear algebra, modern quant finance is impossible.

What Does a Quant Actually Do With This?

Risk Management: A risk manager at a hedge fund computes the covariance matrix of 300 stocks every morning to estimate how much money the fund could lose in a bad day.

Factor Models: PCA is used to find 5–10 "risk factors" (like market, size, value) from thousands of stock returns, drastically simplifying the risk model.

Vectors in Finance — The Core Intuition

A portfolio weight vector is one of the most important vectors in quant finance:

w = [w₁, w₂, w₃, ..., wₙ]ᵀ where w₁ + w₂ + ... + wₙ = 1

Each wᵢ is the fraction of your capital invested in stock i. If you have $1M and invest $300K in Apple, $400K in Google, $300K in Tesla: w = [0.3, 0.4, 0.3]ᵀ

Dot Product — The Most Important Vector Operation

The dot product (inner product) of two vectors multiplies them element-wise and sums the result:

a · b = Σᵢ aᵢbᵢ = a₁b₁ + a₂b₂ + ... + aₙbₙ
🎯 Financial Analogy: Portfolio Return

The portfolio return is a dot product! If your weights are w = [0.3, 0.4, 0.3] and today's returns are r = [0.02, -0.01, 0.03], then:

Portfolio Return = w · r = 0.3×0.02 + 0.4×(-0.01) + 0.3×0.03 = 0.006 - 0.004 + 0.009 = 0.011 = 1.1%

Summation Notation in Finance

You'll see this everywhere. It's just a compact way to write addition:

μ_portfolio = Σᵢ wᵢ μᵢ

This says: the portfolio expected return equals the weighted sum of individual asset expected returns. Every formula in portfolio theory is secretly a dot product or matrix product.

Matrix Operations

The language of financial data

A matrix is a rectangular array of numbers arranged in rows and columns. In finance, matrices are everywhere: a returns matrix (days × assets), a covariance matrix (assets × assets), a factor loading matrix. Mastering these operations is non-negotiable for quant work.

What Is a Matrix?

📊 Spreadsheet Analogy

If you've ever used Excel or a DataFrame in Python/Pandas, you've used a matrix. Think of a matrix as a spreadsheet where every cell contains a number. Each row is usually an observation (a day, a simulation), each column is a variable (a stock, a feature).

An m × n matrix has m rows and n columns. We write it as A ∈ ℝᵐˣⁿ. The element in row i, column j is written Aᵢⱼ or A[i,j].

Matrix Addition & Scalar Multiplication

Addition: Add element-by-element. Matrices must have the same shape.

C = A + B ⟹ Cᵢⱼ = Aᵢⱼ + Bᵢⱼ

Scalar multiplication: Multiply every element by the same number.

C = αA ⟹ Cᵢⱼ = α · Aᵢⱼ
🏦 Finance Use Case

If A is the returns matrix for Portfolio 1 and B for Portfolio 2, then 0.5A + 0.5B is the blended portfolio — a 50/50 mix.

Matrix Multiplication — The Most Important Operation

This is not element-wise multiplication. To multiply A (m×k) by B (k×n), the number of columns in A must equal the number of rows in B. The result C is (m×n).

C = AB ⟹ Cᵢⱼ = Σₖ Aᵢₖ · Bₖⱼ

🏦 Finance Use: wᵀ

Your weights vector w is usually a column vector (n×1). When computing portfolio return, you write rₚ = wᵀr — the transpose turns it into a row vector so you can multiply by the returns column vector. This is just the dot product in matrix notation.

Identity Matrix & Inverse

The identity matrix I is the matrix equivalent of the number 1 — multiplying any matrix by it leaves it unchanged:

AI = IA = A

The inverse of a square matrix A (written A⁻¹) is the matrix such that AA⁻¹ = I. Not all matrices have inverses — a matrix must be non-singular (non-zero determinant) to be invertible.

⚠ When Inverses Fail in Finance

In practice, covariance matrices can become singular (non-invertible) when you have more assets than time periods, or when assets are perfectly correlated. This is a major practical challenge in quant work — it's why techniques like shrinkage estimation and PCA regularization exist. You will encounter this the first time you try to implement Markowitz optimization with too many assets.

Covariance Matrices

The DNA of portfolio risk

The covariance matrix (often written as Σ — capital sigma) encodes how all pairs of assets in a portfolio move together. It is the single most important object in quantitative finance. Every risk model, every portfolio optimizer, every options pricer touches the covariance matrix.

From Covariance to Covariance Matrix

Recall: the covariance between two assets i and j is:

σᵢⱼ = Cov(rᵢ, rⱼ) = E[(rᵢ − μᵢ)(rⱼ − μⱼ)]

When you have n assets, you have a covariance between every pair. The covariance matrix stacks all of these into a single n×n matrix:

The Covariance Matrix Σ for 3 Assets

σ₁²
σ₁₂
σ₁₃
σ₁₂
σ₂²
σ₂₃
σ₁₃
σ₂₃
σ₃²

Diagonal entries = each asset's own variance (how volatile it is). Off-diagonal entries = covariance between pairs (how they move together). The matrix is always symmetric because Cov(A,B) = Cov(B,A).

Computing the Covariance Matrix From Data

Given a returns matrix R (T × n, where T = number of days, n = number of assets), the sample covariance matrix is:

Σ = (1/(T−1)) · (R − μ̄)ᵀ (R − μ̄)

where μ̄ is the mean return vector replicated across rows (we subtract the mean from each column first — this is called demeaning).

Diagonal: Asset 1 has variance 0.04 → std dev = √0.04 = 20%. Asset 2: 30%. Asset 3: 40%. (TSLA is most volatile!)

Assets 1 and 3 have low covariance 0.01 — they barely move together.

Correlation Matrix vs Covariance Matrix

Covariances are hard to interpret because they depend on the scale of returns. The correlation matrix normalizes each entry to lie between −1 and +1:

ρᵢⱼ = σᵢⱼ / (σᵢ · σⱼ)

In matrix form: P = D⁻¹ Σ D⁻¹ where D is the diagonal matrix of standard deviations. You can always go back: Σ = D P D.

✅ Properties of a Valid Covariance Matrix

1. Symmetric: Σ = Σᵀ (Cov(i,j) always equals Cov(j,i))
2. Positive Semi-Definite (PSD): For any portfolio weights w, wᵀΣw ≥ 0 (variance can't be negative)
3. Diagonal entries ≥ 0: Individual variances are non-negative
4. |ρᵢⱼ| ≤ 1: Implied correlations are bounded

⚠️ Estimated covariance matrices from noisy financial data often violate PSD. This is a major source of problems in practice and is addressed with regularization techniques.

Practical Challenge: Estimating Σ From Financial Data

The Curse of Dimensionality

For a portfolio of n assets, Σ has n(n+1)/2 unique entries. For n=500 assets: 125,250 unique covariances to estimate! With only 252 trading days of history (1 year), you'd need to estimate 500+ parameters with only 252 observations — impossible without constraints.

Solutions: Factor models (reduce to k<shrinkage (blend sample Σ with a simpler target), and PCA (our next topic). This is exactly why PCA is so important in quant finance.

Eigenvectors & Eigenvalues

The natural axes of a transformation

Eigenvectors are the most conceptually important topic in this guide. They reveal the intrinsic structure of a matrix — the special directions that a matrix transformation simply stretches (never rotates). For the covariance matrix, eigenvectors are the directions of maximum variance in your data. This is the heart of PCA.

The Core Definition

Av = λv

Where:

  • A is a square matrix (n×n)
  • v is the eigenvector — a non-zero vector that doesn't change direction when multiplied by A
  • λ (lambda) is the eigenvalue — the scalar factor by which the eigenvector is stretched
🔍
The Key Intuition

When you multiply a general vector by a matrix A, it gets stretched AND rotated. But eigenvectors are special — they only get stretched (scaled by λ), never rotated. They are the "natural directions" or "axes" of the transformation. For a covariance matrix, they point in the directions of maximum data variance.

Geometric Intuition — Visualizing Eigenvectors

Finding Eigenvalues — The Characteristic Equation

To find eigenvalues, we solve the characteristic equation:

det(A − λI) = 0

This gives a polynomial of degree n in λ. The roots are the n eigenvalues.

Worked Example: 2×2 Matrix

# Find eigenvalues of A = [[3, 1], [1, 3]]
A = [[3, 1], [1, 3]]

# Characteristic equation: det(A - λI) = 0
# A - λI = [[3-λ, 1], [1, 3-λ]]
# det = (3-λ)(3-λ) - 1*1 = 0
# (3-λ)² - 1 = 0
# λ² - 6λ + 8 = 0
# (λ-4)(λ-2) = 0

# Eigenvalues: λ₁ = 4,  λ₂ = 2

# For λ₁ = 4: (A - 4I)v = 0 → [[-1,1],[1,-1]]v=0 → v₁=[1,1]/√2
# For λ₂ = 2: (A - 2I)v = 0 → [[1,1],[1,1]]v=0  → v₂=[1,-1]/√2

Key Properties of Eigenvalues for Covariance Matrices

Since the covariance matrix Σ is symmetric and positive semi-definite:

  • All eigenvalues are real numbers (not complex)
  • All eigenvalues are ≥ 0 (variance can't be negative)
  • Eigenvectors corresponding to different eigenvalues are orthogonal (perpendicular)
  • We can arrange them as λ₁ ≥ λ₂ ≥ ... ≥ λₙ ≥ 0
  • Sum of eigenvalues = trace(Σ) = total variance in the system

Eigendecomposition (Spectral Decomposition)

Any symmetric matrix can be written as:

Σ = Q Λ Qᵀ

Where:

  • Q is the matrix of eigenvectors (columns are the eigenvectors v₁, v₂, ..., vₙ)
  • Λ (Lambda) is a diagonal matrix with eigenvalues λ₁, λ₂, ..., λₙ on the diagonal
  • Qᵀ = Q⁻¹ since Q is orthogonal

Finance Interpretation of Eigenvalues

What Each Eigenvalue Tells You

For the covariance matrix of stock returns:

  • λ₁ (largest eigenvalue): The variance in the direction of greatest dispersion. The corresponding eigenvector v₁ is usually the "market factor" — a roughly equal-weighted portfolio. In S&P 500, λ₁ might explain 30-40% of total variance.
  • λ₂: Often corresponds to "large vs small cap" or "growth vs value" contrast.
  • Σ λᵢ = Trace(Σ): The sum of all eigenvalues equals the total variance. So λₖ / Σλᵢ is the fraction of total variance explained by component k.
  • Near-zero eigenvalues: Indicate near-linear dependencies — assets that are almost perfectly correlated with combinations of other assets. This makes Σ nearly singular and hard to invert.

Python: Computing Eigenvalues of Covariance Matrix

import numpy as np
import matplotlib.pyplot as plt

# Simulate returns for 3 assets (250 days)
np.random.seed(42)
returns = np.random.multivariate_normal(
    mean=[0, 0, 0],
    cov=[[0.04, 0.03, 0.01],
         [0.03, 0.09, 0.02],
         [0.01, 0.02, 0.16]],
    size=250
)

# Compute sample covariance matrix
Sigma = np.cov(returns.T)  # Note: .T because np.cov expects (n_vars, n_obs)
print("Covariance Matrix:\n", Sigma)

# Eigendecomposition
eigenvalues, eigenvectors = np.linalg.eigh(Sigma)  # eigh = for symmetric

# Sort descending
idx = eigenvalues[::-1].argsort()  # reverse order
eigenvalues = eigenvalues[::-1]
eigenvectors = eigenvectors[:, ::-1]

# Fraction of variance explained by each
explained = eigenvalues / eigenvalues.sum()
print("\nEigenvalues:", eigenvalues)
print("Variance explained:", explained)
print("First eigenvector (market factor?):", eigenvectors[:, 0])

# Verify reconstruction: Sigma ≈ Q * Lambda * Q.T
Lambda = np.diag(eigenvalues)
Q = eigenvectors
Sigma_reconstructed = Q @ Lambda @ Q.T
print("\nMax reconstruction error:", np.max(np.abs(Sigma - Sigma_reconstructed)))

Principal Component Analysis (PCA)

Compressing 500 dimensions into 5 risk factors

PCA is the direct application of eigendecomposition to data. It finds a new set of uncorrelated axes (principal components) that are aligned with the directions of maximum variance in your data. In quant finance, PCA is used to build statistical factor models, denoise covariance matrices, and visualize multi-dimensional risk.

The Intuition: Rotating to a Better Coordinate System

🌀
The Core Idea

Imagine plotting daily returns for two highly correlated stocks (e.g., Visa and Mastercard). The data forms an elongated ellipse pointing diagonally. PCA rotates your coordinate system so that the first new axis (PC1) points along the longest dimension of the ellipse — the direction of maximum variance. The second axis (PC2) is perpendicular to it. Now most variation is captured in just PC1.

Step-by-Step in Math

Step 1: Demean — X = R − μ̄ (subtract mean return of each asset)

Step 2: Covariance matrix — Σ = XᵀX / (T−1)

Step 3: Eigendecomposition — Σ = QΛQᵀ, sort eigenvectors by λ descending

Step 4: Keep only top k eigenvectors — Q_k = first k columns of Q

Step 5: Project — Z = X Q_k — each row of Z is a "factor score" for that day

Z = X Qₖ (T × n) × (n × k) = (T × k)

Z has only k columns (k << n) but captures most of the variance.

PCA in Quantitative Finance

Statistical Risk Factor Models

PCA on equity returns typically reveals:

PCApproximate Financial MeaningTypical Variance Explained
PC1Market factor — all stocks go up/down together30–50%
PC2Size factor — large vs small cap5–10%
PC3Sector rotation (e.g., tech vs utilities)3–7%
PC4–PC10Other systematic factors1–3% each
PC11+Idiosyncratic noiseNoise

By keeping only the first 10–20 PCs, you can explain 80%+ of portfolio variance with only 10–20 numbers instead of 500. This is the power of PCA!

Explained Variance Ratio — Choosing k

EVR(k) = λₖ / Σᵢ λᵢ | Cumulative EVR(k) = Σⱼ₌₁ᵏ λⱼ / Σᵢ λᵢ

Plot cumulative EVR vs k. Look for the "elbow" — the point where adding more components gives diminishing returns. In practice for equities, k=10 components often explains 80% of variance.

]) * 0.0004  # daily variance scale

PCA for Covariance Matrix Denoising

One of the most powerful uses of PCA in practice is denoising the covariance matrix. Random Matrix Theory tells us that if you have T observations and n assets, many eigenvalues of the sample covariance matrix are pure noise. The cutoff is given by the Marchenko-Pastur distribution:

λ_max = σ² (1 + √(n/T))²

Eigenvalues below this threshold are likely noise. We can reconstruct a cleaner covariance matrix using only the signal eigenvalues:

Σ_clean = Qₛ Λₛ Qₛᵀ + trace_noise/n · I

This denoised covariance matrix produces much more stable portfolio weights in optimization.

Portfolio Variance

Putting it all together: the quadratic form

This is the payoff chapter. Every concept we've built — vectors, matrix multiplication, covariance matrices, eigenvectors, PCA — converges here in the single most important formula in portfolio theory. Understanding this formula, in all its depth, is what separates a quant engineer from someone who just knows finance.

The Portfolio Variance Formula

σ²ₚ = wᵀ Σ w

What Every Symbol Means

SymbolTypeMeaning
σ²ₚScalarPortfolio variance (total risk squared). Take √ to get volatility.
wVector (n×1)Portfolio weight vector. wᵢ = fraction invested in asset i. Σwᵢ = 1.
wᵀVector (1×n)Transpose of w — row vector for matrix algebra to work out.
ΣMatrix (n×n)Covariance matrix of asset returns. Encodes all pairwise co-movements.

Expanding the Formula — 2 Assets

For a 2-asset portfolio, let's expand wᵀΣw fully to see exactly what it computes:

[w₁, w₂]
σ₁²
σ₁₂
σ₁₂
σ₂²
[w₁
w₂]
=
w₁²σ₁² + 2w₁w₂σ₁₂ + w₂²σ₂²
σ²ₚ = w₁²σ₁² + w₂²σ₂² + 2w₁w₂ρ₁₂σ₁σ₂
📊 Economic Interpretation

w₁²σ₁²: Risk from asset 1, scaled by how much you hold of it.
w₂²σ₂²: Risk from asset 2, scaled similarly.
2w₁w₂ρ₁₂σ₁σ₂: The diversification term. When ρ₁₂ < 0 (negatively correlated assets), this term is negative, reducing total portfolio risk below the weighted average of individual risks. This is the mathematical foundation of diversification!

The Efficient Frontier

If we plot all possible (return, risk) combinations of portfolios by varying weights, we get a curve. The efficient frontier is the upper portion — portfolios that maximize return for each level of risk (or minimize risk for each level of return).

Minimum Variance Portfolio — The Math

The minimum variance portfolio solves:

min w → wᵀΣw subject to: 1ᵀw = 1

Using Lagrange multipliers, the solution is:

w* = (Σ⁻¹ 1) / (1ᵀ Σ⁻¹ 1)

Here 1 is a vector of ones and Σ⁻¹ is the inverse of the covariance matrix. This is why matrix inversion matters so much in quant finance!

Mean-Variance Optimization (Markowitz)

For a target return μ_target, the optimal portfolio solves:

min wᵀΣw s.t. μᵀw = μ_target, 1ᵀw = 1

Solution: w* = Σ⁻¹(aμ + b·1) / normalization. The set of all optimal portfolios for different μ_target traces out the efficient frontier. Every point on the frontier corresponds to solving one matrix equation involving Σ⁻¹.

Portfolio Variance Decomposition via PCA

Recall from Tab 4: Σ = QΛQᵀ. Substituting into portfolio variance:

σ²ₚ = wᵀ(QΛQᵀ)w = (Qᵀw)ᵀ Λ (Qᵀw) = zᵀΛz = Σᵢ λᵢ zᵢ²

where z = Qᵀw is the vector of factor loadings of your portfolio. The portfolio variance is a weighted sum of eigenvalues, weighted by how much exposure you have to each principal component!

🎯 The Key Insight for Risk Management

Since λ₁ >> λ₂ >> ... (eigenvalues decay fast), the portfolio variance is dominated by its exposure to PC1 (usually the market factor). To reduce risk, you must reduce z₁ = v₁ᵀw — your portfolio's inner product with the first eigenvector. This is why market-neutral strategies (strategies with z₁ ≈ 0) have much lower overall variance than long-only strategies.

Complete Python Portfolio Optimizer

import numpy as np
from scipy.optimize import minimize
import matplotlib.pyplot as plt

# ─── Setup ────────────────────────────────────────────────
np.random.seed(42)
n = 5   # number of assets

# True covariance matrix
Sigma = np.array([
    [0.04,  0.02,  0.01,  0.005, 0.002],
    [0.02,  0.09,  0.03,  0.01,  0.004],
    [0.01,  0.03,  0.16,  0.02,  0.01 ],
    [0.005, 0.01,  0.02,  0.06,  0.015],
    [0.002, 0.004, 0.01,  0.015, 0.025]
])
mu = np.array([0.10, 0.14, 0.18, 0.12, 0.09])  # annual returns

# ─── Portfolio variance function ──────────────────────────
def portfolio_variance(w, Sigma):
    return w @ Sigma @ w  # wᵀΣw — the key formula!

def portfolio_return(w, mu):
    return w @ mu         # wᵀμ

# ─── Minimum Variance Portfolio ───────────────────────────
constraints = {'type': 'eq', 'fun': lambda w: w.sum() - 1}
bounds = [(0, 1)] * n   # long-only
w0 = np.ones(n) / n    # equal weight starting point

mvp_result = minimize(
    portfolio_variance, w0,
    args=(Sigma,),
    method='SLSQP',
    bounds=bounds,
    constraints=constraints
)
w_mvp = mvp_result.x
print(f"MVP weights: {w_mvp.round(3)}")
print(f"MVP volatility: {np.sqrt(portfolio_variance(w_mvp, Sigma))*100:.2f}%")
print(f"MVP return:     {portfolio_return(w_mvp, mu)*100:.2f}%")

# ─── Efficient Frontier ────────────────────────────────────
target_returns = np.linspace(mu.min(), mu.max(), 50)
frontier_vols = []

for mu_target in target_returns:
    cons = [
        {'type': 'eq', 'fun': lambda w: w.sum() - 1},
        {'type': 'eq', 'fun': lambda w: portfolio_return(w, mu) - mu_target}
    ]
    res = minimize(portfolio_variance, w0, args=(Sigma,),
                   method='SLSQP', bounds=bounds, constraints=cons)
    if res.success:
        frontier_vols.append(np.sqrt(res.fun))

plt.figure(figsize=(10, 6))
plt.plot([v*100 for v in frontier_vols], target_returns*100, 
         color='cyan', lw=2.5, label='Efficient Frontier')
plt.scatter(np.sqrt(np.diag(Sigma))*100, mu*100, 
            s=80, color='violet', zorder=5, label='Individual Assets')
plt.xlabel('Volatility (%)'); plt.ylabel('Return (%)')
plt.title('Efficient Frontier — wᵀΣw'); plt.legend(); plt.grid(alpha=0.3)
plt.show()

Risk Decomposition — Contribution of Each Asset

In risk management, a key task is understanding which assets contribute most to portfolio variance. The marginal contribution to risk of asset i is:

MCR_i = (Σw)_i (the i-th element of Σw)

The percentage risk contribution is:

RC_i = wᵢ(Σw)ᵢ / (wᵀΣw) = wᵢ MCR_i / σ²ₚ

A Risk Parity portfolio equalizes all RC_i values so that each asset contributes equally to total risk. This is widely used by risk parity funds like Bridgewater's All Weather Fund.

🔬 Lab 5 — Efficient Frontier Full Build ADVANCED

Goal: Compute and visualize the full Markowitz efficient frontier. Understand the minimum variance portfolio and Sharpe ratio.

import numpy as np
import matplotlib.pyplot as plt
from scipy.optimize import minimize

# 4 assets with different risk/return profiles
mu = np.array([0.08, 0.12, 0.18, 0.22])  # annual expected returns
Sigma = np.array([
    [0.0256, 0.0100, 0.0064, 0.0036],
    [0.0100, 0.0625, 0.0175, 0.0125],
    [0.0064, 0.0175, 0.1024, 0.0288],
    [0.0036, 0.0125, 0.0288, 0.1600]
])
rf = 0.04   # risk-free rate
n = len(mu)

def port_stats(w):
    ret = w @ mu
    vol = np.sqrt(w @ Sigma @ w)
    return ret, vol

# Random portfolios (Monte Carlo sampling)
N_random = 10000
rand_rets, rand_vols = [], []
for _ in range(N_random):
    w = np.random.dirichlet(np.ones(n))  # random weights summing to 1
    r, v = port_stats(w)
    rand_rets.append(r); rand_vols.append(v)

# Efficient Frontier via optimization
target_rets = np.linspace(0.08, 0.22, 80)
eff_vols, eff_weights = [], []
for tr in target_rets:
    res = minimize(lambda w: w@Sigma@w, np.ones(n)/n,
               constraints=[{'type':'eq','fun':lambda w:w.sum()-1},
                            {'type':'eq','fun':lambda w:w@mu-tr}],
               bounds=[(0,1)]*n, method='SLSQP')
    if res.success: eff_vols.append(np.sqrt(res.fun))

# Find Tangency Portfolio (max Sharpe)
best_sharpe_idx = np.argmax((np.array(rand_rets) - rf) / np.array(rand_vols))

# Plot
plt.figure(figsize=(11, 7))
sharpes = (np.array(rand_rets) - rf) / np.array(rand_vols)
sc = plt.scatter(rand_vols, rand_rets, c=sharpes, cmap='viridis', alpha=0.3, s=8)
plt.colorbar(sc, label='Sharpe Ratio')
plt.plot(eff_vols, target_rets[:len(eff_vols)], 'w-', lw=3, label='Efficient Frontier')
plt.scatter(rand_vols[best_sharpe_idx], rand_rets[best_sharpe_idx],
            s=200, color='red', zorder=5, marker='*', label='Max Sharpe')
plt.xlabel('Volatility σ'); plt.ylabel('Expected Return μ')
plt.title('Markowitz Efficient Frontier — Color = Sharpe Ratio')
plt.legend(); plt.grid(alpha=0.3); plt.show()
