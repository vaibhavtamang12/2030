Prerequisite 1: Vectors & Matrices — The Essentials

Think of a vector as a list of numbers. In our context, it will represent a probability distribution — i.e., "what's the chance of being in each state right now?"

π = [0.3, 0.5, 0.2] ← probability of being in State A, B, C

A matrix is a grid (table) of numbers. In our context, it will represent transition probabilities — "if I'm in state X, what's the probability of moving to state Y?"

P =
0.70.20.1
0.30.40.3
0.10.60.3
Row i, Column j = probability of going from state i to state j

Matrix Multiplication (in 60 seconds)

When we multiply a row vector (a distribution) by a matrix, we get a new row vector — the distribution at the next time step. This is the core engine of Markov Chains.

π_next = π_current × P
Concrete Example
If today there's a 60% chance I'm in State A and 40% in State B, and P says "from A, go to B with prob 0.3" and "from B, go to A with prob 0.5", then tomorrow's distribution is:

P(A tomorrow) = 0.6 × (stay in A) + 0.4 × (B→A) = 0.6 × 0.7 + 0.4 × 0.5 = 0.42 + 0.20 = 0.62

Eigenvectors & Eigenvalues (Intuition Only)

A special vector v is called an eigenvector of matrix M if multiplying M by v just scales v — it doesn't change its direction. The scaling factor is the eigenvalue λ.

M × v = λ × v

For Markov Chains, the stationary distribution is essentially the eigenvector of the transition matrix corresponding to eigenvalue 1. You don't need to compute it by hand — just know this geometric fact: multiplying by P doesn't change the stationary distribution.

Prerequisite 2: Probability — The Language

Sample Space & Events

A sample space Ω is the set of all possible outcomes. An event is any subset of Ω. A probability measure P assigns a number between 0 and 1 to every event.

Rolling a die

Ω = {1, 2, 3, 4, 5, 6}
Event "roll even" = {2, 4, 6}
P(even) = 3/6 = 0.5

Rules of Probability

P(Ω) = 1    (something always happens)
P(A∪B) = P(A) + P(B) – P(A∩B)
P(A) + P(Aᶜ) = 1    (complement rule)

Conditional Probability — The Most Important Concept Here

P(B | A) means "the probability of B, given that A already happened." It's the most natural thing in the world — your estimates change when you learn new information.

P(B | A) = P(A ∩ B) / P(A)
Intuition
Imagine you're standing in a room. P(it will rain) might be 0.3. But P(it will rain | I see dark clouds) = 0.85. Conditioning narrows the world to only those scenarios where A happened, then asks: in that narrowed world, how often does B happen?

Independence

Events A and B are independent if knowing A happened tells you nothing about B. Mathematically: P(A ∩ B) = P(A) × P(B).

Random Variables

A random variable X is a function that assigns a number to each outcome in the sample space. The expected value E[X] is the probability-weighted average.

E[X] = Σ x · P(X = x)     (for discrete X)

Variance and Standard Deviation

Var(X) = E[(X - E[X])²] = E[X²] - (E[X])² σ = √Var(X)

Variance measures spread. Low variance = values cluster near mean. High variance = values spread widely. This is crucial for understanding how random walks spread over time.

Prerequisite 3: Key Probability Distributions

Bernoulli Distribution

A single coin flip. Takes value 1 with prob p, and 0 with prob (1-p).

E[X] = p    Var(X) = p(1-p)

Each step of a simple random walk is a Bernoulli: +1 or -1.

Normal (Gaussian) Distribution

The famous bell curve. Described by mean μ and variance σ². Most naturally occurring averages follow this.

X ~ N(μ, σ²)

Brownian motion has normally distributed increments — this is its defining feature.

The Central Limit Theorem (CLT) — A Preview

The CLT says: add up many independent random variables and the sum approaches a normal distribution, regardless of what the individual distributions look like. This is WHY Brownian motion (the sum/integral of infinitely many tiny steps) is normal. We'll revisit this in Tab 04.

Connection Map
Bernoulli steps → sum many → CLT → Normal distribution → Random Walk limit → Brownian Motion. This is the entire thread connecting Tab 01 through Tab 04.

Prerequisite 4: Logarithms & Exponentials (Brief)

In finance and quant work, prices often move in percentage terms (multiplicative), not additive terms. We use log(price) to convert multiplicative changes into additive ones — which is necessary for applying all our additive tools (like Brownian motion) to stock prices.

log-return = ln(S_t / S_{t-1}) = ln(S_t) - ln(S_{t-1})

When stock prices are log-normally distributed, their log-returns are normally distributed. This is the foundation of the Black-Scholes model you'll encounter later.

Probability Foundations for Stochastic Processes

Before chains, walks, and motion — we need to deeply understand how randomness works over time. This tab bridges basic probability to dynamic, evolving random systems.

Stochastic Processes: Randomness That Evolves

A stochastic process is a collection of random variables indexed by time: {X_t, t ∈ T}. At each time t, X_t is a random variable. Together, they describe how a random quantity evolves.

Real-World Examples
  • Stock price S_t at each minute (continuous time)
  • Number of customers in a queue at each hour (discrete time)
  • Credit rating of a company each month (discrete states)
  • Temperature at a weather station over time (continuous states)

State Space: Where Can the Process Be?

The state space is the set of all values the process can take. This is crucial for categorizing processes:

The Markov Property: Memory and Memorylessness

This is THE most important concept for everything in this course. The Markov Property states:

Markov Property (Formal)
P(X_{n+1} = j | X_0 = i_0, X_1 = i_1, ..., X_n = i) = P(X_{n+1} = j | X_n = i)

In plain English: given where you are right now, where you go next is independent of your entire history of how you got here.

✅ Markov Process

The weather today only depends on yesterday's weather. Not last week, not last month. Only the current state determines the future.

The future is a function of the present alone.

❌ Non-Markov Process

A person's health depends on their entire medical history: what diseases they had years ago, their cumulative diet, etc.

The future depends on the entire past.

Is the Random Walk Markov?

Yes! At each step, we flip a coin: +1 or -1. The next position only depends on where we are now, not on the path we took to get here. This is why all our processes in this guide are Markov.

Conditional Expectation: The Key Tool

E[X | Y = y] is the expected value of X, given that Y takes value y. It's a number that summarizes the average of X in the world where Y = y.

E[X_{n+1} | X_n = x] = Σ_j j · P(X_{n+1} = j | X_n = x)

For a fair random walk (equal prob of +1 and -1):

E[X_{n+1} | X_n = x] = x · (+1) · (1/2) + x · (-1) · (1/2) = x

The expected next position equals the current position. This makes the random walk a martingale — a fair game where there's no drift.

Martingales: Fair Games

A stochastic process {M_n} is a martingale if E[M_{n+1} | M_0, ..., M_n] = M_n. The best prediction of tomorrow is today's value — no systematic trend up or down.

Quant Connection
In financial mathematics, stock prices under the "risk-neutral measure" are martingales. This is the foundational idea behind derivatives pricing. You'll see this concept everywhere in quant finance!

Distributions We'll Use — Deeper Look

The Binomial Distribution

If we take n independent Bernoulli steps (each +1 or -1 with equal probability), the sum S_n follows a scaled binomial distribution. After n steps, S_n takes values: -n, -n+2, ..., n-2, n.

P(S_n = k) = C(n, (n+k)/2) · (1/2)^n    for k and n of the same parity

The Fundamental Bridge: Sums Become Normals

The Central Limit Theorem is the bridge from everything discrete in Tabs 02-03 to the continuous world of Tab 04.

1

Start with many independent, identically distributed random variables

X_1, X_2, ..., X_n — each with mean μ and variance σ²

2

Form the sum S_n = X_1 + X_2 + ... + X_n

Each random walk position IS this sum, where X_i = ±1 at each step

3

As n → ∞, the standardized sum converges to N(0,1)

(S_n - nμ) / (σ√n) → Normal(0,1). This is the CLT!

4

Scale time continuously → Brownian Motion

When we let the step size shrink as steps get infinitesimally small, the limit is Brownian Motion B(t)

Markov Chains

A Markov Chain is a random process that jumps between states, where the next state depends only on the current state — not on any history. Simple rules, surprisingly deep consequences.

One-Sentence Summary
A Markov Chain is a random walk through a set of discrete states, governed by a matrix of transition probabilities, and — under mild conditions — always eventually settles into a steady-state distribution.

Building Intuition: The Weather Model

Imagine a city with just two types of weather: Sunny (S) and Rainy (R). From observing historical data, we know:

  • If today is Sunny: 70% chance tomorrow is Sunny, 30% chance it's Rainy
  • If today is Rainy: 40% chance tomorrow is Sunny, 60% chance it's Rainy
S
Sunny
0.30 →
0.40 ←
R
Rainy
S→S: 0.70  |  R→R: 0.60

This is a Markov Chain! Notice: to predict tomorrow's weather, you only need to know today's weather. What the weather was last week is irrelevant.

The Transition Matrix P

We encode all probabilities in the Transition Matrix P. Each entry P[i][j] = probability of moving from state i to state j.

Critical Property
Every row of a transition matrix must sum to 1. Why? Because from any state, you must go somewhere — including staying in the same state.
P =
→S →R
S→ 0.70.3
R→ 0.40.6
Row sums: 0.7+0.3=1 ✓
Row sums: 0.4+0.6=1 ✓

This is called a stochastic matrix.

Multi-Step Transitions: Powers of P

What's the probability of being in Sunny weather in 2 days, given it's Sunny today? We use P², the matrix squared:

P(State after n steps) = P^n P² = P × P → P^n = P × P × ... × P (n times)

Computing P² for our weather example

P²[S][S] = P[S][S]×P[S][S] + P[S][R]×P[R][S] = 0.7×0.7 + 0.3×0.4 = 0.49 + 0.12 = 0.61

P²[S][R] = P[S][S]×P[S][R] + P[S][R]×P[R][R] = 0.7×0.3 + 0.3×0.6 = 0.21 + 0.18 = 0.39

So if today is Sunny, in 2 days: 61% chance Sunny, 39% chance Rainy.

State Classification

Before we can find the long-run behavior, we need to classify states:

State TypeDefinitionExample
Transient Probability of returning to this state is less than 1. You'll eventually leave and never come back. Starting position in a biased random walk near an absorbing barrier
Recurrent You're guaranteed to return to this state eventually (return probability = 1) All states in a finite, irreducible Markov Chain
Absorbing Once you enter, you can never leave. P[i][i] = 1. Bankruptcy in a company credit model
Periodic You can only return in multiples of d steps (period d) A 2-state chain where S→R and R→S always (period 2)
Aperiodic You can return in any number of steps (period = 1) Most real-world Markov chains

Irreducibility

A Markov Chain is irreducible if every state can be reached from every other state (possibly in multiple steps). This means the chain forms one connected component.

Why It Matters
An irreducible + aperiodic + finite Markov Chain is guaranteed to converge to a unique stationary distribution. These conditions are the checklist for "nice" long-run behavior.

The Stationary Distribution π

As we run the chain for many, many steps, the probability distribution over states converges to a fixed vector π called the stationary distribution. This is the long-run fraction of time spent in each state.

The Defining Equation

π · P = π     (matrix equation) Equivalently: π_j = Σ_i π_i · P[i][j] Subject to: Σ_j π_j = 1 (probabilities sum to 1)

In words: the stationary distribution is unchanged when you multiply by the transition matrix. The distribution reproduces itself — it's in "equilibrium."

Solving for π — Weather Example

We need to solve π · P = π with π_S + π_R = 1:

System of equations:

π_S = 0.7·π_S + 0.4·π_R    (from column S)

π_R = 0.3·π_S + 0.6·π_R    (from column R)

π_S + π_R = 1

From equation 1: 0.3·π_S = 0.4·π_R → π_S = (4/3)·π_R

Substituting: (4/3)·π_R + π_R = 1 → (7/3)·π_R = 1 → π_R = 3/7 ≈ 0.429

Answer: π = [π_S, π_R] = [4/7 ≈ 0.571, 3/7 ≈ 0.429]

Long run: ~57% of days are Sunny, ~43% are Rainy, regardless of today's weather.

Detailed Example: Credit Rating Transitions

A credit rating agency models company ratings (AAA, BBB, CCC, Default) as a Markov Chain. Here's a simplified transition matrix:

P =
AAA BBB CCC Def
AAA 0.920.070.010.00
BBB 0.050.850.090.01
CCC 0.010.100.700.19
Def 0.000.000.001.00

Notice "Default" is an absorbing state (row = [0, 0, 0, 1]). Once defaulted, always defaulted. This is a classic application in fixed-income quant work.

Mean First Passage Time

A key quantity: how many steps on average does it take to first reach state j, starting from state i? This is the Mean First Passage Time m_{ij}.

m_{ij} = 1 + Σ_{k ≠ j} P[i][k] · m_{kj}    for i ≠ j m_{jj} = 1 / π_j (mean recurrence time for state j)

Practical use: "On average, how long does it take for a BBB-rated company to default?" — directly answerable with this formula.

Detailed Worked Example: 3-State Random Walk

Consider states {0, 1, 2, 3} where 0 and 3 are absorbing (barriers), and at each interior state we go left (−1) or right (+1) with equal probability.

Gambler's Ruin
This is the famous Gambler's Ruin Problem. A gambler starts with $i, plays a fair game, stops when broke (state 0) or reaches $3 goal. What's the probability of ruin?

Answer: P(ruin | start at i) = 1 - i/3. Starting with $1: 67% ruin. Starting with $2: 33% ruin.

Key Theorems (Intuition, No Proof)

Ergodic Theorem

For an irreducible, aperiodic Markov chain, the time-average equals the ensemble average. The long-run fraction of time in state j is π_j.

Convergence Theorem

For ergodic chains: lim_{n→∞} P^n[i][j] = π_j for all i. The starting state becomes irrelevant!

Perron-Frobenius

For positive stochastic matrices, the largest eigenvalue is 1, and the corresponding eigenvector is the stationary distribution.

Random Walks

A random walk is the simplest model of a process that moves randomly over time — one step at a time, each step unpredictable. It's the discrete ancestor of Brownian motion and the conceptual heart of most financial models.

Connection to Markov Chains
A random walk IS a special Markov chain. The state is the current position (an integer), and the transition probabilities are: P(move right) = p, P(move left) = 1-p. The Markov property holds — next position only depends on current position, not history.

The Simple Symmetric Random Walk

Start at position 0. At each time step n = 1, 2, 3, ..., flip a fair coin:

  • Heads: move +1 (step right)
  • Tails: move −1 (step left)

Let X_i be the i-th step (X_i = +1 or -1 with equal probability). The position after n steps is:

S_n = X_1 + X_2 + ... + X_n = Σ_{i=1}^{n} X_i

Properties of S_n

Expected Position

E[S_n] = 0

Since each step has expected value 0 (equal chance of ±1), the expected position is always 0. No drift.

Variance Grows Linearly

Var(S_n) = n    σ = √n

Since the X_i are independent: Var(S_n) = n × Var(X_i) = n × 1. The walk spreads out at rate √n.

The √n Rule — Crucial Insight!
After n steps, the typical distance from the origin is √n, NOT n. This sublinear spread is one of the most important facts in probability. After 100 steps, you're typically ~10 units away. After 10,000 steps, ~100 units. The walk spreads, but slowly.

The Asymmetric Random Walk (Biased Walk)

If p ≠ 1/2, we have a biased random walk. Each step is +1 with probability p and −1 with probability q = 1 − p.

E[X_i] = p·(+1) + q·(-1) = p - q = μ (drift) Var(X_i) = 1 - (p-q)² = 4pq E[S_n] = n·(p-q) = n·μ Var(S_n) = n·4pq

A biased walk drifts in one direction. Set p = 0.55 in the simulator above and watch the walk trend upward! This models a stock with positive expected return.

Financial Analogy
In finance, (p-q) is analogous to the drift of an asset (expected return above risk-free rate). The variance 4pq is related to volatility. The Biased Random Walk is the discrete version of Brownian motion with drift — the building block of the Black-Scholes model!

Return to Origin: A Surprising Result

For the symmetric random walk in 1D, the probability of ever returning to the origin is exactly 1. The walk is recurrent. You'll always come back — it just might take an incredibly long time.

Pólya's Theorem
  • 1D walk: Recurrent (returns to origin with probability 1)
  • 2D walk: Recurrent (returns to origin with probability 1)
  • 3D+ walk: Transient (returns to origin with probability < 1 ≈ 0.34 for 3D)
The famous quote: "A drunk man will find his way home, but a drunk bird may get lost forever."

The Arc-Sine Law: Where You Spend Time

This is one of the most counterintuitive results in probability theory. Consider a fair random walk for 2n steps. The fraction of time the walk spends on the positive side follows an arc-sine distribution.

Intuition: The probability that you spend roughly half the time positive is actually the smallest value — not the largest! You're most likely to spend almost ALL the time on one side or the other. The walk "commits" to a side for long stretches.

Quant Implication
This means momentum is natural in random processes! A fair random walk is more likely to stay above zero for 90% of the time (or below for 90% of the time) than to spend equal time on both sides. Don't mistake random momentum for predictability.

2D Random Walk on a Grid

Scaling Limit: Random Walk → Brownian Motion

This is the most profound connection in this entire guide. As we make the steps smaller and more frequent — shrinking both time and space — the random walk converges to Brownian motion.

1

Start with a simple random walk S_n

Steps at times 1, 2, 3, ..., n. Each step ±1.

2

Rescale time: compress n steps into [0, 1]

Let t ∈ [0,1] and look at S_{⌊nt⌋}. This is a continuous-time process.

3

Rescale space: shrink step size by 1/√n

Define B_n(t) = S_{⌊nt⌋} / √n. This keeps variance = 1 at time t = 1.

4

Take the limit n → ∞

By Donsker's theorem (the functional CLT), B_n(t) → B(t) = Brownian Motion!

Donsker's Theorem: B_n(t) = S_{⌊nt⌋} / √n → B(t) as n → ∞

Brownian Motion

Brownian motion is the continuous-time limit of the random walk. It is the mathematical foundation of modern quantitative finance — underlying options pricing, interest rate models, and virtually every model of financial risk.

The Big Picture — Where We Are
Markov Chains → (continuous states) → Random Walk → (continuous time + space, CLT limit) → Brownian Motion. Every concept we've built leads here. Brownian motion is where discrete probability meets continuous analysis.

Historical Background: From Physics to Finance

1827 — Robert Brown

Botanist Robert Brown observed pollen grains suspended in water moving in irregular, unpredictable zigzag patterns under a microscope. This "Brownian motion" puzzled scientists for 80 years.

1905 — Albert Einstein

Einstein explained the motion: it results from countless collisions with water molecules. He predicted the variance would grow as 2Dt (where D is the diffusion coefficient). This proved atoms exist!

1923 — Norbert Wiener

Wiener gave the first rigorous mathematical construction of Brownian motion as a continuous stochastic process. It's also called the Wiener process in his honor.

1973 — Black, Scholes & Merton

Used Brownian motion to model stock prices, leading to the Black-Scholes options pricing formula. This earned Scholes and Merton the 1997 Nobel Prize in Economics.

Formal Definition: What Makes B(t) Brownian Motion?

A stochastic process {B(t), t ≥ 0} is a standard Brownian motion (Wiener process) if it satisfies these four properties:

①

Starts at zero: B(0) = 0 (almost surely)

Every path starts at the origin.

②

Independent increments

For any 0 ≤ s < t ≤ u < v, the increment B(t)-B(s) is independent of B(v)-B(u). Non-overlapping time periods produce independent changes. This is the continuous Markov property.

③

Stationary Gaussian increments

For any 0 ≤ s < t: B(t) - B(s) ~ Normal(0, t-s). The increment over any interval is normally distributed with variance = length of interval. This is Donsker's theorem in the limit!

④

Continuous paths

Almost surely, B(t) is a continuous function of t. No jumps! (But the paths are nowhere differentiable — infinitely rough.)

Key Properties of Brownian Motion

1. Scaling (Self-Similarity)

For any constant c > 0, the process {c⁻¹ · B(c²t)} has the same distribution as {B(t)}. Brownian motion looks the same at any time scale — it's a fractal.

c⁻¹ · B(c²t) ≡ B(t) in distribution

Zoom in on a Brownian path — it looks as irregular as the original. Zoom out — same irregularity. This scale-free property makes it both beautiful and mathematically tractable.

2. Nowhere Differentiable Paths

Despite being continuous, Brownian motion is differentiable nowhere with probability 1. The "velocity" B'(t) does not exist! You can never speak of the instantaneous rate of change of Brownian motion in the classical sense.

Why? Intuition
Think of the random walk: the slope between consecutive steps is ±(1/Δt) for step size Δx = 1 and time step Δt → 0. This ratio → ±∞. In the limit, the path oscillates infinitely fast — it has infinite total variation over any finite interval.

3. Quadratic Variation

This is where Brownian motion departs from classical calculus. The quadratic variation of B(t) over [0, T] equals T, not zero (as it would for smooth functions).

[B, B]_T = Σ (B(t_{i+1}) - B(t_i))² → T as mesh → 0

For comparison, any smooth function f has quadratic variation 0. This fact — (dB)² = dt — is what makes Itô calculus different from ordinary calculus. (We won't go into Itô calculus here, but this is the gateway concept.)

4. The Reflection Principle

For any level a > 0 and time T:

P(max_{0≤t≤T} B(t) ≥ a) = 2·P(B(T) ≥ a) = 2·(1 - Φ(a/√T))

where Φ is the standard normal CDF. This is used extensively for computing barrier option prices and first passage times in finance.

5. The 2D and 3D Brownian Motion

In 2D, Brownian motion is recurrent (like the 2D random walk). In 3D, it's transient — with probability 1, a 3D Brownian particle never returns to the origin. Same result as Pólya's theorem, in the continuous limit.

Brownian Motion with Drift (The Finance Model)

Standard BM has no trend. We add drift μ and diffusion coefficient σ to model real assets:

X(t) = μt + σ·B(t) where X(t) - X(s) ~ Normal(μ(t-s), σ²(t-s))

This is Brownian motion with drift. The μt term is deterministic drift (trend), and σB(t) is the random part.

μ > 0: Upward Drift

The process trends upward on average. Models a stock with positive expected return (bull market).

σ: Volatility

Controls the "noisiness" of the path. High σ = wide price swings. Low σ = smooth trend. In finance, σ is called volatility.

Geometric Brownian Motion: The Stock Price Model

One problem: real stock prices can't go negative. If we model log(S_t) as BM with drift:

log(S_t) = log(S_0) + μt + σ·B(t) ⟺ S_t = S_0 · exp(μt + σ·B(t))

This is Geometric Brownian Motion (GBM). It ensures prices stay positive, has log-normal marginal distributions, and produces the Black-Scholes model when combined with Itô's formula.

def simulate_markov(P, n_steps, start_state=0):
    """Simulate a Markov chain for n_steps steps."""
    states = [start_state]
    current = start_state
    for _ in range(n_steps - 1):
        # np.random.choice picks next state based on row probabilities
        current = np.random.choice(len(P), p=P[current])

    return np.array(states)

def brownian_motion(T=1.0, n=1000, mu=0.0, sigma=1.0):
    """
    Simulate Brownian motion with drift.
    dX = mu*dt + sigma*dB
    """
    dt = T / n
    t = np.linspace(0, T, n+1)
    # Each increment ~ N(0, dt)
    increments = np.random.normal(0, np.sqrt(dt), n)
    B = np.concatenate([[0], np.cumsum(increments)])
    # Add drift: X(t) = mu*t + sigma*B(t)
    X = mu * t + sigma * B
    return t, X

# Lab 5: Quant Application — Monte Carlo Option Pricing
# ─────────────────────────────────────────────────────────────────
# Price a European Call using GBM simulations, compare to Black-Scholes

from scipy.stats import norm

def black_scholes_call(S0, K, T, r, sigma):
    """Black-Scholes formula for European call option price."""
    d1 = (np.log(S0/K) + (r + 0.5*sigma**2)*T) / (sigma*np.sqrt(T))
    d2 = d1 - sigma*np.sqrt(T)
    return S0*norm.cdf(d1) - K*np.exp(-r*T)*norm.cdf(d2)

def mc_call_price(S0, K, T, r, sigma, n_simulations=100000, n_steps=252):
    """Monte Carlo pricing of European call option."""
    dt = T / n_steps
    # Simulate final prices directly (log-normal shortcut)
    Z = np.random.standard_normal(n_simulations)
    ST = S0 * np.exp((r - 0.5*sigma**2)*T + sigma*np.sqrt(T)*Z)
    # Call payoff: max(S_T - K, 0)
    payoff = np.maximum(ST - K, 0)
    # Discount back to present value
    price = np.exp(-r*T) * np.mean(payoff)
    stderr = np.exp(-r*T) * np.std(payoff) / np.sqrt(n_simulations)
    return price, stderr

# Parameters
S0 = 100    # Current stock price
K  = 105    # Strike price  
T  = 1.0    # Time to expiry (years)
r  = 0.05   # Risk-free rate
sigma = 0.2  # Volatility

bs_price = black_scholes_call(S0, K, T, r, sigma)
mc_price, mc_err = mc_call_price(S0, K, T, r, sigma)

print(f"Black-Scholes Price:   ${bs_price:.4f}")
print(f"Monte Carlo Price:     ${mc_price:.4f} ± {1.96*mc_err:.4f} (95% CI)")
print(f"Error:                 {abs(mc_price-bs_price):.4f}")

# Convergence analysis: how many simulations do we need?
n_sims_list = [100, 500, 1000, 5000, 10000, 50000, 100000]
mc_prices = []
for n in n_sims_list:
    p, _ = mc_call_price(S0, K, T, r, sigma, n_simulations=n)
    mc_prices.append(p)

plt.figure(figsize=(10, 5))
plt.semilogx(n_sims_list, mc_prices, 'o-', color='#4f8ef7', label='MC Price')
plt.axhline(bs_price, color='#f74f8e', linestyle='--', linewidth=2,
            label=f'Black-Scholes = ${bs_price:.4f}')
plt.xlabel('Number of Simulations')
plt.ylabel('Call Price ($)')
plt.title('Monte Carlo Convergence to Black-Scholes', color='white')
plt.legend()
plt.grid(True, alpha=0.3)

# GBM parameters per regime
regime_params = {
    0: {'mu': 0.20/252,  'sigma': 0.12/np.sqrt(252)},   # Bull: positive drift, low vol
    1: {'mu': -0.30/252, 'sigma': 0.35/np.sqrt(252)}    # Bear: negative drift, high vol
}

def regime_switching_gbm(S0=100, n_days=1260, start_regime=0):
    S = [S0]
    regimes = [start_regime]
    current_regime = start_regime
    for t in range(1, n_days):
        # Step 1: Update regime (Markov Chain step)
        current_regime = np.random.choice(2, p=P_regime[current_regime])
        regimes.append(current_regime)
        # Step 2: Update price (GBM step with regime's parameters)
        p = regime_params[current_regime]
        log_ret = p['mu'] + p['sigma'] * np.random.randn()
        S.append(S[-1] * np.exp(log_ret))
    return np.array(S), np.array(regimes)

fig, axes = plt.subplots(2, 1, figsize=(16, 10), gridspec_kw={'height_ratios': [3, 1]})

prices, regimes = regime_switching_gbm(n_days=1260)  # 5 years
t = np.arange(1260)

# Color price path by regime
for i in range(1, 1260):
    color = '#5ef7a4' if regimes[i] == 0 else '#f74f8e'
    axes[0].plot([t[i-1], t[i]], [prices[i-1], prices[i]], 
              color=color, linewidth=0.8, alpha=0.8)

axes[0].set_title('Regime-Switching GBM: Green=Bull, Red=Bear', color='white', fontsize=13)
axes[0].set_ylabel('Price ($)')
axes[0].grid(True, alpha=0.3)

# Show the regime path
axes[1].fill_between(t, regimes, color='#f74f8e', alpha=0.7, label='Bear Regime')
axes[1].set_yticks([0, 1])
axes[1].set_yticklabels(['Bull', 'Bear'])
axes[1].set_xlabel('Trading Days')
axes[1].set_title('Regime (Markov Chain State)', color='white')
axes[1].grid(True, alpha=0.3)

plt.tight_layout()
plt.show()

# Compute stationary distribution for regime chain
evals, evecs = np.linalg.eig(P_regime.T)
stat_idx = np.argmin(np.abs(evals - 1))
pi_regime = np.real(evecs[:, stat_idx])
pi_regime /= pi_regime.sum()
print(f"Theoretical stationary: Bull={pi_regime[0]:.3f}, Bear={pi_regime[1]:.3f}")
print(f"Simulated fraction:     Bull={np.mean(regimes==0):.3f}, Bear={np.mean(regimes==1):.3f}")

What You've Built in Lab 6
The regime-switching model unifies all three topics: (1) a Markov Chain governs which "regime" (Bull/Bear) the market is in, (2) within each regime, the stock price follows Geometric Brownian Motion (Random Walk → BM), and (3) the stationary distribution of the Markov Chain determines long-run market fractions in each regime. This is a real model used in quantitative asset management!
