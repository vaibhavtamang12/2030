Prerequisites

These foundational concepts are woven throughout OLS theory. Each is explained from scratch below.

⚠ What You Need First

Linear regression, matrix algebra basics, expected values, variance, and the normal distribution appear constantly. All five are explained below — take 5–10 minutes per section before diving into the main content.

1. Linear Regression — The Core Idea

Linear regression models the relationship between a dependent variable Y and one or more independent variables X. The idea: Y can be approximated as a weighted sum of the X variables plus an error term.

Simple Linear Regression
Y_i = β₀ + β₁X_i + ε_i
β₀ = intercept, β₁ = slope, ε_i = error term (the part Y can't explain). i indexes observations 1 to n.
Multiple Regression (Matrix Form)
Y = Xβ + ε
Y is an n×1 vector, X is an n×k matrix (includes a column of 1s for intercept), β is k×1, ε is n×1. This compact form handles any number of predictors.

The Scatter Plot Intuition

Imagine plotting house price (Y) against size in sq ft (X). Each house is a dot. The regression line is the best fitting straight line through the cloud of dots. "Best fitting" means minimising the sum of squared vertical distances from each dot to the line. Those vertical distances are the residuals ê_i = Y_i − Ŷ_i.

2. OLS — Ordinary Least Squares

OLS finds the β that minimises the sum of squared residuals (SSR):

OLS Objective
β̂ = argmin_β Σᵢ (Yᵢ − Xᵢβ)² = argmin_β ‖Y − Xβ‖²
Solution: β̂ = (XᵀX)⁻¹XᵀY
The analytical solution — no iteration needed. Requires XᵀX to be invertible (no perfect multicollinearity).

3. Residuals vs Errors

Error ε_i
The unobservable true deviation from the population regression line. We can never see ε_i.
Residual ê_i
The observable estimated deviation from the sample regression line. ê_i = Y_i − Ŷ_i = Y_i − X_iβ̂. We use residuals to diagnose regression problems.
Why residuals matter
Violations of OLS assumptions leave patterns in the residuals. Plotting residuals vs fitted values, time, or X is the primary diagnostic tool.
Good Residuals — Random Scatter
No pattern, constant variance — OLS assumptions satisfied
Bad Residuals — Pattern Present
Structure visible — assumptions violated (something is wrong)

4. Expected Value & Variance of Estimators

Unbiasedness
E[β̂] = β. On average, the estimator hits the true value. A biased estimator systematically misses.
Efficiency
Among all unbiased estimators, the one with lowest variance is "efficient." The Gauss-Markov theorem tells us OLS is efficient under its assumptions.
Consistency
β̂ → β as n → ∞. Even if biased in small samples, a consistent estimator converges to the truth.

5. The Error Covariance Structure

The variance-covariance matrix of the error vector ε is central to understanding heteroskedasticity and autocorrelation:

Error Covariance Matrix
Var(ε) = E[εεᵀ] = Ω
Under ideal OLS assumptions: Ω = σ²I (scalar times identity). Heteroskedasticity: σᵢ² vary (diagonal elements differ). Autocorrelation: off-diagonal elements ≠ 0.

✅ Foundation set! With linear regression, OLS, residuals, estimator properties, and the error covariance structure understood — you're ready to master OLS assumptions and their violations: heteroskedasticity and autocorrelation.

Topic 1

OLS & the BLUE Property

Ordinary Least Squares is more than just a fitting method — it is the provably optimal estimator under a specific set of conditions. Understanding why it's optimal tells you exactly what breaks when those conditions fail.

🎯 Why Quants Care About BLUE

In factor models, risk premia estimation, and strategy alpha regression, you want your β estimates to be as precise as possible. OLS delivers that — but only when its assumptions hold. Violations (heteroskedasticity in cross-sectional returns, autocorrelation in time-series regressions) can make OLS dangerously misleading.

The Gauss-Markov Theorem

This is the cornerstone result that justifies OLS:

Gauss-Markov Theorem
Under assumptions A1–A5, β̂_OLS is BLUE:
Best Linear Unbiased Estimator
Best = minimum variance. Linear = linear function of Y. Unbiased = E[β̂] = β. No other linear unbiased estimator has smaller variance.

The BLUE Analogy — Archery

Unbiased: On average, arrows land on the bullseye (E[β̂]=β). A biased archer consistently aims left — systematically wrong.

Best (minimum variance): Arrows cluster tightly together. A high-variance archer is sometimes far right, sometimes far left — even if on average centred.

BLUE: Among all unbiased linear archers, OLS is the one whose arrows cluster most tightly. When assumptions fail, this tightness guarantee disappears — OLS may still be unbiased, but other estimators might cluster more tightly.

The OLS Estimator — Derivation Sketch

  1. Define SSR

    SSR(β) = Σᵢ(Yᵢ − Xᵢβ)² = (Y−Xβ)ᵀ(Y−Xβ) = YᵀY − 2βᵀXᵀY + βᵀXᵀXβ

  2. Differentiate w.r.t. β

    ∂SSR/∂β = −2XᵀY + 2XᵀXβ. Set to zero.

  3. Solve the Normal Equations

    XᵀXβ̂ = XᵀY → β̂ = (XᵀX)⁻¹XᵀY. This is a system of k linear equations in k unknowns.

  4. Verify it's a minimum

    Second derivative XᵀX is positive semidefinite → SSR is convex → any critical point is a global minimum.

Properties Under A1–A5
E[β̂] = β                           (Unbiasedness)
Var(β̂) = σ²(XᵀX)⁻¹              (Covariance matrix)
β̂ ~ N(β, σ²(XᵀX)⁻¹)            (Normality, under A6)
These three properties together make statistical inference (t-tests, F-tests, confidence intervals) valid.

What "Minimum Variance" Means Geometrically

OLS Sampling Distribution
β̂ across 500 different samples — tight cluster around truth
Inefficient Estimator (High Variance)
Same data — alternative estimator spreads wider

🧪 Colab Experiment 1 — OLS Properties Verification

experiment_1_ols_properties.py ▶ Open Colab 
import numpy as np
import matplotlib.pyplot as plt
from scipy import stats

np.random.seed(42)

# ═══════════════════════════════════════════════════════
# Verify OLS is BLUE — simulate many samples, check:
# 1) Unbiasedness: E[β̂] = β_true
# 2) Minimum Variance: Var(β̂_OLS) ≤ Var(β̂_alt)
# ═══════════════════════════════════════════════════════

n      = 50
b_true = np.array([2.0, 0.8])  # [intercept, slope]
sigma  = 1.5
REPS   = 3000

X_fixed = np.column_stack([np.ones(n), np.random.uniform(0, 10, n)])

ols_betas, alt_betas = [], []

for _ in range(REPS):
    eps = np.random.normal(0, sigma, n)
    Y = X_fixed @ b_true + eps

    # OLS estimator: β̂ = (XᵀX)⁻¹XᵀY
    b_ols = np.linalg.solve(X_fixed.T @ X_fixed, X_fixed.T @ Y)
    ols_betas.append(b_ols[1])  # slope

    # Alternative (inefficient) estimator: just use first-half data
    b_alt = np.mean(Y[n//2:] - Y[:n//2]) / np.mean(X_fixed[n//2:,1] - X_fixed[:n//2,1])
    alt_betas.append(b_alt)

ols_betas = np.array(ols_betas)
alt_betas = np.array(alt_betas)

print("=== OLS BLUE Verification ===")
print(f"True β₁ = {b_true[1]}")
print(f"OLS:  E[β̂₁] = {ols_betas.mean():.4f}  (bias = {ols_betas.mean()-b_true[1]:.4f})")
print(f"Alt:  E[β̂₁] = {alt_betas.mean():.4f}  (bias = {alt_betas.mean()-b_true[1]:.4f})")
print(f"OLS:  Var(β̂₁) = {ols_betas.var():.5f}")
print(f"Alt:  Var(β̂₁) = {alt_betas.var():.5f}")
print(f"Theoretical Var(β̂₁_OLS) = σ²/Σ(xᵢ-x̄)² = {sigma**2/np.var(X_fixed[:,1])/n:.5f}")
print(f"→ OLS variance is {'LOWER' if ols_betas.var() < alt_betas.var() else 'higher'} — BLUE confirmed!")

fig, axes = plt.subplots(1, 2, figsize=(13, 4))
for ax, betas, name, col in zip(axes,
    [ols_betas, alt_betas], ['OLS (BLUE)', 'Alternative'],
    ['#2980b9', '#c0392b']):
    ax.hist(betas, bins=60, color=col, alpha=0.7, density=True)
    ax.axvline(b_true[1], color='black', lw=2, ls='--', label=f'True β₁={b_true[1]}')
    ax.axvline(betas.mean(), color=col, lw=2, label=f'Mean={betas.mean():.3f}')
    ax.set_title(f'{name}\nVar={betas.var():.5f}'); ax.legend(fontsize=8)
plt.suptitle('OLS vs Alternative: Sampling Distributions of β̂₁', fontsize=13)
plt.tight_layout(); plt.show()
Topic 2

The OLS Assumptions

The BLUE result depends on six assumptions — Gauss-Markov (A1–A5) plus normality (A6 for inference). Each assumption has a specific role, and each violation has a specific consequence.

The Six Assumptions

A1 — LINEARITY
Linear in Parameters

The model is Y = Xβ + ε. The relationship between Y and the betas is linear (X can be transformed).

Violation: True model is Y = β₀ + β₁X² but you fit Y = β₀ + β₁X → omitted non-linearity appears in residuals.

Required for BLUE
A2 — RANDOM SAMPLING
Data is i.i.d. / Exogenous X

Observations are independent draws. The X matrix is non-stochastic or at least exogenous: E[ε|X] = 0.

Violation: Endogeneity — X is correlated with ε (omitted variable bias, simultaneity). OLS is biased AND inconsistent.

Required for Unbiasedness
A3 — NO PERFECT COLLINEARITY
Full Column Rank

No predictor is an exact linear combination of others. X has rank k, so (XᵀX) is invertible.

Violation: β̂ = (XᵀX)⁻¹XᵀY doesn't exist! Or near-collinearity: estimates exist but with huge variance.

Required for Existence
A4 — ZERO CONDITIONAL MEAN
E[ε|X] = 0

The errors have mean zero given the predictors. Equivalent to saying X contains all relevant predictors.

Violation: Omitted variable bias. β̂ estimates the direct effect plus the indirect effect through the omitted variable.

Violation → Biased OLS
A5 — SPHERICAL ERRORS
Var(ε|X) = σ²I

Two conditions: (i) Homoskedasticity: Var(εᵢ|X) = σ² for all i. (ii) No autocorrelation: Cov(εᵢ,εⱼ|X) = 0 for i≠j.

Violation of (i): Heteroskedasticity. Violation of (ii): Autocorrelation. Both make OLS inefficient; standard errors wrong.

Violation → Not BLUE
A6 — NORMALITY
ε|X ~ N(0, σ²I)

Errors are normally distributed. NOT required for BLUE — required for exact finite-sample inference (t-tests, F-tests).

Violation: t and F tests are approximate (CLT saves us asymptotically for large n).

Required for Exact Inference

The Consequence Hierarchy

Assumption ViolatedOLS Biased?OLS Still BLUE?Std Errors Valid?Fix
A4: E[ε|X]≠0 (endogeneity)YESNONOIV/2SLS, control variables
A5(i): HeteroskedasticityNoNO (inefficient)NO (wrong)WLS, robust std errors
A5(ii): AutocorrelationNoNO (inefficient)NO (wrong)GLS, Newey-West std errors
A6: Non-normalityNoYESApprox. (large n)Bootstrap, robust tests
A3: MulticollinearityNoYES (in theory)Very wideDrop variables, ridge regression

⚠ The Critical Distinction: Biased vs Inefficient

Heteroskedasticity and autocorrelation do NOT bias OLS — the estimates β̂ are still centred on the truth on average. But they make OLS inefficient (too wide or too narrow variance) and crucially, they make the standard errors wrong. Wrong standard errors → wrong t-statistics → wrong p-values → wrong decisions. This is why they matter enormously in practice.

Topic 3

Heteroskedasticity

When error variance is not constant across observations, standard OLS inference breaks down — confidence intervals become either too wide or too narrow, and hypothesis tests mislead.

🔗 Violates Assumption A5(i)

Heteroskedasticity means Var(εᵢ|X) = σᵢ² ≠ σ² — the error variance differs across observations. OLS remains unbiased but is no longer efficient, and its standard errors are wrong. This makes t-tests and F-tests invalid.

What Heteroskedasticity Looks Like

The Income-Spending Analogy

Regress household spending Y on income X. Low-income households have limited flexibility — spending is tightly constrained near income. High-income households have huge variability — one millionaire spends $50K on travel, another $500K. The spread of errors grows with income. That's heteroskedasticity.

In finance: cross-sectional returns during calm periods have small dispersion; during a crisis, returns spread wildly. Volatility clustering in GARCH models is another manifestation.

Homoskedastic Residuals
Constant spread — A5(i) satisfied
Heteroskedastic Residuals
Spread increases with X — A5(i) violated

Consequences for OLS

True Variance Under Heteroskedasticity
True Var(β̂) = (XᵀX)⁻¹ (Σᵢ σᵢ²xᵢxᵢᵀ) (XᵀX)⁻¹
OLS assumes: Var(β̂) = σ²(XᵀX)⁻¹
The true variance is a "sandwich" matrix. The OLS formula σ²(XᵀX)⁻¹ is generally wrong — it can be too large or too small depending on the pattern of heteroskedasticity.
OLS Under HeteroskedasticityStatusImplication
β̂ estimatesStill unbiasedE[β̂] = β — no bias problem
β̂ estimatesNOT efficient (not BLUE)WLS has lower variance
Standard errorsWRONG (biased)t-stats invalid — can be too large or small
Confidence intervalsWrong widthUnder or over-state uncertainty
t-tests and F-testsInvalidFalse positives or false negatives

Forms of Heteroskedasticity

Common Functional Forms
Multiplicative: σᵢ² = σ²·Xᵢ² (variance proportional to X²)
Power form: σᵢ² = σ²·Xᵢᵞ (γ unknown)
Linear form: σᵢ² = α₀ + α₁Xᵢ (White's general form)
Group-wise: σᵢ² = σₐ² if Xᵢ in group A, σᵦ² if in group B

Detection Tests

Breusch-Pagan Test
Regress ê² on X → LM = n·R² ~ χ²(k)
H₀: homoskedasticity. H₁: σᵢ² is a linear function of X. R² is from the auxiliary regression of squared residuals on regressors.
White's General Test
Regress ê² on X, X², and cross-products → LM = n·R² ~ χ²(p)
More general than Breusch-Pagan — detects all forms of heteroskedasticity. p = number of auxiliary regressors (can be large).
Goldfeld-Quandt Test
Sort by X, compare SSR of top vs bottom thirds → F-statistic
Assumes monotone relationship between variance and X. Intuitive but less general.

🧪 Colab Experiment 2 — Heteroskedasticity Complete Workflow

experiment_2_heteroskedasticity.py ▶ Open Colab 
import numpy as np
import matplotlib.pyplot as plt
from scipy import stats
import statsmodels.api as sm
from statsmodels.stats.diagnostic import het_breuschpagan, het_white

np.random.seed(42)

# ════════════════════════════════════════════════════════
# Generate heteroskedastic data: σᵢ² = σ²·Xᵢ²
# ════════════════════════════════════════════════════════
n = 200
X = np.random.uniform(1, 10, n)
X_with_const = sm.add_constant(X)

beta_true = np.array([2.0, 1.5])
sigma_base = 0.5

# Heteroskedastic errors: std dev proportional to X
sigma_i = sigma_base * X  # σᵢ = 0.5·Xᵢ (larger X → larger error)
eps = np.random.normal(0, sigma_i, n)
Y = X_with_const @ beta_true + eps

# ════════════════════════════════════════════════════════
# Fit OLS
# ════════════════════════════════════════════════════════
model_ols = sm.OLS(Y, X_with_const).fit()
resid = model_ols.resid

print("=== OLS Results (No Correction) ===")
print(model_ols.summary().tables[1])

# ════════════════════════════════════════════════════════
# Tests for Heteroskedasticity
# ════════════════════════════════════════════════════════
lm_bp, p_bp, fval_bp, fp_bp = het_breuschpagan(resid, X_with_const)
print(f"\n=== Breusch-Pagan Test ===")
print(f"LM statistic: {lm_bp:.4f}, p-value: {p_bp:.6f}")
print(f"{'REJECT H₀: Heteroskedasticity detected!' if p_bp < 0.05 else 'Fail to reject H₀: No evidence of heteroskedasticity'}")

lm_w, p_w, fval_w, fp_w = het_white(resid, X_with_const)
print(f"\n=== White's Test ===")
print(f"LM statistic: {lm_w:.4f}, p-value: {p_w:.6f}")

# ════════════════════════════════════════════════════════
# Compare Standard Errors: OLS vs HC0 vs HC3
# ════════════════════════════════════════════════════════
model_hc0 = sm.OLS(Y, X_with_const).fit(cov_type='HC0')
model_hc3 = sm.OLS(Y, X_with_const).fit(cov_type='HC3')

print("\n=== Standard Error Comparison ===")
print(f"{'':20} {'OLS SE':>10} {'HC0 SE':>10} {'HC3 SE':>10}")
for i, name in enumerate(['Intercept', 'X slope']):
    print(f"{name:20} {model_ols.bse[i]:>10.4f} {model_hc0.bse[i]:>10.4f} {model_hc3.bse[i]:>10.4f}")

# ════════════════════════════════════════════════════════
# WLS: If we know σᵢ ∝ Xᵢ, weight by 1/Xᵢ²
# ════════════════════════════════════════════════════════
weights = 1 / X**2
model_wls = sm.WLS(Y, X_with_const, weights=weights).fit()
print("\n=== WLS (Optimal Weights) ===")
print(f"WLS slope: {model_wls.params[1]:.4f} (true: {beta_true[1]})")
print(f"OLS slope: {model_ols.params[1]:.4f}")
print(f"WLS SE: {model_wls.bse[1]:.4f} vs OLS SE: {model_ols.bse[1]:.4f}")

# Diagnostic plots
fig, axes = plt.subplots(2, 2, figsize=(13, 10))
axes[0,0].scatter(X, resid, alpha=0.4, color='#c0392b')
axes[0,0].axhline(0, color='black', lw=2)
axes[0,0].set_title('Residuals vs X (Heteroskedastic Fan)')
axes[0,0].set_xlabel('X'); axes[0,0].set_ylabel('Residual')

axes[0,1].scatter(X, resid**2, alpha=0.4, color='#e67e22')
axes[0,1].set_title('Squared Residuals vs X\n(Breusch-Pagan auxiliary regression)')

axes[1,0].scatter(model_ols.fittedvalues, resid, alpha=0.4, color='#8e44ad')
axes[1,0].axhline(0, color='black', lw=2)
axes[1,0].set_title('Residuals vs Fitted (Scale-Location)')

axes[1,1].scatter(np.sqrt(np.abs(resid)), model_ols.fittedvalues, alpha=0.4, color='#27ae60')
axes[1,1].set_title('Scale-Location Plot\n(sqrt|ê| vs fitted)')

plt.tight_layout(); plt.show()

Topic 4

Autocorrelation

When errors are correlated across time or space, OLS standard errors can be dramatically underestimated — making results appear far more significant than they really are.

🔗 Violates Assumption A5(ii)

Autocorrelation (serial correlation) means Cov(εᵢ, εⱼ|X) ≠ 0 for i≠j. Most common in time-series data where today's error is correlated with yesterday's. Like heteroskedasticity, OLS remains unbiased but is inefficient and has wrong standard errors.

What Autocorrelation Looks Like

The Weather Analogy

If your regression of returns on interest rates has errors today that are large and positive, the next day's errors are also likely to be large and positive — because the shock that caused the large residual today (e.g., a new technology emerging) doesn't vanish overnight. The errors have "memory."

In a model with autocorrelation, consecutive residuals look like a smooth wave or trend rather than white noise. This persistence is the signature.

White Noise Residuals — No Autocorrelation
Residuals jump randomly — A5(ii) satisfied
AR(1) Residuals — Positive Autocorrelation
Runs of same sign — A5(ii) violated (ρ=0.7)

The AR(1) Error Model — Most Common Case

First-Order Autoregressive Errors
εₜ = ρεₜ₋₁ + uₜ, where uₜ ~ i.i.d. N(0, σᵤ²)
Cov(εₜ, εₜ₋ₖ) = ρᵏ · σ²/(1−ρ²)
|ρ| < 1 for stationarity. ρ > 0: positive autocorrelation (most common in economics — residuals move together). ρ < 0: alternating (less common). σ² = σᵤ²/(1−ρ²) is the marginal variance.

Consequences for OLS

True vs Estimated Variance Under AR(1)
True Var(β̂₁) ≈ (σ²/Σ(xₜ-x̄)²) · (1 + 2ρ·(n−1)/n + 2ρ²·(n−2)/n + ...)
OLS Assumes: Var(β̂₁) = σ²/Σ(xₜ-x̄)²
The variance inflation factor (VIF for autocorrelation) ≈ (1+ρ)/(1−ρ) for large n. With ρ=0.5: VIF = 3 — OLS standard errors are 1/√3 ≈ 0.58 of true → t-stats inflated by √3 ≈ 1.73.
ρ (AR(1))Variance Inflation (1+ρ)/(1-ρ)SE Deflationt-Stat Inflation

⚠ Autocorrelation Makes Everything Look Significant

With ρ=0.7, your t-statistics are inflated by a factor of ~2.4. This means a true t-stat of 1.0 (not significant) appears as 2.4 (highly significant). In time-series financial regressions, this routinely causes researchers to find "significant" return predictability that is entirely spurious — driven by autocorrelated errors, not true relationships.

Detection Tests

Durbin-Watson Test
DW = Σₜ(êₜ − êₜ₋₁)² / Σₜêₜ² ≈ 2(1−ρ̂)
DW ≈ 2: no autocorrelation. DW < 2: positive autocorrelation. DW > 2: negative autocorrelation. Critical values dL and dU from Durbin-Watson tables. Range: [0, 4]. Inconclusive region between dL and dU.
Breusch-Godfrey LM Test (More General)
Regress ê on X and ê₋₁,...,ê₋ₚ → LM = n·R² ~ χ²(p)
Tests for AR(p) autocorrelation. More powerful than DW. Works when lagged Y appears as regressor (DW is invalid then). Most commonly used in practice.

experiment_3_autocorrelation.py ▶ Open Colab 
import numpy as np
import matplotlib.pyplot as plt
from statsmodels import api as sm
from statsmodels.stats.stattools import durbin_watson
from statsmodels.stats.diagnostic import acorr_breusch_godfrey

np.random.seed(0)
T = 200; rho = 0.7; beta_true = [1.0, 2.0]; sigma_u = 1.0

# Generate time series with AR(1) errors
t_idx = np.arange(T)
X_ts = 0.5 + 0.02*t_idx + np.random.normal(0, 0.5, T)  # trending X

eps = np.zeros(T)
u = np.random.normal(0, sigma_u, T)
for t in range(1, T):
    eps[t] = rho * eps[t-1] + u[t]

Xm = sm.add_constant(X_ts)
Y_ts = Xm @ np.array(beta_true) + eps

# ════════════════════════════════════════════════════════
# Fit OLS and run diagnostics
# ════════════════════════════════════════════════════════
model = sm.OLS(Y_ts, Xm).fit()
resid = model.resid

dw = durbin_watson(resid)
lm_bg, p_bg, f_bg, fp_bg = acorr_breusch_godfrey(model, nlags=5)

print(f"=== OLS with ρ={rho} AR(1) Errors ===")
print(f"True β₁={beta_true[1]}. OLS β̂₁={model.params[1]:.4f}")
print(f"Durbin-Watson: {dw:.4f} (≈2(1-ρ) = {2*(1-rho):.2f})")
print(f"Breusch-Godfrey LM: {lm_bg:.4f}, p-value: {p_bg:.6f}")
print(f"→ {'AUTOCORRELATION DETECTED' if p_bg < 0.05 else 'No autocorrelation detected'}")

# Compare OLS vs Newey-West standard errors
model_nw = sm.OLS(Y_ts, Xm).fit(cov_type='HAC', cov_kwds={'maxlags': 5})
print(f"\n=== Standard Error Comparison ===")
print(f"OLS SE(β₁) = {model.bse[1]:.4f},  t = {model.tvalues[1]:.2f}")
print(f"NW  SE(β₁) = {model_nw.bse[1]:.4f},  t = {model_nw.tvalues[1]:.2f}")
print(f"Variance inflation: {(model_nw.bse[1]/model.bse[1])**2:.2f}x")

# Cochrane-Orcutt correction
rho_hat = np.corrcoef(resid[1:], resid[:-1])[0,1]
Y_co = Y_ts[1:] - rho_hat * Y_ts[:-1]
X_co = sm.add_constant(X_ts[1:] - rho_hat * X_ts[:-1])
model_co = sm.OLS(Y_co, X_co).fit()
print(f"\n=== Cochrane-Orcutt Correction ===")
print(f"Estimated ρ̂ = {rho_hat:.4f} (True ρ = {rho})")
print(f"CO β̂₁ = {model_co.params[1]:.4f}, SE = {model_co.bse[1]:.4f}")

# ACF Plot of Residuals
fig, axes = plt.subplots(2, 2, figsize=(13, 8))
axes[0,0].plot(t_idx, resid, color='#c0392b', lw=1)
axes[0,0].axhline(0, color='black', lw=1.5)
axes[0,0].set_title('Residuals Over Time (AR(1) pattern visible)')

sm.graphics.tsa.plot_acf(resid, lags=25, ax=axes[0,1])
axes[0,1].set_title('ACF of Residuals (exponential decay)')

axes[1,0].scatter(resid[:-1], resid[1:], alpha=0.4, color='#2980b9')
axes[1,0].set_title(f'Residual(t) vs Residual(t-1): ρ̂={rho_hat:.3f}')

sm.graphics.tsa.plot_acf(model_co.resid, lags=25, ax=axes[1,1])
axes[1,1].set_title('ACF After Cochrane-Orcutt (cleaned up)')

plt.tight_layout(); plt.show()

experiment_4_combined_diagnostics.py ▶ Open Colab 
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
import statsmodels.api as sm
from statsmodels.stats.diagnostic import (
    het_breuschpagan, het_white, acorr_breusch_godfrey)
from statsmodels.stats.stattools import durbin_watson
from statsmodels.graphics.tsaplots import plot_acf

def full_ols_diagnostics(Y, X, title="OLS Diagnostics"):
    """Complete OLS diagnostic workflow"""
    Xc = sm.add_constant(X)
    model = sm.OLS(Y, Xc).fit()
    resid = model.resid

    print(f"\n{'='*60}")
    print(f"{title}")
    print(f"{'='*60}")

    # 1. Heteroskedasticity tests
    lm_bp, p_bp, _, _ = het_breuschpagan(resid, Xc)
    lm_w,  p_w,  _, _ = het_white(resid, Xc)
    print(f"\n📊 HETEROSKEDASTICITY:")
    print(f"  Breusch-Pagan: LM={lm_bp:.3f}, p={p_bp:.4f} → {'⚠ DETECTED' if p_bp<0.05 else '✓ OK'}")
    print(f"  White's Test:  LM={lm_w:.3f},  p={p_w:.4f} → {'⚠ DETECTED' if p_w<0.05 else '✓ OK'}")

    # 2. Autocorrelation tests
    dw = durbin_watson(resid)
    lm_bg, p_bg, _, _ = acorr_breusch_godfrey(model, nlags=5)
    print(f"\n📈 AUTOCORRELATION:")
    print(f"  Durbin-Watson: {dw:.4f} (2=none, <2=pos, >2=neg)")
    print(f"  Breusch-Godfrey: LM={lm_bg:.3f}, p={p_bg:.4f} → {'⚠ DETECTED' if p_bg<0.05 else '✓ OK'}")

    # 3. Compare standard errors
    m_hc3 = sm.OLS(Y, Xc).fit(cov_type='HC3')
    m_hac = sm.OLS(Y, Xc).fit(cov_type='HAC', cov_kwds={'maxlags':5})
    print(f"\n📐 STANDARD ERROR COMPARISON (slope):")
    print(f"  OLS SE = {model.bse[1]:.4f}, t = {model.tvalues[1]:.2f}")
    print(f"  HC3 SE = {m_hc3.bse[1]:.4f}, t = {m_hc3.tvalues[1]:.2f}")
    print(f"  HAC SE = {m_hac.bse[1]:.4f}, t = {m_hac.tvalues[1]:.2f}")

    # 4. Diagnostic plots
    fig, axes = plt.subplots(2, 3, figsize=(15, 8))
    fitted = model.fittedvalues

    axes[0,0].scatter(fitted, resid, alpha=0.4); axes[0,0].axhline(0,c='r')
    axes[0,0].set_title('Residuals vs Fitted')
    axes[0,1].scatter(fitted, np.sqrt(np.abs(resid)), alpha=0.4)
    axes[0,1].set_title('Scale-Location')
    sm.qqplot(resid, line='45', ax=axes[0,2], alpha=0.4); axes[0,2].set_title('Q-Q Plot')
    axes[1,0].plot(resid, alpha=0.7); axes[1,0].axhline(0,c='r')
    axes[1,0].set_title('Residuals Over Time')
    plot_acf(resid, lags=20, ax=axes[1,1]); axes[1,1].set_title('ACF of Residuals')
    axes[1,2].scatter(resid[:-1], resid[1:], alpha=0.4); axes[1,2].set_title('Lag-1 Residuals')

    plt.suptitle(title, fontsize=14, fontweight='bold')
    plt.tight_layout(); plt.show()
    return model

# ════════ Run on three scenarios ════════
