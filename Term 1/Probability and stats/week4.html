Prerequisites

Before we dive in, make sure these building blocks feel familiar. We'll explain each one simply — even if they're brand new to you.

⚠️ What you need to know first

These topics appear constantly in everything that follows. We'll give you a quick-start understanding of each — enough to confidently build on them.

1. Random Variables

A random variable is just a variable whose value is determined by a random experiment. Think of it as a rule that assigns a number to each outcome.

Real-World Analogy

Roll a standard die. The outcome is random — you don't know if you'll get 1, 2, 3, 4, 5, or 6. But if you define X = "number shown on die", then X is a random variable. It's not random after you roll — it becomes a fixed number. Before rolling, it's a variable with possible values {1, 2, 3, 4, 5, 6}, each equally likely.

In quant finance: Tomorrow's stock price is a random variable. You don't know it yet, but you can reason about its probability distribution.

Discrete Random Variable
Takes countable, distinct values. Examples: number of heads in 10 coin flips, number of trades in an hour.
Continuous Random Variable
Can take any value in a range. Examples: the exact return on a stock today, a person's exact height.

2. Probability Mass Function (PMF) — for Discrete

The PMF of a discrete random variable X tells you: "What is the probability that X equals exactly this value?"

PMF Definition
P(X = x) = p(x)
Must satisfy: p(x) ≥ 0 for all x, and Σ p(x) = 1 (all probabilities sum to 1)

Think of PMF as a histogram of a die

For a fair die: P(X=1) = P(X=2) = ... = P(X=6) = 1/6. Each bar in a histogram has height 1/6. They all add up to 1. That's a PMF.

PMF of a Fair Die (X ∈ {1,2,3,4,5,6})

3. Probability Density Function (PDF) — for Continuous

For continuous variables, P(X = x) = 0 for any exact value. Instead we use a PDF. The PDF f(x) gives the density of probability, and we find probabilities by computing areas under the curve.

PDF Key Property
P(a ≤ X ≤ b) = ∫ₐᵇ f(x) dx
Total area = 1: ∫₋∞^∞ f(x) dx = 1. Density f(x) ≥ 0 everywhere.

Think of PDF as a density map

Imagine spreading 1 kg of sand over a table. The "density" at each point tells you how much sand is there per unit area. The total sand is 1 kg. Probability is like sand — it's spread out, and to get how much falls in a range, you "collect" (integrate) the density over that range.

PDF of Standard Normal N(0,1) — the famous bell curve

4. Expected Value E[X]

The expected value (or mean) is the long-run average value of a random variable if you repeated the experiment many times.

Expected Value
E[X] = Σ x · P(X=x)    (discrete)
E[X] = ∫ x · f(x) dx    (continuous)
Weight each value by how likely it is, then sum everything up.

Weighted Average Example

Fair die: E[X] = 1·(1/6) + 2·(1/6) + 3·(1/6) + 4·(1/6) + 5·(1/6) + 6·(1/6) = 21/6 = 3.5

You'll never actually roll a 3.5, but if you roll millions of times, the average of all outcomes converges to 3.5. This is the Law of Large Numbers.

5. Independence

Two events A and B are independent if knowing one happened tells you nothing about whether the other happened.

Independence
P(A and B) = P(A) · P(B)
For random variables: X and Y are independent if P(X=x, Y=y) = P(X=x)·P(Y=y) for all x,y

Coin Flip Independence

Flipping two fair coins: the first coin landing heads does NOT affect the second coin. P(H₁ and H₂) = ½ × ½ = ¼. But stock returns on two related stocks are NOT independent — a market crash affects both.

✅ You're ready to proceed! With these five concepts — random variables, PMF, PDF, expected value, and independence — you have everything you need to master Joint Distributions and Conditional Expectation. Let's go!

Topic 1 of 3

Joint PMF / PDF

What happens when we study two (or more) random variables at the same time? That's exactly what joint distributions answer.

🎯 Why Quants Need This

In finance, assets don't live in isolation. A bond's yield and a stock's return are studied simultaneously. Options pricing depends on the joint behavior of underlying assets. Risk management requires understanding how multiple risks co-move.

The Core Idea — Two Variables at Once

When you have a single random variable X, you describe it with a PMF or PDF. When you have two random variables X and Y — both depending on the same random experiment — you need a joint distribution that describes them together.

The Weather Example

Imagine a weather station that records two things each day: X = temperature (High/Low) and Y = whether it rains (Yes/No). These aren't independent — rainy days tend to be cooler. To capture their relationship, you need a joint distribution over all four combinations: (High, Rain), (High, No Rain), (Low, Rain), (Low, No Rain).

Joint PMF — Discrete Case

For two discrete random variables X and Y, the Joint PMF is a function that gives the probability for every combination of values:

Joint PMF
p(x, y) = P(X = x, Y = y)
Read as: "The probability that X equals x AND Y equals y simultaneously."

The Three Requirements of a Valid Joint PMF

  1. Non-negativity

    p(x, y) ≥ 0 for all (x, y). Probabilities can't be negative.

  2. Normalization

    Σₓ Σᵧ p(x, y) = 1. All probabilities summed over every possible combination equal exactly 1.

  3. Completeness

    p(x, y) is defined for every possible pair (x, y) in the sample space.

Concrete Example — Dice Roll

Roll two fair dice. Let X = value on die 1, Y = value on die 2. Since the dice are independent, P(X=x, Y=y) = (1/6)(1/6) = 1/36 for all pairs. But let's try something more interesting.

New example: Let X = number of heads in 2 fair coin flips, Y = number of tails in those same 2 flips. Note: Y = 2 − X always!

p(x,y)Y=0Y=1Y=2
X=0 001/4
X=1 02/40
X=2 1/400

📌 Notice: X and Y are NOT independent here!

If X=2 (both heads), then Y must be 0. Knowing X completely determines Y. Their joint distribution is highly structured, not just a product of marginals.

A Richer Example — Bivariate Distribution

Let X = number of software bugs found in module A, Y = bugs found in module B. After historical analysis, the joint PMF is:

p(x,y)Y=0Y=1Y=2Row Sum
X=0 0.100.150.050.30
X=1 0.200.250.050.50
X=2 0.050.100.050.20
Col Sum 0.350.500.151.00 ✓

The green values are the marginal distributions — a preview of our next topic! Everything must sum to 1 (bottom-right corner).

Joint PDF — Continuous Case

When X and Y are both continuous, we have a Joint PDF f(x, y) — a surface over the (x,y) plane where the height represents the density of probability.

Joint PDF Key Property
P(a ≤ X ≤ b, c ≤ Y ≤ d) = ∫ₐᵇ ∫꜀ᵈ f(x,y) dy dx
Normalization: ∫∫ f(x,y) dy dx = 1 over the entire plane. f(x,y) ≥ 0 everywhere.

Famous Example: Bivariate Normal Distribution

In finance, stock returns are often modeled as jointly normal. The joint PDF of two correlated normal random variables is a 3D bell-shaped surface (an "elliptical hill").

Contour Lines = Same Density Level

The elliptical contour lines on the base of the 3D surface are like topographic map lines — each loop represents points with the same probability density. The more circular the ellipses, the lower the correlation. Tilted/elongated ellipses signal strong correlation.

Key Relationships to Remember

Everything connects: Joint → Marginals → Conditional → Conditional Expectation

Topic 2 of 3

Marginal Distributions

If you know the joint behavior of X and Y, how do you "zoom out" and ask only about X — ignoring Y entirely? That's exactly what marginal distributions do.

🔗 Connection to Joint Distribution

Marginal distributions are derived from the joint distribution. You cannot have a marginal without first having a joint. This is why we studied joint distributions first.

The Core Idea — Collapsing One Dimension

Imagine your joint distribution as a spreadsheet with X values as rows and Y values as columns. Each cell has a probability p(x,y). The row totals give you the marginal distribution of X. The column totals give you the marginal distribution of Y.

The Spreadsheet Marginals Analogy

In Excel, if column A is "product" and column B is "region" and the cells contain "revenue", then the row sum tells you total revenue per product (ignoring region). That's the marginal distribution of "product." You've marginalized out the region variable.

Marginal PMF — Discrete Case

Marginal PMF Formulas
p_X(x) = Σᵧ p(x, y)     (sum over all Y values)
p_Y(y) = Σₓ p(x, y)     (sum over all X values)
You "sum out" the variable you don't care about. The remaining function is the marginal.

Step-by-Step: Computing Marginals from the Bug Example

We return to our software bug example. Let's compute both marginals explicitly:

p(x,y)Y=0Y=1Y=2p_X(x)
X=0 0.100.150.05 0.10+0.15+0.05 = 0.30
X=1 0.200.250.05 0.20+0.25+0.05 = 0.50
X=2 0.050.100.05 0.05+0.10+0.05 = 0.20
p_Y(y) 0.35 0.50 0.15 1.00 ✓

📌 Interpretation

p_X(x): If you only care about module A (ignoring module B), there's a 30% chance of 0 bugs, 50% chance of 1 bug, 20% chance of 2 bugs.

p_Y(y): If you only care about module B, there's a 35% chance of 0 bugs, 50% chance of 1 bug, 15% chance of 2 bugs.

Marginal Distributions vs Joint Distribution

Marginal PDF — Continuous Case

For continuous random variables, "summing out" becomes "integrating out":

Marginal PDF Formulas
f_X(x) = ∫₋∞^∞ f(x,y) dy
f_Y(y) = ∫₋∞^∞ f(x,y) dx
Integrate (sum continuously) over the variable you want to eliminate.

Example — Uniform over a Triangle

Let f(x,y) = 2 for 0 ≤ x ≤ 1, 0 ≤ y ≤ x (a triangle region). Let's find the marginal of X:

Finding f_X(x)
f_X(x) = ∫₀ˣ 2 dy = 2x    for 0 ≤ x ≤ 1
We integrated over y from 0 to x (the valid range of y given x). Result: f_X(x) = 2x.

💡 Key insight: Two different joint distributions can have the SAME marginals! The marginals alone don't tell you the full story of X and Y together.

The Most Important Lesson About Marginals: Knowing the marginal distributions f_X(x) and f_Y(y) does NOT mean you know the joint distribution f(x,y). You also need to know how X and Y are related (their dependence structure, often captured by correlation or copulas). In quant finance, this is why copula models were so important (and why their misuse contributed to the 2008 financial crisis).

Independence in Terms of Marginals

Here's a beautiful and very useful result:

Independence ↔ Factorization
X ⊥ Y ⟺ f(x,y) = f_X(x) · f_Y(y)
X and Y are independent if and only if the joint distribution FACTORS into the product of marginals.

This gives you a simple test for independence: compute the marginals, multiply them, and check if the product equals the joint distribution everywhere.

🧪 Colab Experiment 2: Computing and Plotting Marginals

experiment_2_marginal_distributions.py · Google Colab ▶ Open Colab 
import numpy as np
import matplotlib.pyplot as plt
from scipy.stats import multivariate_normal

# ── Discrete: Compute marginals from joint PMF ──
joint_pmf = np.array([
    [0.10, 0.15, 0.05],
    [0.20, 0.25, 0.05],
    [0.05, 0.10, 0.05],
])

marginal_X = joint_pmf.sum(axis=1)  # sum over columns (Y)
marginal_Y = joint_pmf.sum(axis=0)  # sum over rows (X)

print("Marginal PMF of X:", marginal_X)
print("Marginal PMF of Y:", marginal_Y)
print("Sum of X marginal:", marginal_X.sum(), "(must be 1)")

# ── Test for Independence ──
product = np.outer(marginal_X, marginal_Y)
print("\nProduct of Marginals:")
print(product.round(4))
print("\nJoint PMF:")
print(joint_pmf)
print("\nAre they equal? (Independence test)")
print(np.allclose(joint_pmf, product))  # False → NOT independent

# ── Visualization: Joint + Marginals in one plot ──
fig = plt.figure(figsize=(10, 8))
gs = fig.add_gridspec(2, 2, hspace=0.05, wspace=0.05,
                      width_ratios=[4, 1], height_ratios=[1, 4])

ax_joint = fig.add_subplot(gs[1, 0])
ax_marg_x = fig.add_subplot(gs[0, 0], sharex=ax_joint)
ax_marg_y = fig.add_subplot(gs[1, 1], sharey=ax_joint)

# Continuous bivariate normal for a nicer plot
for rho, title in [(0.0, 'Independent (ρ=0)'), (0.8, 'Correlated (ρ=0.8)')]:
    x_range = np.linspace(-3, 3, 200)
    y_range = np.linspace(-3, 3, 200)
    X, Y = np.meshgrid(x_range, y_range)
    pos = np.dstack((X, Y))
    
    rv = multivariate_normal([0, 0], [[1, rho],[rho, 1]])
    Z = rv.pdf(pos)
    
    fig2, axes2 = plt.subplots(1, 3, figsize=(15, 4))
    
    # Joint
    axes2[0].contourf(X, Y, Z, levels=15, cmap='Blues')
    axes2[0].set_title(f'Joint f(x,y): {title}')
    
    # Marginal X — integrate over y (same for both!)
    from scipy.stats import norm
    x_vals = np.linspace(-3, 3, 200)
    axes2[1].plot(x_vals, norm.pdf(x_vals), 'b-', linewidth=2)
    axes2[1].set_title(f'Marginal f_X(x) — same regardless of ρ!')
    axes2[1].fill_between(x_vals, norm.pdf(x_vals), alpha=0.3)
    
    # Marginal Y
    axes2[2].plot(x_vals, norm.pdf(x_vals), 'g-', linewidth=2)
    axes2[2].set_title(f'Marginal f_Y(y) — also same!')
    axes2[2].fill_between(x_vals, norm.pdf(x_vals), alpha=0.3, color='green')
    
    plt.suptitle(title, fontsize=14, fontweight='bold')
    plt.tight_layout()
    plt.show()
    print(f"\n{title}: Joint changes with rho, but BOTH marginals stay N(0,1)!")

Topic 3 of 3

Conditional Expectation E[X | Y]

You've learned what X looks like on average. But what if you already know Y? Conditional expectation answers: "Given what I know about Y, what's my best guess for X?"

🔗 Builds On Everything Before This

Conditional expectation requires: (1) a joint distribution to define conditioning, (2) marginal distributions to normalize, and (3) conditional distributions as an intermediate step. This is why we covered those topics first.

Step 0: Conditional Distributions

Before defining E[X|Y], we first need to understand the conditional distribution — the distribution of X given that Y has taken a specific value.

Conditional PMF
p(x | y) = P(X=x | Y=y) = p(x,y) / p_Y(y)
Read: "The probability X=x, GIVEN that we know Y=y." We divide by p_Y(y) to re-normalize.

The "Slicing" Analogy

Imagine the joint PMF table as a spreadsheet. Conditioning on Y=1 means: zoom in to just the column where Y=1. Extract only those values, then rescale them so they sum to 1. That rescaled column is your conditional distribution P(X | Y=1).

For the continuous case: conditioning on Y=y is like taking a vertical slice through the 3D surface f(x,y) at height y, then normalizing the slice so its area equals 1.

Computing P(X | Y=1) from Bug Example

From our table, the Y=1 column is: p(0,1)=0.15, p(1,1)=0.25, p(2,1)=0.10. Marginal p_Y(1)=0.50.

xp(x,1)p_Y(1)P(X=x | Y=1)
00.150.500.15/0.50 = 0.30
10.250.500.25/0.50 = 0.50
20.100.500.10/0.50 = 0.20
Sum =1.00 ✓

Now: Conditional Expectation E[X | Y = y]

Once you have the conditional distribution, computing E[X | Y=y] is exactly like computing a regular expected value — but using the conditional distribution instead of the marginal.

Conditional Expectation — Discrete
E[X | Y=y] = Σₓ x · P(X=x | Y=y) = Σₓ x · p(x,y) / p_Y(y)
Conditional Expectation — Continuous
E[X | Y=y] = ∫ x · f(x|y) dx = ∫ x · f(x,y)/f_Y(y) dx

Computing E[X | Y=1] for Bug Example

Step-by-Step Calculation
E[X | Y=1] = 0×0.30 + 1×0.50 + 2×0.20 = 0 + 0.50 + 0.40 = 0.90
Compare to unconditional mean: E[X] = 0×0.30 + 1×0.50 + 2×0.20 = 0.90. (In this case similar; they'd differ more in strongly dependent distributions.)

E[X|Y] as a Random Variable — The Key Insight

Here's what trips up most first-time learners: E[X|Y=y] is a number (for a fixed y). But E[X|Y] — without fixing Y — is a random variable that is a function of Y.

The Forecasting Analogy

A weather forecaster says: "If it rained yesterday (Y=1), the expected high temperature today is 18°C. If it didn't rain (Y=0), expected high is 24°C."

This defines a function: g(Y) = E[Temp | Y]. Before you know yesterday's rain, g(Y) is random (you don't know which value it will take). After you know it rained, g(1) = 18°C is just a number. Both interpretations are correct — they're just at different stages of observation.

In quant finance: E[Stock Return | Market Return = r] is your prediction of a stock's return given a specific market return. It varies as r varies — it's a function (called the regression function).

E[X|Y=y] as a Function of y — Conditional Expectation Curve

The Tower Property (Law of Total Expectation)

This is one of the most powerful and frequently used results in probability theory and quantitative finance:

Tower Property / Law of Total Expectation
E[X] = E[ E[X|Y] ]
The expectation of the conditional expectation equals the unconditional expectation. You can "average out" the conditioning variable.

Salary Example

E[Salary] = E[E[Salary | Department]]

Step 1: Compute average salary within each department (that's E[Salary | Dept = d] for each d). Step 2: Take a weighted average across departments (weighted by how many employees are in each). That grand average equals the overall mean salary. The tower property guarantees this always works.

Tower Property — Proof Sketch (Discrete)

Proof Sketch
E[E[X|Y]] = Σᵧ E[X|Y=y] · P(Y=y)
           = Σᵧ [Σₓ x · P(X=x|Y=y)] · P(Y=y)
           = Σᵧ Σₓ x · P(X=x,Y=y)
           = Σₓ x · [Σᵧ P(X=x,Y=y)]
           = Σₓ x · P(X=x) = E[X] ✓

The Geometric Interpretation

For a bivariate normal, E[X|Y=y] traces out a straight line — this is literally the linear regression line! That's no coincidence.

For Bivariate Normal: Conditional Expectation
E[X | Y=y] = μ_X + ρ · (σ_X/σ_Y) · (y − μ_Y)
This is the regression line of X on Y! The slope is ρ·(σ_X/σ_Y). When ρ=0: E[X|Y]=μ_X (knowing Y tells you nothing new about X).

E[X|Y] and the Best Predictor

Fundamental Theorem: Among all possible functions g(Y) you could use to predict X from Y, the one that minimizes the Mean Squared Error E[(X − g(Y))²] is g(Y) = E[X|Y]. Conditional expectation is the optimal predictor.

This is why linear regression works: if X and Y are bivariate normal, the best predictor is linear — and it's exactly the regression line. For non-normal distributions, the best predictor may be non-linear, and E[X|Y] captures that.

import numpy as np
import matplotlib.pyplot as plt
from scipy.stats import multivariate_normal

# ══════════════════════════════════════════════════
# Part A: Discrete Conditional Expectation
# ══════════════════════════════════════════════════
joint_pmf = np.array([
    [0.10, 0.15, 0.05],
    [0.20, 0.25, 0.05],
    [0.05, 0.10, 0.05],
])

marginal_Y = joint_pmf.sum(axis=0)
x_vals = np.array([0, 1, 2])

print("=== Conditional Distributions P(X | Y=y) ===")
cond_expectations = []
for y in [0, 1, 2]:
    cond_pmf = joint_pmf[:, y] / marginal_Y[y]
    e_x_given_y = (x_vals * cond_pmf).sum()
    cond_expectations.append(e_x_given_y)
    print(f"\nP(X | Y={y}): {cond_pmf.round(3)}")
    print(f"E[X | Y={y}] = {e_x_given_y:.4f}")

# Verify Tower Property
unconditional_E_X = (x_vals * joint_pmf.sum(axis=1)).sum()
tower = (np.array(cond_expectations) * marginal_Y).sum()
print(f"\n=== Tower Property Verification ===")
print(f"E[X] directly = {unconditional_E_X:.4f}")
print(f"E[E[X|Y]] = {tower:.4f}")
print(f"Equal? {np.isclose(unconditional_E_X, tower)}")

# ══════════════════════════════════════════════════
# Part B: Continuous — Bivariate Normal
# Visualize E[X|Y=y] as regression line
# ══════════════════════════════════════════════════
np.random.seed(42)
rho = 0.75
cov = [[1, rho], [rho, 1]]
samples = np.random.multivariate_normal([0, 0], cov, size=2000)
X_sim, Y_sim = samples[:, 0], samples[:, 1]

# Theoretical E[X|Y=y] = rho * y (since μ=0, σ=1)
y_line = np.linspace(-3, 3, 100)
cond_exp_line = rho * y_line

# Empirical: bin Y values and compute mean X in each bin
bins = np.linspace(-3, 3, 20)
bin_centers, empirical_means = [], []
for i in range(len(bins)-1):
    mask = (Y_sim >= bins[i]) & (Y_sim < bins[i+1])
    if mask.sum() > 10:
        bin_centers.append((bins[i] + bins[i+1])/2)
        empirical_means.append(X_sim[mask].mean())

fig, ax = plt.subplots(figsize=(9, 6))
ax.scatter(Y_sim, X_sim, alpha=0.05, s=8, color='steelblue', label='Simulated (X,Y) pairs')
ax.plot(y_line, cond_exp_line, 'r-', linewidth=2.5,
        label=f'Theoretical E[X|Y=y] = {rho}·y')
ax.scatter(bin_centers, empirical_means, color='orange', s=50, zorder=5,
           label='Empirical E[X|Y≈y] (bin means)')
ax.set_xlabel('Y'); ax.set_ylabel('X')
ax.set_title(f'Conditional Expectation E[X|Y] = Regression Line (ρ={rho})')
ax.legend()
ax.axhline(0, color='gray', linewidth=0.8)
ax.axvline(0, color='gray', linewidth=0.8)
plt.tight_layout()
plt.show()

The Big Picture

How Everything Connects

Let's step back and see how joint distributions, marginals, and conditional expectations form a complete, powerful framework — and why every quant needs them.

The Unified Roadmap

Quant Finance Applications

📈
Options Pricing (Black-Scholes)

Call option price = e^(-rT)·E[max(S_T-K, 0)]. This is a conditional expectation of the payoff under the risk-neutral measure. Computing it requires the joint distribution of paths.

⚖️
Portfolio Risk (VaR, CVaR)

Value at Risk and Conditional VaR require the joint distribution of asset returns. CVaR = E[Loss | Loss > VaR] — literally a conditional expectation!

🔗
Copulas & Tail Dependence

Copulas are joint distributions that separate marginals from dependence structure. They're used to model default correlation in CDOs and structured credit products.

📊
Factor Models (CAPM, APT)

CAPM states E[R_i | R_M] = α + β·R_M. This is the conditional expectation of asset return given the market return — the systematic risk component.

🤖
Machine Learning

Every regression model estimates E[Y|X=x]. Bayesian inference updates joint → conditional. Neural networks implicitly estimate conditional expectations.

🎲
Monte Carlo Pricing

MC methods estimate E[f(X,Y)] by simulating from the joint distribution and averaging. Variance reduction techniques use conditional expectations to reduce sampling noise.

  1. Strengthen Foundations

    Complete all exercises in this guide (Tier 1 first). Make sure the Tower Property feels natural. Work through Harvard Stat 110 lectures 7-10.

  2. Move to Multivariate Distributions

    Generalize to 3+ variables. Study the multivariate normal distribution in detail — it's the workhorse of quantitative finance. Learn about covariance matrices.

  3. Study Copulas

    Learn Sklar's Theorem: any joint distribution can be decomposed into marginals + copula. Gaussian copula, t-copula, Archimedean copulas. Critical for credit risk modeling.

  4. Stochastic Processes

    Conditional expectation is the foundation of martingales. Study Brownian motion, Itô's Lemma, and the Feynman-Kac formula. All require deep understanding of E[X|Y].

  5. Bayesian Statistics

    Bayesian inference is fundamentally about computing conditional distributions. Bayes' theorem: P(θ|data) = P(data|θ)·P(θ)/P(data) — a conditional distribution over parameters.

  6. Quantitative Finance Applications

    Apply everything to Black-Scholes derivation, risk-neutral pricing, VaR/CVaR computation, and portfolio optimization. Read Hull's "Options, Futures, and Other Derivatives."
