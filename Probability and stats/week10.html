Before We Begin: Essential Foundations

These are the building blocks you must understand before the main topics make sense. We explain each concept in plain language — no prior knowledge assumed beyond high-school algebra.

📖 How to use this guide
This tab is your foundation. Read it fully before moving on. Every term introduced here will appear repeatedly in later sections — understanding it deeply now means you'll never feel lost later.

1. What is a Random Variable?

A random variable is simply a quantity whose value is uncertain — it could take different values in different "runs" of an experiment. Think of it like a dice roll. Before you roll, you don't know the outcome. The set of all possible outcomes (1–6) and how likely each one is constitutes the distribution of the random variable.

🎲 Analogy: Stock Prices
In finance, tomorrow's stock price is a random variable. We don't know exactly what it will be, but we can model it with a distribution based on historical data and assumptions. This is the heart of quantitative finance.

2. Probability Distributions

A probability distribution tells you how likely each possible value of a random variable is.

Normal (Gaussian)Bell-shaped, symmetricDaily stock returns

3. Expected Value (Mean)

The expected value E[X] is the long-run average of a random variable. If you ran an experiment a million times and averaged the results, you'd get (approximately) E[X].

E[X] = Σ x · P(X = x)   (discrete)

E[X] = ∫ x · f(x) dx    (continuous)
Expected Value — the weighted average of all possible outcomes

4. Variance and Standard Deviation

Variance Var(X) measures how spread out a distribution is. A high variance means the outcomes are very unpredictable. Standard deviation σ is just the square root of variance — it's in the same units as the original variable, making it more interpretable.

Var(X) = E[(X - E[X])²] = E[X²] - (E[X])²

σ = √Var(X)
Variance & Standard Deviation — measuring uncertainty
⚠️ Why Variance Matters to Quants
As a quant, when you estimate something (like an option price) using simulation, your estimate has its own variance. Smaller variance = more accurate estimate = you need fewer simulations = cheaper computation. This is why "variance reduction" is such a big deal.

5. The Law of Large Numbers

If you repeat an experiment many times and average the results, the average converges to the true expected value. This is the mathematical justification for why simulation (Monte Carlo) works at all.

Formally: as n → ∞,  (X₁ + X₂ + ... + Xₙ) / n → E[X]

6. The Central Limit Theorem (CLT)

Even if individual outcomes have any distribution, when you average many of them, the distribution of that average approaches a Normal (Gaussian) distribution. This is why the normal distribution is everywhere in statistics and finance.

🔑 CLT in Practice
The CLT tells us that our Monte Carlo estimate's error follows a normal distribution, which means we can compute confidence intervals. The standard error of a Monte Carlo estimate is σ/√n — to halve the error, you need 4× more samples. This is the core limitation that variance reduction techniques try to overcome.

7. What is Monte Carlo Simulation? (Big Picture)

Monte Carlo simulation is the idea of using random sampling to estimate quantities that are hard (or impossible) to compute analytically. The name comes from the famous Monte Carlo casino — because randomness is central to the approach.

8. Pseudo-Random vs. True Random Numbers

Computers can't generate truly random numbers — they use deterministic algorithms that produce sequences which look random (they pass statistical tests). These are called pseudo-random number generators (PRNGs). They're completely predictable if you know the "seed" — the starting value. This is actually useful: by fixing the seed, you get reproducible simulations.

9. Probability Density Function (PDF) vs. Cumulative Distribution (CDF)

The PDF f(x) tells you the relative likelihood of a value being near x. The CDF F(x) = P(X ≤ x) tells you the probability of being at most x. The CDF is always between 0 and 1 and is crucial for the "inverse transform" method of generating random numbers.

Generating Random Numbers

Before you can run any simulation, you need a source of randomness. This section explains exactly how computers generate "random" numbers, and how to transform uniform random numbers into any distribution you need.

Why Uniform [0,1] is the Starting Point

Everything in random number generation starts with the uniform distribution on [0,1]. A uniform random number U has equal probability of being anywhere between 0 and 1. From this single primitive, we can generate random numbers from any distribution — the key methods are described below.

💡 The Key Insight
Once you can generate U ~ Uniform[0,1], you can generate anything. The uniform distribution is the "master key" to all random sampling. Every technique below is just a transformation of uniform samples.

The Linear Congruential Generator (LCG)

The most famous PRNG algorithm — still used inside many software packages. It generates a sequence of integers using the recurrence relation:

X_{n+1} = (a · X_n + c) mod m
Linear Congruential Generator · where a = multiplier, c = increment, m = modulus, X₀ = seed

To get a uniform [0,1] number: U_n = X_n / m. The quality of the generator depends critically on the choice of a, c, and m. Bad choices lead to short periods or structured patterns.

The Mersenne Twister

Python's random module and NumPy both use the Mersenne Twister — a far more sophisticated PRNG with period 2¹⁹⁹³⁷ − 1 (astronomically large). It passes rigorous statistical tests and is the industry standard for simulations. You'll use it without thinking about it via np.random.uniform().

Method 1: Inverse Transform Sampling

The most fundamental technique. Given a CDF F(x), we use the fact that if U ~ Uniform[0,1], then X = F⁻¹(U) has distribution F. In plain English: generate a uniform number, then "look up" the corresponding value on the inverse CDF curve.

If U ~ Uniform[0,1], then X = F⁻¹(U) has CDF F
Inverse Transform Theorem — works for any distribution with an invertible CDF
📐 Example: Exponential Distribution
CDF: F(x) = 1 - e^(-λx). Inverse: F⁻¹(U) = -ln(1-U)/λ. So to generate Exp(λ): generate U ~ U[0,1], return X = -ln(1-U)/λ. That's it!

Method 2: Box-Muller Transform (Generating Normal RVs)

There's no closed-form inverse CDF for the normal distribution, so we need a different trick. Box-Muller converts two uniform samples into two independent normal samples:

Z₁ = √(-2 ln U₁) · cos(2π U₂)
Z₂ = √(-2 ln U₁) · sin(2π U₂)
Box-Muller Transform · Z₁, Z₂ ~ N(0,1) independently

Method 3: Acceptance-Rejection Sampling

When inverse transform is hard and Box-Muller doesn't apply, we use acceptance-rejection. The idea: sample from a simpler "proposal" distribution g(x), then accept or reject based on a ratio involving the target f(x).

  1. Find a "cover" distribution g(x) and constant M such that f(x) ≤ M·g(x) for all x.
  2. Generate a candidate X ~ g and U ~ Uniform[0,1].
  3. Accept X if U ≤ f(X) / (M·g(X)), else reject and repeat.
  4. The accepted X values follow distribution f.

Quasi-Random (Low-Discrepancy) Numbers

Standard PRNGs are "random" but can have clumping — areas with many samples and areas with few. Quasi-random sequences (like the Halton or Sobol sequences) are deterministically designed to fill space more uniformly. This can dramatically improve convergence in high-dimensional problems.

Monte Carlo Simulation

With random number generation covered, we now have everything we need to perform Monte Carlo simulation — the most powerful tool in the quant's toolbox.

The Core Idea

Monte Carlo simulation uses repeated random sampling to numerically approximate quantities that are difficult or impossible to calculate analytically. The name honors the famous Monte Carlo Casino in Monaco — a nod to the role of chance.

🔑 The Monte Carlo Principle
To estimate E[f(X)]: generate many samples X₁, X₂, ..., Xₙ from the distribution of X, compute f(Xᵢ) for each, and average them. By the Law of Large Numbers, this average converges to E[f(X)].
μ̂ = (1/n) Σᵢ f(Xᵢ) → E[f(X)] as n → ∞
Monte Carlo Estimator · unbiased and consistent

The Error of Monte Carlo

Our estimate μ̂ has an error. The standard error (SE) tells us how far off our estimate is likely to be:

SE = σ / √n
Standard Error of MC Estimator · σ = std dev of f(X), n = number of samples

This is the fundamental limitation: to halve the error, you need 4× more samples (since √4 = 2). This O(n⁻¹/²) convergence is slow. It's the reason variance reduction techniques exist.

Classic Example: Estimating π

Throw random darts at a unit square. The circle inscribed in the square has area π/4. The fraction of darts inside the circle approaches π/4. So π ≈ 4 × (darts inside) / (total darts).

Monte Carlo in Quantitative Finance: Option Pricing

The most important application for quant engineers. An option is a contract giving the right to buy/sell an asset at a fixed price (strike K) at expiry T. Under the Black-Scholes model, the asset price follows:

S_T = S₀ · exp((r - σ²/2)T + σ√T · Z)

Z ~ N(0,1)
Asset price at expiry T under risk-neutral measure · S₀=spot, r=rate, σ=volatility

The payoff of a European call option is max(S_T - K, 0). The option price is:

C = e^(-rT) · E[max(S_T - K, 0)]
Call Option Price · discounted expected payoff under risk-neutral measure

Why Standard MC is Sometimes Not Enough

Monte Carlo works, but it can be slow. The standard error decreases as 1/√n — you need 100× more samples to get 10× more accuracy. For complex financial products (exotic options, multi-asset portfolios), each simulation can be expensive, so reducing the variance of the estimator is crucial for practical use.

⚠️ The Variance Problem
Suppose your MC option pricer has a std error of ±$0.50 with 10,000 paths. To get ±$0.05 accuracy, you'd need 1,000,000 paths — 100× more computation. Variance reduction techniques can achieve the same accuracy with much fewer paths.

Variance Reduction: Antithetic Variates

Antithetic variates is the simplest and most elegant variance reduction technique. Once you understand the intuition, it feels almost obvious — yet it can halve variance with virtually no extra cost.

The Intuition: Fighting Randomness with Its Mirror

Standard MC samples can "all go the same way by chance" — sometimes you get unlucky and all your samples cluster on one side, giving a biased estimate. The antithetic variates idea: for every random sample you generate, also generate its "mirror" — the scenario that goes the other way.

💡 The Intuition with a Dice Example
Imagine estimating the mean of a fair die (true answer = 3.5). Instead of rolling once and getting "5", roll and also "mirror" it: if you rolled 5, also include 7-5=2. Now your pair averages to 3.5 exactly — no variance! This is the idea behind antithetic variates.

How It Works Mathematically

If U is Uniform[0,1], then 1-U is also Uniform[0,1] — and they are negatively correlated. This is the key: negatively correlated samples partially cancel each other's errors.

Generate paired samples: (U₁, 1-U₁), (U₂, 1-U₂), etc. For each pair, average the two outcomes:

Ŷ = [f(X(U)) + f(X(1-U))] / 2
Antithetic Estimator for a single pair

The variance of this pair-estimator:

Var(Ŷ) = [Var(f(U)) + Var(f(1-U)) + 2·Cov(f(U), f(1-U))] / 4
If f is monotone, Cov is negative → Var(Ŷ) < Var(f(U))/2

When f is a monotone function (which payoff functions often are), the covariance term is negative, making the variance smaller than using two independent samples. The variance reduction compared to standard MC:

Var(AV) / Var(MC) = (1 + ρ) / 2   where ρ = Corr(f(U), f(1-U))
The more negative ρ, the greater the variance reduction

In Practice: Normal Samples

For normal random variables: if Z ~ N(0,1), then -Z ~ N(0,1) too, and they're perfectly negatively correlated (ρ = -1 for Z vs -Z themselves). So for each Z you generate, also use -Z as the antithetic.

Application: Option Pricing with Antithetic Variates

For a European call under Black-Scholes: instead of generating just Z ~ N(0,1) for each path, generate Z and -Z:

  • Path 1: S_T⁺ = S₀ · exp((r - σ²/2)T + σ√T · Z) → payoff₁
  • Path 2 (antithetic): S_T⁻ = S₀ · exp((r - σ²/2)T - σ√T · Z) → payoff₂
  • Estimate: (payoff₁ + payoff₂) / 2 for this pair
✅ When Antithetic Works Well
Antithetic variates work best when the integrand f is monotone in the random inputs. For call options (monotone in S_T, which is monotone in Z), it works beautifully. For non-monotone payoffs (e.g., butterfly spreads), it may not help or could even hurt.

The Cost: None!

A huge advantage: antithetic variates add virtually no computational cost. You generate n/2 base samples and n/2 mirror samples — the same total sample count n, but with lower variance. It's essentially free variance reduction.

Variance Reduction: Control Variates

Control variates is arguably the most powerful and versatile variance reduction technique. The idea is to use known information — something you already know the exact expected value of — to correct and improve your estimate.

The Intuition: Using a Friend Who Knows the Answer

Imagine you're estimating something hard (price of an exotic option). But there's a simpler version of the problem for which you know the exact answer. You can use the difference between your estimate of the simple problem and the known exact answer to correct your estimate of the hard problem.

💡 Everyday Analogy: Estimating Commute Time
You want to estimate tomorrow's commute, but traffic is uncertain. You know exactly how long the drive takes with zero traffic (20 min). You observe today's drive took 27 min, and with zero traffic it would have been 21 min. So extra time today = 6 min. Your estimate for tomorrow = 20 min (baseline) + expected extra time. You're using the "zero-traffic drive" as a control variate.

The Mathematics

We want to estimate μ = E[Y]. We have a control variate C, a random variable with known expectation E[C] = μ_C. The control variate estimator is:

Ŷ_CV = Ȳ - b·(C̄ - μ_C)
Control Variate Estimator · b = control coefficient, C̄ = sample mean of control

Note that E[Ŷ_CV] = E[Ȳ] - b·(E[C̄] - μ_C) = μ - b·0 = μ. So the estimator is still unbiased for any b!

The variance of this estimator:

Var(Ŷ_CV) = Var(Y) + b²·Var(C) - 2b·Cov(Y, C)
Variance of CV estimator

Optimal Control Coefficient b*

Minimize with respect to b by taking the derivative and setting to zero:

b* = Cov(Y, C) / Var(C)
Optimal control coefficient · same formula as OLS regression coefficient!

With this optimal b*, the variance becomes:

Var(Ŷ_CV) = Var(Y) · (1 - ρ²_{YC})
Variance reduction factor · ρ_{YC} = correlation between Y and C
🔑 The Correlation is Everything
The variance reduction is (1 - ρ²). If Y and C are 90% correlated (ρ=0.9), variance drops by 81%! If ρ=0.5, variance drops by 25%. The control variate works best when you can find a variable C that is highly correlated with the target Y and has a known exact mean.

Application: Option Pricing with Control Variates

The most famous application in quant finance: use the Black-Scholes European option price as a control variate for pricing exotic options.

To price an Asian option (whose exact price isn't known analytically):

  1. Simulate paths for the Asian option payoff Y_Asian and simultaneously compute the European option payoff Y_Euro on the same paths.
  2. The European option has a known exact price (Black-Scholes formula) → this is our E[C].
  3. Estimate b* = Cov(Y_Asian, Y_Euro) / Var(Y_Euro) from a pilot run.
  4. Apply: Ŷ_Asian = Ȳ_Asian - b*(Ȳ_Euro - BS_price)
✅ Great Control Variates in Finance
European call/put (for exotics) · Geometric Asian option (for arithmetic Asian) · Delta-hedging strategy · Underlying asset itself (E[S_T] = S₀·e^(rT) is known)

Multiple Control Variates

You can use multiple control variates simultaneously. If C₁, C₂, ..., Cₖ all have known means, you get a system of equations for optimal coefficients. This is exactly multiple linear regression of Y on C₁,...,Cₖ. The variance reduction is 1 - R², where R² is the coefficient of determination from the regression.

Variance Reduction: Importance Sampling

Importance sampling is the most sophisticated of the three techniques, and arguably the most powerful when applied correctly. The idea: instead of sampling uniformly (wasting samples on "unimportant" regions), sample more heavily from the regions that matter most.

The Problem With Standard MC in Rare Events

Consider estimating the probability of a very rare event — say, a stock portfolio losing more than 30% in a day. Standard MC would require an astronomically large number of paths to even observe a single such event. Most samples are "wasted" on normal days that don't contribute to our estimate.

⚠️ The Rare Event Problem
If P(loss > 30%) = 0.001 (one in a thousand), you'd need ~100,000 paths to get 100 observations of the rare event. This is expensive! Importance sampling can reduce this to ~1,000 paths by "zooming in" on the dangerous region.

The Mathematical Foundation

We want to compute E_p[f(X)] = ∫ f(x) · p(x) dx. We can rewrite this as an expectation under a different distribution q(x):

E_p[f(X)] = ∫ f(x) · p(x)/q(x) · q(x) dx = E_q[f(X) · w(X)]
Importance Sampling Identity · w(x) = p(x)/q(x) is the likelihood ratio (weight)

The IS estimator:

μ̂_IS = (1/n) Σᵢ f(Xᵢ) · w(Xᵢ) where Xᵢ ~ q, w(x) = p(x)/q(x)
Importance Sampling Estimator — still unbiased, but with different variance

Choosing the Proposal Distribution q

The variance of the IS estimator:

Var(μ̂_IS) = (1/n) · [E_q[(f(X)·w(X))²] - μ²]
IS variance depends on choice of q

The optimal proposal q*(x) = f(x)·p(x) / μ would give zero variance — but this requires knowing μ, which is what we're trying to estimate! In practice, we choose a q that approximates this optimal distribution.

🔑 The Heuristic Rule
Choose q(x) to be large where f(x)·p(x) is large — i.e., where the integrand is large. For rare event estimation (f = 1 for rare event), shift the distribution toward the rare region. For option pricing, shift the distribution so that the option pays off more often.

Pitfall: When IS Breaks Down

If q(x) = 0 somewhere that p(x)·f(x) > 0, the weights w(x) become infinite — and the estimator can have infinite variance. This is the main danger of IS: a poorly chosen proposal can make things much worse than standard MC.

⚠️ The Cardinal Rule of IS
The support of q must include the support of p·f. Never let q assign zero probability to regions where the integrand is nonzero. "Cover more than you need, never cover less."

Self-Normalized IS (SNIS)

When the normalizing constant of p is unknown (common in Bayesian inference), use the self-normalized estimator:

μ̂_SNIS = Σᵢ f(Xᵢ) · w̃(Xᵢ) / Σᵢ w̃(Xᵢ) where w̃(x) = p̃(x)/q(x)
Self-Normalized IS — biased but consistent; more robust in practice

Application: Option Pricing with IS (Drift Shifting)

For deep out-of-the-money options, most paths yield zero payoff. We shift the drift of the underlying process toward the strike to generate more in-the-money paths, then reweight:

  1. Choose a new drift μ̃ = r + θ·σ (shifted toward in-the-money)
  2. Generate paths under the new measure Q̃
  3. Compute the Radon-Nikodym derivative w = dQ/dQ̃ = exp(-θZ - θ²/2)
  4. Estimator: e^(-rT) · (1/n) Σ payoff(S_T) · w

Putting It All Together: Method Comparison

Now that you understand each technique, let's compare them systematically — when to use each, their relative strengths, and how to combine them.

The Relationship Between Methods

Side-by-Side Comparison

Property Antithetic Variates Control Variates Importance Sampling
Core Idea Use U and 1-U symmetry Correct with known E[C] Shift sampling distribution
Extra Info Needed None Known E[C] for a related variable Good proposal distribution q
Implementation Effort Very low Low-Medium Medium-High
Potential Variance Reduction Up to 50%+ Up to 99%+ (if high ρ) Unlimited (theoretically)
Risk of Backfire Low (non-monotone f) Low (always non-negative) High (infinite variance if bad q)
Best For Simple options, monotone payoffs Exotic options with European analog Rare events, deep OTM options
Works for Path-Dependent? Yes (mirror entire path) Yes (geometric Asian as CV) Yes (measure change over path)
Computational Overhead ~0% ~5-10% (regression) ~10-20% (weight computation)

Which to Use When?

📊 Decision Framework

USE ANTITHETIC WHEN:
  • Quick improvement needed
  • Payoff is monotone in inputs
  • No domain knowledge available
  • Prototyping or first pass
USE CONTROL VARIATES WHEN:
  • Related simpler problem has known answer
  • High correlation with target is expected
  • Pricing exotic options
  • Consistent high accuracy needed
USE IMPORTANCE SAMPLING WHEN:
  • Estimating rare event probabilities
  • Pricing deep out-of-the-money options
  • Standard MC yields mostly zero samples
  • Can design a good proposal

Combined: All Three Together

These methods are complementary and can be combined for multiplicative gains. For example, when pricing an Asian option:

  1. Use Importance Sampling to shift the drift toward in-the-money paths
  2. Apply Antithetic Variates by mirroring each simulated path
  3. Use the geometric Asian option (known analytically) as a control variate
  4. Result: variance reduction of 100× or more vs. plain MC

Lab 2: Monte Carlo Option Pricing

# ── LAB 2: Monte Carlo European Option Pricer ─────────────────
import numpy as np
from scipy.stats import norm
import matplotlib.pyplot as plt

def black_scholes_call(S0, K, r, sigma, T):
    """Analytical Black-Scholes call price"""
    d1 = (np.log(S0/K) + (r + 0.5*sigma**2)*T) / (sigma*np.sqrt(T))
    d2 = d1 - sigma*np.sqrt(T)
    return S0*norm.cdf(d1) - K*np.exp(-r*T)*norm.cdf(d2)

def mc_call(S0, K, r, sigma, T, n):
    """Standard Monte Carlo"""
    Z = np.random.standard_normal(n)
    ST = S0 * np.exp((r - 0.5*sigma**2)*T + sigma*np.sqrt(T)*Z)
    payoffs = np.maximum(ST - K, 0) * np.exp(-r*T)
    return payoffs.mean(), payoffs.std() / np.sqrt(n)

# Parameters
S0, K, r, sigma, T = 100, 105, 0.05, 0.20, 1.0
true_price = black_scholes_call(S0, K, r, sigma, T)
print(f"True BS price: ${true_price:.4f}")

# Convergence study
sample_sizes = [100, 500, 1000, 5000, 10000, 50000]
estimates, std_errors = [], []

for n in sample_sizes:
    est, se = mc_call(S0, K, r, sigma, T, n)
    estimates.append(est)
    std_errors.append(se)
    print(f"n={n:6d}: price=${est:.4f}, SE={se:.4f}, error={abs(est-true_price):.4f}")

# Plot convergence
fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(12, 4))

ax1.semilogx(sample_sizes, estimates, 'o-', color='#00d4ff')
ax1.axhline(true_price, color='#ff6b35', linestyle='--', label='True Price')
ax1.set_title('MC Estimate vs Sample Size')
ax1.set_xlabel('n (log scale)'); ax1.legend()

ax2.loglog(sample_sizes, std_errors, 's-', color='#a855f7', label='Std Error')
ax2.loglog(sample_sizes, [std_errors[0]/np.sqrt(n/sample_sizes[0]) for n in sample_sizes], 
           '--', color='#f5c842', label='O(1/√n) reference')
ax2.set_title('Standard Error Convergence')
ax2.set_xlabel('n (log scale)'); ax2.legend()

plt.tight_layout()
plt.show()

Lab 3: All Three Variance Reduction Methods

# ── LAB 3: Variance Reduction Comparison ─────────────────────
import numpy as np
from scipy.stats import norm
import matplotlib.pyplot as plt

S0, K, r, sigma, T = 100, 105, 0.05, 0.20, 1.0
n_trials = 500   # number of independent runs
n_paths  = 1000  # paths per run
disc = np.exp(-r*T)
drift = (r - 0.5*sigma**2)*T
vol = sigma*np.sqrt(T)

def true_price():
    d1 = (np.log(S0/K)+(r+0.5*sigma**2)*T)/(sigma*np.sqrt(T))
    return S0*norm.cdf(d1)-K*disc*norm.cdf(d1-vol)
C_true = true_price()

mc_estimates, av_estimates, cv_estimates, is_estimates = [], [], [], []

for _ in range(n_trials):
    Z = np.random.standard_normal(n_paths)

    # 1. Standard MC
    ST = S0*np.exp(drift+vol*Z)
    mc = disc*np.maximum(ST-K,0).mean()
    mc_estimates.append(mc)

    # 2. Antithetic Variates (n/2 pairs)
    Z_av = Z[:n_paths//2]
    ST_p = S0*np.exp(drift+vol*Z_av)
    ST_m = S0*np.exp(drift-vol*Z_av)  # antithetic
    av = disc*((np.maximum(ST_p-K,0)+np.maximum(ST_m-K,0))/2).mean()
    av_estimates.append(av)

    # 3. Control Variates (control = S_T, E[S_T] = S0*e^(rT))
    E_ST = S0*np.exp(r*T)
    payoffs = disc*np.maximum(ST-K,0)
    b_star = np.cov(payoffs,ST)[0,1]/np.var(ST)
    cv = payoffs.mean() - b_star*(ST.mean()-E_ST)
    cv_estimates.append(cv)

    # 4. Importance Sampling (drift shift toward K)
    theta = np.log(K/S0)/(sigma*np.sqrt(T)) - (r/sigma-sigma/2)*np.sqrt(T)
    Z_is = Z + theta                 # shifted draws
    ST_is = S0*np.exp(drift+vol*Z_is)
    weights = np.exp(-theta*Z-0.5*theta**2)  # Radon-Nikodym
    is_est = disc*(np.maximum(ST_is-K,0)*weights).mean()
    is_estimates.append(is_est)

# ── Results ──
results = {
    'Standard MC': mc_estimates,
    'Antithetic Variates': av_estimates,
    'Control Variates': cv_estimates,
    'Importance Sampling': is_estimates,
}
print(f"True Price: ${C_true:.4f}\n")
for name, ests in results.items():
    v = np.var(ests)
    base_v = np.var(mc_estimates)
    print(f"{name:25s}: mean={np.mean(ests):.4f}, var={v:.6f}, reduction={100*(1-v/base_v):.1f}%")

Lab 4: Importance Sampling for Rare Events

# ── LAB 4: IS for Rare Event Probability Estimation ──────────
import numpy as np
from scipy.stats import norm
import matplotlib.pyplot as plt

# Estimate P(X > t) for X~N(0,1) as t gets large
thresholds = [1.0, 1.5, 2.0, 2.5, 3.0, 3.5, 4.0]
n = 10000
n_rep = 200  # repetitions to estimate variance

def estimate_mc(t, n):
    X = np.random.standard_normal(n)
    return (X > t).mean()

def estimate_is(t, n):
    # Optimal IS proposal for tail: N(t, 1)
    X = np.random.normal(t, 1, n)
    # Likelihood ratio: p(X)/q(X) = N(0,1)/N(t,1)
    w = norm.pdf(X, 0, 1) / norm.pdf(X, t, 1)
    return ((X > t) * w).mean()

print("t\t\tTrue Prob\tMC Std Err\tIS Std Err\tSpeedup")
print("-"*70)
for t in thresholds:
    true_p = 1 - norm.cdf(t)
    mc_ests  = [estimate_mc(t,  n//10) for _ in range(n_rep)]  # cheaper MC
    is_ests  = [estimate_is(t, n//10) for _ in range(n_rep)]
    mc_se   = np.std(mc_ests)
    is_se   = np.std(is_ests)
    speedup = (mc_se/is_se)**2 if is_se > 0 else np.inf
    print(f"t={t:.1f}\t{true_p:.2e}\t{mc_se:.2e}\t{is_se:.2e}\t{speedup:.0f}x")
